## 15.1 Вступ до SQLite

Є безліч SQL-баз даних, і деякі з них краще підходять для певних цілей. Одна з найпростіших і найлегших SQL-баз даних — SQLite, що запускається безпосередньо на вашому комп'ютері і автоматично постачається разом із Python.

У цьому розділі ви навчитеся використовувати пакет `sqlite3` для створення нової бази даних, зберігання та отримання даних.

### Основи SQLite

Є чотири основні кроки для роботи з SQLite:

1. Імпортувати пакет `sqlite3`.
2. Підключитися до наявної бази даних або створити нову.
3. Виконати SQL-оператори на базі даних.
4. Закрити з'єднання з базою даних.

Відкрийте IDLE і введіть таке:

```python
>>> import sqlite3
>>> connection = sqlite3.connect("test_database.db")
```

Функція `sqlite3.connect()` використовується для підключення до бази даних або її створення. При виконанні `.connect("test_database.db")` Python шукає наявну базу даних. Якщо вона не знайдена, створюється нова в поточному робочому каталозі. Щоб створити базу даних в іншому каталозі, потрібно вказати повний шлях.

> **Примітка**
>
> Якщо ви хочете створити одноразову базу даних для тестування, можна використати спеціальну назву `":memory:"` для створення бази у тимчасовій пам'яті:
>
> ```python
> connection = sqlite3.connect(":memory:")
> ```

Функція `.connect()` повертає об'єкт `sqlite3.Connection`. Для зберігання та отримання даних потрібен об'єкт `Cursor`, що отримується методом `Connection.cursor()`:

```python
>>> cursor = connection.cursor()
>>> type(cursor)
<class 'sqlite3.Cursor'>
```

Об'єкт `sqlite3.Cursor` — це ваш шлях до взаємодії з базою даних. За допомогою `Cursor` можна створювати таблиці, виконувати SQL-оператори та отримувати результати запитів.

Скористаємося функцією SQLite `datetime` для отримання поточного місцевого часу:

```python
>>> query = "SELECT datetime('now', 'localtime');"
>>> cursor.execute(query)
<sqlite3.Cursor object at 0x000001A27EB85E30>
```

Для отримання результатів запиту використовується метод `cursor.fetchone()`, що повертає кортеж із першим рядком результатів:

```python
>>> cursor.fetchone()
('2018-11-20 23:07:21',)
```

Можна об'єднати `.execute()` і `.fetchone()` в одному рядку:

```python
>>> time = cursor.execute(query).fetchone()[0]
>>> time
'2018-11-20 23:09:45'
```

Для закриття з'єднання з базою даних використовується `connection.close()`.

### Використання with для керування з'єднанням із базою даних

Так само як оператор `with` з функцією `open()` автоматично закриває файл, він також застосовується до з'єднань із базою даних SQLite:

```python
>>> with sqlite3.connect("test_database.db") as connection:
...     cursor = connection.cursor()
...     query = "SELECT datetime('now', 'localtime');"
...     time = cursor.execute(query).fetchone()[0]
>>> time
'2018-11-20 23:14:37'
```

Керування з'єднанням за допомогою оператора `with` має кілька переваг: код часто стає чистішим і коротшим, а будь-які зміни в базі даних автоматично зберігаються (commit) після виходу з блоку `with`.

### Робота з таблицями бази даних

Для зберігання даних у базі потрібно створити таблицю. Створімо таблицю `People` із трьома стовпцями: `FirstName`, `LastName` і `Age`:

```sql
CREATE TABLE People(FirstName TEXT, LastName TEXT, Age INT);
```

Після створення таблиці її можна заповнити даними командою `INSERT INTO`:

```sql
INSERT INTO People VALUES('Ron', 'Obvious', 42);
```

> **Примітка**
>
> Рядки в SQL обмежуються одинарними лапками. Коли ви пишете SQL-запити у вигляді рядків Python, переконайтеся, що вони обмежені подвійними лапками, щоб всередині можна було використовувати одинарні.

Ось як виконати ці оператори та зберегти зміни:

```python
import sqlite3

with sqlite3.connect("test_database.db") as connection:
    cursor = connection.cursor()
    cursor.execute(
        """CREATE TABLE People(
        FirstName TEXT,
        LastName TEXT,
        Age INT
        );"""
    )
    cursor.execute(
        """INSERT INTO People VALUES(
        'Ron',
        'Obvious',
        42
        );"""
    )
```

При використанні оператора `with` не потрібно викликати `connection.commit()` і `connection.close()` — зміни зберігаються автоматично при виході з блоку `with`.

### Виконання кількох SQL-операторів

Якщо потрібно виконати більше одного SQL-оператора одночасно, є кілька варіантів. Перший — метод `.executescript()`, якому передається рядок із повним SQL-скриптом:

```python
import sqlite3

with sqlite3.connect("test_database.db") as connection:
    cursor = connection.cursor()
    cursor.executescript(
        """DROP TABLE IF EXISTS People;
        CREATE TABLE People(
        FirstName TEXT,
        LastName TEXT,
        Age INT
        );
        INSERT INTO People VALUES(
        'Ron',
        'Obvious',
        '42'
        );"""
    )
```

Також можна виконати багато схожих операторів за допомогою методу `.executemany()`, передавши кортеж кортежів, де кожен внутрішній кортеж містить значення для окремої команди:

```python
people_values = (
    ("Ron", "Obvious", 42),
    ("Luigi", "Vercotti", 43),
    ("Arthur", "Belling", 28)
)

cursor.executemany("INSERT INTO People VALUES(?, ?, ?)", people_values)
```

Тут знаки питання виступають як заповнювачі для кортежів у `people_values`. Це називається параметризованим оператором (parameterized statement).

### Уникнення проблем безпеки за допомогою параметризованих операторів

З міркувань безпеки, особливо при взаємодії з SQL-таблицею на основі введення користувача, завжди слід використовувати параметризовані SQL-оператори. Це пов'язано з тим, що користувач потенційно може ввести значення, що виглядає як SQL-код і спричинить несподівану поведінку. Це називається SQL-ін'єкцією (SQL injection attack).

Неправильний спосіб — з використанням f-рядка:

```python
# НЕБЕЗПЕЧНО: вразливо до SQL-ін'єкцій!
cursor.execute(
    f"INSERT INTO People Values('{first_name}', '{last_name}', {age});"
)
```

Правильний спосіб — з параметризованим оператором:

```python
data = (first_name, last_name, age)
cursor.execute("INSERT INTO People VALUES(?, ?, ?);", data)
```

Також можна оновлювати вміст рядка за допомогою параметризованого SQL-оператора `UPDATE`:

```python
cursor.execute(
    "UPDATE People SET Age=? WHERE FirstName=? AND LastName=?;",
    (45, 'Luigi', 'Vercotti')
)
```

### Отримання даних

Для отримання даних із бази можна використати методи `.fetchone()` і `.fetchall()`. `.fetchone()` повертає один рядок результатів, `.fetchall()` — усі результати запиту відразу.

```python
import sqlite3

values = (
    ("Ron", "Obvious", 42),
    ("Luigi", "Vercotti", 43),
    ("Arthur", "Belling", 28),
)

with sqlite3.connect("test_database.db") as connection:
    cursor = connection.cursor()
    cursor.execute("DROP TABLE IF EXISTS People")
    cursor.execute(
        """CREATE TABLE People(
        FirstName TEXT,
        LastName TEXT,
        Age INT
        );"""
    )
    cursor.executemany("INSERT INTO People VALUES(?, ?, ?);", values)

    # Вибрати імена та прізвища всіх людей старших за 30
    cursor.execute(
        "SELECT FirstName, LastName FROM People WHERE Age > 30;"
    )
    for row in cursor.fetchall():
        print(row)
```

`.fetchall()` повертає результати запиту у вигляді списку кортежів, де кожен кортеж містить дані з одного рядка. Результат:

```
('Ron', 'Obvious')
('Luigi', 'Vercotti')
```

## 15.2 Бібліотеки для роботи з іншими SQL-базами даних

Якщо ви хочете працювати з певним типом SQL-бази даних через Python, більша частина базового синтаксису, мабуть, буде ідентична тому, що ви щойно вивчили для SQLite. Однак вам потрібно встановити додатковий пакет, оскільки SQLite — єдиний вбудований варіант.

Є багато варіантів SQL і відповідних Python-пакетів. Ось кілька найбільш поширених і надійних альтернатив SQLite з відкритим кодом:

- **pyodbc** — підключається до ODBC-баз даних, зокрема Microsoft SQL Server;
- **psycopg2** — підключається до бази даних PostgreSQL;
- **PyMySQL** — підключається до баз даних MySQL.

Одна відмінність між SQLite та іншими базами даних полягає в тому, що більшість баз потребують імені користувача та пароля для підключення.

Пакет **SQLAlchemy** — ще один популярний варіант для роботи з базами даних. SQLAlchemy — це об'єктно-реляційне відображення (ORM), що використовує об'єктно-орієнтовану парадигму для побудови запитів до бази даних. Воно може бути налаштоване для підключення до різних баз даних і дозволяє робити запити без написання «сирого» SQL.

## 15.3 Підсумок та додаткові ресурси

У цьому розділі ви навчилися взаємодіяти з базою даних SQLite, що постачається разом із Python. SQLite — це мала та легка SQL-база, що може використовуватися для зберігання та отримання даних у програмах Python. Для взаємодії з SQLite у Python потрібно імпортувати модуль `sqlite3`.

Для роботи з базою даних SQLite спочатку потрібно підключитися до наявної або створити нову за допомогою функції `sqlite3.connect()`, що повертає об'єкт `Connection`. Потім можна використати метод `Connection.cursor()` для отримання нового об'єкта `Cursor`.

Об'єкти `Cursor` використовуються для виконання SQL-операторів і отримання результатів запитів. `Cursor.execute()` і `Cursor.executescript()` використовуються для виконання SQL-запитів. Результати отримуються методами `Cursor.fetchone()` і `Cursor.fetchall()`.
