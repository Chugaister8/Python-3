## 5.1 Цілі числа та числа з рухомою комою

У Python є три вбудованих числових типи даних: цілі числа, числа з рухомою комою та комплексні числа. У цьому розділі ви дізнаєтеся про цілі числа та числа з рухомою комою — два найбільш уживаних числових типи. Про комплексні числа ви дізнаєтеся в розділі 5.6.

### Цілі числа

Ціле число — це число без знаків після коми. Наприклад, `1` є цілим числом, а `1.0` — ні. Тип даних для цілих чисел має назву `int`, що можна побачити за допомогою функції type():

```python
>>> type(1)
<class 'int'>
```

Ціле число можна створити, просто вписавши число явно або використавши функцію int(). У Розділі 4 ви навчилися перетворювати рядок із цілим числом на число за допомогою int(). Наприклад, наступний код перетворює рядок `"25"` на ціле число `25`:

```python
>>> int("25")
25
```

Цілочисельний літерал — це значення цілого числа, явно записане у вашому коді, так само як рядковий літерал — це рядок, явно записаний у коді. Наприклад, `1` є цілочисельним літералом, а `int("1")` — ні.

Цілочисельні літерали можна записувати двома способами:

```python
>>> 1000000
1000000
>>> 1_000_000
1000000
```

Перший приклад простий: просто вводиться цифра 1, а після неї шість нулів. Недоліком такого запису є те, що великі числа важко читати.

Коли ви записуєте великі числа від руки, ви, мабуть, групуєте цифри по три, розділяючи їх комою. `1 000 000` читається набагато легше, ніж `1000000`.

У Python не можна використовувати коми для групування цифр у цілочисельних літералах, але можна використовувати підкреслення (`_`). Значення `1_000_000` виражає мільйон у зручнішому для читання вигляді.

Розмір цілого числа не обмежений, що може здивувати, враховуючи скінченну пам'ять комп'ютерів. Спробуйте ввести в інтерактивне вікно IDLE найбільше число, яке тільки можете придумати. Python впорається з ним без проблем!

### Числа з рухомою комою

Число з рухомою комою, або скорочено float, — це число з десятковою крапкою. `1.0` є числом з рухомою комою, як і `-2.75`. Тип даних для чисел з рухомою комою має назву `float`:

```python
>>> type(1.0)
<class 'float'>
```

Числа з рухомою комою можна створити, вписавши число безпосередньо у код або за допомогою функції float(). Як і int(), float() можна використовувати для перетворення рядка, що містить число, на число з рухомою комою:

```python
>>> float("1.25")
1.25
```

Літерал числа з рухомою комою — це значення числа з рухомою комою, явно записане у коді. `1.25` є літералом числа з рухомою комою, а `float("1.25")` — ні.

Літерали чисел з рухомою комою можна записувати трьома способами. Кожен із наступних записів створює літерал числа з рухомою комою зі значенням один мільйон:

```python
>>> 1000000.0
1000000.0
>>> 1_000_000.0
1000000.0
>>> 1e6
1000000.0
```

Перші два способи схожі на два методи запису цілочисельних літералів, які ви бачили раніше. Другий метод із підкресленнями для розділення цифр на групи по три зручний для створення літералів чисел з рухомою комою з великою кількістю цифр.

Для дуже великих чисел можна використовувати E-запис. Третій метод у попередньому прикладі використовує E-запис для створення літерала числа з рухомою комою.

Щоб записати літерал числа з рухомою комою в E-записі, введіть число, після нього літеру `e`, а потім ще одне число. Python бере число зліва від `e` і множить його на 10 в ступені числа після `e`. Таким чином, `1e6` еквівалентно `1×10⁶`.

> **Примітка**
>
> E-запис є скороченням від exponential notation («експоненціальний запис») і є більш поширеною назвою для способу відображення великих чисел на багатьох калькуляторах і в мовах програмування.

Python також використовує E-запис для відображення великих чисел з рухомою комою:

```python
>>> 200000000000000000.0
2e+17
```

Число з рухомою комою `200000000000000000.0` відображається як `2e+17`. Знак `+` вказує на те, що показник ступеня `17` є позитивним числом. Також можна використовувати від'ємні числа як показник ступеня:

```python
>>> 1e-4
0.0001
```

Літерал `1e-4` інтерпретується як 10 в ступені `-4`, що дорівнює `1/10000` або, еквівалентно, `0.0001`.

На відміну від цілих чисел, числа з рухомою комою мають максимальний розмір. Максимальне число з рухомою комою залежить від вашої системи, але щось на кшталт `2e400` має бути далеко за межами можливостей більшості машин. `2e400` — це `2×10⁴⁰⁰`, що значно більше за загальну кількість атомів у Всесвіті!

Коли ви досягаєте максимального числа з рухомою комою, Python повертає спеціальне значення `inf`:

```python
>>> 2e400
inf
```

`inf` означає «нескінченність» і просто вказує на те, що число, яке ви намагалися створити, перевищує максимальне допустиме значення числа з рухомою комою на вашому комп'ютері. Тип `inf` — все одно `float`:

```python
>>> n = 2e400
>>> n
inf
>>> type(n)
<class 'float'>
```

Також існує `-inf`, що означає від'ємну нескінченність і представляє від'ємне число з рухомою комою, що виходить за межі мінімального допустимого значення на вашому комп'ютері:

```python
>>> -2e400
-inf
```

Як програміст, ви навряд чи часто стикатиметеся з `inf` і `-inf`, якщо тільки не працюєте регулярно з надзвичайно великими числами.

## 5.2 Арифметичні оператори та вирази

У цьому розділі ви навчитеся виконувати базову арифметику з числами в Python: додавання, віднімання, множення та ділення. Попутно ви дізнаєтеся деякі угоди про запис математичних виразів у коді.

### Додавання

Додавання виконується за допомогою оператора `+`:

```python
>>> 1 + 2
3
```

Два числа по обидва боки від оператора `+` називаються операндами. У попередньому прикладі обидва операнди є цілими числами, але операнди не обов'язково мають бути одного типу. Ціле число можна додати до числа з рухомою комою без жодних проблем:

```python
>>> 1.0 + 2
3.0
```

Зверніть увагу: результат `1.0 + 2` дорівнює `3.0`, що є числом з рухомою комою. Будь-коли, коли число з рухомою комою додається до числа, результат також є числом з рухомою комою. Додавання двох цілих чисел завжди дає `int`.

> **Примітка**
>
> PEP 8 рекомендує відокремлювати обидва операнди від оператора пробілом.
>
> Python чудово обчислить `1+1`, але `1 + 1` є бажаним форматом, оскільки загалом вважається більш зрозумілим. Це практичне правило стосується всіх операторів у цьому розділі.

### Віднімання

Щоб відняти два числа, просто поставте `-` між ними:

```python
>>> 1 - 1
0
>>> 5.0 - 3
2.0
```

Так само як і при додаванні двох цілих чисел, віднімання двох цілих чисел завжди дає `int`. Якщо один із операндів є числом з рухомою комою, результат також є числом з рухомою комою.

Оператор `-` також використовується для позначення від'ємних чисел:

```python
>>> -3
-3
```

Від'ємне число можна відняти від іншого числа, але, як видно нижче, це іноді може виглядати заплутано:

```python
>>> 1 - -3
4
>>> 1 - (-3)
4
```

Використання дужок є гарною ідеєю, оскільки робить код більш явним. Комп'ютери виконують код, але люди його читають. Все, що ви можете зробити для полегшення читання та розуміння коду, — це добра справа.

### Множення

Для множення двох чисел використовується оператор `*`:

```python
>>> 3 * 3
9
>>> 2 * 8.0
16.0
```

Тип числа, яке отримується при множенні, підпорядковується тим самим правилам, що і при додаванні та відніманні. Множення двох цілих чисел дає `int`, а множення числа на float дає `float`.

### Ділення

Оператор `/` використовується для ділення двох чисел:

```python
>>> 9 / 3
3.0
>>> 5.0 / 2
2.5
```

На відміну від додавання, віднімання та множення, ділення за допомогою оператора `/` завжди повертає float. Якщо потрібно отримати ціле число після ділення двох чисел, можна скористатися int() для перетворення результату:

```python
>>> int(9 / 3)
3
```

Майте на увазі, що int() відкидає будь-яку дробову частину числа:

```python
>>> int(5.0 / 2)
2
```

`5.0 / 2` повертає float `2.5`, а `int(2.5)` повертає ціле число `2` з відкинутою частиною `.5`.

### Цілочисельне ділення

Якщо запис `int(5.0 / 2)` здається вам трохи громіздким, Python надає другий оператор ділення `//`, що називається оператором цілочисельного ділення:

```python
>>> 9 // 3
3
>>> 5.0 // 2
2.0
>>> -3 // 2
-2
```

Оператор `//` спочатку ділить число зліва на число справа, а потім округлює результат до цілого числа. При від'ємних числах це може дати не той результат, який ви очікуєте.

Наприклад, `-3 // 2` повертає `-2`. Спочатку `-3` ділиться на `2`, і отримується `-1.5`. Потім `-1.5` округлюється в бік зменшення до `-2`. З іншого боку, `3 // 2` повертає `1`.

Також зверніть увагу: `//` повертає число з рухомою комою, якщо один із операндів є float. Саме тому `9 // 3` повертає ціле число `3`, а `5.0 // 2` повертає float `2.0`.

Подивімося, що відбудеться, коли ви спробуєте поділити число на `0`:

```python
>>> 1 / 0
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ZeroDivisionError: division by zero
```

Python генерує `ZeroDivisionError`, повідомляючи вас про спробу порушити фундаментальний закон всесвіту.

### Піднесення до степеня

Число можна піднести до степеня за допомогою оператора `**`:

```python
>>> 2 ** 2
4
>>> 2 ** 3
8
>>> 2 ** 4
16
```

Показник ступеня не обов'язково має бути цілим числом. Він також може бути float:

```python
>>> 3 ** 1.5
5.196152422706632
>>> 9 ** 0.5
3.0
```

Піднесення числа до степеня `0.5` еквівалентне вилученню квадратного кореня, але зверніть увагу: хоча квадратний корінь із `9` є цілим числом, Python повертає float `3.0`.

Для позитивних операндів оператор `**` повертає ціле число, якщо обидва операнди є цілими числами, і float, якщо хоча б один із операндів є числом з рухомою комою.

Числа також можна піднести до від'ємного степеня:

```python
>>> 2 ** -1
0.5
>>> 2 ** -2
0.25
```

Піднесення числа до від'ємного степеня еквівалентно діленню `1` на число, піднесене до відповідного позитивного степеня. Таким чином, `2 ** -1` — це те саме, що `1 / (2 ** 1)`, що дорівнює `1 / 2`, або `0.5`. Аналогічно, `2 ** -2` — це те саме, що `1 / (2 ** 2)`, що дорівнює `1 / 4`, або `0.25`.

### Оператор залишку від ділення

Оператор `%`, або оператор залишку від ділення (modulus), повертає залишок від ділення лівого операнда на правий:

```python
>>> 5 % 3
2
>>> 20 % 7
6
>>> 16 % 8
0
```

`3` ділиться на `5` один раз із залишком `2`, тому `5 % 3` дорівнює `2`. Аналогічно, `7` ділиться на `20` двічі із залишком `6`.

В останньому прикладі `16` ділиться на `8` без залишку, тому `16 % 8` дорівнює `0`. Коли число зліва від `%` ділиться на число справа без залишку, результат дорівнює `0`.

Одне з найбільш поширених застосувань `%` — визначення того, чи ділиться одне число на інше. Наприклад, число `n` є парним тоді й лише тоді, коли `n % 2` дорівнює `0`.

Як ви думаєте, що повертає `1 % 0`? Перевіримо:

```python
>>> 1 % 0
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ZeroDivisionError: integer division or modulo by zero
```

Це логічно: `1 % 0` є залишком від ділення `1` на `0`. Але ділити `1` на `0` не можна, тому Python генерує `ZeroDivisionError`.

> **Примітка**
>
> Під час роботи в інтерактивному вікні IDLE помилки на кшталт `ZeroDivisionError` не спричиняють великих проблем. Помилка відображається, і з'являється новий рядок запрошення, що дає змогу продовжувати писати код.
>
> Однак щоразу, коли Python стикається з помилкою під час виконання скрипту, виконання зупиняється. Говорять, що програма аварійно завершилась (crashed). У Розділі 8 ви навчитеся обробляти помилки, щоб програми не завершувалися аварійно несподівано.

### Арифметичні вирази

Оператори можна комбінувати для формування складних виразів. Вираз — це комбінація чисел, операторів та дужок, яку Python може обчислити (evaluate) для отримання значення.

Ось кілька прикладів арифметичних виразів:

```python
>>> 2*3 - 1
5
>>> 4/2 + 2**3
10.0
>>> -1 + (-3*2 + 4)
-3
```

Правила обчислення виразів такі самі, як і в звичайній арифметиці. У школі ви, мабуть, вивчали ці правила під назвою «порядок дій».

Оператори `*`, `/`, `//` і `%` мають однаковий пріоритет у виразі, і всі вони мають вищий пріоритет, ніж оператори `+` і `-`. Саме тому `2*3 - 1` повертає `5`, а не `4`: вираз `2*3` обчислюється першим, оскільки `*` має вищий пріоритет, ніж `-`.

Ви можете помітити, що вирази в попередньому прикладі не дотримуються правила про пробіл з обох боків від усіх операторів. PEP 8 говорить таке про пробіли у складних виразах:

> «Якщо використовуються оператори з різними пріоритетами, розгляньте можливість додавання пробілів навколо операторів із нижчим пріоритетом. Керуйтеся власним судженням; однак ніколи не використовуйте більше одного пробілу і завжди зберігайте однакову кількість пробілів з обох боків бінарного оператора.»
>
> — PEP 8, Інші рекомендації

## 5.4 Змусимо Python «збрехати»

Як ви думаєте, чому дорівнює `0.1 + 0.2`? Відповідь `0.3`, правда? Подивімося, що на це скаже Python. Спробуйте в інтерактивному вікні:

```python
>>> 0.1 + 0.2
0.30000000000000004
```

Ну, це… майже правильно! Що взагалі відбувається? Це помилка в Python?

Ні, це не помилка! Це помилка представлення числа з рухомою комою (floating-point representation error), і вона не має нічого спільного з Python. Вона пов'язана зі способом зберігання чисел з рухомою комою в пам'яті комп'ютера.

Число `0.1` можна представити у вигляді дробу `1/10`. І число `0.1`, і дріб `1/10` є десятковими представленнями, або представленнями за основою 10. Однак комп'ютери зберігають числа з рухомою комою у двійковому представленні (база 2), яке більш відоме як бінарне.

У бінарному представленні з десятковим числом `0.1` відбувається щось знайоме і водночас несподіване. Дріб `1/3` не має скінченного десяткового представлення: `1/3 = 0.3333...` з нескінченною кількістю трійок після коми. Те саме відбувається з дробом `1/10` у двійковій системі числення.

Двійкове представлення `1/10` — це нескінченно повторюваний дріб:

```
0.00011001100110011001100110011...
```

Комп'ютери мають скінченну пам'ять, тому число `0.1` має зберігатися як наближення, а не як точне значення. Збережене наближення трохи більше за справжнє значення і виглядає так:

```
0.1000000000000000055511151231257827021181583404541015625
```

Однак ви, мабуть, помітили: коли Python виводить `0.1`, він відображає `0.1`, а не наближене значення вище:

```python
>>> 0.1
0.1
```

Python не просто обрізає цифри двійкового представлення `0.1`. Те, що насправді відбувається, трохи тонше.

Оскільки наближення `0.1` у двійковій системі є саме наближенням, цілком можливо, що більше ніж одне десяткове число матиме однакове двійкове наближення. Наприклад, числа `0.1` і `0.10000000000000001` мають однакове двійкове наближення. Python виводить найкоротше десяткове число, яке відповідає цьому наближенню.

Це пояснює, чому в першому прикладі цього розділу `0.1 + 0.2` не дорівнює `0.3`. Python складає двійкові наближення `0.1` і `0.2`, що дає число, яке не є двійковим наближенням `0.3`.

Якщо від усього цього голова йде обертом — не хвилюйтеся! Якщо ви не пишете програми для фінансів або наукових обчислень, вам не потрібно турбуватися про неточність арифметики чисел з рухомою комою.

## 5.5 Математичні функції та методи чисел

У Python є кілька вбудованих функцій для роботи з числами. У цьому розділі ви дізнаєтеся про три найбільш поширені з них:

1. round() — для округлення чисел до заданої кількості знаків після коми;
2. abs() — для отримання абсолютного значення числа;
3. pow() — для піднесення числа до певного степеня.

Ви також дізнаєтеся про метод, який мають числа з рухомою комою, для перевірки того, чи є значення цілочисельним.

Вперед!

### Функція round()

За допомогою round() можна округлити число до найближчого цілого:

```python
>>> round(2.3)
2
>>> round(2.7)
3
```

round() має дещо несподівану поведінку, коли число закінчується на `.5`:

```python
>>> round(2.5)
2
>>> round(3.5)
4
```

`2.5` округлюється до `2`, а `3.5` — до `4`. Більшість людей очікують, що число, яке закінчується на `.5`, округляється вгору, тому розгляньмо ближче, що тут відбувається.

Python 3 округлює числа відповідно до стратегії «округлення до парного» (rounding ties to even). «Рівне значення» (tie) — це будь-яке число, остання цифра якого дорівнює п'яти. `2.5` і `3.1415` є «рівними значеннями», а `1.37` — ні.

При округленні «рівного значення» до парного спочатку дивляться на цифру на одну десяткову позицію лівіше останньої цифри. Якщо ця цифра парна — округлення відбувається вниз. Якщо непарна — вгору. Саме тому `2.5` округлюється до `2`, а `3.5` — до `4`.

> **Примітка**
>
> Округлення «рівних значень» до парного — це стратегія округлення для чисел з рухомою комою, рекомендована IEEE (Інститутом інженерів електротехніки та електроніки), оскільки вона допомагає обмежити вплив округлення на операції, що включають велику кількість чисел.
>
> IEEE підтримує стандарт IEEE 754 для роботи з числами з рухомою комою на комп'ютері. Він був опублікований у 1985 році і досі широко використовується виробниками апаратного забезпечення.

Число можна округлити до заданої кількості знаків після коми, передавши другий аргумент у round():

```python
>>> round(3.14159, 3)
3.142
>>> round(2.71828, 2)
2.72
```

Число `3.14159` округлюється до 3 знаків після коми і дає `3.142`, а число `2.71828` округлюється до 2 знаків після коми і дає `2.72`.

Другий аргумент round() має бути цілим числом. Якщо це не так, Python генерує `TypeError`:

```python
>>> round(2.65, 1.4)
Traceback (most recent call last):
  File "<pyshell#0>", line 1, in <module>
round(2.65, 1.4)
TypeError: 'float' object cannot be interpreted as an integer
```

Іноді round() дає не зовсім правильну відповідь:

```python
>>> # Очікуване значення: 2.68
>>> round(2.675, 2)
2.67
```

`2.675` є «рівним значенням», оскільки лежить рівно посередині між `2.67` і `2.68`. Оскільки Python округлює «рівні значення» до найближчого парного числа, можна очікувати, що `round(2.675, 2)` поверне `2.68`, але він повертає `2.67`. Ця помилка є результатом помилки представлення числа з рухомою комою, а не помилки в функції round().

Робота з числами з рухомою комою може бути досить виснажливою, але ця проблема не є специфічною для Python. Всі мови, що реалізують стандарт IEEE для чисел з рухомою комою, мають ті самі проблеми, включно з C/C++, Java та JavaScript.

Однак у більшості випадків незначні помилки, що виникають при роботі з числами з рухомою комою, є несуттєвими, і результати round() цілком задовільні.

### Функція abs()

Абсолютне значення числа `n` — це просто `n`, якщо `n` є позитивним, і `-n`, якщо `n` є від'ємним. Наприклад, абсолютне значення `3` дорівнює `3`, а абсолютне значення `-5` дорівнює `5`.

Щоб отримати абсолютне значення числа в Python, використовується функція abs():

```python
>>> abs(3)
3
>>> abs(-5.0)
5.0
```

abs() завжди повертає позитивне число того самого типу, що й аргумент. Тобто абсолютне значення цілого числа — завжди позитивне ціле число, а абсолютне значення float — завжди позитивний float.

### Функція pow()

У розділі 5.2 ви навчилися піднімати число до степеня за допомогою оператора `**`. Також можна використовувати функцію pow(). pow() приймає два аргументи. Перший — основа, тобто число, яке потрібно піднести до степеня, другий аргумент — показник степеня.

Наприклад, наступний код використовує pow() для піднесення `2` до степеня `3`:

```python
>>> pow(2, 3)
8
```

Так само як і `**`, показник степеня в pow() може бути від'ємним:

```python
>>> pow(2, -2)
0.25
```

Яка ж різниця між `**` і pow()? Функція pow() приймає необов'язковий третій аргумент, що обчислює перше число, піднесене до степеня другого числа, а потім бере залишок від ділення на третє число.

Іншими словами, `pow(x, y, z)` еквівалентно `(x ** y) % z`. Ось приклад із `x = 2`, `y = 3` і `z = 2`:

```python
>>> pow(2, 3, 2)
0
```

Спочатку `2` зводиться до степеня `3`, і отримується `8`. Потім обчислюється `8 % 2`, що дорівнює `0`, оскільки `2` ділить `8` без залишку.

### Перевірка того, чи є float цілочисельним

У Розділі 3 ви дізналися про методи рядків, такі як .lower(), .upper() і .find(). Цілі числа та числа з рухомою комою також мають методи.

Методи чисел використовуються не дуже часто, але один із них буває корисним. Числа з рухомою комою мають метод .is_integer(), що повертає `True`, якщо число є цілочисельним, тобто не має дробової частини, і `False` в іншому випадку:

```python
>>> num = 2.5
>>> num.is_integer()
False
>>> num = 2.0
>>> num.is_integer()
True
```

Метод .is_integer() може бути корисним для перевірки введення користувача. Наприклад, якщо ви пишете застосунок для кошика покупок в магазині, що продає піцу, вам потрібно перевірити, що кількість піц, яку вводить покупець, є цілим числом. Як виконувати такі перевірки, ви дізнаєтеся в Розділі 8.

## 5.6 Виведення чисел у гарному вигляді

Відображення чисел для користувача вимагає вставки чисел у рядок. У Розділі 3 ви дізналися, як це зробити за допомогою f-рядків, оточивши змінну, якій присвоєно число, фігурними дужками:

```python
>>> n = 7.125
>>> f"The value of n is {n}"
'The value of n is 7.125'
```

Ці фігурні дужки підтримують просту мову форматування, за допомогою якої можна змінити вигляд значення у фінальному форматованому рядку.

Наприклад, щоб відформатувати значення `n` у наведеному вище прикладі до двох знаків після коми, замініть вміст фігурних дужок у f-рядку на `{n:.2f}`:

```python
>>> n = 7.125
>>> f"The value of n is {n:.2f}"
'The value of n is 7.12'
```

Двокрапка (`:`) після змінної `n` вказує на те, що все після неї є специфікатором форматування. У цьому прикладі специфікатор форматування — `.2f`.

`.2` у `.2f` округлює число до двох знаків після коми, а `f` вказує Python відображати `n` як число з фіксованою крапкою. Це означає, що число відображається рівно з двома знаками після коми, навіть якщо оригінальне число має менше знаків.

Коли `n = 7.125`, результат `{n:.2f}` дорівнює `7.12`. Так само як і round(), Python при форматуванні чисел у рядках округлює «рівні значення» до парного. Тому, якщо замінити `n = 7.125` на `n = 7.126`, результат `{n:.2f}` буде `"7.13"`:

```python
>>> n = 7.126
>>> f"The value of n is {n:.2f}"
'The value of n is 7.13'
```

Щоб округлити до одного знаку після коми, замініть `.2` на `.1`:

```python
>>> n = 7.126
>>> f"The value of n is {n:.1f}"
'The value of n is 7.1'
```

Коли ви форматуєте число як число з фіксованою крапкою, воно завжди відображається з точно вказаною кількістю знаків після коми:

```python
>>> n = 1
>>> f"The value of n is {n:.2f}"
'The value of n is 1.00'
>>> f"The value of n is {n:.3f}"
'The value of n is 1.000'
```

Великі числа можна групувати тисячними роздільниками за допомогою опції `,`:

```python
>>> n = 1234567890
>>> f"The value of n is {n:,}"
'The value of n is 1,234,567,890'
```

Щоб округлити до певної кількості знаків після коми і водночас групувати за тисячами, поставте `,` перед крапкою `.` у специфікаторі форматування:

```python
>>> n = 1234.56
>>> f"The value of n is {n:,.2f}"
'The value of n is 1,234.56'
```

Специфікатор `,.2f` зручний для відображення грошових значень:

```python
>>> balance = 2000.0
>>> spent = 256.35
>>> remaining = balance - spent
>>> f"After spending ${spent:.2f}, I was left with ${remaining:,.2f}"
'After spending $256.35, I was left with $1,743.65'
```

Ще одна корисна опція — `%`, що використовується для відображення відсотків. Опція `%` множить число на `100` і відображає його у форматі числа з фіксованою крапкою, після чого додає знак відсотка.

Опція `%` завжди має стояти в кінці специфікатора форматування, і її не можна комбінувати з опцією `f`. Наприклад, `.1%` відображає число як відсоток з одним знаком після коми:

```python
>>> ratio = 0.9
>>> f"Over {ratio:.1%} of Pythonistas say 'Real Python rocks!'"
"Over 90.0% of Pythonistas say 'Real Python rocks!'"
>>> # Відображення відсотка з двома знаками після коми
>>> f"Over {ratio:.2%} of Pythonistas say 'Real Python rocks!'"
"Over 90.00% of Pythonistas say 'Real Python rocks!'"
```

Мова форматування є потужною та обширною. Тут показані лише основи. Для отримання додаткової інформації рекомендуємо звернутися до офіційної документації.

## 5.7 Комплексні числа

Python є однією з небагатьох мов програмування, що забезпечують вбудовану підтримку комплексних чисел. Хоча комплексні числа рідко трапляються за межами наукових обчислень та комп'ютерної графіки, їхня підтримка є однією з переваг Python.

> **Примітка**
>
> Комплексні числа зустрічаються лише в кількох специфічних ситуаціях. Багатьом програмістам вони ніколи не потрібні.
>
> Ви можете пропустити цей розділ, якщо вас не цікавить робота з комплексними числами в Python. Жодна інша частина книги не залежить від інформації в цьому розділі.

Якщо ви колись вивчали математику на рівні передобчислення або вищої алгебри, то пам'ятаєте, що комплексне число — це число з двома окремими компонентами: дійсною частиною та уявною частиною.

Для позначення комплексних чисел існує кілька способів, але поширеним методом є позначення дійсної частини літерою `i`, а уявної — літерою `j`. Наприклад, `1i + 2j` — це комплексне число з дійсною частиною `1` і уявною частиною `2`.

Щоб створити комплексне число в Python, просто запишіть дійсну частину, після неї знак плюс і уявну частину з літерою `j` наприкінці:

```python
>>> n = 1 + 2j
```

Коли ви перевіряєте значення `n`, Python відображає число в дужках:

```python
>>> n
(1+2j)
```

Ця угода допомагає уникнути плутанини: виведене значення не є рядком або математичним виразом.

Уявні числа мають дві властивості — `.real` і `.imag`, що повертають відповідно дійсну та уявну частину числа:

```python
>>> n.real
1.0
>>> n.imag
2.0
```

Зверніть увагу: Python повертає обидві частини як float, хоча вони були вказані як цілі числа.

Комплексні числа також мають метод .conjugate(), що повертає комплексно спряжене число:

```python
>>> n.conjugate()
(1-2j)
```

Для будь-якого комплексного числа його спряжене — це комплексне число з тією самою дійсною частиною і уявною частиною, однаковою за абсолютним значенням, але з протилежним знаком. Таким чином, комплексно спряженим до `1 + 2j` є `1 - 2j`.

> **Примітка**
>
> Властивості `.real` і `.imag` не вимагають дужок після себе, на відміну від методу .conjugate().
>
> Метод .conjugate() є функцією, що виконує певну дію над комплексним числом. `.real` і `.imag` не виконують жодних дій — вони просто повертають певну інформацію про число.
>
> Різниця між методами і властивостями є частиною об'єктно-орієнтованого програмування, про яке ви дізнаєтеся в Розділі 10.

Усі арифметичні оператори, що працюють з float та цілими числами, працюють і з комплексними числами, за винятком оператора цілочисельного ділення `//`. Оскільки це не книга з математики, механіку комплексної арифметики розглядати не будемо. Натомість ось кілька прикладів використання комплексних чисел з арифметичними операторами:

```python
>>> a = 1 + 2j
>>> b = 3 - 4j
>>> a + b
(4-2j)
>>> a - b
(-2+6j)
>>> a * b
(11+2j)
>>> a ** b
(932.1391946432212+95.9465336603419j)
>>> a / b
(-0.2+0.4j)
>>> a // b
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: can't take floor of complex number.
```

Цікаво (хоча з математичної точки зору і не дивно), що об'єкти `int` і `float` також мають властивості `.real` і `.imag`, а також метод .conjugate():

```python
>>> x = 42
>>> x.real
42
>>> x.imag
0
>>> x.conjugate()
42
>>> y = 3.14
>>> y.real
3.14
>>> y.imag
0.0
>>> y.conjugate()
3.14
```

Для float та цілих чисел `.real` і `.conjugate()` завжди повертають саме число, а `.imag` завжди повертає `0`. Варто зазначити: `n.real` і `n.imag` повертають ціле число, якщо `n` є цілим числом, і float, якщо `n` є float.

Тепер, коли ви ознайомилися з основами комплексних чисел, вам може бути цікаво, коли вони взагалі можуть знадобитися. Якщо ви вивчаєте Python для веброзробки або автоматизації, правда в тому, що вам може ніколи не знадобитися використовувати комплексні числа.

З іншого боку, комплексні числа важливі в таких сферах, як наукові обчислення та комп'ютерна графіка. Якщо вам доведеться працювати в цих сферах, вбудована підтримка комплексних чисел у Python буде корисною.

Детальний розгляд цих тем виходить за межі даної книги. Однак вступ до пакету NumPy — поширеного інструменту для наукових обчислень у Python — ви отримаєте в Розділі 17.
