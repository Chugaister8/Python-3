# Розділ 10. Об'єктно-орієнтоване програмування (ООП)

ООП, або об'єктно-орієнтоване програмування, — це метод структурування програми шляхом об'єднання пов'язаних властивостей та поведінок в окремі об'єкти.

Концептуально об'єкти схожі на компоненти системи. Уявіть програму як щось на зразок заводського конвеєра. Компонент системи на кожному кроці конвеєра обробляє матеріал трохи, в підсумку перетворюючи сировину на готовий продукт.

Об'єкт містить дані — подібно до сировини або напівфабрикатів на кожному кроці конвеєра — і поведінку, що нагадує дію, яку виконує кожен компонент конвеєра.

У цьому розділі ви навчитеся:

- створювати клас, що є своєрідним планом для створення об'єкта;
- використовувати класи для створення нових об'єктів;
- моделювати системи за допомогою успадкування класів.

Розпочнемо!

## 10.1 Визначення класу

Фундаментальні структури даних — числа, рядки та списки — призначені для представлення простих речей: вартості чогось, назви вірша, улюблених кольорів тощо. Але що робити, якщо потрібно представити щось значно складніше?

Наприклад, припустимо, вам потрібно відстежувати співробітників організації. Для кожного співробітника потрібно зберігати деяку базову інформацію: ім'я, вік, посаду та рік початку роботи.

Один зі способів — представити кожного співробітника у вигляді списку:

```python
kirk = ["James Kirk", 34, "Captain", 2265]
spock = ["Spock", 35, "Science Officer", 2254]
mccoy = ["Leonard McCoy", "Chief Medical Officer", 2266]
```

Із таким підходом є ряд проблем.

По-перше, коли ви посилаєтеся на `kirk[0]` за кілька рядків від місця, де оголошено список `kirk`, чи пам'ятатимете ви, що нульовий елемент — це ім'я співробітника? А що, якщо не кожен співробітник має однакову кількість елементів у списку?

По-друге, у списку `mccoy` вище відсутній вік, тому `mccoy[1]` поверне `"Chief Medical Officer"` замість віку доктора МакКоя.

Чудовим способом зробити такий код більш керованим і зручним для підтримки є використання класів.

### Класи та екземпляри

Класи використовуються для створення визначених користувачем структур даних. Класи також мають спеціальні функції, що називаються методами і визначають поведінки та дії, які може виконувати об'єкт, створений із класу.

У цьому розділі ви створите клас `Dog`, що зберігатиме деяку базову інформацію про собаку.

Важливо зазначити: клас лише надає структуру. Клас — це план того, як щось має бути визначено. Сам по собі він не надає жодного реального вмісту. Клас `Dog` може вказувати, що ім'я та вік є необхідними для визначення собаки, але не стверджуватиме, яке конкретно ім'я або вік у певної собаки.

Хоча клас — це план, екземпляр — це об'єкт, побудований із класу, що містить реальні дані. Екземпляр класу `Dog` — це вже не план. Це справжня собака з іменем, наприклад Miles, якому чотири роки.

Інакше кажучи, клас — це як бланк або анкета. Він визначає потрібну інформацію. Після заповнення бланку ваша конкретна копія є екземпляром класу. Вона містить реальну інформацію, що стосується вас.

Можна заповнити кілька копій бланку, щоб створити багато різних екземплярів, але без бланку як орієнтира ви б загубилися, не знаючи, яка інформація потрібна. Таким чином, перш ніж створювати окремі екземпляри об'єкта, необхідно спочатку визначити, що потрібно, задавши клас.

### Як визначити клас

Усі визначення класів починаються з ключового слова `class`, за яким слідують ім'я класу і двокрапка. Це схоже на сигнатуру функції, за винятком того, що не потрібно додавати параметри в дужках. Будь-який код із відступом нижче визначення класу вважається частиною тіла класу.

Ось приклад простого класу `Dog`:

```python
class Dog:
    pass
```

Тіло класу `Dog` складається з одного оператора: ключового слова `pass`. `pass` часто використовується як заповнювач там, де згодом з'явиться код. Воно дозволяє запускати цей код без помилок.

> **Примітка**
>
> На відміну від функцій і змінних, угода про іменування класів у Python — використання нотації CamelCase, починаючи з великої літери. Наприклад, клас для конкретної породи собаки, як-от джек-рассел-тер'єр, записується як `JackRussellTerrier`.

Клас `Dog` поки що не дуже цікавий, тому додамо деякі властивості, які мають мати всі об'єкти `Dog`. Щоб зберегти простоту, обмежимося двома: іменем і віком.

Щоб визначити властивості, або атрибути екземпляра (instance attributes), які повинні мати всі об'єкти `Dog`, необхідно визначити спеціальний метод .__init__(). Цей метод запускається щоразу, коли створюється новий об'єкт `Dog`, і повідомляє Python, яким має бути початковий стан об'єкта.

Першим позиційним аргументом .__init__() завжди є змінна, що посилається на екземпляр класу. Ця змінна майже завжди називається `self`. Після аргументу `self` можна вказати будь-які інші аргументи, необхідні для створення екземпляра класу.

Наступне оновлене визначення класу `Dog` показує, як написати метод .__init__(), що створює два атрибути екземпляра: `.name` і `.age`:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age
```

Зверніть увагу: сигнатура функції — частина, що починається з ключового слова `def` — має відступ у чотири пробіли. Тіло функції має відступ у вісім пробілів. Цей відступ надзвичайно важливий. Він повідомляє Python, що метод .__init__() належить класу `Dog`.

> **Примітка**
>
> Функції, що належать класу, називаються методами екземпляра (instance methods), оскільки вони належать до екземпляра класу. Наприклад, list.append() і string.find() є методами екземпляра.

Тоді як атрибути екземпляра є специфічними для кожного об'єкта, атрибути класу (class attributes) є однаковими для всіх екземплярів. У наступному прикладі створюється атрибут класу `species` зі значенням `"Canis familiaris"`:

```python
class Dog:
    # Атрибут класу
    species = "Canis familiaris"

    def __init__(self, name, age):
        self.name = name
        self.age = age
```

Атрибути класу визначаються безпосередньо під першим рядком класу і поза межами будь-якого визначення методу.

Атрибути класу слід використовувати, коли властивість повинна мати однакове початкове значення для всіх екземплярів класу. Атрибути екземпляра використовуються для властивостей, які необхідно вказати до створення екземпляра.

## 10.2 Створення екземпляра об'єкта

Після визначення класу у вас є план для створення — також відомого як інстанціювання (instantiating) — нових об'єктів. Щоб інстанціювати об'єкт, введіть ім'я класу в оригінальному CamelCase, після чого — дужки, що містять будь-які значення, які потрібно передати методу .__init__() класу.

Розгляньмо практичний приклад. Відкрийте інтерактивне вікно IDLE і введіть таке:

```python
>>> class Dog:
...     pass
...
```

Це створює новий клас `Dog` без атрибутів і методів. Тепер інстанціюємо новий об'єкт `Dog`:

```python
>>> Dog()
<__main__.Dog object at 0x106702d30>
```

Виведення вказує, що тепер є новий об'єкт `Dog` за адресою пам'яті `0x106702d30`. Зверніть увагу: адреса на вашому екрані, найімовірніше, відрізнятиметься.

Два нові об'єкти `Dog`, присвоєні змінним `a` і `b`, є різними об'єктами в пам'яті:

```python
>>> a = Dog()
>>> b = Dog()
>>> a == b
False
```

Для визначених користувачем класів стандартна поведінка оператора `==` — порівнювати адреси пам'яті двох об'єктів.

Визначити клас об'єкта можна за допомогою функції type():

```python
>>> type(a)
<class '__main__.Dog'>
```

### Атрибути класу та екземпляра

Розгляньмо трохи складніший приклад із класом `Dog`, що має атрибути екземпляра `.name` і `.age`:

```python
>>> class Dog:
...     species = "Canis familiaris"
...     def __init__(self, name, age):
...         self.name = name
...         self.age = age
...
>>> buddy = Dog("Buddy", 9)
>>> miles = Dog("Miles", 4)
```

Після оголошення нового класу `Dog` створюються два нові екземпляри — собака на ім'я Buddy, якому дев'ять років, і собака на ім'я Miles, якому чотири роки.

Метод .__init__() має три параметри, але чому в прикладі передаються лише два аргументи? Коли ви інстанціюєте об'єкт `Dog`, Python створює новий екземпляр і передає його першому параметру .__init__(). Це фактично видаляє параметр `self`, тому вам потрібно турбуватися лише про параметри `name` і `age`.

Після створення екземплярів `Dog` можна отримати доступ до їхніх атрибутів за допомогою нотації з крапкою:

```python
>>> buddy.name
'Buddy'
>>> buddy.age
9
>>> miles.name
'Miles'
>>> miles.age
4
>>> buddy.species
'Canis familiaris'
```

Однією з найбільших переваг використання класів для організації даних є те, що екземпляри гарантовано матимуть очікувані атрибути:

```python
>>> buddy.species == miles.species
True
```

Атрибути екземпляра та класу можна змінювати динамічно:

```python
>>> buddy.age = 10
>>> miles.species = "Felis silvestris"
```

Важливий висновок: користувацькі об'єкти є змінними за замовчуванням.

### Методи екземпляра

Методи екземпляра — це функції, визначені всередині класу. Вони існують лише в контексті самого об'єкта і не можуть бути викликані без посилання на об'єкт. Так само як і .__init__(), першим аргументом методу екземпляра завжди є `self`:

```python
class Dog:
    species = "Canis familiaris"

    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Метод екземпляра
    def description(self):
        return f"{self.name} is {self.age} years old"

    # Ще один метод екземпляра
    def speak(self, sound):
        return f"{self.name} says {sound}"
```

У цьому прикладі визначаються два нових методи екземпляра:

1. .description() повертає рядок із іменем і віком собаки.
2. .speak() має один параметр `sound` і повертає рядок із іменем собаки та звуком, який вона видає.

Подивімося, як методи екземпляра працюють на практиці:

```python
>>> miles = Dog("Miles", 4)
>>> miles.description()
'Miles is 4 years old'
>>> miles.speak("Woof Woof")
'Miles says Woof Woof'
>>> miles.speak("Bow Wow")
'Miles says Bow Wow'
```

При написанні власних класів рекомендується мати метод, що повертає рядок із корисною інформацією про екземпляр класу. Однак .description() — не найбільш Pythonic-спосіб це зробити.

Подивімося, що відбудеться при виклику print() для об'єкта `miles`:

```python
>>> print(miles)
<__main__.Dog object at 0x00aeff70>
```

Повідомлення, що відображається, не дуже інформативне. Змінити те, що виводиться, можна, визначивши спеціальний метод екземпляра .__str__().

Замінімо .description() на .__str__() в класі `Dog`:

```python
class Dog:
    # Інші частини класу Dog залишаються без змін

    # Замінюємо .description() на __str__()
    def __str__(self):
        return f"{self.name} is {self.age} years old"
```

Тепер при виклику `print(miles)` отримуємо значно зручніший результат:

```python
>>> miles = Dog("Miles", 4)
>>> print(miles)
Miles is 4 years old
```

> **Примітка**
>
> Методи на кшталт .__str__() зазвичай називають dunder-методами, оскільки вони починаються і закінчуються подвійними підкресленнями. Є низка дunder-методів, що дозволяють вашим класам добре взаємодіяти з іншими можливостями мови Python. Dunder-методи є потужними та важливою частиною опанування ООП у Python.

## 10.3 Успадкування від інших класів

Успадкування (inheritance) — це процес, за яким один клас бере атрибути та методи іншого. Щойно утворені класи називаються дочірніми класами (child classes), а класи, від яких вони похідні, — батьківськими класами (parent classes).

Дочірні класи можуть перевизначати та розширювати атрибути і методи батьківських класів. Іншими словами, дочірні класи успадковують усі атрибути та методи батьківського класу, але також можуть вказувати власні атрибути та методи або навіть перевизначати методи батьківського класу.

Концепцію успадкування об'єктів можна уявити схожою на генетичне успадкування, хоча ця аналогія й не є бездоганною.

### Клас object

Найбільш базовим типом класу є `object`, від якого загалом успадковують всі інші класи. При визначенні нового класу Python 3 неявно використовує `object` як батьківський клас, тому такі два визначення є еквівалентними:

```python
class Dog(object):
    pass

# У Python 3 це те саме, що й:
class Dog:
    pass
```

> **Примітка**
>
> У Python 2 є відмінність між класами нового і старого стилю. У Python 3 є клас `object`, від якого успадковують усі класи, хоча це не потрібно явно вказувати у своєму коді.

### Приклад: собачий майданчик

Уявімо, що ви на собачому майданчику. Там є багато собак різних порід, що демонструють різну поведінку.

Припустимо, ви хочете змоделювати собачий майданчик за допомогою класів Python. Клас `Dog` зі попереднього розділу може розрізняти собак за іменем і віком, але не за породою.

Можна змінити клас `Dog`, додавши атрибут `.breed`:

```python
class Dog:
    species = "Canis familiaris"

    def __init__(self, name, age, breed):
        self.name = name
        self.age = age
        self.breed = breed
```

Тепер можна інстанціювати різних собак:

```python
>>> miles = Dog("Miles", 4, "Jack Russell Terrier")
>>> buddy = Dog("Buddy", 9, "Dachshund")
>>> jack = Dog("Jack", 3, "Bulldog")
>>> jim = Dog("Jim", 5, "Bulldog")
```

Кожна порода собак має дещо іншу поведінку. Наприклад, бульдоги гавкають низьким «гав», а таксонди — вищим «гав-гав».

Використовуючи лише клас `Dog`, потрібно вказувати рядок для аргументу `sound` методу .speak() щоразу при його виклику:

```python
>>> buddy.speak("Yap")
'Buddy says Yap'
>>> jim.speak("Woof")
'Jim says Woof'
```

Передавання рядка при кожному виклику .speak() є повторюваним і незручним. Можна спростити роботу з класом `Dog`, створивши дочірній клас для кожної породи.

### Батьківські та дочірні класи

Для довідки — повне визначення класу `Dog`:

```python
class Dog:
    species = "Canis familiaris"

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __str__(self):
        return f"{self.name} is {self.age} years old"

    def speak(self, sound):
        return f"{self.name} says {sound}"
```

Щоб створити дочірній клас, потрібно створити новий клас із власним іменем, а потім помістити ім'я батьківського класу в дужки. Наступний код створює три нових дочірніх класи класу `Dog`:

```python
class JackRussellTerrier(Dog):
    pass

class Dachshund(Dog):
    pass

class Bulldog(Dog):
    pass
```

Після визначення дочірніх класів можна інстанціювати собак конкретних порід:

```python
>>> miles = JackRussellTerrier("Miles", 4)
>>> buddy = Dachshund("Buddy", 9)
>>> jack = Bulldog("Jack", 3)
>>> jim = Bulldog("Jim", 5)
```

Екземпляри дочірніх класів успадковують усі атрибути та методи батьківського класу:

```python
>>> miles.species
'Canis familiaris'
>>> buddy.name
'Buddy'
>>> print(jack)
Jack is 3 years old
>>> jim.speak("Woof")
'Jim says Woof'
```

Щоб визначити, якому класу належить певний об'єкт, використовується вбудована функція type():

```python
>>> type(miles)
<class '__main__.JackRussellTerrier'>
```

Щоб визначити, чи є `miles` також екземпляром класу `Dog`, використовується вбудована функція isinstance():

```python
>>> isinstance(miles, Dog)
True
>>> isinstance(miles, Bulldog)
False
>>> isinstance(jack, Dachshund)
False
```

Загалом усі об'єкти, створені з дочірнього класу, є екземплярами батьківського класу, хоча можуть не бути екземплярами інших дочірніх класів.

### Розширення функціональності батьківського класу

Всі три дочірніх класи успадковують кожен атрибут і метод від батьківського класу, включно з методом .speak(). Оскільки різні породи собак гавкають по-різному, хочеться надати значення за замовчуванням для аргументу `sound` їхніх відповідних методів .speak().

Для цього потрібно перевизначити метод .speak() у визначенні кожного дочірнього класу. Щоб перевизначити метод, визначений у батьківському класі, треба визначити метод із тим самим іменем у дочірньому класі:

```python
class JackRussellTerrier(Dog):
    def speak(self, sound="Arf"):
        return f"{self.name} says {sound}"
```

Тепер можна викликати .speak() для екземпляра `JackRussellTerrier` без передавання аргументу:

```python
>>> miles = JackRussellTerrier("Miles", 4)
>>> miles.speak()
'Miles says Arf'
>>> miles.speak("Grrr")
'Miles says Grrr'
```

Однією з переваг успадкування класів є те, що зміни в батьківському класі автоматично поширюються на дочірні, якщо відповідний атрибут або метод не перевизначений у дочірньому класі.

Наприклад, якщо змінити рядок, який повертає .speak() у класі `Dog`:

```python
class Dog:
    # Інші атрибути та методи опущено...
    def speak(self, sound):
        return f"{self.name} barks: {sound}"
```

Тепер `jim.speak("Woof")` поверне `'Jim barks: Woof'` замість `'Jim says Woof'`. Однак виклик .speak() для екземпляра `JackRussellTerrier` не відобразить нового формату виведення, оскільки метод перевизначено.

Іноді повне перевизначення методу батьківського класу є доцільним. Але в цьому випадку ми не хочемо, щоб клас `JackRussellTerrier` втрачав будь-які зміни у форматуванні рядка виведення `Dog.speak()`.

Для цього можна отримати доступ до батьківського класу зсередини методу дочірнього класу за допомогою функції super():

```python
class JackRussellTerrier(Dog):
    def speak(self, sound="Arf"):
        return super().speak(sound)
```

Коли викликається `super().speak(sound)` всередині `JackRussellTerrier`, Python шукає метод .speak() у батьківському класі `Dog` і викликає його зі змінною `sound`. Тепер при виклику `miles.speak()` відображається результат із новим форматуванням класу `Dog`:

```python
>>> miles = JackRussellTerrier("Miles", 4)
>>> miles.speak()
'Miles barks: Arf'
```

> **Важливо**
>
> Функція super() робить значно більше, ніж просто шукає метод або атрибут у батьківському класі. Вона проходить по всій ієрархії класів у пошуку відповідного методу або атрибута. Якщо бути неуважним, super() може дати несподівані результати.

## 10.5 Підсумок та додаткові ресурси

У цьому розділі ви дізналися про об'єктно-орієнтоване програмування (ООП) у Python — парадигму програмування, що не є специфічною для Python. Більшість сучасних мов програмування — таких як Java, C# і C++ — дотримуються принципів ООП.

Ви побачили, як визначати клас, що є своєрідним «планом» для об'єкта, і як інстанціювати об'єкт із класу. Також ви дізналися про атрибути, що відповідають властивостям об'єкта, і методи, що відповідають поведінкам та діям об'єкта.

Нарешті, ви дізналися про роботу успадкування шляхом створення дочірніх класів від батьківського. Ви побачили, як посилатися на метод батьківського класу за допомогою super() і як перевіряти, чи успадковує об'єкт від якогось класу, за допомогою isinstance().

ООП — це велика і часом складна тема. Деякі програмісти вважають ООП фундаментальною частиною сучасного програмування, але ця точка зору не позбавлена критики.
