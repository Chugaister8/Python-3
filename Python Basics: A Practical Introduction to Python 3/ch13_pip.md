# Розділ 13. Встановлення пакетів за допомогою pip

На цьому етапі ви працювали в межах стандартної бібліотеки Python. У другій половині цього курсу ви будете працювати з різними пакетами, що не входять до Python за замовчуванням.

Багато мов програмування пропонують менеджер пакетів, що автоматизує процес встановлення, оновлення та видалення сторонніх пакетів. Python — не виняток.

Де-факто менеджером пакетів для Python є pip. Історично pip потрібно було завантажувати та встановлювати окремо від Python. Починаючи з Python 3.4, він входить до більшості дистрибутивів мови.

У цьому розділі ви навчитеся:

- встановлювати сторонні пакети та керувати ними за допомогою pip;
- розуміти переваги та ризики використання сторонніх пакетів.

Вперед!

## 13.1 Встановлення сторонніх пакетів за допомогою pip

Менеджер пакетів Python pip використовується для встановлення сторонніх пакетів і керування ними. Це окрема програма від Python, хоча pip, найімовірніше, було встановлено на вашому комп'ютері разом із Python.

pip — це інструмент командного рядка, тобто його потрібно запускати з командного рядка або термінального додатку.

**Windows:** натисніть клавішу Windows, введіть `cmd` і натисніть `Enter`, щоб відкрити командний рядок. Як альтернативу можна використати PowerShell.

**macOS:** натисніть `Cmd+Пробіл`, щоб відкрити пошук Spotlight. Введіть `terminal` і натисніть `Enter`.

**Ubuntu Linux:** натисніть кнопку «Показати програми», знайдіть «термінал» і натисніть на значок застосунку Terminal.

Відкривши термінальний додаток, введіть таку команду, щоб перевірити, чи встановлено pip:

```
$ python3 -m pip --version
```

Якщо pip встановлено, ви побачите щось на кшталт:

```
pip 19.3.1 from c:\users\David\appdata\local\programs\python\
python38-32\lib\site-packages\pip (python 3.8)
```

> **Важливо**
>
> На macOS та Ubuntu Linux важливо запускати команди pip, використовуючи команду `python3`, а не `python`. Це гарантує, що використовується версія pip для Python 3, а не Python 2.
>
> На Windows команда `python3` може не працювати. Якщо ви не бачите жодного виведення або отримуєте помилку, спробуйте команду з `python`:
>
> ```
> $ python -m pip --version
> ```
>
> Якщо це спрацювало, замінюйте всі команди `python3` у цьому розділі на `python`.

Якщо ваша операційна система повідомляє, що `pip3` — нерозпізнана команда, то pip не встановлено разом із вашим дистрибутивом Python. У такому випадку перегляньте інструкції з встановлення Python у Розділі 2.

### Оновлення pip до останньої версії

Перш ніж рухатися далі, переконайтеся, що у вас встановлена остання версія pip. Щоб оновити pip, введіть таке у термінал і натисніть `Enter`:

```
$ python3 -m pip install --upgrade pip
```

Якщо доступна новіша версія pip, вона буде завантажена та встановлена. Інакше ви побачите повідомлення:

```
Requirement already satisfied.
```

### Перегляд усіх встановлених пакетів

Введіть таку команду у термінал:

```
$ python3 -m pip list
```

Якщо ви ще не встановлювали жодних пакетів, ви побачите щось на кшталт:

```
Package    Version
---------- ------
pip        19.3.1
setuptools 41.2.0
```

Ви бачите pip у списку, тому що pip сам є пакетом. Також може відображатися setuptools — пакет, що використовується pip для налаштування та встановлення інших пакетів.

### Встановлення пакету

Встановімо пакет requests — один із найпопулярніших Python-пакетів. У терміналі введіть:

```
$ python3 -m pip install requests
```

Поки pip встановлює requests, ви побачите такий результат:

```
Collecting requests
Downloading https://.../requests-2.22.0-py2.py3-none-any.whl (57kB)
...
Installing collected packages: urllib3, certifi, chardet, idna, requests
Successfully installed certifi-2019.11.28 chardet-3.0.4 idna-2.8
requests-2.22.0 urllib3-1.25.7
```

Зверніть увагу: pip встановлює ще чотири пакети: `chardet`, `certifi`, `idna` і `urllib3`. Це залежності requests — пакети, що потрібні requests для правильної роботи.

Після встановлення знову запустіть `pip list`:

```
Package    Version
---------- ---------
certifi    2019.11.28
chardet    3.0.4
idna       2.8
pip        19.3.1
requests   2.22.0
setuptools 41.2.0
urllib3    1.25.7
```

### Встановлення конкретних версій пакетів за допомогою специфікаторів версій

За замовчуванням pip встановлює останню версію пакету. Можна контролювати, яка версія встановлюється, за допомогою специфікаторів версій:

| Специфікатор версії | Опис |
|--------------------|------|
| `<=`, `>=` | Включаючий менший або більший специфікатор |
| `<`, `>` | Виключаючий менший або більший специфікатор |
| `==` | Специфікатор точної рівності |

Приклади:

```
# Встановити останню версію 2.0 або вище
$ python3 -m pip install requests>=2.0

# Встановити останню версію не вище 3.0
$ python3 -m pip install requests<=3.0

# Встановити останню версію строго нижче 3.0
$ python3 -m pip install requests<3.0

# Встановити останню версію в діапазоні 1.x
$ python3 -m pip install requests>=1.0,<2.0

# Встановити точну версію 2.22.0
$ python3 -m pip install requests==2.22.0
```

### Перегляд деталей пакету

Після встановлення requests можна переглянути деякі відомості про пакет:

```
$ python3 -m pip show requests
Name: requests
Version: 2.22.0
Summary: Python HTTP for Humans.
Home-page: http://python-requests.org
Author: Kenneth Reitz
Author-email: me@kennethreitz.org
License: Apache 2.0
Location: c:\users\David\...\python\python38-32\lib\site-packages
Requires: chardet, idna, certifi, urllib3
Required-by:
```

Пакет requests використовується для виконання HTTP-запитів із Python-програми. Він надзвичайно корисний у різних сферах і є залежністю великої кількості інших Python-пакетів.

### Видалення пакету

Щоб видалити requests, введіть таке у термінал:

```
$ python3 -m pip uninstall requests
```

Перш ніж щось видалити, pip запитає підтвердження:

```
Uninstalling requests-2.22.0:
  Would remove:
    c:\users\...\requests-2.22.0.dist-info\*
    c:\users\...\requests\*
Proceed (y/n)?
```

Введіть `y` і натисніть `Enter`. Ви побачите підтвердження:

```
Successfully uninstalled requests-2.22.0
```

> **Примітка**
>
> pip видалив requests, але не видалив жодну з його залежностей! Це задумана поведінка. Уявіть, що у вашому середовищі встановлено кілька пакетів зі спільними залежностями. Якби pip видаляв залежності разом із пакетом, це призвело б до непрацездатності інших пакетів, що їх використовують!

Видалити кілька пакетів одночасно можна однією командою:

```
$ python3 -m pip uninstall certifi chardet idna urllib3
```

## 13.2 Підводні камені сторонніх пакетів

Краса сторонніх пакетів у тому, що вони дозволяють додавати функціональність до вашого проєкту без необхідності реалізовувати все з нуля. Це дає колосальне підвищення продуктивності.

Але з великими можливостями приходить велика відповідальність. Щойно ви включаєте чужий пакет у свій проєкт, ви надаєте величезний кредит довіри тим, хто відповідає за його розробку та підтримку.

Використовуючи пакет, який ви не розробляли, ви втрачаєте контроль над певними аспектами свого проєкту. Зокрема, мейнтейнери пакету можуть випустити нову версію зі змінами, несумісними з тією версією, що використовується у вашому проєкті.

За замовчуванням pip встановлює останній реліз пакету, тому якщо ви поширюєте свій код комусь іншому і вони встановлюють новішу версію пакету, що потрібна вашому проєкту, вони можуть не зуміти запустити ваш код.

Ця проблема є суттєвим викликом — і для кінцевого користувача, і для вас. На щастя, Python пропонує рішення цієї поширеної проблеми: **віртуальні середовища** (virtual environments).

Віртуальне середовище створює ізольоване та, що найважливіше, відтворюване середовище для розробки проєкту. Середовище може містити конкретну версію Python, а також конкретні версії залежностей вашого проєкту.

Коли ви поширюєте свій код комусь іншому, вони можуть відтворити це середовище та бути впевнені, що зможуть запустити ваш код без помилок.

Віртуальні середовища є більш просунутою темою, що виходить за межі цієї книги. Щоб дізнатися більше про них, перегляньте курс Real Python «Managing Python Dependencies With Pip and Virtual Environments». У ньому ви навчитеся:

- встановлювати, використовувати та керувати сторонніми Python-пакетами за допомогою менеджера пакетів pip на Windows, macOS та Linux;
- ізолювати залежності проєкту за допомогою так званих віртуальних середовищ для уникнення конфліктів версій;
- застосовувати повний 7-крокових процес пошуку та визначення якісних сторонніх пакетів для використання у власних проєктах;
- налаштовувати відтворювані середовища розробки та деплой застосунків за допомогою менеджера пакетів pip і файлів requirements.

## 13.3 Підсумок та додаткові ресурси

У цьому розділі ви навчилися встановлювати сторонні пакети за допомогою менеджера пакетів Python pip. Ви ознайомилися з кількома корисними командами pip: `pip install`, `pip list`, `pip show` і `pip uninstall`.

Ви також дізналися про деякі підводні камені, пов'язані зі сторонніми пакетами. Не кожен пакет, що можна завантажити за допомогою pip, є хорошим вибором для вашого проєкту. Оскільки ви не контролюєте код у встановленому пакеті, потрібно довіряти, що пакет є безпечним і буде добре працювати для користувачів вашої програми.
