# Розділ 11. Модулі та пакети

У міру накопичення досвіду написання коду ви неминуче працюватимете над проєктами, настільки великими, що зберігати весь код в одному файлі стає незручно.

Замість одного файлу можна розміщувати пов'язаний код в окремих файлах, що називаються модулями. Окремі модулі можна складати разом, наче будівельні блоки, для створення більшого застосунку.

Є чотири основні переваги поділу програми на модулі:

1. **Простота:** модулі зосереджені на одній задачі.
2. **Зручність підтримки:** невеликі файли кращі за великі.
3. **Повторне використання:** модулі зменшують дублювання коду.
4. **Scoping:** модулі мають власні namespace.

У цьому розділі ви навчитеся:

- створювати власні модулі;
- використовувати модулі в іншому файлі за допомогою оператора `import`;
- організовувати кілька модулів у пакет.

Розпочнемо!

## 11.1 Робота з модулями

Модуль — це файл із кодом Python, що може повторно використовуватися в інших файлах коду Python.

Технічно кожен файл скрипту Python, що ви створювали під час читання цієї книги, є модулем, але ви ще не бачили, як використовувати код одного модуля в іншому.

У цьому розділі ви детальніше дослідите модулі. Ви навчитеся створювати їх за допомогою IDLE, імпортувати один модуль в інший і зрозумієте, як модулі створюють namespace.

### Створення модулів

Відкрийте IDLE і почніть нове вікно скрипту, вибравши Файл → Новий файл (File → New File) або натиснувши `Ctrl+N`. У вікні скрипту визначте функцію add(), що повертає суму двох параметрів:

```python
# adder.py
def add(x, y):
    return x + y
```

Збережіть файл як `adder.py` у новій директорії `myproject/` десь на вашому комп'ютері. `adder.py` — це Python-модуль! Це не завершена програма, але не всі модулі нею мають бути.

Тепер відкрийте ще одне нове вікно скрипту та введіть такий код:

```python
# main.py
value = add(2, 2)
print(value)
```

Збережіть файл як `main.py` у тій самій папці `myproject/`. Потім натисніть `F5`, щоб запустити модуль.

При запуску ви побачите `NameError`, оскільки функція add() визначена в `adder.py`, а не в `main.py`. Щоб використовувати add() у `main.py`, спочатку потрібно імпортувати модуль `adder`.

### Імпортування одного модуля в інший

У вікні скрипту для `main.py` додайте такий рядок на початок файлу:

```python
# main.py
import adder               # <-- Додайте цей рядок

# Залиште код нижче без змін
value = add(2, 2)
print(value)
```

Коли ви імпортуєте один модуль в інший, вміст імпортованого модуля стає доступним в іншому. Модуль із оператором `import` називається викликаючим модулем (calling module).

`NameError` все ще виникатиме, тому що add() можна отримати лише з namespace `adder`.

**Namespace** — це колекція імен, таких як імена змінних, функцій і класів. Кожен Python-модуль має власний namespace.

Щоб отримати доступ до імені в імпортованому модулі з викликаючого, введіть ім'я імпортованого модуля, після нього крапку (`.`) і потрібне ім'я:

```
<модуль>.<ім'я>
```

Наприклад, щоб використати функцію add() з модуля `adder`, потрібно написати `adder.add()`.

> **Важливо**
>
> Ім'я модуля при імпорті збігається з назвою файлу модуля. З цієї причини назви файлів модулів мають бути допустимими Python-ідентифікаторами: лише великі та малі літери, цифри та підкреслення, і не починатися з цифри.

Оновіть код у `main.py`:

```python
# main.py
import adder
value = adder.add(2, 2)    # <-- Змініть цей рядок
print(value)
```

Збережіть файл і запустіть модуль. У інтерактивному вікні відобразиться значення `4`.

Коли ви пишете `import <модуль>` на початку файлу, імпортується весь namespace модуля. Будь-які нові змінні або функції, додані до `adder.py`, будуть доступні в `main.py` без необхідності імпортувати щось нове.

Додамо функцію `double()` до `adder.py`:

```python
# adder.py

# Залиште цей код без змін
def add(x, y):
    return x + y

def double(x):              # <-- Додайте цю функцію
    return x + x
```

А в `main.py`:

```python
# main.py
import adder
value = adder.add(2, 2)
double_value = adder.double(value)   # <-- Додайте цей рядок
print(double_value)                  # <-- Змініть цей рядок
```

Збережіть і запустіть `main.py`. При запуску модуля в інтерактивному вікні відобразиться значення `8`.

### Варіанти оператора import

Оператор `import` є гнучким. Є два варіанти, які варто знати:

1. `import <модуль> as <інша_назва>`
2. `from <модуль> import <ім'я>`

#### import \<модуль\> as \<інша_назва\>

Можна змінити ім'я імпорту за допомогою ключового слова `as`:

```python
import adder as a
value = a.add(2, 2)
double_value = a.double(value)
print(double_value)
```

При такому імпорті namespace модуля доступний через `a`, а не через `adder`.

Є дві причини використовувати цей формат:
1. Ім'я модуля довге і ви хочете імпортувати його скорочену версію.
2. Ім'я модуля конфліктує з наявним іменем у викликаючому модулі.

#### from \<модуль\> import \<ім'я\>

Замість імпортування всього namespace можна імпортувати лише певне ім'я з модуля:

```python
from adder import add, double

value = add(2, 2)
double_value = double(value)
print(double_value)
```

Імпортовані імена додаються безпосередньо до локального namespace викликаючого модуля.

### Зведена таблиця операторів import

| Оператор import | Результат |
|----------------|-----------|
| `import <модуль>` | Імпортує весь namespace `<модуля>` під ім'ям `<модуль>`. Доступ через `<модуль>.<ім'я>`. |
| `import <модуль> as <інша_назва>` | Імпортує весь namespace `<модуля>` під ім'ям `<інша_назва>`. Доступ через `<інша_назва>.<ім'я>`. |
| `from <модуль> import <ім'я1>, <ім'я2>, ...` | Імпортує лише `<ім'я1>`, `<ім'я2>` тощо з `<модуля>`. Імена додаються до локального namespace і доступні безпосередньо. |

### Навіщо потрібні namespace?

Уявіть, що кожній людині на планеті присвоюється ID-номер. Щоб розрізняти людей, кожен ID-номер має бути унікальним.

Світ поділений на країни. Якщо кожній країні присвоїти унікальний код і включити його до ID-номера особи, то дві людини з різних країн можуть мати однаковий ID-номер: наприклад, `US-357` і `GB-246`.

Коди країн у цьому сценарії — це приклади namespace. Вони ілюструють три основні причини використання namespace:

1. Вони групують імена в логічні контейнери.
2. Вони запобігають конфліктам між однаковими іменами.
3. Вони надають контекст іменам.

Загалом `import <модуль>` є кращим способом імпортування, оскільки він зберігає namespace імпортованого модуля повністю відокремленим від namespace викликаючого.

Наприклад, стандартна бібліотека Python має модуль `datetime`, що містить клас `datetime`. Щоб уникнути повторення `datetime.datetime`, програмісти часто імпортують з псевдонімом:

```python
import datetime as dt
dt.datetime(2020, 2, 2)
```

Або імпортують клас безпосередньо:

```python
from datetime import datetime
datetime(2020, 2, 2)
```

Завжди використовуйте здоровий глузд при імпорті модулів, щоб максимально зберегти контекст.

## 11.2 Робота з пакетами

Модулі дозволяють розділити програму на окремі файли, які можна повторно використовувати за потреби. Пов'язаний код можна організувати в один модуль і зберігати окремо від іншого коду.

Пакети роблять цю організаційну структуру ще на крок далі, дозволяючи групувати пов'язані модулі під одним namespace.

### Створення пакетів

Пакет — це папка, що містить один або кілька Python-модулів. Вона також повинна містити спеціальний модуль `__init__.py`.

Модуль `__init__.py` не обов'язково має містити будь-який код! Він лише має існувати, щоб Python розпізнав папку `mypackage/` як Python-пакет.

Використовуючи файловий провідник свого комп'ютера, створіть нову папку `packages_example/`. Всередині неї створіть ще одну папку `mypackage/`.

Папка `packages_example/` називається папкою проєкту (project folder), або кореневою папкою проєкту. Папка `mypackage/` з часом стане Python-пакетом.

Відкрийте IDLE і створіть такі файли:
- `main.py` — у папці `packages_example/`
- `__init__.py` — у папці `mypackage/`
- `module1.py` — у папці `mypackage/`
- `module2.py` — у папці `mypackage/`

У файлі `module1.py` додайте таку функцію:

```python
# module1.py
def greet(name):
    print(f"Hello, {name}!")
```

У файлі `module2.py` додайте таке:

```python
# module2.py
def depart(name):
    print(f"Goodbye, {name}!")
```

### Імпортування модулів із пакетів

У файлі `main.py` спочатку спробуємо:

```python
# main.py
import mypackage

mypackage.module1.greet("Pythonista")
mypackage.module2.depart("Pythonista")
```

При запуску виникне `AttributeError`: при імпорті `mypackage` namespace `module1` і `module2` не імпортуються автоматично. Потрібно імпортувати їх окремо:

```python
# main.py
import mypackage.module1
import mypackage.module2

mypackage.module1.greet("Pythonista")
mypackage.module2.depart("Pythonista")
```

Тепер при збереженні та запуску `main.py` відображається:

```
Hello, Pythonista!
Goodbye, Pythonista!
```

Загалом модулі імпортуються з пакетів за допомогою імен модулів із крапками:

```python
import <назва_пакета>.<назва_модуля>
```

> **Важливо**
>
> Так само як назви файлів модулів, назви папок пакетів мають бути допустимими Python-ідентифікаторами.

### Варіанти оператора import для пакетів

Є чотири варіанти оператора `import` для імпортування модулів із пакетів:

1. `import <пакет>`
2. `import <пакет> as <інша_назва>`
3. `from <пакет> import <модуль>`
4. `from <пакет> import <модуль> as <інша_назва>`

Наприклад, замість окремого імпортування `mypackage.module1` і `mypackage.module2` можна імпортувати обидва модулі з пакета в одному рядку:

```python
# main.py
from mypackage import module1, module2

module1.greet("Pythonista")
module2.depart("Pythonista")
```

Також можна перейменувати:

```python
# main.py
from mypackage import module1 as m1, module2 as m2

m1.greet("Pythonista")
m2.depart("Pythonista")
```

Або імпортувати окремі імена:

```python
# main.py
from mypackage.module1 import greet
from mypackage.module2 import depart

greet("Pythonista")
depart("Pythonista")
```

### Рекомендації щодо імпортування пакетів

Загалом найбільш явним форматом є:

```python
import <пакет>.<модуль>
```

Доступ до імен — через `<пакет>.<модуль>.<ім'я>`. Коли ви зустрічаєте ім'я в коді, немає жодних сумнівів щодо його походження.

Наступний формат дозволяє пропустити назву пакета:

```python
from <пакет> import <модуль>
```

Доступ — через `<модуль>.<ім'я>`. Втрачається контекст пакета, але зберігається контекст модуля.

Нарешті, такий формат загалом неоднозначний і варто використовувати його лише коли немає ризику конфлікту імен:

```python
from <пакет>.<модуль> import <ім'я>
```

### Імпортування модулів із підпакетів

Пакет — це просто папка з одним або кількома Python-модулями, один із яких має називатися `__init__.py`. Тому цілком можливо мати таку структуру пакета:

```
packages_example/
│
├── main.py
│
└── mypackage/
    ├── __init__.py
    ├── module1.py
    ├── module2.py
    │
    └── mysubpackage/
        ├── __init__.py
        └── module3.py
```

Пакет, вкладений в інший пакет, називається підпакетом (subpackage).

Додамо до `module3.py`:

```python
# module3.py
people = ["John", "Paul", "George", "Ringo"]
```

Тепер відкрийте файл `main.py` і замініть наявний код таким:

```python
# main.py
from mypackage.module1 import greet
from mypackage.mysubpackage.module3 import people

for person in people:
    greet(person)
```

При збереженні та запуску `main.py` відображається:

```
Hello, John!
Hello, Paul!
Hello, George!
Hello, Ringo!
```

Підпакети чудово підходять для організації коду у великих пакетах. Однак глибоко вкладені підпакети призводять до довгих імен модулів із крапками. Рекомендується обмежуватися максимум одним-двома рівнями підпакетів.

## 11.3 Підсумок та додаткові ресурси

У цьому розділі ви навчилися створювати власні Python-модулі та пакети, а також імпортувати об'єкти з одного модуля в інший.

Ви побачили, що поділ коду на модулі та пакети є вигідним, оскільки:

- невеликі файли коду простіші за великі;
- невеликі файли коду зручніші в підтримці, ніж великі;
- модулі можна повторно використовувати по всьому проєкту;
- модулі групують пов'язані об'єкти в ізольовані namespace.
