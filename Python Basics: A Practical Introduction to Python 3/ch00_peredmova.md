## Передмова

Привіт і ласкаво просимо до книги Python Basics: Практичний вступ до Python 3. Сподіваюся, ви готові дізнатися, чому стільки професійних та аматорських розробників тяжіють до Python і як ви можете одразу почати використовувати його у своїх проєктах — малих і великих.

Ця книга розрахована на початківців: як на тих, хто вже трохи знайомий із програмуванням, але не знає мову Python та її екосистему, так і на повних новачків.

Якщо у вас немає диплому з інформатики — не хвилюйтеся. Fletcher, David, Dan і Joanna проведуть вас через важливі концепції програмування, одночасно навчаючи основам Python і, що не менш важливо, пропускаючи на перших порах зайві подробиці.

### Python — це мова повного спектру

Вивчаючи нову мову програмування, ви ще не маєте досвіду, щоб оцінити, наскільки добре вона прислужиться вам у довгостроковій перспективі. Якщо ви розглядаєте Python, запевняю вас: це хороший вибір. Одна з ключових причин полягає в тому, що Python — це мова повного спектру.

Що я маю на увазі? Одні мови дуже добре підходять для початківців. Вони ведуть вас за руку і роблять програмування надзвичайно простим. Якщо взяти крайній приклад, то можна звернутися до візуальних мов, таких як Scratch. Тут ви отримуєте блоки, що представляють концепції програмування — змінні, цикли, виклики методів тощо — і перетягуєте їх на візуальну поверхню. Scratch може бути простим для початку роботи з простими програмами. Але будувати на ньому професійні застосунки неможливо. Назвіть хоча б одну компанію зі списку Fortune 500, яка будує свою основну бізнес-логіку на Scratch.

Не можете? Я теж — тому що це було б безумством.

Інші мови надзвичайно потужні для досвідчених розробників. Найпопулярніша в цій категорії — мабуть, C++ та її близький родич C. Браузер, яким ви користувалися сьогодні, швидше за все, написаний на C або C++. Операційна система, що запускає цей браузер, також, найімовірніше, створена на C/C++. Ваш улюблений шутер від першої особи або стратегія? Так і є: C/C++.

Цими мовами можна досягти неймовірного. Але вони зовсім недружні до новачків, які шукають м'яке введення.

Можливо, ви не читали багато коду на C++. Від нього майже очі болять. Ось приклад — справжній, хоча й складний:

```cpp
template <typename T>
_Defer<void(*(PID<T>, void (T::*)(void)))
(const PID<T>&, void (T::*)(void))>
defer(const PID<T>& pid, void (T::*method)(void))
{
void (*dispatch)(const PID<T>&, void (T::*)(void)) =
&process::template dispatch<T>;
return std::tr1::bind(dispatch, pid, method);
}
```

Ні, дякую.

І Scratch, і C++ не є тим, що я назвав би мовами повного спектру. На рівні Scratch починати легко, але щоб будувати справжні застосунки, доводиться переходити на «справжню» мову. І навпаки: на C++ можна будувати справжні застосунки, але плавного входу немає — ви одразу потрапляєте в усю складність цієї мови, яка існує для підтримки таких масштабних застосунків.

Python, однак, особливий. Це мова повного спектру. Ми часто оцінюємо простоту мови за тестом «hello world»: який синтаксис і які дії потрібні, щоб вивести «hello world» для користувача? У Python простіше не буває:

```python
print("Hello world")
```

І все. Утім, я вважаю цей тест недостатньо переконливим.

Тест «hello world» корисний, але насправді недостатній, щоб продемонструвати потужність або складність мови. Спробуємо інший приклад. Не все в ньому має бути зрозумілим одразу — просто слідкуйте за ходом думки, щоб відчути його дух. Книга охоплює ці концепції та більше в міру того, як ви просуватиметеся вперед. Наступний приклад — це щось, що ви цілком могли б написати ближче до кінця.

Ось новий тест: що знадобиться, щоб написати програму, яка звертається до зовнішнього сайту, завантажує вміст у пам'ять застосунку, а потім відображає певну частину цього вмісту для користувача? Спробуймо цей експеримент у Python 3 за допомогою пакета requests (який потрібно встановити — докладніше в розділі 12):

```python
import requests
resp = requests.get("https://realpython.com")
html = resp.text
print(html[205:294])
```

Неймовірно, але це все. Під час виконання програми результат буде приблизно таким:

```
<title>Python Tutorials – Real Python</title>
<meta name="author" content="Real Python">
```

Це легкий, початковий бік спектру Python. Кілька простих рядків — і відкривається неймовірна потужність. Оскільки Python має доступ до багатьох потужних, але добре запакованих бібліотек, таких як requests, його часто описують як мову з «батарейками в комплекті» (batteries included).

Отже, перед вами простий потужний стартовий приклад. Якщо говорити про реальні застосунки, то й тут написано чимало видатних програм на Python.

YouTube — найпопулярніший у світі сервіс відеостримінгу — написаний на Python і обробляє понад 1 000 000 запитів на секунду. Instagram — ще один приклад Python-застосунку. Ближче до нас — realpython.com і мої сайти, зокрема talkpython.fm.

Ця властивість повного спектру Python означає, що ви можете починати з простого і поступово освоювати більш просунуті можливості в міру того, як ростуть вимоги до вашого застосунку.

### Python популярний

Ви, мабуть, чули, що Python популярний. З одного боку, може здатися, що популярність мови не має великого значення, якщо з її допомогою можна будувати потрібні застосунки.

Але в розробці програмного забезпечення популярність — це вагомий показник якості доступних бібліотек і кількості вакансій. Коротко кажучи, варто тяжіти до більш популярних технологій, оскільки там більше варіантів та інтеграцій.

Отже, Python справді такий популярний? Так. Звісно, можна знайти багато галасу і перебільшень. Але є чимало статистики, що це підтверджує. Подивімося на аналітику, що публікується на StackOverflow.com.

Вони ведуть сайт StackOverflow Trends. Тут можна переглядати тенденції для різних технологій за тегами. Порівнюючи Python з іншими вірогідними кандидатами для вивчення програмування, ви помітите, що одна з них — не як усі інші.

Дослідити цей графік і створити схожі можна на insights.stackoverflow.com/trends.

Зверніть увагу на неймовірне зростання Python порівняно з рівною лінією або навіть низхідною тенденцією інших звичних кандидатів! Якщо ви ставите на успіх певної технології, яку б ви обрали зі цього списку?

Це лише один графік — що він насправді говорить нам? Давайте подивимося ще один. StackOverflow щороку проводить опитування розробників. Воно всебічне і дуже добре організоване. Повні результати за 2018 рік можна знайти на insights.stackoverflow.com/survey/2018/. З цього огляду я хотів би звернути вашу увагу на розділ «Найбільш улюблені, найбільш небажані та найбільш бажані мови». У розділі «найбільш бажані» наведено відповіді:

> Розробників, які не працюють із цією мовою або технологією, але висловили зацікавленість у роботі з нею.

І знову, на графіку нижче видно, що Python очолює рейтинг і значно випереджає навіть друге місце.

Отже, якщо ви згодні зі мною, що відносна популярність мови програмування має значення — Python є очевидно хорошим вибором.

### Нам не потрібно, щоб ви були вченим-комп'ютерником

Ще один момент, який хочу підкреслити, поки ви починаєте цей шлях вивчення Python: нам не потрібно, щоб ви були вченим-комп'ютерником. Якщо це ваша мета — чудово. Вивчення Python — потужний крок у цьому напрямку. Але навчання програмуванню часто подається у вигляді «у нас незайняті вакансії розробників, нам потрібні програмісти!»

Це може бути правдою, а може й ні. Але важливіше для вас особисто те, що програмування (навіть трохи програмування) може стати для вас надзвичайною перевагою.

Щоб проілюструвати цю ідею, уявімо, що ви біолог. Чи варто вам кидати біологію і влаштовуватися фронтенд-розробником? Напевно, ні. Але такі навички, як та, з якої я починав цю передмову, — використання requests для отримання даних з вебу, — будуть неймовірно корисні для вас як біолога.

Замість того, щоб вручну експортувати й зчитувати дані з вебу або електронних таблиць, за допомогою Python ви можете обробляти тисячі джерел даних або таблиць за той час, що витрачаєте на одну вручну. Навички роботи з Python здатні підсилити вашу потужність як біолога і вивести її далеко за межі можливостей ваших колег — зробивши її вашою надзвичайною перевагою.
