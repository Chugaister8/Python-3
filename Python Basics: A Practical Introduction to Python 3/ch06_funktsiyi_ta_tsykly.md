## 6.1 Що таке функція насправді?

У попередніх кількох розділах ви використовували такі функції, як print() і len(), для відображення тексту та визначення довжини рядка. Але що таке функція насправді?

У цьому розділі ви детальніше розглянете len(), щоб краще зрозуміти, що являє собою функція і як вона виконується.

### Функції є значеннями

Одна з найважливіших властивостей функції в Python полягає в тому, що функції є значеннями і можуть бути присвоєні змінній.

В інтерактивному вікні IDLE перевірте ім'я `len`, ввівши таке у рядку запрошення:

```python
>>> len
<built-in function len>
```

Після натискання `Enter` Python повідомляє, що ім'я `len` — це змінна, значенням якої є вбудована функція.

Так само як значення цілих чисел мають тип `int`, а рядки — тип `str`, значення функцій також мають тип:

```python
>>> type(len)
<class 'builtin_function_or_method'>
```

Як і будь-якій іншій змінній, `len` можна присвоїти будь-яке значення:

```python
>>> len = "I'm not the len you're looking for."
>>> len
"I'm not the len you're looking for."
```

Тепер `len` має рядкове значення, і можна перевірити, що тип — `str`, за допомогою type():

```python
>>> type(len)
<class 'str'>
```

Ім'я змінної `len` є ключовим словом у Python, і хоча його значення можна змінити, зазвичай це погана ідея. Зміна значення `len` може зробити код заплутаним, оскільки нове `len` легко сплутати з вбудованою функцією.

> **Важливо**
>
> Якщо ви вводили попередні приклади коду, у вас більше немає доступу до вбудованої функції `len` в IDLE.
>
> Повернути її можна таким кодом:
>
> ```python
> >>> del len
> ```
>
> Ключове слово `del` використовується для відв'язування змінної від значення. `del` означає «видалити», але воно не видаляє значення. Натомість воно від'єднує ім'я від значення і видаляє ім'я.

Зазвичай після використання `del` спроба скористатися видаленим іменем змінної спричиняє `NameError`. Однак у цьому випадку ім'я `len` не видаляється:

```python
>>> len
<built-in function len>
```

Оскільки `len` — це ім'я вбудованої функції, воно повторно присвоюється оригінальному значенню функції.

Пройшовши через кожен із цих кроків, ми побачили, що ім'я функції є відокремленим від самої функції.

### Як Python виконує функції

Тепер детальніше подивімося на те, як Python виконує функцію.

Перше, що слід зазначити: виконати функцію, просто ввівши її ім'я, не можна. Потрібно викликати функцію, щоб наказати Python фактично її виконати.

Подивімося, як це працює з len():

```python
>>> # Введення лише імені не виконує функцію.
>>> # IDLE перевіряє змінну як зазвичай.
>>> len
<built-in function len>
>>> # Використовуйте дужки для виклику функції.
>>> len()
Traceback (most recent call last):
  File "<pyshell#3>", line 1, in <module>
    len()
TypeError: len() takes exactly one argument (0 given)
```

У цьому прикладі Python генерує `TypeError` при виклику `len()`, оскільки len() очікує аргумент.

Аргумент — це значення, що передається функції як вхідні дані. Деякі функції можна викликати без аргументів, а деякі приймають скільки завгодно аргументів. len() вимагає рівно одного аргументу.

Коли функція завершує виконання, вона повертає значення як результат. Значення, що повертається, зазвичай, але не завжди, залежить від значень будь-яких переданих аргументів.

Процес виконання функції можна описати трьома кроками:

1. Функція викликається, і будь-які аргументи передаються їй як вхідні дані.
2. Функція виконується, і з аргументами виконуються певні дії.
3. Функція повертає значення, і виклик функції замінюється значенням, що повертається.

Подивімося на практиці, як Python виконує такий рядок коду:

```python
>>> num_letters = len("four")
```

Спочатку len() викликається з аргументом `"four"`. Обчислюється довжина рядка `"four"`, що дорівнює числу `4`. Потім len() повертає число `4` і замінює виклик функції цим значенням.

Таким чином, після виконання функції рядок коду виглядає так:

```python
>>> num_letters = 4
```

Потім Python присвоює значення `4` змінній `num_letters` і продовжує виконувати рядки коду, що залишилися.

### Функції можуть мати побічні ефекти

Ви дізналися, як викликати функцію і що вони повертають значення після виконання. Однак іноді функції роблять більше, ніж просто повертають значення.

Коли функція змінює або впливає на щось зовнішнє щодо самої функції, говорять, що вона має побічний ефект (side effect). Ви вже бачили одну функцію з побічним ефектом: print().

Коли ви викликаєте print() з рядковим аргументом, рядок відображається в оболонці Python як текст. Але print() не повертає жодного тексту як значення.

Щоб побачити, що повертає print(), можна присвоїти значення, що повертається print(), змінній:

```python
>>> return_value = print("What do I return?")
What do I return?
>>> return_value
>>>
```

Коли ви присвоюєте `print("What do I return?")` змінній `return_value`, рядок `"What do I return?"` відображається. Однак при перевірці значення `return_value` нічого не відображається.

print() повертає спеціальне значення `None`, що вказує на відсутність даних. `None` має тип `NoneType`:

```python
>>> type(return_value)
<class 'NoneType'>
>>> print(return_value)
None
```

Коли ви викликаєте print(), текст, що відображається, — це не значення, що повертається. Це побічний ефект print().

Тепер, знаючи, що функції є значеннями — так само як рядки та числа — і розуміючи, як функції викликаються та виконуються, давайте розглянемо, як можна створювати власні функції.

## 6.2 Написання власних функцій

У міру написання довших і складніших програм ви можете помітити, що вам потрібно повторно використовувати кілька рядків коду. Або, можливо, потрібно кілька разів обчислити одну й ту саму формулу з різними значеннями.

Спокуса скопіювати й вставити аналогічний код в інші місця програми і підправити за потреби є, але зазвичай це погана ідея!

Повторюваний код може стати справжнім кошмаром у підтримці. Якщо ви знайдете помилку в коді, що скопійовано і вставлено скрізь, вам доведеться вносити виправлення в усі місця. Це багато роботи, і ви можете щось пропустити!

У цьому розділі ви навчитеся визначати власні функції, щоб уникати повторення коду. Вперед!

### Анатомія функції

Кожна функція складається з двох частин:

1. **Сигнатура функції** — визначає ім'я функції та будь-які вхідні дані, які вона очікує.
2. **Тіло функції** — містить код, що виконується щоразу при використанні функції.

Почнімо з написання функції, що приймає два числа як вхідні дані і повертає їхній добуток. Ось як така функція може виглядати, з позначеними в коментарях сигнатурою, тілом і оператором повернення:

```python
def multiply(x, y):  # Сигнатура функції

    # Тіло функції
    product = x * y
    return product
```

Може здатися дивним писати функцію для чогось настільки простого, як оператор `*`. Насправді `multiply` — це функція, яку ви, мабуть, не стали б писати в реальному сценарії. Але вона є чудовим першим прикладом для розуміння того, як створюються функції!

Розберімо функцію, щоб зрозуміти, що відбувається.

### Сигнатура функції

Перший рядок коду у функції називається сигнатурою функції. Він завжди починається з ключового слова `def`, що є скороченням від «define» («визначити»).

Подивімося уважніше на сигнатуру функції `multiply`:

```python
def multiply(x, y):
```

Сигнатура функції складається з чотирьох частин:

1. Ключове слово `def`
2. Ім'я функції — `multiply`
3. Список параметрів — `(x, y)`
4. Двокрапка (`:`) наприкінці рядка

Коли Python зустрічає рядок, що починається з ключового слова `def`, він створює нову функцію. Функція присвоюється змінній з тим самим іменем, що й ім'я функції.

> **Примітка**
>
> Оскільки імена функцій стають змінними, вони мають дотримуватися тих самих правил для імен змінних, що ви вивчили в Розділі 3.
>
> Таким чином, ім'я функції може містити лише числа, літери та підкреслення і не повинно починатися з числа.

Список параметрів — це список імен параметрів, укладений у відкриваючу та закриваючу дужки. Він визначає очікувані вхідні дані функції. `(x, y)` — це список параметрів функції `multiply`. Він створює два параметри: `x` і `y`.

Параметр — це дещо схоже на змінну, але без значення. Він є заповнювачем для фактичних значень, що передаються щоразу при виклику функції з одним або кількома аргументами.

Код у тілі функції може використовувати параметри так, ніби вони є змінними з реальними значеннями. Наприклад, тіло функції може містити рядок коду з виразом `x * y`.

Оскільки `x` і `y` не мають значення, `x * y` також не має значення. Python зберігає вираз як шаблон і підставляє відсутні значення під час виконання функції.

Функція може мати будь-яку кількість параметрів, включно з їхньою повною відсутністю!

### Тіло функції

Тіло функції — це код, що виконується щоразу, коли функція використовується у вашій програмі. Ось тіло функції `multiply`:

```python
def multiply(x, y):
    # Тіло функції
    product = x * y
    return product
```

`multiply` — досить проста функція. Її тіло містить лише два рядки коду!

Перший рядок створює змінну `product` і присвоює їй значення `x * y`. Оскільки `x` і `y` ще не мають значень, цей рядок є шаблоном для значення, яке буде присвоєне `product` під час виконання функції.

Другий рядок коду називається оператором повернення (return statement). Він починається з ключового слова `return`, за яким слідує змінна `product`. Коли Python досягає оператора `return`, він припиняє виконання функції і повертає значення `product`.

Зверніть увагу: обидва рядки коду в тілі функції мають відступ. Це надзвичайно важливо! Кожен рядок, що має відступ нижче сигнатури функції, розуміється як частина тіла функції.

Наприклад, функція print() у наступному прикладі не є частиною тіла функції, оскільки вона не має відступу:

```python
def multiply(x, y):
    product = x * y
    return product

print("Where am I?")  # Не в тілі функції.
```

Якщо print() має відступ, вона стає частиною тіла функції, навіть якщо між print() і попереднім рядком є порожній рядок:

```python
def multiply(x, y):
    product = x * y
    return product

    print("Where am I?")  # В тілі функції.
```

Є одне правило, якого необхідно дотримуватися при розстановці відступів у тілі функції: кожен рядок має мати однакову кількість пробілів для відступу.

Спробуйте зберегти наступний код у файл `multiply.py` і запустити його з IDLE:

```python
def multiply(x, y):
    product = x * y
     return product  # Додатковий пробіл для відступу.
```

IDLE не запустить код! З'явиться діалогове вікно з помилкою «unexpected indent». Python не очікував, що оператор `return` матиме інший відступ, ніж попередній рядок.

Інша помилка виникає, коли рядок коду має відступ менший, ніж рядок вище, але він не відповідає жодному попередньому рівню відступу. Змініть файл `multiply.py` так:

```python
def multiply(x, y):
    product = x * y
   return product  # Відступ менший за попередній рядок.
```

Тепер збережіть і запустіть файл. IDLE зупиняє його з помилкою «unindent does not match any outer indentation level». Оператор `return` не має відступу з тією самою кількістю пробілів, що й будь-який інший рядок у тілі функції.

> **Примітка**
>
> Хоча Python не має правил щодо кількості пробілів для відступу в тілі функції, PEP 8 рекомендує відступ у чотири пробіли.
>
> Упродовж усієї книги ми дотримуємося цієї угоди.

Як тільки Python виконає оператор `return`, функція припиняє роботу і повертає значення. Якщо нижче оператора `return` є код із відступом, що робить його частиною тіла функції, він ніколи не виконається.

Наприклад, функція print() ніколи не буде виконана в наступній функції:

```python
def multiply(x, y):
    product = x * y
    return product
    print("You can't see me!")
```

Якщо викликати цю версію `multiply()`, рядок `"You can't see me!"` ніколи не відобразиться.

### Виклик власної функції

Виклик власної функції відбувається так само, як і виклик будь-якої іншої функції. Введіть ім'я функції, після якого в дужках перелічте аргументи. Наприклад, щоб викликати `multiply()` з аргументами `2` і `4`, просто введіть:

```python
multiply(2, 4)
```

На відміну від вбудованих функцій, власні функції недоступні до їхнього визначення за допомогою ключового слова `def`. Функцію потрібно визначити до її виклику.

Спробуйте зберегти і запустити наступний скрипт:

```python
num = multiply(2, 4)
print(num)
def multiply(x, y):
    product = x * y
    return product
```

Коли Python читає рядок `num = multiply(2, 4)`, він не розпізнає ім'я `multiply` і генерує `NameError`:

```
Traceback (most recent call last):
  File "C:Usersdaveamultiply.py", line 1, in <module>
    num = multiply(2, 4)
NameError: name 'multiply' is not defined
```

Щоб виправити помилку, перемістіть визначення функції на початок файлу:

```python
def multiply(x, y):
    product = x * y
    return product
num = multiply(2, 4)
print(num)
```

Тепер після збереження та запуску скрипту в інтерактивному вікні відображається значення `8`.

### Функції без оператора return

Усі функції в Python повертають значення, навіть якщо воно дорівнює `None`. Однак не всім функціям потрібен оператор `return`.

Наприклад, наступна функція є цілком допустимою:

```python
def greet(name):
    print(f"Hello, {name}!")
```

`greet()` не має оператора `return`, але працює чудово:

```python
>>> greet("Dave")
Hello, Dave!
```

Незважаючи на відсутність оператора `return`, `greet()` все одно повертає значення:

```python
>>> return_value = greet("Dave")
Hello, Dave!
>>> print(return_value)
None
```

Зверніть увагу: рядок `"Hello, Dave!"` виводиться навіть тоді, коли результат `greet("Dave")` присвоюється змінній. Це пов'язано з тим, що виклик print() усередині тіла функції `greet()` завжди спричиняє побічний ефект виведення в консоль.

Якщо ви не очікували побачити `"Hello, Dave!"`, то саме ви щойно зіткнулися з однією з проблем побічних ефектів. Вони не завжди є очікуваними!

### Документування функцій

При створенні власних функцій завжди слід документувати, що вони роблять. Таким чином інші розробники зможуть прочитати документацію і знатимуть, як використовувати функцію та чого очікувати при її виклику.

Щоб отримати довідку про функцію в інтерактивному вікні IDLE, можна використати функцію help():

```python
>>> help(len)
Help on built-in function len in module builtins:

len(obj, /)
    Return the number of items in a container.
```

Коли ви передаєте ім'я змінної або функції до help(), вона відображає корисну інформацію про неї. У цьому випадку help() повідомляє, що `len` — це вбудована функція, що повертає кількість елементів у контейнері.

> **Примітка**
>
> Контейнер — це спеціальна назва для об'єкта, що містить інші об'єкти. Рядок є контейнером, оскільки він містить символи. Про інші типи контейнерів ви дізнаєтеся в Розділі 9.

Подивімося, що відбудеться, коли ви викличете help() для функції `multiply()`:

```python
>>> help(multiply)
Help on function multiply in module __main__:

multiply(x, y)
```

help() відображає сигнатуру функції, але жодної інформації про те, що вона робить, немає. Щоб краще задокументувати `multiply()`, потрібно надати docstring.

Docstring — це рядковий літерал із потрійними лапками, розміщений на початку тіла функції. Docstrings використовуються для документування того, що робить функція і які типи параметрів вона очікує.

Ось як виглядає `multiply()` із доданим docstring:

```python
def multiply(x, y):
    """Повернути добуток двох чисел x і y."""
    product = x * y
    return product
```

Оновіть скрипт `multiply.py`, додавши docstring, збережіть і запустіть скрипт. Тепер можна використати help() в інтерактивному вікні, щоб побачити docstring:

```python
>>> help(multiply)
Help on function multiply in module __main__:

multiply(x, y)
    Повернути добуток двох чисел x і y.
```

PEP 8 мало що говорить про docstrings, за винятком того, що кожна функція повинна мати їх.

Існує низка стандартизованих форматів docstrings, але тут ми не будемо їх розглядати. Деякі загальні рекомендації щодо написання docstrings можна знайти в PEP 257.

## 6.4 Цикли

Одна з чудових речей у комп'ютерах — це те, що можна змусити їх робити одне й те саме знову і знову, і вони рідко скаржаться або втомлюються.

Цикл — це блок коду, що повторюється знову і знову або задану кількість разів, або доки виконується якась умова. У Python є два типи циклів: цикл `while` і цикл `for`. У цьому розділі ви навчитеся використовувати обидва.

Почнімо з розгляду того, як працюють цикли `while`.

### Цикл while

Цикли `while` повторюють ділянку коду, доки виконується деяка умова. Кожен цикл `while` складається з двох частин:

1. **Оператор while** — починається з ключового слова `while`, за яким слідує умова перевірки, і закінчується двокрапкою (`:`).
2. **Тіло циклу** — містить код, що повторюється на кожному кроці циклу. Кожен рядок має відступ у чотири пробіли.

Під час виконання циклу `while` Python обчислює умову перевірки і визначає, чи є вона істинною або хибною. Якщо умова перевірки є істинною, виконується код у тілі циклу. В іншому випадку код у тілі пропускається і виконується решта програми.

Якщо умова перевірки є істинною і тіло циклу виконується, то після того, як Python досягає кінця тіла, він повертається до оператора `while` і повторно обчислює умову перевірки. Якщо умова все ще є істинною, тіло виконується знову. Якщо вона є хибною, тіло пропускається.

Цей процес повторюється знову і знову до тих пір, поки умова перевірки не стає хибною, змушуючи Python виходити з циклу.

Розгляньмо приклад. Введіть наступний код в інтерактивне вікно:

```python
>>> n = 1
>>> while n < 5:
...     print(n)
...     n = n + 1
...
1
2
3
4
```

Спочатку цілому числу `1` присвоюється змінна `n`. Потім створюється цикл `while` із умовою перевірки `n < 5`, що перевіряє, чи значення `n` менше за `5`.

Якщо `n` менше за `5`, виконується тіло циклу. У тілі циклу є два рядки коду. У першому рядку значення `n` виводиться на екран, а потім `n` збільшується на `1` в другому рядку.

Виконання циклу відбувається в п'ять кроків, описаних у таблиці нижче:

| Крок | Значення n | Умова перевірки | Що відбувається |
|------|-----------|----------------|----------------|
| 1 | 1 | 1 < 5 (істина) | Виводиться 1; n збільшується до 2 |
| 2 | 2 | 2 < 5 (істина) | Виводиться 2; n збільшується до 3 |
| 3 | 3 | 3 < 5 (істина) | Виводиться 3; n збільшується до 4 |
| 4 | 4 | 4 < 5 (істина) | Виводиться 4; n збільшується до 5 |
| 5 | 5 | 5 < 5 (хиба) | Нічого не виводиться; цикл завершується |

Якщо бути неуважним, можна створити нескінченний цикл. Це відбувається, коли умова перевірки завжди є істинною. Нескінченний цикл ніколи не завершується: тіло циклу повторюється вічно.

Ось приклад нескінченного циклу:

```python
>>> n = 1
>>> while n < 5:
...     print(n)
```

Єдина відмінність між цим циклом `while` і попереднім полягає в тому, що `n` ніколи не збільшується в тілі циклу. На кожному кроці циклу `n` дорівнює `1`. Це означає, що умова перевірки `n < 5` завжди є істинною, і число `1` виводиться знову і знову нескінченно.

> **Примітка**
>
> Нескінченні цикли самі по собі не є поганими. Іноді вони саме той тип циклу, який вам потрібен.
>
> Наприклад, код, що взаємодіє з апаратним забезпеченням, може використовувати нескінченний цикл для постійної перевірки того, чи активовано кнопку або перемикач.

Якщо ви запускаєте програму, що потрапляє в нескінченний цикл, можна примусово завершити Python натисканням `Ctrl+C`. Python зупиняє виконання програми і генерує помилку `KeyboardInterrupt`:

```
Traceback (most recent call last):
  File "<pyshell#8>", line 2, in <module>
    print(n)
KeyboardInterrupt
```

Розгляньмо практичний приклад циклу `while`. Один із способів застосування циклу `while` — перевірка того, чи відповідає введення користувача певній умові, і, якщо ні, повторний запит на введення доти, доки не буде отримано допустиме введення.

Наприклад, наступна програма безперервно запитує у користувача позитивне число, доки він не введе його:

```python
num = float(input("Enter a positive number: "))
while num <= 0:
    print("That's not a positive number!")
    num = float(input("Enter a positive number: "))
```

Спочатку користувачу пропонується ввести позитивне число. Умова перевірки `num <= 0` визначає, чи є `num` меншим або рівним `0`.

Якщо `num` є позитивним, умова перевірки є хибною. Тіло циклу пропускається і програма завершується.

В іншому випадку, якщо `num` дорівнює `0` або є від'ємним, тіло циклу виконується. Користувачу повідомляють, що введення було неправильним, і знову пропонують ввести позитивне число.

Цикли `while` ідеально підходять для повторення ділянки коду доти, доки виконується певна умова. Однак вони не дуже підходять для повторення ділянки коду задану кількість разів.

### Цикл for

Цикл `for` виконує ділянку коду один раз для кожного елемента в колекції елементів. Кількість виконань коду визначається кількістю елементів у колекції.

Як і його аналог `while`, цикл `for` складається з двох основних частин:

1. **Оператор for** — починається з ключового слова `for`, за яким слідує вираз належності (membership expression), і закінчується двокрапкою (`:`).
2. **Тіло циклу** — містить код, що виконується на кожному кроці циклу, і має відступ у чотири пробіли.

Розгляньмо приклад. Наступний цикл `for` виводить по одній кожну літеру рядка `"Python"`:

```python
for letter in "Python":
    print(letter)
```

У цьому прикладі оператором `for` є `for letter in "Python"`. Виразом належності є `letter in "Python"`.

На кожному кроці циклу змінній `letter` присвоюється наступна літера рядка `"Python"`, а потім значення `letter` виводиться.

Цикл виконується один раз для кожного символу рядка `"Python"`, тому тіло циклу виконується шість разів. Наступна таблиця підсумовує виконання цього циклу `for`:

| Крок | Значення letter | Що відбувається |
|------|----------------|----------------|
| 1 | `"P"` | Виводиться P |
| 2 | `"y"` | Виводиться y |
| 3 | `"t"` | Виводиться t |
| 4 | `"h"` | Виводиться h |
| 5 | `"o"` | Виводиться o |
| 6 | `"n"` | Виводиться n |

Щоб зрозуміти, чому цикли `for` краще підходять для перебору колекцій елементів, перепишемо цикл `for` з попереднього прикладу як цикл `while`.

Для цього можна використати змінну для зберігання індексу наступного символу в рядку. На кожному кроці циклу виводитимемо символ за поточним індексом, а потім збільшуватимемо індекс.

Цикл зупиниться, коли значення змінної-індексу буде дорівнювати довжині рядка. Пам'ятайте, що індекси починаються з `0`, тому останній індекс рядка `"Python"` дорівнює `5`.

Ось як можна написати такий код:

```python
word = "Python"
index = 0
while index < len(word):
    print(word[index])
    index = index + 1
```

Це значно складніше, ніж версія з циклом `for`!

Не лише цикл `for` менш складний — сам код виглядає більш природним. Він ближче до того, як можна описати цикл англійською мовою.

> **Примітка**
>
> Іноді ви можете почути, як люди описують якийсь код як особливо «Pythonic». Термін «Pythonic» зазвичай використовується для опису коду, що є чітким, лаконічним і використовує переваги вбудованих можливостей Python.
>
> У цих термінах використання циклу `for` для перебору колекції елементів є більш Pythonic, ніж використання циклу `while`.

Іноді зручно перебирати діапазон чисел. У Python є зручна вбудована функція range(), що повертає саме це — діапазон чисел!

Наприклад, `range(3)` повертає діапазон цілих чисел, починаючи з `0` і до `3` не включно. Тобто `range(3)` — це діапазон чисел `0`, `1` і `2`.

Можна використовувати `range(n)`, де `n` — будь-яке позитивне число, щоб виконати цикл рівно `n` разів. Наприклад, наступний цикл `for` виводить рядок `"Python"` тричі:

```python
for n in range(3):
    print("Python")
```

Також діапазону можна задати початкову точку. Наприклад, `range(1, 5)` — це діапазон чисел `1`, `2`, `3` і `4`. Перший аргумент — початкове число, другий — кінцева точка, що не включається до діапазону.

Використовуючи дворгументну версію range(), наступний цикл `for` виводить квадрат кожного числа від `10` до `19` включно:

```python
for n in range(10, 20):
    print(n * n)
```

Розгляньмо практичний приклад. Наступна програма просить користувача ввести суму і відображає, як розподілити її між `2`, `3`, `4` і `5` людьми:

```python
amount = float(input("Enter an amount: "))
for num_people in range(2, 6):
    print(f"{num_people} people: ${amount / num_people:,.2f} each")
```

Цикл `for` перебирає числа `2`, `3`, `4` і `5` і виводить кількість людей та суму, яку має заплатити кожна людина. Специфікатор форматування `,.2f` використовується для відображення суми як числа з фіксованою крапкою, округленого до двох знаків після коми, з тисячними роздільниками.

Запуск програми з введенням `10` дає такий результат:

```
Enter an amount: 10
2 people: $5.00 each
3 people: $3.33 each
4 people: $2.50 each
5 people: $2.00 each
```

Цикли `for` загалом використовуються в Python частіше, ніж цикли `while`. У більшості випадків цикл `for` є більш лаконічним і зрозумілим, ніж еквівалентний цикл `while`.

### Вкладені цикли

За умови правильного відступу цикли можна поміщати всередину інших циклів.

Введіть таке в інтерактивне вікно IDLE:

```python
for n in range(1, 4):
    for j in range(4, 7):
        print(f"n = {n} and j = {j}")
```

Коли Python входить у тіло першого циклу `for`, змінній `n` присвоюється значення `1`. Потім виконується тіло другого циклу `for`, і `j` присвоюється значення `4`. Перше, що виводиться, — `n = 1 and j = 4`.

Після виконання функції print() Python повертається до внутрішнього циклу `for`, присвоює `j` значення `5`, а потім виводить `n = 1 and j = 5`. Python не повертається до зовнішнього циклу `for`, оскільки внутрішній цикл, що знаходиться в тілі зовнішнього, ще не завершив виконання.

Далі `j` присвоюється значення `6`, і Python виводить `n = 1 and j = 6`. На цьому внутрішній цикл `for` завершує виконання, і керування повертається до зовнішнього.

Змінній `n` присвоюється значення `2`, і внутрішній цикл `for` виконується вдруге. Тобто `j` присвоюється значення `4`, і в консоль виводиться `n = 2 and j = 4`.

Два цикли продовжують виконуватися таким чином, і кінцевий результат виглядає так:

```
n = 1 and j = 4
n = 1 and j = 5
n = 1 and j = 6
n = 2 and j = 4
n = 2 and j = 5
n = 2 and j = 6
n = 3 and j = 4
n = 3 and j = 5
n = 3 and j = 6
```

Цикл усередині іншого циклу називається вкладеним циклом (nested loop), і вони трапляються частіше, ніж можна очікувати. Цикли `while` можна вкладати в цикли `for` і навпаки, а також вкладати цикли глибше ніж на два рівні!

> **Важливо**
>

>
> Використання вкладених циклів іноді є єдиним способом досягти потрібного результату, але надмірна кількість вкладених циклів може негативно вплинути на продуктивність програми.

Цикли є потужним інструментом. Вони використовують одну з найбільших переваг, що надають комп'ютери: здатність повторювати одне й те саме завдання величезну кількість разів, не втомлюючись і не скаржачись.

## 6.6 Scope у Python

Будь-яке обговорення функцій і циклів у Python буде неповним без згадки про питання scope.

Scope може бути одним із складніших понять у програмуванні, тому в цьому розділі ви отримаєте плавне введення в нього. Наприкінці розділу ви знатимете, що таке scope і чому це важливо. Також ви вивчите правило LEGB для визначення scope.

### Що таке scope?

Коли ви присвоюєте значення змінній, ви даєте цьому значенню ім'я. Імена є унікальними. Наприклад, не можна присвоїти одне й те саме ім'я двом різним числам.

```python
>>> x = 2
>>> x
2
>>> x = 3
>>> x
3
```

Коли ви присвоюєте `3` до `x`, ви більше не можете відновити значення `2` за іменем `x`.

Така поведінка є логічною. Адже якщо змінна `x` одночасно має значення `2` і `3`, як обчислити `x + 2`? Результат мав би бути `4` чи `5`?

Виявляється, є спосіб присвоїти одне й те саме ім'я двом різним значенням. Спробуйте запустити наступний скрипт:

```python
x = "Hello World"
def func():
    x = 2
    print(f"Inside 'func', x has the value {x}")

func()
print(f"Outside 'func', x has the value {x}")
```

У цьому прикладі змінній `x` присвоєно два різних значення. На початку `x` присвоюється `"Hello, World"`, а всередині `func()` — `2`.

Виведення скрипту, яке може вас здивувати, виглядає так:

```
Inside 'func', x has the value 2
Outside 'func', x has the value Hello World
```

Як `x` досі має значення `"Hello World"` після виклику `func()`, що змінює значення `x` на `2`?

Відповідь полягає в тому, що функція `func()` має інший scope, ніж код за межами функції. Тобто можна назвати об'єкт усередині `func()` тим самим іменем, що і щось зовні `func()`, і Python зможе розрізнити їх.

Тіло функції має так звану локальну область видимості (local scope) з власним набором доступних імен. Код за межами тіла функції знаходиться в глобальній області видимості (global scope).

Scope можна вважати набором імен, відображених на об'єкти. Коли ви використовуєте певне ім'я у своєму коді, наприклад ім'я змінної або функції, Python перевіряє поточний scope, щоб визначити, чи існує це ім'я.

### Визначення scope

Scope утворюють ієрархію. Наприклад, розгляньмо таке:

```python
x = 5
def outer_func():
    y = 3
    def inner_func():
        z = x + y
        return z
    return inner_func()
```

> **Примітка**
>
> Функція `inner_func()` називається внутрішньою функцією (inner function), оскільки вона визначена всередині іншої функції. Так само як можна вкладати цикли, можна також визначати функції всередині інших функцій!
>
> Докладніше про внутрішні функції можна прочитати у статті Real Python «Inner Functions—What Are They Good For?».

Змінна `z` знаходиться в локальному scope `inner_func()`. Коли Python виконує рядок `z = x + y`, він шукає змінні `x` і `y` в локальному scope. Однак їх там немає, тому він переміщається вгору до scope функції `outer_func()`.

Scope для `outer_func()` є охоплюючим scope (enclosing scope) для `inner_func()`. Він не є глобальним scope і не є локальним scope для `inner_func()`. Він розташований між ними.

Змінна `y` визначена в scope для `outer_func()` і має значення `3`. Однак `x` у цьому scope не існує, тому Python переміщається вгору ще раз до глобального scope. Там він знаходить ім'я `x`, яке має значення `5`. Тепер, коли імена `x` і `y` знайдені, Python може виконати рядок `z = x + y`, що присвоює `z` значення `8`.

### Правило LEGB

Зручним способом запам'ятати, як Python визначає scope, є правило LEGB. Це правило є абревіатурою від Local (локальний), Enclosing (охоплюючий), Global (глобальний), Built-in (вбудований).

Python визначає scope в тому порядку, в якому кожен scope з'являється в списку LEGB. Ось короткий огляд:

**Local (L) — локальний scope.** Це поточний, або локальний, scope. Це може бути тіло функції або scope верхнього рівня скрипту. Він завжди представляє scope, в якому наразі працює інтерпретатор Python.

**Enclosing (E) — охоплюючий scope.** Це scope на один рівень вище локального. Якщо локальний scope є внутрішньою функцією, то охоплюючий scope — це scope зовнішньої функції. Якщо scope є функцією верхнього рівня, охоплюючий scope збігається з глобальним.

**Global (G) — глобальний scope.** Це найвищий scope у скрипті. Він містить усі імена, визначені в скрипті, що не знаходяться в тілі функції.

**Built-in (B) — вбудований scope.** Вбудований scope містить усі імена, наприклад ключові слова, що вбудовані в Python. Функції на кшталт round() і abs() знаходяться у вбудованому scope. Все, що ви можете використовувати без попереднього визначення, міститься у вбудованому scope.

### Порушення правил

Розгляньте наступний скрипт. Як ви думаєте, яким буде результат?

```python
total = 0
def add_to_total(n):
    total = total + n

add_to_total(5)
print(total)
```

Здається, скрипт має вивести значення `5`, правда? Спробуйте запустити його і подивіться, що відбудеться.

Сталося щось несподіване — виникла помилка!

```
Traceback (most recent call last):
  File "C:/Users/davea/stuff/python/scope.py", line 6, in <module>
    add_to_total(5)
  File "C:/Users/davea/stuff/python/scope.py", line 4, in add_to_total
    total = total + n
UnboundLocalError: local variable 'total' referenced before assignment
```

Зачекайте! Відповідно до правила LEGB, Python мав би розпізнати, що ім'я `total` не існує в локальному scope функції `add_to_total()`, і перейти до глобального scope для знаходження імені, правда?

Проблема тут у тому, що скрипт намагається виконати присвоєння змінній `total`, що створює нове ім'я в локальному scope. Потім, коли Python виконує праву частину присвоєння, він знаходить ім'я `total` в локальному scope, якому ще нічого не присвоєно.

Такі помилки є хитрими і є однією з причин того, що найкраще використовувати унікальні імена змінних і функцій незалежно від того, в якому scope ви знаходитеся.

Цю проблему можна обійти за допомогою ключового слова `global`. Спробуйте запустити наступний змінений скрипт:

```python
total = 0
def add_to_total(n):
    global total
    total = total + n

add_to_total(5)
print(total)
```

Цього разу ви отримаєте очікуваний результат `5`. Чому?

Рядок `global total` наказує Python шукати ім'я `total` в глобальному scope. Таким чином, рядок `total = total + n` не створює нову локальну змінну.

Хоча це «виправляє» скрипт, використання ключового слова `global` загалом вважається поганою практикою.

Якщо ви виявляєте, що використовуєте `global` для вирішення подібних проблем, зупиніться і подумайте, чи є кращий спосіб написати код. Часто виявиться, що він є!
