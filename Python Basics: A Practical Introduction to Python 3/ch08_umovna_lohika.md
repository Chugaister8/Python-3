## 8.1 Порівняння значень

Умовна логіка ґрунтується на виконанні різних дій залежно від того, є певний вираз, що називається умовою (conditional), істинним чи хибним. Ця ідея не є специфічною для комп'ютерів. Люди постійно використовують умовну логіку для прийняття рішень.

Наприклад, мінімальний вік для придбання алкогольних напоїв у США — 21 рік. Твердження «Якщо вам принаймні 21 рік, ви можете придбати пиво» є прикладом умовної логіки. Фраза «вам принаймні 21 рік» є умовою, оскільки вона може бути або істинною, або хибною.

У комп'ютерному програмуванні умови часто набувають форми порівняння двох значень, наприклад визначення того, чи одне значення більше за інше, або чи рівні два значення. Для порівнянь використовується стандартний набір символів, що називаються булевими компараторами (boolean comparators), більшість із яких вам, мабуть, вже знайомі.

Наступна таблиця описує ці булеві компаратори:

| Булевий компаратор | Приклад | Значення |
|-------------------|---------|----------|
| `>` | `a > b` | `a` більше за `b` |
| `<` | `a < b` | `a` менше за `b` |
| `>=` | `a >= b` | `a` більше або рівне `b` |
| `<=` | `a <= b` | `a` менше або рівне `b` |
| `!=` | `a != b` | `a` не рівне `b` |
| `==` | `a == b` | `a` рівне `b` |

Термін «булевий» походить від прізвища англійського математика Джорджа Буля, чиї праці допомогли закласти основи сучасних обчислень. На честь Буля умовну логіку іноді називають булевою логікою, а умови — булевими виразами.

Також існує фундаментальний тип даних, що називається булевим (boolean), або скорочено `bool`, який може мати лише одне з двох значень. У Python ці значення зручно названі `True` і `False`:

```python
>>> type(True)
<class 'bool'>
>>> type(False)
<class 'bool'>
```

Зверніть увагу: `True` і `False` обидва починаються з великої літери.

Результат обчислення умови завжди є булевим значенням:

```python
>>> 1 == 1
True
>>> 3 > 5
False
```

У першому прикладі, оскільки `1` рівне `1`, результат `1 == 1` є `True`. У другому прикладі `3` не більше `5`, тому результат — `False`.

> **Важливо**
>
> Поширена помилка при написанні умов — використання оператора присвоєння `=` замість `==` для перевірки рівності двох значень.
>
> На щастя, Python генерує `SyntaxError` при виявленні такої помилки, тому ви дізнаєтеся про неї до запуску програми.

Булеві компаратори зручно уявляти як запитання про два значення. `a == b` запитує, чи мають `a` і `b` однакове значення. Аналогічно, `a != b` запитує, чи мають `a` і `b` різні значення.

Умовні вирази не обмежуються порівнянням чисел. Також можна порівнювати значення, наприклад рядки:

```python
>>> "a" == "a"
True
>>> "a" == "b"
False
>>> "a" < "b"
True
>>> "a" > "b"
False
```

Два останніх приклади можуть здатися вам дивними. Як один рядок може бути більшим або меншим за інший?

Компаратори `<` і `>` представляють поняття «більше» і «менше» при роботі з числами, але більш загально вони представляють поняття порядку. У цьому відношенні `"a" < "b"` перевіряє, чи рядок `"a"` стоїть перед рядком `"b"`. Але як упорядковуються рядки?

У Python рядки впорядковані лексикографічно — це вишуканий спосіб сказати, що вони впорядковані так, як з'являлися б у словнику. Тому `"a" < "b"` можна розуміти як запитання, чи літера `a` стоїть перед літерою `b` у словнику.

Лексикографічне впорядкування поширюється на рядки з двох і більше символів, шляхом порівняння кожної складової літери рядка:

```python
>>> "apple" < "astronaut"
True
>>> "beauty" > "truth"
False
```

На практиці компаратори `<` і `>` найчастіше використовуються з числами, а не з рядками.

## 8.2 Логічні оператори

На додаток до булевих компараторів, Python має спеціальні ключові слова, що називаються логічними операторами (logical operators) і можуть використовуватися для об'єднання булевих виразів. Є три логічних оператори: `and`, `or` і `not`.

Логічні оператори використовуються для побудови складних логічних виразів. Здебільшого їхні значення схожі на значення відповідних слів у природній мові, хоча правила їхнього використання в Python є значно точнішими.

### Ключове слово and

Розгляньмо такі твердження:

1. Коти мають чотири лапи.
2. Коти мають хвости.

Загалом обидва ці твердження є істинними.

Коли ми об'єднуємо ці два твердження за допомогою `and`, результуюче речення «коти мають чотири лапи і коти мають хвости» також є істинним. Якщо обидва твердження заперечити, складне твердження «коти не мають чотирьох лап і коти не мають хвостів» є хибним.

Навіть якщо ми комбінуємо хибні та істинні твердження, складне твердження є хибним.

Коли два твердження `P` і `Q` об'єднуються за допомогою `and`, значення істинності складного твердження «P and Q» є істинним тоді й лише тоді, коли обидва `P` і `Q` є істинними.

Оператор `and` у Python працює точно так само:

```python
>>> 1 < 2 and 3 < 4    # Обидва True
True
>>> 2 < 1 and 4 < 3    # Обидва False
False
>>> 1 < 2 and 4 < 3    # Друге твердження False
False
>>> 2 < 1 and 3 < 4    # Перше твердження False
False
```

Наступна таблиця підсумовує правила для оператора `and`:

| Комбінація з and | Результат |
|------------------|-----------|
| `True and True` | `True` |
| `True and False` | `False` |
| `False and True` | `False` |
| `False and False` | `False` |

### Ключове слово or

Коли ми використовуємо слово «або» в повсякденній розмові, іноді маємо на увазі виключне «або». Тобто лише перший варіант або другий може бути істинним.

У Python ключове слово `or` є включним. Тобто, якщо `P` і `Q` — це два вирази, твердження «P or Q» є істинним, якщо будь-яке з наступного є істинним:

1. `P` є істинним
2. `Q` є істинним
3. Обидва `P` і `Q` є істинними

```python
>>> 1 < 2 or 3 < 4     # Обидва True
True
>>> 2 < 1 or 4 < 3     # Обидва False
False
>>> 1 < 2 or 4 < 3     # Друге твердження False
True
>>> 2 < 1 or 3 < 4     # Перше твердження False
True
```

Якщо будь-яка частина складного твердження є `True`, навіть якщо інша частина є `False`, результат завжди є `True`. Наступна таблиця підсумовує ці результати:

| Комбінація з or | Результат |
|-----------------|-----------|
| `True or True` | `True` |
| `True or False` | `True` |
| `False or True` | `True` |
| `False or False` | `False` |

### Ключове слово not

Ключове слово `not` інвертує значення істинності одного виразу:

| Використання not | Результат |
|-----------------|-----------|
| `not True` | `False` |
| `not False` | `True` |

```python
>>> not True
False
>>> not False
True
```

Однак слід мати на увазі, що `not` не завжди поводиться так, як можна очікувати, у поєднанні з компараторами на кшталт `==`. Наприклад, `not True == False` повертає `True`, але `False == not True` генерує помилку:

```python
>>> not True == False
True
>>> False == not True
  File "<stdin>", line 1
    False == not True
                 ^
SyntaxError: invalid syntax
```

Це відбувається тому, що Python обробляє логічні оператори відповідно до пріоритету операторів (operator precedence), так само як арифметичні оператори мають порядок пріоритетів у звичайній математиці.

Порядок пріоритетів для логічних і булевих операторів від найвищого до найнижчого описано в наступній таблиці. Оператори в одному рядку мають однаковий пріоритет.

| Порядок пріоритетів (від найвищого до найнижчого) |
|--------------------------------------------------|
| `<`, `<=`, `==`, `>=`, `>` |
| `not` |
| `and` |
| `or` |

У виразі `False == not True` оператор `not` має нижчий пріоритет, ніж `==`. Це означає, що Python спочатку намагається обчислити `False == not`, що є синтаксично некоректним.

Щоб уникнути `SyntaxError`, оточіть `not True` дужками:

```python
>>> False == (not True)
True
```

Групування виразів у дужки — чудовий спосіб уточнити, які оператори до якої частини складного виразу належать.

### Побудова складних виразів

Можна комбінувати ключові слова `and`, `or` і `not` із `True` і `False` для створення складніших виразів. Ось приклад:

```python
True and not (1 != 1)
```

Яке значення матиме цей вираз?

Щоб з'ясувати, розбийте вираз, починаючи з крайнього правого боку. `1 != 1` є `False`, оскільки `1` має те саме значення, що й воно само. Тому можна спростити вираз:

```python
True and not (False)
```

Тепер `not (False)` — це те саме, що `not False`, що є `True`. Спрощуємо ще раз:

```python
True and True
```

Нарешті, `True and True` — просто `True`. Таким чином, після кількох кроків можна бачити, що `True and not (1 != 1)` обчислюється як `True`.

При роботі зі складними виразами найкрача стратегія — починати з найскладнішої частини виразу і рухатися назовні.

Наприклад, спробуйте обчислити такий вираз:

```python
("A" != "A") or not (2 >= 3)
```

Почніть із двох виразів у дужках. `"A" != "A"` є `False`, оскільки `"A"` рівне собі. `2 >= 3` також є `False`, оскільки `2` менше `3`. Це дає еквівалентний, але простіший вираз:

```python
(False) or not (False)
```

Оскільки `not` має вищий пріоритет, ніж `or`, цей вираз еквівалентний:

```python
False or (not False)
```

`not False` є `True`, тому можна спростити ще раз:

```python
False or True
```

Нарешті, оскільки будь-який складний вираз з `or` є `True`, якщо будь-який із виразів зліва або справа від `or` є `True`, можна зробити висновок, що `("A" != "A") or not (2 >= 3)` є `True`.

Групування виразів у дужки в складному умовному операторі покращує читаність. Іноді, однак, дужки необхідні для отримання очікуваного значення.

Наприклад, на перший погляд можна очікувати, що наступне виводить `True`, але насправді воно повертає `False`:

```python
>>> True and False == True and False
False
```

Причина цього в тому, що оператор `==` має вищий пріоритет, ніж `and`, тому Python інтерпретує вираз як `True and (False == True) and False`. Оскільки `False == True` є `False`, це еквівалентно `True and False and False`, що обчислюється як `False`.

Ось як додати дужки, щоб вираз дорівнював `True`:

```python
>>> (True and False) == (True and False)
True
```

## 8.3 Керування потоком виконання програми

Тепер, коли ми можемо порівнювати значення між собою за допомогою булевих компараторів і будувати складні умовні вирази за допомогою логічних операторів, можна додати логіку в код, щоб він виконував різні дії за різних умов.

### Оператор if

Оператор `if` наказує Python виконувати певну частину коду лише за умови виконання умови.

Наприклад, наступний оператор `if` виведе `2 and 2 is 4`, якщо умова `2 + 2 == 4` є `True`:

```python
if 2 + 2 == 4:
    print("2 and 2 is 4")
```

Англійською це можна прочитати як: «якщо 2 + 2 дорівнює 4, то вивести рядок "2 and 2 is 4"».

Так само як цикли `while`, оператор `if` складається з трьох частин:

1. Ключове слово `if`
2. Умова перевірки, після якої стоїть двокрапка
3. Блок коду з відступом, що виконується, якщо умова перевірки є `True`

У наведеному прикладі умова перевірки — `2 + 2 == 4`. Оскільки цей вираз є `True`, виконання оператора `if` в IDLE відображає текст `2 and 2 is 4`.

Якщо умова перевірки є `False` (наприклад, `2 + 2 == 5`), Python пропускає блок коду з відступом і продовжує виконання на наступному рядку без відступу.

Наприклад, наступний оператор `if` нічого не виводить:

```python
if 2 + 2 == 5:
    print("Is this the mirror universe?")
```

> **Примітка**
>
> Пропуск двокрапки (`:`) після умови перевірки в операторі `if` генерує `SyntaxError`:
>
> ```python
> >>> if 2 + 2 == 4
> SyntaxError: invalid syntax
> ```

Після виконання блоку коду з відступом в операторі `if` Python продовжує виконувати решту програми.

Розгляньмо такий скрипт:

```python
grade = 95
if grade >= 70:
    print("You passed the class!")

print("Thank you for attending.")
```

Результат виглядає так:

```
You passed the class!
Thank you for attending.
```

Оскільки `grade` дорівнює `95`, умова `grade >= 70` є `True`, і рядок `"You passed the class!"` виводиться. Потім виконується решта коду і виводиться `"Thank you for attending."`.

Якщо змінити значення `grade` на `40`, результат виглядатиме так:

```
Thank you for attending.
```

Рядок `print("Thank you for attending.")` виконується незалежно від того, чи `grade` більше або рівне `70`, оскільки він стоїть після блоку коду з відступом в операторі `if`.

Студент, що не склав іспит, не знатиме про це, якщо в результаті виконання коду він побачить лише `"Thank you for attending."`.

Додамо ще один оператор `if`, щоб повідомити студента про те, що він не склав іспит, якщо його оцінка нижча за `70`:

```python
grade = 40
if grade >= 70:
    print("You passed the class!")
if grade < 70:
    print("You did not pass the class :(")

print("Thank you for attending.")
```

Тепер результат виглядає так:

```
You did not pass the class :(
Thank you for attending.
```

### Ключове слово else

Ключове слово `else` використовується після оператора `if` для виконання коду лише в тому випадку, якщо умова перевірки оператора `if` є `False`.

Наступний скрипт використовує `else`, щоб скоротити код попереднього прикладу для відображення того, чи склав студент іспит:

```python
grade = 40
if grade >= 70:
    print("You passed the class!")
else:
    print("You did not pass the class :(")

print("Thank you for attending.")
```

Зверніть увагу: ключове слово `else` не має умови перевірки і закінчується двокрапкою. Умова не потрібна, оскільки `else` виконується для будь-якої умови, що не відповідає умові перевірки оператора `if`.

> **Важливо**
>
> Пропуск двокрапки (`:`) після ключового слова `else` генерує `SyntaxError`.

Результат наведеного скрипту:

```
You did not pass the class :(
Thank you for attending.
```

Рядок із виведенням `"Thank you for attending."` виконується навіть у тому випадку, коли виконується блок коду після `else`.

Ключові слова `if` і `else` добре працюють разом, якщо потрібно перевірити умову рівно з двома станами.

Іноді потрібно перевірити три та більше умов. Для цього використовується `elif`.

### Ключове слово elif

Ключове слово `elif` є скороченням від «else if» і може використовуватися для додавання додаткових умов після оператора `if`.

Так само як оператори `if`, оператори `elif` мають три частини:

1. Ключове слово `elif`
2. Умова перевірки, після якої стоїть двокрапка
3. Блок коду з відступом, що виконується, якщо умова є `True`

Наступний скрипт об'єднує `if`, `elif` і `else` для виведення буквеної оцінки студента:

```python
grade = 85

if grade >= 90:                              # 1
    print("You passed the class with a A.")
elif grade >= 80:                            # 2
    print("You passed the class with a B.")
elif grade >= 70:                            # 3
    print("You passed the class with a C.")
else:                                        # 4
    print("You did not pass the class :(")

print("Thanks for attending.")               # 5
```

Як `grade >= 80`, так і `grade >= 70` є `True`, коли `grade` дорівнює `85`, тому можна очікувати, що блоки обох `elif` на рядках 2 і 3 будуть виконані.

Однак виконується лише перший блок, для якого умова перевірки є `True`. Усі наступні блоки `elif` і `else` пропускаються, тому виконання скрипту дає такий результат:

```
You passed the class with a B.
Thanks for attending.
```

Розберімо покроково виконання скрипту:

1. `grade` присвоюється значення `85` у рядку 1.
2. `grade >= 90` є `False`, тому оператор `if` у рядку 1 пропускається.
3. `grade >= 80` є `True`, тому виконується блок під оператором `elif` у рядку 2 і виводиться `"You passed the class with a B."`.
4. Оператори `elif` і `else` у рядках 3 і 4 пропускаються, оскільки умова для `elif` у рядку 2 вже виконалась.
5. Нарешті, виконується рядок 5 і виводиться `"Thanks for attending."`.

Ключові слова `if`, `elif` і `else` є одними з найбільш уживаних у Python. Вони дозволяють писати код, що реагує на різні умови по-різному.

Оператор `if` дозволяє вирішувати складніші задачі, ніж код без умовної логіки. Можна навіть вкладати оператор `if` всередині іншого, щоб писати код, що обробляє надзвичайно складну логіку!

### Вкладені оператори if

Так само як цикли `for` і `while` можна вкладати один в одного, оператор `if` можна вкласти в інший для створення складних структур прийняття рішень.

Розгляньмо такий сценарій. Двоє людей грають один проти одного в гру. Потрібно визначити, хто з двох гравців виграє, залежно від рахунків гравців та виду спорту:

- Якщо гравці грають у баскетбол, перемагає той, у кого більший рахунок.
- Якщо гравці грають у гольф, перемагає той, у кого менший рахунок.
- В обох видах спорту, якщо рахунки рівні, гра — нічия.

Наступна програма вирішує це за допомогою вкладених операторів `if`:

```python
sport = input("Enter a sport: ")
p1_score = int(input("Enter player 1 score: "))
p2_score = int(input("Enter player 2 score: "))

# 1
if sport.lower() == "basketball":
    if p1_score == p2_score:
        print("The game is a draw.")
    elif p1_score > p2_score:
        print("Player 1 wins.")
    else:
        print("Player 2 wins.")
# 2
elif sport.lower() == "golf":
    if p1_score == p2_score:
        print("The game is a draw.")
    elif p1_score < p2_score:
        print("Player 1 wins.")
    else:
        print("Player 2 wins.")
# 3
else:
    print("Unknown sport")
```

Ця програма спочатку просить користувача ввести вид спорту і рахунки двох гравців.

У `(#1)` рядок, присвоєний `sport`, перетворюється на нижній регістр за допомогою `.lower()` і порівнюється з рядком `"basketball"`. Це забезпечує однакове трактування введених значень типу `"Basketball"` або `"BasketBall"`.

Потім порівнюються рахунки гравців. Якщо вони рівні — нічия. Якщо рахунок гравця 1 більший — він виграє. В іншому разі виграє гравець 2.

У `(#2)` виконується аналогічна логіка для гольфу. Якщо рахунок гравця 1 менший — він виграє. В іншому разі виграє гравець 2.

Нарешті, у `(#3)`, якщо `sport` містить рядок, відмінний від `"basketball"` або `"golf"`, відображається повідомлення `"Unknown sport"`.

> **Примітка**
>
> Вкладення операторів `if` за своєю природою збільшує складність коду, і кількість можливих шляхів виконання зростає дуже швидко.
>
> Вкладені оператори `if` іноді є необхідними, але, як правило, вони не рекомендовані. Якщо ви помічаєте, що пишете багато вкладених операторів `if`, варто зупинитися і подумати, як можна спростити код.

Ось спрощена версія попередньої програми, яка використовує складні умовні вирази замість вкладених операторів `if`:

```python
if p1_score == p2_score:
    print("The game is a draw.")

elif (sport.lower() == "basketball") or (sport.lower() == "golf"):
    sport = sport.lower()
    p1_wins_basketball = (sport == "basketball") and (p1_score > p2_score)
    p1_wins_golf = (sport == "golf") and (p1_score < p2_score)
    p1_wins = p1_wins_basketball or p1_wins_golf
    if p1_wins:
        print("Player 1 wins.")
    else:
        print("Player 2 wins.")
else:
    print("Unknown sport")
```

У цій переробленій версії програма може виконуватися лише чотирма способами, і код зрозуміліший.

## 8.5 Вийти зі зразка

У Розділі 6 ви навчилися повторювати блок коду багаторазово за допомогою циклів `for` або `while`. Цикли корисні для виконання повторюваних завдань і для обробки різних вхідних даних.

Поєднання операторів `if` із циклами `for` відкриває потужні методи керування виконанням коду.

У цьому розділі ви навчитеся писати оператори `if`, вкладені в цикли `for`, а також дізнаєтеся про два ключові слова — `break` і `continue` — що дозволяють точніше керувати потоком виконання всередині циклу.

### Оператори if і цикли for

Блок коду в циклі `for` такий самий, як і будь-який інший блок коду. Це означає, що оператор `if` можна вкладати в цикл `for` так само, як і в будь-якому іншому місці коду.

Наступний приклад використовує цикл `for` з оператором `if` для обчислення та відображення суми всіх парних цілих чисел, менших за 100:

```python
sum_of_evens = 0
for n in range(1, 100):
    if n % 2 == 0:
        sum_of_evens = sum_of_evens + n

print(sum_of_evens)
```

Спочатку змінна `sum_of_evens` ініціалізується значенням `0`. Потім програма перебирає числа від `1` до `99` і додає парні значення до `sum_of_evens`. Фінальне значення `sum_of_evens` дорівнює `2450`.

### break

Ключове слово `break` наказує Python буквально «вийти» з циклу. Тобто цикл повністю зупиняється і виконується будь-який код після циклу.

Наприклад, наступний код перебирає числа від `0` до `3`, але зупиняє цикл при зустрічі числа `2`:

```python
for n in range(0, 4):
    if n == 2:
        break
    print(n)

print(f"Finished with n = {n}")
```

У результаті виводяться лише перші два числа:

```
0
1
Finished with n = 2
```

### continue

Ключове слово `continue` використовується для пропуску коду, що залишився в тілі циклу, і переходу до наступної ітерації.

Наприклад, наступний код перебирає числа від `0` до `3`, виводячи кожне число по ходу, але пропускає число `2`:

```python
for i in range(0, 4):
    if i == 2:
        continue
    print(i)

print(f"Finished with i = {i}")
```

Виводяться всі числа, крім `2`:

```
0
1
3
Finished with i = 3
```

> **Примітка**
>
> Завжди варто давати змінним короткі, але описові назви, що полегшують розуміння їхнього призначення.
>
> Літери `i`, `j` і `k` є винятком, оскільки вони дуже поширені в програмуванні. Вони майже завжди використовуються, коли потрібне «тимчасове» число виключно для підрахунку кроків у циклі.

Підсумовуючи: ключове слово `break` використовується для зупинки циклу при виконанні певної умови, а ключове слово `continue` — для пропуску ітерації циклу при виконанні певної умови.

### Цикли for...else

Цикли в Python можуть мати власний блок `else`, хоча така конструкція використовується нечасто.

Розгляньмо приклад:

```python
phrase = "it marks the spot"
for character in phrase:
    if character == "X":
        break
else:
    print("There was no 'X' in the phrase")
```

Цикл `for` у цьому прикладі перебирає символи рядка `"it marks the spot"` і зупиняється, якщо знаходить літеру `"X"`.

Якщо запустити цей код, у консолі виведеться `There was no 'X' in the phrase`.

Тепер спробуйте змінити `phrase` на рядок `"X marks the spot"`. При запуску того самого коду з цією фразою жодного виведення не відбудеться. Що відбувається?

Будь-який код у блоці `else` після циклу `for` виконується лише в тому випадку, якщо цикл `for` завершився без виконання оператора `break`.

Ось практичний приклад, що дає користувачу три спроби ввести пароль:

```python
for n in range(3):
    password = input("Password: ")
    if password == "I<3Bieber":
        break
    print("Password is incorrect.")
else:
    print("Suspicious activity. The authorities have been alerted.")
```

Цей приклад перебирає числа від `0` до `2`. На кожній ітерації користувача просять ввести пароль. Якщо введений пароль правильний, `break` використовується для виходу з циклу. В іншому випадку користувача повідомляють, що пароль неправильний, і дають ще одну спробу.

Після трьох невдалих спроб цикл `for` завершується, ніколи не виконавши рядок із `break`. У цьому випадку виконується блок `else` і користувача попереджають.

> **Примітка**
>
> У цьому розділі ми зосередилися на циклах `for`, оскільки вони загалом є найбільш поширеним типом циклів.
>
> Однак усе, що тут обговорювалося, також застосовується до циклів `while`. Тобто можна використовувати `break` і `continue` всередині циклу `while`. Цикли `while` можуть навіть мати блок `else`!

## 8.6 Відновлення після помилок

Стикатися з помилками в коді може бути неприємно, але це цілком нормально! Це трапляється навіть із найкращими програмістами.

Програмісти часто називають помилки часу виконання винятками (exceptions). Тому, коли ви стикаєтеся з помилкою, — привітайте себе! Ви щойно змусили код зробити щось виняткове!

Помилки — не завжди погана річ. Тобто вони не завжди означають, що ви зробили щось не так. Наприклад, спроба поділити `1` на `0` призводить до `ZeroDivisionError`. Якщо дільник вводиться користувачем, ви заздалегідь не можете знати, чи введе користувач `0`!

Щоб створювати надійні програми, потрібно вміти обробляти помилки, спричинені неприпустимим введенням користувача або будь-яким іншим непередбачуваним джерелом. У цьому розділі ви навчитеся це робити.

### Зоопарк винятків

При зустрічі з винятком корисно знати, що пішло не так. У Python є низка вбудованих типів винятків, що описують різні види помилок.

Упродовж цієї книги ви бачили кілька різних помилок. Зберімо їх тут і додамо кілька нових.

**ValueError**

`ValueError` виникає, коли операція стикається з неприпустимим значенням. Наприклад, спроба перетворити рядок `"not a number"` на ціле число призводить до `ValueError`:

```python
>>> int("not a number")
Traceback (most recent call last):
  File "<pyshell#1>", line 1, in <module>
    int("not a number")
ValueError: invalid literal for int() with base 10: 'not a number'
```

Ім'я виключення відображається на останньому рядку, за ним слідує опис конкретної виниклої проблеми. Це загальний формат для всіх винятків Python.

**TypeError**

`TypeError` виникає, коли операція виконується над значенням неправильного типу. Наприклад, спроба додати рядок і ціле число призведе до `TypeError`:

```python
>>> "1" + 2
Traceback (most recent call last):
  File "<pyshell#1>", line 1, in <module>
    "1" + 2
TypeError: can only concatenate str (not "int") to str
```

**NameError**

`NameError` виникає, коли ви намагаєтеся використати ім'я змінної, яке ще не було визначено:

```python
>>> print(does_not_exist)
Traceback (most recent call last):
  File "<pyshell#3>", line 1, in <module>
    print(does_not_exist)
NameError: name 'does_not_exist' is not defined
```

**ZeroDivisionError**

`ZeroDivisionError` виникає, коли дільник в операції ділення дорівнює `0`:

```python
>>> 1 / 0
Traceback (most recent call last):
  File "<pyshell#4>", line 1, in <module>
    1 / 0
ZeroDivisionError: division by zero
```

**OverflowError**

`OverflowError` виникає, коли результат арифметичної операції є надто великим. Наприклад, спроба підняти значення `2.0` до степеня `1_000_000` призводить до `OverflowError`:

```python
>>> pow(2.0, 1_000_000)
Traceback (most recent call last):
  File "<pyshell#6>", line 1, in <module>
    pow(2.0, 1_000_000)
OverflowError: (34, 'Result too large')
```

Нагадаємо з Розділу 5: цілі числа в Python мають необмежену точність. Це означає, що `OverflowError` може виникати лише з числами з рухомою комою.

Список вбудованих винятків Python можна знайти в документації.

### Ключові слова try і except

Іноді можна передбачити, що може виникнути певний виняток. Замість того щоб давати програмі аварійно завершитися, можна перехопити помилку, якщо вона виникне, і зробити щось інше.

Наприклад, може знадобитися попросити користувача ввести ціле число. Якщо користувач введе нецілочисельне значення, наприклад рядок `"a"`, потрібно повідомити його про введення неприпустимого значення.

Щоб запобігти аварійному завершенню програми, можна використовувати ключові слова `try` і `except`. Розгляньмо приклад:

```python
try:
    number = int(input("Enter an integer: "))
except ValueError:
    print("That was not an integer")
```

Ключове слово `try` використовується для позначення блоку `try` і закінчується двокрапкою. Виконується код із відступом після `try`. У цьому випадку користувача просять ввести ціле число. Оскільки input() повертає рядок, введення користувача перетворюється на ціле число за допомогою int() і результат присвоюється змінній `number`.

Якщо користувач вводить нецілочисельне значення, операція int() генерує `ValueError`. Якщо це відбувається, виконується код із відступом після рядка `except ValueError`. Таким чином, замість аварійного завершення програми відображається рядок `"That was not an integer"`.

Якщо користувач вводить допустиме цілочисельне значення, код у блоці `except ValueError` ніколи не виконується.

З іншого боку, якщо виникає інший тип виключення, наприклад `TypeError`, програма аварійно завершиться. Наведений приклад обробляє лише один тип виключення — `ValueError`.

Можна обробляти кілька типів виключень, розділяючи їхні назви комами і поміщаючи список назв у дужки:

```python
def divide(num1, num2):
    try:
        print(num1 / num2)
    except (TypeError, ZeroDivisionError):
        print("encountered a problem")
```

У цьому прикладі функція `divide()` приймає два параметри `num1` і `num2` і виводить результат ділення `num1` на `num2`.

Якщо `divide()` викликається з аргументом-рядком, операція ділення генерує `TypeError`. Крім того, якщо `num2` дорівнює `0`, генерується `ZeroDivisionError`.

Рядок `except (TypeError, ZeroDivisionError)` обробить обидва ці виключення і відобразить рядок `"encountered a problem"`, якщо будь-яке з них виникне.

Однак часто корисніше перехоплювати кожну помилку окремо, щоб відображати більш зрозуміле для користувача повідомлення. Для цього можна використовувати кілька блоків `except` після блоку `try`:

```python
def divide(num1, num2):
    try:
        print(num1 / num2)
    except TypeError:
        print("Both arguments must be numbers")
    except ZeroDivisionError:
        print("num2 must not be 0")
```

У цьому прикладі `TypeError` і `ZeroDivisionError` обробляються окремо, і відображається більш описове повідомлення при виникненні чогось нежданого.

### «Голий» блок except

Можна використовувати ключове слово `except` самостійно, не вказуючи конкретних винятків:

```python
try:
    # Робимо багато небезпечних речей, які можуть зламатися
except:
    print("Something bad happened!")
```

Якщо під час виконання коду в блоці `try` виникає будь-який виняток, буде виконано блок `except` і відображено повідомлення `"Something bad happened!"`.

Це може здатися чудовим способом гарантувати, що програма ніколи не аварійно завершується, але насправді це погана ідея, і така практика загалом засуджується!

Є кілька причин, але найважливіша для програмістів-початківців полягає в тому, що перехоплення кожного виключення може приховати баги у вашому коді, даючи вам хибне відчуття впевненості в тому, що код працює як очікується.

Якщо ви перехоплюєте лише конкретні виключення, то при виникненні несподіваних помилок Python виведе трасування та інформацію про помилку, надаючи більше інформації для налагодження.

## 8.7 Симуляція подій та обчислення ймовірностей

У цьому розділі ми застосуємо деякі концепції про цикли та умовну логіку до реальної задачі: симуляція подій та обчислення ймовірностей.

Ми проведемо просту симуляцію, відому як експеримент Монте-Карло (Monte Carlo experiment). Кожен експеримент складається з випробування (trial) — деякого процесу, що можна повторювати, наприклад підкидання монети, — що дає певний результат (outcome), наприклад орел або решка. Випробування повторюється знову і знову для обчислення ймовірності виникнення певного результату.

Для цього нам потрібно додати деяку випадковість у наш код.

### Модуль random

Python надає кілька функцій для генерування випадкових чисел у модулі `random`. Модуль — це колекція пов'язаного коду. Стандартна бібліотека Python — це організована колекція модулів, які можна імпортувати у власний код для вирішення різних задач.

Щоб імпортувати модуль `random`, введіть у інтерактивне вікно IDLE:

```python
>>> import random
```

Тепер можна використовувати функції з модуля `random` у коді. Наприклад, функція `randint()` має два обов'язкових параметри `a` і `b` і повертає випадкове ціле число, більше або рівне `a` і менше або рівне `b`. Обидва параметри мають бути цілими числами.

Наприклад, наступний код повертає випадкове ціле число від `1` до `10`:

```python
>>> random.randint(1, 10)
9
```

Оскільки результат є випадковим, ваш результат, ймовірно, відрізнятиметься від `9`. Якщо ввести той самий код ще раз, швидше за все отримаєте інше число.

Оскільки `randint()` знаходиться в модулі `random`, для її використання потрібно ввести `random`, потім крапку (`.`) і ім'я функції.

Важливо пам'ятати, що при використанні `randint()` обидва параметри `a` і `b` мають бути цілими числами, а результат може дорівнювати одному з `a` або `b`, або будь-якому числу між ними. Наприклад, `random.randint(0, 1)` випадковим чином повертає або `0`, або `1`.

Крім того, кожне ціле число між `a` і `b` має однакову ймовірність бути повернутим `randint()`. Тому для `randint(1, 10)` кожне ціле число від `1` до `10` має 10% шанс бути повернутим. Для `randint(0, 1)` є 50% шанс повернення `0`.

### Підкидання чесної монети

Подивімося, як використовувати `randint()` для симуляції підкидання чесної монети. Під чесною монетою маємо на увазі монету, яка при підкиданні має однаковий шанс випасти орлом або решкою.

Одне випробування нашого експерименту — підкидання монети. Результатом є або орел, або решка. Питання таке: загалом, на багатьох підкиданнях, яке співвідношення орлів до решок?

Подумаємо, як розв'язати цю задачу. Нам потрібно відстежувати, скільки разів випадає орел або решка, тому потрібен лічильник орлів і лічильник решок. Кожне випробування складається з двох кроків:

1. Підкинути монету.
2. Якщо монета впала орлом — оновити лічильник орлів. В іншому випадку монета впала решкою — оновити лічильник решок.

Потрібно повторити випробування багато разів, наприклад `10 000`. Цикл `for` над `range(10_000)` — хороший вибір для такого завдання.

Тепер, маючи план, напишімо функцію `coin_flip()`, що випадковим чином повертає рядок `"heads"` або рядок `"tails"`. Можна використати `random.randint(0, 1)`. Використаємо `0` для представлення орла і `1` для решки.

Ось код функції `coin_flip()`:

```python
import random

def coin_flip():
    """Випадковим чином повернути 'heads' або 'tails'."""
    if random.randint(0, 1) == 0:
        return "heads"
    else:
        return "tails"
```

Якщо `random.randint(0, 1)` повертає `0`, `coin_flip()` повертає `"heads"`. В іншому випадку `coin_flip()` повертає `"tails"`.

Тепер напишімо цикл `for`, що підкидає монету `10 000` разів і відповідно оновлює лічильник орлів або решок:

```python
# Спочатку ініціалізуємо лічильники нулями
heads_tally = 0
tails_tally = 0
for trial in range(10_000):
    if coin_flip() == "heads":
        heads_tally = heads_tally + 1
    else:
        tails_tally = tails_tally + 1
```

Спочатку створюються дві змінні `heads_tally` і `tails_tally`, обидві ініціалізовані цілим числом `0`.

Потім цикл `for` виконується `10 000` разів. Щоразу викликається функція `coin_flip()`. Якщо вона повертає рядок `"heads"`, змінна `heads_tally` збільшується на `1`. В іншому випадку збільшується `tails_tally`.

Нарешті, виведемо співвідношення орлів і решок:

```python
ratio = heads_tally / tails_tally
print(f"The ratio of heads to tails is {ratio}")
```

Якщо зберегти наведений код у скрипт і запустити кілька разів, результат зазвичай буде між `.98` і `1.02`. Якщо збільшити `range(10_000)` у циклі `for` до, скажімо, `range(50_000)`, результати мають наближатися до `1.0`.

### Підкидання нечесної монети

`randint()` повертає `0` або `1` з однаковою ймовірністю. Щоб симулювати нечесну монету, нам потрібен спосіб повертати одне з `0` або `1` з більшою ймовірністю.

Функція `random()` може викликатися без аргументів і повертає число з рухомою комою, більше або рівне `0.0`, але менше `1.0`. Кожне можливе значення, що повертається, є однаково ймовірним. У теорії ймовірностей це відоме як рівномірний розподіл.

Наслідком цього є те, що для заданого числа `n` від `0` до `1`, ймовірність того, що `random()` поверне число менше `n`, дорівнює самому `n`. Наприклад, ймовірність того, що `random()` менше `0.8`, дорівнює `0.8`, а ймовірність того, що `random()` менше `0.25`, дорівнює `0.25`.

Використовуючи цей факт, напишемо функцію, що симулює підкидання монети, але повертає решку із заданою ймовірністю:

```python
import random

def unfair_coin_flip(probability_of_tails):
    if random.random() < probability_of_tails:
        return "tails"
    else:
        return "heads"
```

Наприклад, `unfair_coin_flip(.7)` має 70% шанс повернути `"tails"`.

Перепишемо попередній експеримент із підкиданням монети, використовуючи `unfair_coin_flip()` для кожного випробування:

```python
heads_tally = 0
tails_tally = 0
for trial in range(10_000):
    if unfair_coin_flip(.7) == "heads":
        heads_tally = heads_tally + 1
    else:
        tails_tally = tails_tally + 1
ratio = heads_tally / tails_tally
print(f"The ratio of heads to tails is {ratio}")
```

Кількаразовий запуск цієї симуляції показує, що співвідношення орлів і решок знизилося з `1` в експерименті з чесною монетою приблизно до `0.43`.
