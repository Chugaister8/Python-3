## 12.1 Файли та файлова система

Ви, напевно, давно працюєте з комп'ютерними файлами. Проте є деякі речі про файли, які програмістам необхідно знати, а пересічним користувачам — ні.

### Анатомія файлу

Існує безліч типів файлів: текстові, графічні, аудіофайли, PDF тощо. Але в кінцевому підсумку файл — це просто послідовність байтів, що називається вмістом файлу.

Кожен байт у файлі можна розглядати як ціле число зі значенням від 0 до 255 включно. Байти — це значення, що зберігаються на фізичному пристрої зберігання при збереженні файлу.

Коли ви відкриваєте файл на комп'ютері, вміст файлу зчитується з диска у правильній послідовності байтів. Важливо знати: у самому файлі немає нічого, що диктує, як інтерпретувати вміст. Завдання програміста — правильно інтерпретувати вміст при відкритті файлу. Python бере на себе велику частину цієї роботи.

### Файлова система

Файлова система на комп'ютері виконує дві функції:

1. Забезпечує абстрактне представлення файлів, що зберігаються на комп'ютері та підключених до нього пристроях.
2. Взаємодіє з пристроями для керування зберіганням і отриманням даних файлів.

> **Важливо**
>
> Різні операційні системи використовують різні файлові системи. Це дуже важливо пам'ятати при написанні коду, що виконуватиметься на різних операційних системах.

### Ієрархія файлової системи

Файлові системи організовують файли в ієрархію каталогів, відомих також як папки. На вершині ієрархії знаходиться каталог, що називається кореневим (root directory). Усі інші файли та каталоги у файловій системі містяться в кореневому каталозі.

### Шляхи до файлів

Щоб знайти файл у файловій системі, можна перелічити каталоги в порядку, починаючи з кореневого, а потім вказати ім'я файлу. Рядок, що представляє розташування файлу в такий спосіб, називається шляхом до файлу (file path).

Наприклад, шлях до файлу `jack_russel.gif` у файловій системі: `root/photos/dogs/jack_russel.gif`.

Спосіб запису шляхів до файлів залежить від операційної системи:

1. Windows: `C:\Users\David\Documents\hello.txt`
2. macOS: `/Users/David/Documents/hello.txt`
3. Ubuntu Linux: `/home/David/Documents/hello.txt`

На macOS та Ubuntu Linux операційна система використовує віртуальну файлову систему, що організовує всі файли та каталоги під єдиним кореневим каталогом `/`. На Windows немає єдиного кореневого каталогу — кожен пристрій має окрему файлову систему з унікальним кореневим каталогом, позначеним буквою диска (наприклад, `C:\`).

Інша суттєва відмінність: каталоги у шляхах Windows розділяються зворотними скісними рисками (`\`), а в macOS та Ubuntu — прямими (`/`).

При написанні програм, що мають запускатися на кількох операційних системах, критично важливо правильно обробляти відмінності в шляхах. У Python починаючи з версії 3.4 стандартна бібліотека містить модуль `pathlib`, що знімає складнощі роботи з шляхами на різних операційних системах.

## 12.2 Робота зі шляхами до файлів у Python

Для роботи зі шляхами до файлів у Python використовується модуль `pathlib` зі стандартної бібліотеки. Модуль `pathlib` містить клас `Path`, що використовується для представлення шляху до файлу.

### Створення об'єктів Path

Є кілька способів створити новий об'єкт `Path`:

1. З рядка
2. За допомогою методів класу `Path.home()` і `Path.cwd()`
3. За допомогою оператора `/`

**З рядка:**

```python
>>> path = pathlib.Path("/Users/David/Documents/hello.txt")
```

На Windows каталоги розділяються зворотними косими рисками `\`. Python інтерпретує зворотні косі риски як початок escape-послідовностей. Є два способи це обійти:

```python
# Використовувати прямі косі риски
>>> path = pathlib.Path("C:/Users/David/Desktop/hello.txt")

# Або перетворити рядок на «сирий» рядок за допомогою префікса r
>>> path = pathlib.Path(r"C:\Users\David\Desktop\hello.txt")
```

**Path.home() і Path.cwd():**

```python
>>> home = pathlib.Path.home()
```

`Path.home()` створює об'єкт `Path`, що представляє домашній каталог незалежно від операційної системи. На Windows він повертає `WindowsPath`, на інших системах — `PosixPath`.

`Path.cwd()` повертає об'єкт `Path`, що представляє поточний робочий каталог (CWD).

**За допомогою оператора /:**

```python
>>> home / "Desktop" / "hello.txt"
WindowsPath('C:/Users/David/Desktop/hello.txt')
```

Оператор `/` завжди має мати об'єкт `Path` з лівого боку.

### Абсолютні та відносні шляхи

Шлях, що починається з кореневого каталогу, називається абсолютним шляхом. Шлях, що не є абсолютним, називається відносним.

```python
>>> path = pathlib.Path(r"Photos\image.jpg")
>>> path.is_absolute()
False
```

Відносні шляхи мають сенс лише у контексті якогось іншого каталогу. Розширити відносний шлях до абсолютного можна за допомогою оператора `/`:

```python
>>> home / pathlib.Path(r"Photos\image.png")
WindowsPath('C:/Users/David/Photos/image.png')
```

### Доступ до компонентів шляху до файлу

```python
>>> path = pathlib.Path.home() / "hello.txt"
>>> list(path.parents)
[WindowsPath('C:\\Users\\David'), WindowsPath('C:\\Users'), WindowsPath('C:\\')]
>>> path.parent
'C:\\Users\\David'
>>> path.anchor
'C:\\'
>>> path.name
'hello.txt'
>>> path.stem
'hello'
>>> path.suffix
'.txt'
```

### Перевірка існування шляху до файлу

```python
>>> path = pathlib.Path.home() / "hello.txt"
>>> path.exists()
False
>>> path.is_file()
True
>>> path.is_dir()
False
```

## 12.3 Поширені операції з файловою системою

### Створення каталогів і файлів

Щоб створити новий каталог, використовується метод `Path.mkdir()`:

```python
>>> from pathlib import Path
>>> new_dir = Path.home() / "new_directory"
>>> new_dir.mkdir()
>>> new_dir.exists()
True
>>> new_dir.is_dir()
True
```

Якщо спробувати створити вже існуючий каталог, виникає `FileExistsError`. Щоб уникнути цього, встановіть параметр `exist_ok` у `True`:

```python
>>> new_dir.mkdir(exist_ok=True)
```

Для створення вкладених каталогів, батьківські каталоги яких ще не існують, встановіть параметр `parents` у `True`:

```python
>>> nested_dir = new_dir / "folder_a" / "folder_b"
>>> nested_dir.mkdir(parents=True)
```

Поширений шаблон для створення каталогів:

```python
path.mkdir(parents=True, exist_ok=True)
```

Для створення файлів використовується метод `Path.touch()`:

```python
>>> file_path = new_dir / "file1.txt"
>>> file_path.touch()
>>> file_path.exists()
True
>>> file_path.is_file()
True
```

На відміну від `.mkdir()`, метод `.touch()` не генерує виняток, якщо шлях вже існує. Проте не можна створити файл у каталозі, який не існує — спочатку потрібно створити батьківський каталог:

```python
>>> file_path.parent.mkdir()
>>> file_path.touch()
```

### Перебір вмісту каталогу

Метод `Path.iterdir()` повертає ітератор об'єктів `Path`, що представляють кожен елемент у каталозі:

```python
>>> for path in new_dir.iterdir():
...     print(path)
C:\Users\David\new_directory\file1.txt
C:\Users\David\new_directory\folder_a
C:\Users\David\new_directory\folder_c
```

Зверніть увагу: `.iterdir()` повертає лише елементи, що безпосередньо містяться в папці.

### Пошук файлів у каталозі

Метод `Path.glob()` повертає ітерований об'єкт із вмістом каталогу, що відповідає певному шаблону. Символ узагальнення (wildcard) — це спеціальний символ, що виступає як заповнювач у шаблоні.

| Символ | Опис | Приклад | Збіги |
|--------|------|---------|-------|
| `*` | Будь-яка кількість символів | `"*b*"` | `b`, `ab`, `bc`, `abc` |
| `?` | Один символ | `"?bc"` | `abc`, `bbc`, `cbc` |
| `[abc]` | Один символ із дужок | `[CB]at` | `Cat`, `Bat` |

```python
>>> list(new_dir.glob("*.py"))
[WindowsPath('C:/Users/David/new_directory/program1.py'),
 WindowsPath('C:/Users/David/new_directory/program2.py')]

>>> list(new_dir.glob("program?.py"))
[WindowsPath('C:/Users/David/new_directory/program1.py'),
 WindowsPath('C:/Users/David/new_directory/program2.py')]
```

**Рекурсивне зіставлення з `**`:**

Спеціальний символ `**` робить шаблон рекурсивним і дозволяє шукати у поточному каталозі та всіх його підкаталогах:

```python
>>> list(new_dir.glob("**/*.txt"))
[WindowsPath('C:/Users/David/new_directory/file1.txt'),
 WindowsPath('C:/Users/David/new_directory/folder_c/file2.txt')]
```

Є скорочений метод `.rglob()` (рекурсивний glob):

```python
>>> list(new_dir.rglob("*.py"))
[WindowsPath('C:/Users/David/new_directory/program1.py'),
 WindowsPath('C:/Users/David/new_directory/program2.py'),
 WindowsPath('C:/Users/David/new_directory/folder_a/program3.py')]
```

### Переміщення та видалення файлів і папок

Для переміщення файлу або каталогу використовується метод `.replace()`:

```python
>>> source = new_dir / "file1.txt"
>>> destination = new_dir / "folder_a" / "file1.txt"
>>> source.replace(destination)
WindowsPath('C:/Users/David/new_directory/folder_a/file1.txt')
```

> **Важливо**
>
> Якщо шлях призначення вже існує, `.replace()` перезаписує його вихідним файлом без будь-якого виключення. Це може призвести до небажаної втрати даних. Перевіряйте існування файлу:
>
> ```python
> if not destination.exists():
>     source.replace(destination)
> ```

Для видалення файлу використовується метод `.unlink()`:

```python
>>> file_path = new_dir / "program1.py"
>>> file_path.unlink()
>>> file_path.exists()
False
```

Якщо шлях не існує, виникає `FileNotFoundError`. Щоб ігнорувати помилку, встановіть параметр `missing_ok` у `True`:

```python
>>> file_path.unlink(missing_ok=True)
```

> **Важливо**
>
> При видаленні файлу він зникає назавжди. Переконайтеся, що справді хочете його видалити!

Для видалення **каталогу** використовується метод `.rmdir()`. Зверніть увагу: папка має бути порожньою, інакше виникає `OSError`:

```python
>>> for path in folder_d.iterdir():
...     path.unlink()
>>> folder_d.rmdir()
```

Якщо потрібно видалити непорожній каталог, використовується функція `rmtree()` з вбудованого модуля `shutil`:

```python
>>> import shutil
>>> folder_a = new_dir / "folder_a"
>>> shutil.rmtree(folder_a)
```

## 12.5 Читання та запис файлів

### Що таке файл?

Файл — це послідовність байтів, де байт — це число від 0 до 255. Байти у файлі мають бути декодовані у щось змістовне, щоб зрозуміти вміст файлу.

У Python є модулі стандартної бібліотеки для роботи з текстовими, CSV та аудіофайлами.

### Розуміння текстових файлів

Текстові файли — це файли, що містять лише текст. Є дві проблеми, що можуть ускладнити роботу з ними:

1. **Кодування символів:** при записуванні текстові файли перетворюють символи на байти (encoding), а при читанні — навпаки (decoding). Поширені кодування: ASCII, UTF-8, UTF-16, UTF-32. UTF-8 є найпоширенішим і може кодувати ширший набір символів, ніж ASCII.

2. **Закінчення рядків:** кожен рядок у текстовому файлі закінчується одним або двома символами. У Windows — поєднання `\r\n`, у macOS та Linux — лише `\n`.

### Об'єкти файлів Python

Файли представлені в Python об'єктами-файлами (file objects). Є два типи:
1. Текстові об'єкти-файли — для роботи з текстовими файлами.
2. Двійкові об'єкти-файли — для безпосередньої роботи з байтами.

Є два способи створити об'єкт-файл:

**Метод `Path.open()`:**

```python
>>> from pathlib import Path
>>> path = Path.home() / "hello.txt"
>>> path.touch()
>>> file = path.open(mode="r", encoding="utf-8")
```

**Вбудована функція `open()`:**

```python
>>> file_path = "C:/Users/David/hello.txt"
>>> file = open(file_path, mode="r", encoding="utf-8")
```

Таблиця режимів відкриття файлів:

| Режим | Опис |
|-------|------|
| `"r"` | Створює текстовий об'єкт-файл для читання; генерує помилку, якщо файл не відкривається. |
| `"w"` | Створює текстовий об'єкт-файл для запису; перезаписує всі наявні дані. |
| `"a"` | Створює текстовий об'єкт-файл для додавання даних у кінець файлу. |
| `"rb"` | Двійковий об'єкт-файл для читання. |
| `"wb"` | Двійковий об'єкт-файл для запису. |
| `"ab"` | Двійковий об'єкт-файл для додавання. |

> **Важливо**
>
> Завжди явно закривайте файл методом `.close()` або використовуйте оператор `with`.

### Оператор with

```python
with path.open(mode="r", encoding="utf-8") as file:
    # Робимо щось із файлом
```

Оператор `with` гарантує, що файл буде закрито автоматично — навіть якщо під час виконання коду всередині блоку виникне виняток. Це вважається Pythonic-способом роботи з файлами.

### Читання даних із файлу

```python
>>> path = Path.home() / "hello.txt"
>>> with path.open(mode="r", encoding="utf-8") as file:
...     text = file.read()
>>> text
'Hello World'
```

Метод `.read()` зчитує весь текст файлу та повертає його у вигляді рядка.

Якщо у файлі кілька рядків, можна перебирати їх за допомогою `.readlines()`:

```python
>>> with path.open(mode="r", encoding="utf-8") as file:
...     for line in file.readlines():
...         print(line, end="")
Hello World
Hello again
```

### Запис даних у файл

```python
>>> with path.open(mode="w", encoding="utf-8") as file:
...     file.write("Hi there!")
9
```

Метод `.write()` повертає кількість записаних символів.

> **Важливо**
>
> При встановленні `mode="w"` у `.open()` вміст оригінального файлу перезаписується. Це призводить до втрати всіх наявних даних у файлі!

Для додавання даних у кінець файлу використовується режим `"a"`:

```python
>>> with path.open(mode="a", encoding="utf-8") as file:
...     file.write("\nHello")
```

Для запису кількох рядків одночасно використовується метод `.writelines()`:

```python
>>> lines_of_text = [
...     "Hello from Line 1\n",
...     "Hello from Line 2\n",
...     "Hello from Line 3\n",
... ]
>>> with path.open(mode="w", encoding="utf-8") as file:
...     file.writelines(lines_of_text)
```

Якщо відкрити неіснуючий шлях у режимі запису, Python спробує автоматично створити файл за умови, що всі батьківські папки існують. Якщо потрібно створити і батьківські каталоги:

```python
>>> path.parent.mkdir(parents=True)
>>> with path.open(mode="w", encoding="utf-8") as file:
...     file.write("Hello!")
```

## 12.6 Читання та запис CSV-даних

Формат значень, розділених комами (CSV, comma-separated values), — це зручний спосіб зберігання записів послідовних даних.

Ось як вручну записати список температур у CSV-файл:

```python
>>> temperature_readings = [68, 65, 68, 70, 74, 72]
>>> from pathlib import Path
>>> file_path = Path.home() / "temperatures.txt"
>>> with file_path.open(mode="a", encoding="utf-8") as file:
...     file.write(str(temperature_readings[0]))
...     for temp in temperature_readings[1:]:
...         file.write(f",{temp}")
```

Отриманий рядок `"68,65,68,70,74,72"` можна відновити у список:

```python
>>> temperatures = text.split(",")
>>> int_temperatures = [int(temp) for temp in temperatures]
>>> int_temperatures
[68, 65, 68, 70, 74, 72]
```

Оскільки читання і запис CSV-файлів є дуже поширеними операціями, стандартна бібліотека Python має модуль `csv`.

### Модуль csv

```python
>>> import csv
```

#### Запис CSV-файлів із csv.writer

```python
>>> daily_temperatures = [
...     [68, 65, 68, 70, 74, 72],
...     [67, 67, 70, 72, 72, 70],
...     [68, 70, 74, 76, 74, 73],
... ]
>>> file_path = Path.home() / "temperatures.csv"
>>> with file_path.open(mode="w", encoding="utf-8") as file:
...     writer = csv.writer(file)
...     writer.writerows(daily_temperatures)
```

Головна перевага `csv.writer` — не потрібно переводити значення в рядки перед записом. Об'єкт `csv.writer` робить це за вас. Метод `.writerow()` записує один рядок, `.writerows()` — кілька рядків одночасно.

#### Читання CSV-файлів із csv.reader

```python
>>> daily_temperatures = []
>>> with file_path.open(mode="r", encoding="utf-8") as file:
...     reader = csv.reader(file)
...     for row in reader:
...         int_row = [int(value) for value in row]
...         daily_temperatures.append(int_row)
>>> daily_temperatures
[[68, 65, 68, 70, 74, 72], [67, 67, 70, 72, 72, 70], [68, 70, 74, 76, 74, 73]]
```

Кожен рядок CSV-файлу повертається у вигляді списку рядків.

#### Читання та запис CSV-файлів із заголовками

Для CSV-файлів із рядком заголовка зручніше використовувати `csv.DictReader`:

```python
# Приклад файлу employees.csv:
# name,department,salary
# Lee,Operations,75000.00
# Jane,Engineering,85000.00
# Diego,Sales,80000.00

>>> file_path = Path.home() / "employees.csv"
>>> with file_path.open(mode="r", encoding="utf-8") as file:
...     reader = csv.DictReader(file)
...     for row in reader:
...         print(row)
{'name': 'Lee', 'department': 'Operations', 'salary': '75000.00'}
{'name': 'Jane', 'department': 'Engineering', 'salary': '85000.00'}
{'name': 'Diego', 'department': 'Sales', 'salary': '80000.00'}
```

Замість повернення кожного рядка як списку, `DictReader` повертає кожен рядок як словник.

Для запису словників у CSV-файл використовується `csv.DictWriter`:

```python
>>> people = [
...     {"name": "Veronica", "age": 29},
...     {"name": "Audrey", "age": 32},
...     {"name": "Sam", "age": 24},
... ]
>>> file_path = Path.home() / "people.csv"
>>> with file_path.open(mode="w", encoding="utf-8") as file:
...     writer = csv.DictWriter(file, fieldnames=["name", "age"])
...     writer.writeheader()
...     writer.writerows(people)
```

Результуючий файл `people.csv`:

```
name,age
Veronica,29
Audrey,32
Sam,24
```
