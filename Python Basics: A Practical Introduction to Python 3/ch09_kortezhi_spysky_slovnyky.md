## 9.1 Кортежі — незмінні послідовності

Мабуть, найпростішою складеною структурою даних є послідовність елементів.

Послідовність — це впорядкований список значень. Кожному елементу в послідовності присвоюється ціле число, що називається індексом і визначає порядок появи значень. Так само як і в рядках, індекс першого значення в послідовності дорівнює `0`.

Наприклад, літери англійського алфавіту утворюють послідовність, перший елемент якої — `A`, а останній — `Z`. Рядки також є послідовностями. Рядок `"Python"` має шість елементів, починаючи з `"P"` з індексом `0` і `"n"` з індексом `5`.

Деякі реальні приклади послідовностей: значення, що видає датчик щосекунди, послідовність результатів тестів студента або послідовність щоденних значень акцій компанії за певний проміжок часу.

У цьому розділі ви навчитеся використовувати вбудований тип даних `tuple` для створення послідовностей значень.

### Що таке кортеж?

Слово «кортеж» (tuple) походить із математики, де використовується для опису скінченної впорядкованої послідовності значень.

Зазвичай математики записують кортежі у вигляді списку елементів, розділених комами, у дужках. `(1, 2, 3)` — це кортеж із трьох цілих чисел.

Кортежі є впорядкованими, оскільки їхні елементи з'являються в певному порядку. Перший елемент `(1, 2, 3)` — це `1`, другий — `2`, третій — `3`.

Python запозичив з математики як назву, так і нотацію для кортежів.

### Як створити кортеж

Є кілька способів створити кортеж у Python. Розглянемо два з них:

1. Літерали кортежів
2. Вбудована функція tuple()

#### Літерали кортежів

Так само як рядковий літерал — це рядок, явно створений шляхом оточення тексту лапками, літерал кортежу — це кортеж, явно записаний у вигляді розділеного комами списку значень в дужках.

Ось приклад літерала кортежу:

```python
>>> my_first_tuple = (1, 2, 3)
```

Це створює кортеж із цілих чисел `1`, `2` і `3` і присвоює його імені `my_first_tuple`.

Можна перевірити, що `my_first_tuple` є кортежем, за допомогою type():

```python
>>> type(my_first_tuple)
<class 'tuple'>
```

На відміну від рядків, що є послідовностями символів, кортежі можуть містити значення будь-якого типу, включно зі значеннями різних типів. Кортеж `(1, 2.0, "three")` є цілком допустимим.

Існує спеціальний кортеж, що не містить жодних значень. Він називається порожнім кортежем і створюється парою дужок без нічого між ними:

```python
>>> empty_tuple = ()
```

На перший погляд порожній кортеж може здатися дивною і непотрібною концепцією, але насправді він цілком практичний.

Як ви думаєте, як створити кортеж рівно з одним елементом? Спробуйте в IDLE:

```python
>>> x = (1)
>>> type(x)
<class 'int'>
```

Коли ви оточуєте значення дужками, але не включаєте жодної коми, Python інтерпретує значення не як кортеж, а як тип значення всередині дужок. Тому `(1)` — це просто химерний спосіб записати ціле число `1`.

Щоб створити кортеж із одним елементом `1`, потрібно включити кому після `1`:

```python
>>> x = (1,)
>>> type(x)
<class 'tuple'>
```

#### Вбудована функція tuple()

Також можна використовувати вбудовану функцію tuple() для створення кортежу з іншого типу послідовності, наприклад рядка:

```python
>>> tuple("Python")
('P', 'y', 't', 'h', 'o', 'n')
```

tuple() приймає лише один параметр, тому не можна просто перелічити потрібні значення як окремі аргументи. Якщо спробувати, Python генерує `TypeError`:

```python
>>> tuple(1, 2, 3)
Traceback (most recent call last):
  File "<pyshell#0>", line 1, in <module>
    tuple(1, 2, 3)
TypeError: tuple expected at most 1 arguments, got 3
```

Одиночний параметр tuple() є необов'язковим, і його пропуск повертає порожній кортеж:

```python
>>> tuple()
()
```

Однак більшість Python-програмістів надає перевагу коротшому `()` для створення порожнього кортежу.

### Схожість між кортежами та рядками

Кортежі та рядки мають багато спільного. Обидва є типами послідовностей із скінченною довжиною, підтримують індексування та зрізи, є незмінними і можуть бути перебрані в циклі.

Головна відмінність між рядками та кортежами полягає в тому, що елементи кортежів можуть бути будь-якого типу, тоді як рядки можуть містити лише символи.

**Кортежі мають довжину.** Так само як рядки, для визначення довжини кортежу можна використовувати функцію len():

```python
>>> numbers = (1, 2, 3)
>>> len(numbers)
3
```

**Кортежі підтримують індексування та зрізи.** Кортежі підтримують нотацію індексів і нотацію зрізів так само, як і рядки:

```python
>>> values = (1, 3, 5, 7, 9)
>>> values[2]
5
>>> values[2:4]
(5, 7)
```

**Кортежі є незмінними.** Так само як рядки, кортежі є незмінними. Якщо спробувати змінити значення за якимось індексом кортежу, Python генерує `TypeError`:

```python
>>> values[0] = 2
Traceback (most recent call last):
  File "<pyshell#1>", line 1, in <module>
    values[0] = 2
TypeError: 'tuple' object does not support item assignment
```

**Кортежі є ітерованими.** Так само як рядки, кортежі є ітерованими, тому їх можна перебирати в циклі:

```python
>>> vowels = ("a", "e", "i", "o", "u")
>>> for vowel in vowels:
...     print(vowel.upper())
A
E
I
O
U
```

### Пакування та розпакування кортежів

Є й третій, хоча менш поширений спосіб створення кортежу. Можна записати розділений комами список значень без дужок:

```python
>>> coordinates = 4.21, 9.29
>>> type(coordinates)
<class 'tuple'>
```

Якщо значення можна пакувати в кортеж, логічно, що їх також можна розпаковувати:

```python
>>> x, y = coordinates
>>> x
4.21
>>> y
9.29
```

Тут значення з одного кортежу `coordinates` розпаковуються в дві окремі змінні `x` і `y`.

Поєднуючи пакування та розпакування кортежів, можна виконувати множинне присвоєння змінних в одному рядку:

```python
>>> name, age, occupation = "David", 34, "programmer"
>>> name
'David'
>>> age
34
>>> occupation
'programmer'
```

> **Примітка**
>
> Хоча присвоєння кількох змінних в одному рядку може скоротити кількість рядків програми, варто стримуватися від призначення надто багатьох значень в одному рядку. Присвоєння більше двох-трьох змінних таким чином може ускладнити визначення, яке значення присвоєне якому імені.

Кількість імен змінних зліва від виразу присвоєння має дорівнювати кількості значень у кортежі праворуч, інакше Python генерує `ValueError`:

```python
>>> a, b, c, d = 1, 2, 3
ValueError: not enough values to unpack (expected 4, got 3)
```

### Перевірка наявності значень за допомогою in

Можна перевірити, чи міститься значення в кортежі, за допомогою ключового слова `in`:

```python
>>> vowels = ("a", "e", "i", "o", "u")
>>> "o" in vowels
True
>>> "x" in vowels
False
```

### Повернення кількох значень із функції

Одне з поширених застосувань кортежів — повернення кількох значень із однієї функції:

```python
>>> def adder_subtractor(num1, num2):
...     return (num1 + num2, num1 - num2)
...
>>> adder_subtractor(3, 2)
(5, 1)
```

## 9.2 Списки — змінні послідовності

Структура даних `list` — це ще один тип послідовності в Python. Так само як рядки та кортежі, списки містять елементи, що індексуються цілими числами, починаючи з `0`.

На поверхні списки дуже схожі на кортежі за виглядом і поведінкою. Зі списками можна використовувати нотацію індексів і зрізів, перевіряти існування елемента за допомогою `in` та перебирати списки в циклі `for`.

На відміну від кортежів, однак, списки є змінними (mutable) — це означає, що значення за індексом можна змінити навіть після того, як список створено.

### Створення списків

Літерал списку виглядає майже так само, як літерал кортежу, за винятком того, що він оточений квадратними дужками (`[` і `]`) замість круглих:

```python
>>> colors = ["red", "yellow", "green", "blue"]
>>> type(colors)
<class 'list'>
```

Список можна також створити за допомогою вбудованої функції list() з будь-якого іншого типу послідовності або з рядка за допомогою методу .split():

```python
>>> list((1, 2, 3))
[1, 2, 3]
>>> list("Python")
['P', 'y', 't', 'h', 'o', 'n']
>>> groceries = "eggs, milk, cheese"
>>> grocery_list = groceries.split(", ")
>>> grocery_list
['eggs', 'milk', 'cheese']
```

Рядок, що передається до .split(), називається роздільником (separator). Змінюючи роздільник, можна розбивати рядки на списки різними способами:

```python
>>> "a;b;c".split(";")
['a', 'b', 'c']
>>> "The quick brown fox".split(" ")
['The', 'quick', 'brown', 'fox']
```

### Базові операції зі списками

Операції індексування та зрізів працюють зі списками так само, як із кортежами:

```python
>>> numbers = [1, 2, 3, 4]
>>> numbers[1]
2
>>> numbers[1:3]
[2, 3]
>>> "Bob" in numbers
False
```

### Зміна елементів у списку

Щоб замінити значення в списку, призначте нове значення слоту за допомогою нотації індексів:

```python
>>> colors = ["red", "yellow", "green", "blue"]
>>> colors[0] = "burgundy"
>>> colors
['burgundy', 'yellow', 'green', 'blue']
```

Також можна змінити кілька значень одночасно за допомогою присвоєння зрізів:

```python
>>> colors[1:3] = ["orange", "magenta"]
>>> colors
['burgundy', 'orange', 'magenta', 'blue']
```

Список, що призначається зрізу, не обов'язково має мати ту саму довжину, що й зріз:

```python
>>> colors = ["red", "yellow", "green", "blue"]
>>> colors[1:3] = ["orange", "magenta", "aqua"]
>>> colors
['red', 'orange', 'magenta', 'aqua', 'blue']
```

### Методи списків для додавання та видалення елементів

**list.insert()**

Метод .insert() вставляє нове значення в список на вказаному індексі. Він приймає два параметри — індекс `i` і значення `x` — і вставляє значення `x` за індексом `i`:

```python
>>> colors = ["red", "yellow", "green", "blue"]
>>> colors.insert(1, "orange")
>>> colors
['red', 'orange', 'yellow', 'green', 'blue']
```

> **Важливо**
>
> Викликаючи .insert() на списку, не потрібно присвоювати результат оригінальному списку. Наприклад, наступний код фактично стирає список `colors`:
>
> ```python
> >>> colors = colors.insert(-1, "indigo")
> >>> print(colors)
> None
> ```
>
> .insert() змінює `colors` «на місці» (in place). Це стосується всіх методів списків, що не повертають значення.

**list.pop()**

Метод .pop() приймає один параметр — індекс `i` — і видаляє значення зі списку за цим індексом. Видалене значення повертається методом:

```python
>>> color = colors.pop(3)
>>> color
'green'
>>> colors
['red', 'orange', 'yellow', 'blue', 'indigo', 'violet']
```

Якщо не передати значення до .pop(), він видаляє останній елемент списку:

```python
>>> colors.pop()
'indigo'
>>> colors
['red', 'orange', 'yellow', 'blue']
```

**list.append()**

Метод .append() використовується для додавання нового елемента в кінець списку:

```python
>>> colors.append("indigo")
>>> colors
['red', 'orange', 'yellow', 'blue', 'indigo']
```

**list.extend()**

Метод .extend() використовується для додавання кількох нових елементів у кінець списку:

```python
>>> colors.extend(["violet", "ultraviolet"])
>>> colors
['red', 'orange', 'yellow', 'blue', 'indigo', 'violet', 'ultraviolet']
```

.extend() приймає один параметр, що має бути ітерованим типом. Елементи ітерованого об'єкта додаються до списку в тому порядку, в якому вони з'являються в аргументі.

Підсумкова таблиця методів:

| Метод списку | Опис |
|-------------|------|
| `.insert(i, x)` | Вставити значення `x` за індексом `i` |
| `.append(x)` | Вставити значення `x` в кінець списку |
| `.extend(iterable)` | Вставити всі значення ітерованого об'єкта в кінець списку по порядку |
| `.pop(i)` | Видалити і повернути елемент за індексом `i` |

### Списки чисел

Для підсумовування всіх значень у списку чисел є вбудована функція sum():

```python
>>> sum([1, 2, 3, 4, 5])
15
```

Також є функції min() і max(), що повертають мінімальне та максимальне значення у списку:

```python
>>> min([1, 2, 3, 4, 5])
1
>>> max([1, 2, 3, 4, 5])
5
```

Зверніть увагу: sum(), min() і max() також працюють із кортежами.

### Включення списків

Ще один спосіб створити список з існуючого ітерованого об'єкта — використати включення списку (list comprehension):

```python
>>> numbers = (1, 2, 3, 4, 5)
>>> squares = [num**2 for num in numbers]
>>> squares
[1, 4, 9, 16, 25]
```

Включення списку — це скорочена форма циклу `for`. У наведеному прикладі включення перебирає кожне число в `numbers`, зводить його в квадрат і додає до нового списку `squares`.

Включення списків часто використовуються для перетворення значень одного списку на інший тип:

```python
>>> str_numbers = ["1.5", "2.3", "5.25"]
>>> float_numbers = [float(value) for value in str_numbers]
>>> float_numbers
[1.5, 2.3, 5.25]
```

Якщо ви помічаєте, що створюєте порожній список, перебираєте якийсь ітерований об'єкт і додаєте нові елементи до списку, швидше за все, можна замінити цей код включенням списку!

## 9.3 Вкладення, копіювання та сортування кортежів і списків

### Вкладення списків і кортежів

Списки та кортежі можуть містити значення будь-якого типу, включно з іншими списками та кортежами. Вкладений список або вкладений кортеж — це список або кортеж, що міститься як значення в іншому списку або кортежі.

```python
>>> two_by_two = [[1, 2], [3, 4]]
>>> len(two_by_two)
2
>>> two_by_two[0]
[1, 2]
>>> two_by_two[1][0]
3
```

У дуже спрощеному вигляді список списків або кортеж кортежів можна уявити як таблицю з рядками та стовпцями.

> **Примітка**
>
> Читачів, зацікавлених в аналізі даних або наукових обчисленнях, список списків може нагадати матрицю значень. Хоча для матриць можна використовувати вбудовані типи `list` і `tuple`, існують кращі альтернативи. Дізнайтеся, як працювати з матрицями в Python, у Розділі 17.

### Копіювання списку

Іноді потрібно скопіювати один список в інший. Однак не можна просто присвоїти один об'єкт-список іншому, оскільки можна отримати несподіваний результат:

```python
>>> animals = ["lion", "tiger", "frumious Bandersnatch"]
>>> large_cats = animals
>>> large_cats.append("Tigger")
>>> animals
['lion', 'tiger', 'frumious Bandersnatch', 'Tigger']
```

Коли ви пишете `large_cats = animals`, обидві змінні посилаються на один і той самий об'єкт. Будь-які зміни в одній змінній вплинуть на іншу.

Щоб отримати незалежну копію списку, використовуйте нотацію зрізів для повернення нового списку з тими самими значеннями:

```python
>>> animals = ["lion", "tiger", "frumious Bandersnatch"]
>>> large_cats = animals[:]
>>> large_cats.append("leopard")
>>> large_cats
['lion', 'tiger', 'frumious Bandersnatch', 'leopard']
>>> animals
['lion', 'tiger', 'frumious Bandersnatch']
```

> **Примітка**
>
> Цей метод копіювання списку в програмістській термінології називається поверхневим копіюванням (shallow copy). Для копіювання як самого списку, так і всіх елементів, що він містить, необхідно використати так зване глибоке копіювання (deep copy). Докладніше про поверхневе та глибоке копіювання можна дізнатися в статті «Shallow vs Deep Copying of Python Objects» на realpython.com.

### Сортування списків

Списки мають метод .sort(), що сортує всі елементи у зростаючому порядку. За замовчуванням список сортується в алфавітному або числовому порядку залежно від типу елементів:

```python
>>> colors = ["red", "yellow", "green", "blue"]
>>> colors.sort()
>>> colors
['blue', 'green', 'red', 'yellow']
>>> numbers = [1, 10, 5, 3]
>>> numbers.sort()
>>> numbers
[1, 3, 5, 10]
```

Зверніть увагу: .sort() сортує список «на місці», тому результат не потрібно нікому присвоювати.

.sort() має необов'язковий параметр `key`, що можна використовувати для налаштування сортування. Параметр `key` приймає функцію, і список сортується на основі значення, яке повертає ця функція:

```python
>>> colors = ["red", "yellow", "green", "blue"]
>>> colors.sort(key=len)
>>> colors
['red', 'blue', 'green', 'yellow']
```

Також можна передавати власні функції до `key`:

```python
>>> def get_second_element(item):
...     return item[1]
...
>>> items = [(4, 1), (1, 2), (-9, 0)]
>>> items.sort(key=get_second_element)
>>> items
[(-9, 0), (4, 1), (1, 2)]
```

## 9.6 Зберігання зв'язків у словниках

Одна з найкорисніших структур даних у Python — словник.

### Що таке словник?

У звичайному розумінні словник — це книга, що містить визначення слів. Кожен запис у словнику має дві частини: слово, що визначається, і його визначення.

Словники Python, як списки та кортежі, зберігають колекцію об'єктів. Однак замість зберігання об'єктів у послідовності словники зберігають інформацію у парах даних, що називаються парами ключ-значення (key-value pairs). Тобто кожен об'єкт у словнику має дві частини: ключ і значення.

Ключ у парі ключ-значення — це унікальне ім'я, що ідентифікує частину значення. Порівнюючи це з англійським словником: ключ — це слово, що визначається, а значення — визначення слова.

Наприклад, можна використовувати словник для зберігання назв штатів та їхніх столиць:

| Ключ | Значення |
|------|---------|
| `"California"` | `"Sacramento"` |
| `"New York"` | `"Albany"` |
| `"Texas"` | `"Austin"` |

У таблиці вище ключами словника є назви штатів, а значеннями — назви столиць.

### Створення словників

Наступний код створює літерал словника, що містить назви штатів та їхні столиці:

```python
>>> capitals = {
...     "California": "Sacramento",
...     "New York": "Albany",
...     "Texas": "Austin",
... }
```

Зверніть увагу: кожен ключ відділяється від значення двокрапкою (`:`), кожна пара ключ-значення розділяється комою (`,`), а весь словник укладено у фігурні дужки (`{` і `}`).

Також можна створити словник із послідовності кортежів за допомогою вбудованої функції dict():

```python
>>> key_value_pairs = (
...     ("California", "Sacramento"),
...     ("New York", "Albany"),
...     ("Texas", "Austin"),
... )
>>> capitals = dict(key_value_pairs)
```

> **Примітка**
>
> Якщо ви працюєте з версією Python старіше ніж 3.6, ви помітите, що порядок пар ключ-значення у словнику відрізнятиметься від наведеного в прикладах. До Python 3.6 порядок пар ключ-значення у словнику був випадковим. У пізніших версіях порядок гарантовано відповідає порядку їхнього вставляння.

### Доступ до значень словника

Щоб отримати доступ до значення в словнику, укладіть відповідний ключ у квадратні дужки:

```python
>>> capitals["Texas"]
'Austin'
```

### Додавання та видалення значень у словнику

Словники, як і списки, є змінними структурами даних. Це означає, що елементи можна додавати та видаляти.

Щоб додати нову пару ключ-значення:

```python
>>> capitals["Colorado"] = "Denver"
```

Кожному ключу в словнику можна присвоїти лише одне значення. Якщо ключу присвоюється нове значення, Python просто перезаписує старе.

Щоб видалити елемент зі словника, використовується ключове слово `del`:

```python
>>> del capitals["Texas"]
```

### Перевірка існування ключів словника

Якщо спробувати отримати доступ до значення за неіснуючим ключем, Python генерує `KeyError`:

```python
>>> capitals["Arizona"]
KeyError: 'Arizona'
```

Перевірити наявність ключа можна за допомогою ключового слова `in`:

```python
>>> "Arizona" in capitals
False
>>> "California" in capitals
True
```

Важливо пам'ятати, що `in` перевіряє лише наявність ключів:

```python
>>> "Sacramento" in capitals
False
```

### Перебір словників

При перебиранні словника в циклі `for` ви перебираєте ключі словника:

```python
>>> for key in capitals:
...     print(key)
California
New York
Colorado
```

Для одночасного перебирання ключів і значень використовується метод .items():

```python
>>> for state, capital in capitals.items():
...     print(f"The capital of {state} is {capital}")
The capital of California is Sacramento
The capital of New York is Albany
The capital of Colorado is Denver
```

### Ключі словника та незмінність

Є лише одне обмеження щодо того, що є допустимим ключем словника. Дозволяються лише незмінні типи. Наприклад, список не може бути ключем словника:

```python
>>> capitals[[1, 2, 3]] = "Bad"
TypeError: unhashable type: 'list'
```

Для довідки, ось список усіх типів даних, що є допустимими ключами словника:

- цілі числа
- числа з рухомою комою
- рядки
- булеві значення
- кортежі

На відміну від ключів, значення словника можуть бути будь-яким допустимим типом Python, включно з іншими словниками!

### Вкладені словники

Так само як можна вкладати списки в інші списки, можна створювати вкладені словники:

```python
>>> states = {
...     "California": {
...         "capital": "Sacramento",
...         "flower": "California Poppy"
...     },
...     "New York": {
...         "capital": "Albany",
...         "flower": "Rose"
...     },
...     "Texas": {
...         "capital": "Austin",
...         "flower": "Bluebonnet"
...     },
... }
>>> states["Texas"]["flower"]
'Bluebonnet'
```

## 9.8 Як вибрати структуру даних

У цьому розділі ви дізналися про три структури даних, притаманні Python: списки, кортежі та словники.

Нижче наведено рекомендації щодо вибору відповідної структури:

**Використовуйте список, коли:**

- потрібно оновлювати або змінювати дані під час виконання програми;
- основне призначення структури даних — перебирання.

**Використовуйте кортеж, коли:**
- дані мають природний порядок;
- не потрібно оновлювати або змінювати дані під час виконання програми;
- основне призначення структури даних — перебирання.

**Використовуйте словник, коли:**
- дані є невпорядкованими або порядок не має значення;
- потрібно оновлювати або змінювати дані під час виконання програми;
- основне призначення структури даних — пошук значень.

## 9.10 Підсумок та додаткові ресурси

У цьому розділі ви дізналися про три структури даних: списки, кортежі та словники.

Списки, наприклад `[1, 2, 3, 4]`, — це змінні послідовності об'єктів. Взаємодіяти зі списками можна за допомогою різних методів, таких як .append(), .remove() і .extend(). Списки можна сортувати методом .sort(). Доступ до окремих елементів списку здійснюється за допомогою нотації індексів, так само як і до рядків. Нотація зрізів також працює зі списками.

Кортежі, як і списки, є послідовностями об'єктів. Головна відмінність між списками та кортежами полягає в тому, що кортежі є незмінними. Після створення кортежу його не можна змінити. Так само як списки, можна отримувати доступ до елементів за індексом і за допомогою нотації зрізів.

Словники зберігають дані у вигляді пар ключ-значення. Вони не є послідовностями, тому не можна звертатися до елементів за індексом. Натомість до елементів звертаються за ключем. Словники чудово підходять для зберігання зв'язків або коли потрібен швидкий доступ до даних. Як і списки, словники є змінними.

Списки, кортежі та словники є ітерованими, тобто їх можна перебирати в циклі. Ви побачили, як перебирати всі три ці структури за допомогою циклів `for`.
