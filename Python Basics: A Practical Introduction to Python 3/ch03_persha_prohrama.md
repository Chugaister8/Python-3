## 3.1 Написання Python-скрипту

Якщо IDLE ще не відкрито, відкрийте його. У IDLE є два основні вікна, з якими ви будете працювати: інтерактивне вікно, що відкривається під час запуску IDLE, та вікно скрипту.

Вводити код можна як в інтерактивному вікні, так і у вікні скрипту. Різниця між ними полягає у способі виконання коду. У цьому розділі ви напишете свою першу програму на Python і навчитеся запускати її в обох вікнах.

### Інтерактивне вікно

Інтерактивне вікно містить оболонку Python — текстовий інтерфейс для взаємодії з мовою Python. Звідси й назва «інтерактивне вікно».

Коли ви вперше відкриваєте IDLE, відображається приблизно такий текст:

```
Python 3.8.1 (tags/v3.8.1:1b293b6, Dec 18 2019, 22:39:24)
[MSC v.1916 32 bit (Intel)] on win32

Type "help", "copyright", "credits" or "license" for more information.
>>>
```

Перший рядок повідомляє, яка версія Python запущена. У цьому випадку IDLE працює з Python 3.8.1. Другий і третій рядки містять деяку інформацію про операційну систему та команди, за допомогою яких можна дізнатися більше про Python.

Символ `>>>` в останньому рядку — це запрошення (prompt). Саме тут ви вводитимете код. Введіть `1 + 1` у запрошення і натисніть `Enter`.

Коли ви натиснете `Enter`, Python обчислить вираз, відобразить результат `2`, а потім знову запросить введення:

```python
>>> 1 + 1
2
>>>
```

Зверніть увагу: після результату знову з'являється запрошення Python `>>>`. Python готовий до нових інструкцій! Щоразу після виконання коду новий рядок запрошення з'являється одразу під виведеними даними.

Послідовність подій в інтерактивному вікні можна описати як цикл із трьох кроків:

1. Спочатку Python зчитує код, введений у запрошення.
2. Потім код обчислюється.
3. Нарешті, результат виводиться у вікні і відображається нове запрошення.

Цей цикл зазвичай називають Read-Evaluate-Print Loop, або REPL. Програмісти на Python іноді називають оболонку Python «Python REPL» або просто «REPL».

> **Примітка**
>
> Надалі фінальне запрошення `>>>`, що відображається після виконання коду в інтерактивному вікні, у прикладах коду не наводиться.

Спробуймо щось цікавіше, ніж додавання двох чисел. Обов'язковий ритуал для кожного програміста — написати свою першу програму «Hello, world», яка виводить фразу «Hello, world» на екран.

Щоб вивести текст на екран у Python, використовується функція print(). Функція — це фрагмент коду, який зазвичай приймає якісь вхідні дані, що називаються аргументом, виконує над ними певні дії та повертає результат, що називається значенням, яке повертається.

Грубо кажучи, функції в коді працюють подібно до математичних функцій. Наприклад, математична функція A(r) = πr² приймає радіус r кола як вхідні дані і повертає площу кола.

> **Важливо**
>
> Однак аналогія з математичними функціями не є бездоганною, оскільки функції в коді можуть мати побічні ефекти. Побічний ефект виникає щоразу, коли функція виконує деяку операцію, що змінює щось у програмі або на комп'ютері, де вона виконується.
>
> Наприклад, можна написати функцію на Python, яка приймає ім'я людини як вхідні дані, зберігає ім'я у файлі на комп'ютері, а потім повертає шлях до цього файлу. Операція збереження імені у файлі є побічним ефектом функції.
>
> Докладніше про функції, зокрема про те, як їх писати самостійно, ви дізнаєтеся у Розділі 6.

Функція print() у Python приймає деякий текст як вхідні дані і відображає його на екрані. Щоб скористатися print(), введіть слово `print` у запрошення інтерактивного вікна, а після нього — текст `"Hello, world"` у дужках:

```python
>>> print("Hello, world")
Hello, world
```

Тут `"Hello, world"` — це аргумент, що передається до print(). `"Hello, world"` слід писати в лапках, щоб Python розпізнав це як текст, а не як щось інше.

> **Примітка**
>
> Коли ви вводите код в інтерактивному вікні, ви можете помітити, що колір шрифту змінюється для певних частин коду. IDLE виділяє частини коду різними кольорами, щоб полегшити розпізнавання їхнього призначення.
>
> За замовчуванням вбудовані функції, такі як print(), відображаються фіолетовим кольором, а текст — зеленим.

Інтерактивне вікно може виконувати лише один рядок коду за раз. Це зручно для випробування невеликих прикладів коду та дослідження мови Python, але має суттєве обмеження: код вводиться людиною по одному рядку за раз!

Альтернативно, можна зберегти код Python у текстовому файлі, а потім виконати весь код у файлі однією командою. Код у файлі називається скриптом, а файли, що містять Python-скрипти, — файлами скриптів.

Файли скриптів зручні не лише тим, що спрощують запуск програми, але й тим, що ними можна ділитися з іншими людьми, щоб вони теж могли запускати вашу програму.

### Вікно скрипту

Скрипти пишуться у вікні скрипту IDLE. Відкрити вікно скрипту можна, вибравши Файл → Новий файл (File → New File) у меню у верхній частині інтерактивного вікна.

Зверніть увагу: коли відкривається вікно скрипту, інтерактивне вікно залишається відкритим. Будь-який результат, що генерується кодом, запущеним у вікні скрипту, відображається в інтерактивному вікні, тому, можливо, ви захочете розташувати обидва вікна так, щоб бачити їх одночасно.

У вікні скрипту введіть той самий код, що використовувався для виведення `"Hello, world"` в інтерактивному вікні:

```python
print("Hello, world")
```

Так само як і в інтерактивному вікні, код у вікні скрипту підсвічується.

> **Важливо**
>
> Коли ви пишете код у скрипті, вам не потрібно включати запрошення `>>>`, яке відображається в інтерактивному вікні IDLE. Пам'ятайте про це, якщо копіюєте і вставляєте код із прикладів, що містять запрошення REPL.
>
> Утім, пам'ятайте: копіювати і вставляти приклади з книги не рекомендується. Самостійне набирання кожного прикладу справді дається взнаки!

Перш ніж запустити скрипт, його необхідно зберегти. У меню у верхній частині вікна виберіть Файл → Зберегти (File → Save) і збережіть скрипт під назвою `hello_world.py`. Розширення файлу `.py` — це загальноприйняте розширення, що вказує на те, що файл містить код на Python.

Якщо зберегти скрипт із будь-яким іншим розширенням, підсвічування коду зникне і весь текст у файлі відображатиметься чорним. IDLE підсвічує Python-код лише тоді, коли він зберігається у файлі `.py`.

Після збереження скрипту, щоб запустити програму, достатньо вибрати Запуск → Запустити модуль (Run → Run Module) у вікні скрипту — і ви побачите `Hello, world` в інтерактивному вікні:

```
Hello, world
```

> **Примітка**
>
> Також можна натиснути `F5`, щоб запустити скрипт із вікна скрипту.

Щоразу після запуску скрипту в інтерактивному вікні з'являтиметься приблизно таке виведення:

```
>>> =================== RESTART ===================
```

Так IDLE розділяє результати окремих запусків скрипту. Інакше, якщо запускати один скрипт за іншим, може бути незрозуміло, який результат до якого скрипту належить.

Щоб відкрити наявний скрипт у IDLE, виберіть Файл → Відкрити... (File → Open...) у меню вікна скрипту або інтерактивного вікна. Потім знайдіть і виберіть потрібний файл скрипту. IDLE відкриває скрипти в новому вікні скрипту, тому одночасно можна мати кілька відкритих скриптів.

> **Примітка**
>
> Подвійне клацання на файлі `.py` у файловому менеджері, наприклад у Windows Explorer, справді виконує скрипт у новому вікні. Однак вікно закривається одразу після завершення роботи скрипту — найчастіше до того, як ви встигаєте побачити, що сталося.
>
> Щоб відкрити файл у IDLE і запустити його, бачачи результат, клацніть на значку файлу правою кнопкою миші (`Ctrl+Click` на macOS) і виберіть «Редагувати в IDLE» (Edit with IDLE).

## 3.2 Навмисне зіпсуємо код

Усі припускаються помилок — особливо під час програмування! Якщо ви ще не зробили жодної помилки, давайте навмисне щось зламаємо, щоб побачити, що станеться.

Помилки в програмі називаються похибками (errors), і є два основні їхні типи:

1. Синтаксичні помилки (syntax errors)
2. Помилки часу виконання (run-time errors)

У цьому розділі ви побачите приклади помилок у коді та навчитеся використовувати виведення, яке Python відображає при виникненні помилки, щоб зрозуміти, яка саме помилка сталася і який фрагмент коду її спричинив.

### Синтаксичні помилки

Грубо кажучи, синтаксична помилка виникає, коли ви пишете код, неприпустимий у мові Python. Щоб спричинити синтаксичну помилку, змініть вміст скрипту `hello_world.py` з попереднього розділу на такий:

```python
print("Hello, world)
```

У цьому прикладі видалено подвійні лапки в кінці `"Hello, world"`. Python не зможе визначити, де закінчується рядок тексту. Збережіть змінений скрипт і спробуйте запустити його. Що станеться?

Код не запуститься! IDLE відобразить діалогове вікно з таким повідомленням:

```
EOL while scanning string literal.
```

EOL означає End Of Line («кінець рядка»), тому це повідомлення говорить про те, що Python дійшов до кінця рядка, не знайшовши завершення того, що називається рядковим літералом.

Рядковий літерал — це текст, укладений між двома подвійними лапками. Текст `"Hello, world"` є прикладом рядкового літерала.

> **Примітка**
>
> Для стислості рядкові літерали часто називають рядками (strings), хоча технічно термін «рядок» у Python має ширше значення. Докладніше про рядки ви дізнаєтеся в Розділі 4.

У вікні скрипту зверніть увагу: рядок, що містить `"Hello, world`, виділено червоним кольором. Ця зручна функція допомагає швидко знайти рядок коду, що спричинив синтаксичну помилку.

### Помилки часу виконання

IDLE виявляє синтаксичні помилки до запуску програми, але деякі помилки неможливо виявити до моменту виконання програми. Такі помилки називаються помилками часу виконання, оскільки вони виникають лише під час запуску програми.

Щоб спричинити помилку часу виконання, змініть код у `hello_world.py` на такий:

```python
print(Hello, world)
```

Тепер видалено обидві лапки навколо фрази `"Hello, world"`. Ви помітили, як колір тексту змінився на чорний після видалення лапок? IDLE більше не розпізнає `Hello, world` як рядок.

Як ви думаєте, що станеться при запуску скрипту? Спробуйте і побачите!

В інтерактивному вікні відобразиться червоний текст:

```
Traceback (most recent call last):
  File "/home/hello_world.py", line 1, in <module>
    print(Hello, world)
NameError: name 'Hello' is not defined
```

Що сталося? Під час спроби виконати програму Python згенерував помилку. Щоразу, коли виникає помилка, Python припиняє виконання програми і відображає помилку в інтерактивному вікні IDLE.

Текст, що відображається при помилці, називається трасуванням (traceback). Трасування надає корисну інформацію про помилку. Наведене вище трасування повідомляє нам:

- Помилка сталася на рядку 1 файлу `hello_world.py`.
- Рядок, що спричинив помилку: `print(Hello, world)`.
- Виникла помилка `NameError`.
- Конкретна помилка: `name 'Hello' is not defined`.

Лапки навколо `Hello, world` відсутні, тому Python не розуміє, що це рядок тексту. Натомість Python вважає, що `Hello` і `world` — це імена чогось іншого в коді. Оскільки імена `Hello` і `world` ніде не визначені, програма завершується аварійно.

У наступному розділі ви побачите, як визначати імена для значень у коді. Але перед тим як рухатися далі, можете попрактикуватися із синтаксичними помилками та помилками часу виконання, виконавши завдання для повторення.

## 3.3 Створення змінної

У Python змінні — це імена, яким можна присвоїти значення і використовувати їх для посилання на це значення в будь-якому місці коду. Змінні є фундаментальними для програмування з двох причин:

1. **Змінні зберігають доступ до значень:** наприклад, результат тривалої операції можна присвоїти змінній, щоб не виконувати цю операцію щоразу, коли потрібен результат.
2. **Змінні надають значенням контекст:** число 28 може означати різне — кількість учнів у класі або кількість разів, коли користувач відвідав сайт тощо. Назвавши значення 28 чимось на кшталт `num_students`, ми чітко вказуємо на його зміст.

У цьому розділі ви навчитеся використовувати змінні у своєму коді, а також познайомитеся з деякими угодами, яких дотримуються Python-програмісти при виборі імен змінних.

### Оператор присвоєння

Значення присвоюються змінній за допомогою спеціального символу `=`, що називається оператором присвоєння. Оператор — це символ, наприклад `=` або `+`, який виконує певну операцію над одним або кількома значеннями.

Наприклад, оператор `+` бере два числа — одне зліва від оператора і одне справа — та додає їх. Аналогічно, оператор `=` бере значення праворуч від оператора і присвоює його імені ліворуч.

Щоб побачити оператор присвоєння в дії, змінимо програму «Hello, world» з попереднього розділу. Цього разу ми використаємо змінну для зберігання тексту перед його виведенням на екран:

```python
>>> phrase = "Hello, world"
>>> print(phrase)
Hello, world
```

У першому рядку створюється змінна з іменем `phrase`, якій за допомогою оператора `=` присвоюється значення `"Hello, world"`. Рядок `"Hello, world"`, що спочатку використовувався всередині дужок функції print(), замінено змінною `phrase`.

Виведення `Hello, world` відображається після виконання `print(phrase)`, тому що Python знаходить ім'я `phrase` і бачить, що йому присвоєно значення `"Hello, world"`.

Якби ви не виконали `phrase = "Hello, world"` перед виконанням `print(phrase)`, ви б побачили помилку `NameError`, як у попередньому розділі при спробі виконати `print(Hello, world)`.

> **Примітка**
>
> Хоча `=` виглядає як знак рівності з математики, у Python він має інше значення. Важливо розрізняти оператор `=` і знак рівності — це може стати джерелом плутанини для програмістів-початківців.
>
> Просто пам'ятайте: щоразу, коли ви бачите оператор `=`, те, що стоїть праворуч від нього, присвоюється змінній ліворуч.

Імена змінних чутливі до регістру, тому змінна з іменем `phrase` відрізняється від змінної з іменем `Phrase` (зверніть увагу на велику літеру P). Наприклад, наступний код спричиняє помилку `NameError`:

```python
>>> phrase = "Hello, world"
>>> print(Phrase)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'Phrase' is not defined
```

Якщо у вас виникають проблеми з прикладами коду з цієї книги, обов'язково перевіряйте, чи кожен символ у вашому коді — включно з пробілами — точно відповідає прикладам. Комп'ютер не може здогадатися, що ви мали на увазі, тому «майже правильно» не змусить його зробити правильну річ!

### Правила для допустимих імен змінних

Імена змінних можуть бути будь-якої довжини, але є кілька обов'язкових правил. Імена змінних можуть містити лише великі та малі літери (A–Z, a–z), цифри (0–9) та підкреслення (`_`). Однак ім'я змінної не може починатися з цифри.

Наприклад, `phrase`, `string1`, `_a1p4a` та `list_of_names` — допустимі імена змінних, а `9lives` — ні.

> **Примітка**
>
> Імена змінних у Python можуть містити багато різних допустимих символів Unicode. Unicode — це стандарт цифрового представлення тексту, що використовується в більшості писемностей світу.
>
> Це означає, що імена змінних можуть містити літери з неанглійських алфавітів, наприклад декоровані літери на кшталт é та ü, а також китайські, японські та арабські символи.
>
> Однак не кожна система може відображати декоровані символи, тому краще їх уникати, якщо ваш код буде поширюватися серед людей із різних регіонів.
>
> Докладніше про Unicode можна дізнатися у Вікіпедії. Підтримка Unicode в Python розглядається в офіційній документації Python.

Те, що ім'я змінної є допустимим, не означає, що воно є вдалим. Вибрати гарне ім'я для змінної може бути напрочуд складно. Проте є певні рекомендації, яких можна дотримуватися, щоб обирати кращі імена.

### Описові імена кращі за короткі

Описові імена змінних є вкрай важливими, особливо для складних програм. Часто описові імена потребують використання кількох слів. Не бійтеся довгих імен змінних.

У наступному прикладі значення 3600 присвоєно змінній `s`:

```python
s = 3600
```

Ім'я `s` абсолютно неоднозначне. Використання повного слова значно полегшує розуміння того, що означає код:

```python
seconds = 3600
```

`seconds` — краще ім'я, ніж `s`, тому що надає більше контексту. Але воно все одно не передає повного змісту коду. Чи є 3600 кількістю секунд, необхідних для завершення якогось процесу, чи тривалістю фільму? Сказати неможливо.

Наступне ім'я не залишає жодних сумнівів щодо того, що означає код:

```python
seconds_per_hour = 3600
```

Прочитавши цей код, ви без сумніву розумієте, що 3600 — це кількість секунд в одній годині. Хоча `seconds_per_hour` довше вводити, ніж і одну букву `s`, і слово `seconds`, виграш у зрозумілості — колосальний.

Хоча описові імена змінних означають довші назви, слід уникати надмірно довгих. Що саме є «надмірно довгим» — питання суб'єктивне, але хорошим практичним правилом є тримати імена змінних у межах трьох-чотирьох слів.

### Угоди про іменування змінних у Python

У багатьох мовах програмування прийнято писати імена змінних у стилі camelCase, наприклад `numStudents` або `listOfNames`. Перша буква кожного слова, крім першого, — велика, усі інші літери — малі. Поєднання малих і великих літер нагадує горби верблюда.

У Python, однак, прийнятіше писати імена змінних у стилі snake_case, наприклад `num_students` або `list_of_names`. Усі літери малі, а слова розділяються підкресленням.

Хоча немає жорсткого правила, що вимагає писати імена змінних у стилі snake_case, ця практика закріплена в документі під назвою PEP 8, який широко вважається офіційним посібником зі стилю написання Python-коду.

Дотримання стандартів, викладених у PEP 8, гарантує, що ваш Python-код буде зрозумілим для широкого кола Python-програмістів. Це полегшує обмін кодом і спільну роботу над ним для всіх залучених.

> **Примітка**
>
> Усі приклади коду в цьому курсі відповідають рекомендаціям PEP 8, тому ви отримаєте гарне уявлення про те, як виглядає Python-код, що відповідає стандартним вимогам форматування.

У цьому розділі ви навчилися створювати змінну, ознайомилися з правилами допустимих імен змінних і деякими рекомендаціями щодо вибору гарних імен. Далі ви навчитеся перевіряти значення змінної в інтерактивному вікні IDLE.

## 3.4 Перевірка значень в інтерактивному вікні

Ви вже бачили, як використовувати print() для відображення рядка, присвоєного змінній. Є ще один спосіб відобразити значення змінної під час роботи в оболонці Python.

Введіть у інтерактивне вікно IDLE таке:

```python
>>> phrase = "Hello, world"
>>> phrase
```

Коли ви натискаєте `Enter` після повторного введення `phrase`, відображається такий результат:

```
'Hello, world'
```

Python вивів рядок `"Hello, world"`, і вам не довелося вводити `print(phrase)`!

Тепер введіть таке:

```python
>>> print(phrase)
```

Цього разу, коли ви натискаєте `Enter`, ви бачите:

```
Hello, world
```

Чи помічаєте ви різницю між цим виведенням і виведенням простого введення `phrase`? Тут немає одинарних лапок навколо.

Що відбувається? Коли ви вводите `phrase` і натискаєте `Enter`, ви наказуєте Python перевірити змінну `phrase`. Відображуване виведення — це зручне представлення значення, присвоєного змінній.

У цьому випадку `phrase` присвоєно рядок `"Hello, world"`, тому виведення оточене одинарними лапками, що вказує: `phrase` є рядком.

З іншого боку, коли ви виводите змінну через print(), Python відображає більш зрозуміле для людини представлення її значення. Для рядків обидва способи відображення є зрозумілими для людини, але для деяких інших типів значень це не так.

Іноді виведення через print() і перевірка змінної дають однаковий результат:

```python
>>> x = 2
>>> x
2
>>> print(x)
2
```

Тут `x` присвоєно число 2. Результат як `print(x)`, так і перевірки `x` не оточений лапками, оскільки 2 є числом, а не рядком.

Перевірка змінної замість її виведення через print() корисна з кількох причин. Так можна відобразити значення змінної, не вводячи print(). Але важливіше те, що перевірка зазвичай дає більше корисної інформації, ніж print().

Припустимо, є дві змінні: `x = 2` і `y = "2"`. У цьому випадку `print(x)` і `print(y)` виведуть однакове. Однак перевірка `x` і `y` показує різницю між значенням кожної змінної:

```python
>>> x = 2
>>> y = "2"
>>> print(x)
2
>>> print(y)
2
>>> x
2
>>> y
'2'
```

Головний висновок тут такий: print() відображає зрозуміле для людини представлення значення змінної, тоді як перевірка надає додаткову інформацію про тип значення.

В оболонці Python можна перевіряти не лише змінні. Подивіться, що відбувається, коли ви вводите `print` і натискаєте `Enter`:

```python
>>> print
<built-in function print>
```

Пам'ятайте: перевіряти змінні можна лише в оболонці Python. Наприклад, збережіть і запустіть такий скрипт:

```python
phrase = "Hello, world"
phrase
```

Скрипт виконається без помилок, але жодний результат не відобразиться!

Протягом усієї книги ви побачите приклади, що використовують інтерактивне вікно для перевірки змінних.

## 3.5 Залиште собі корисні нотатки

Програмісти часто читають код, написаний ними кілька місяців тому, і думають: «Що, до біса, це робить?» Навіть із описовими іменами змінних буває важко пригадати, чому ви написали щось саме так, якщо давно не зверталися до цього коду.

Щоб уникнути цієї проблеми, можна залишати коментарі у своєму коді. Коментарі — це рядки тексту, що не впливають на виконання скрипту. Вони служать для документування того, що відбувається в коді.

У цьому розділі ви дізнаєтеся про три способи залишати коментарі в коді. Ви також ознайомитеся з деякими угодами щодо форматування коментарів, а також з поширеними помилками, пов'язаними з надмірним їх використанням.

### Як писати коментар

Найпоширеніший спосіб написати коментар — почати новий рядок у коді з символу `#`. Під час виконання коду будь-які рядки, що починаються з `#`, ігноруються. Коментарі, що починаються на новому рядку, називаються блоковими коментарями.

Також можна писати рядкові коментарі — коментарі, що розміщуються на тому самому рядку, що й код. Для цього просто поставте `#` в кінці рядка коду, а після нього — текст коментаря.

Ось приклад скрипту `hello_world.py` з обома видами доданих коментарів:

```python
# Це блоковий коментар.
phrase = "Hello, world."
print(phrase)  # Це рядковий коментар.
```

Перший рядок нічого не виконує, оскільки починається з `#`. Аналогічно, `print(phrase)` виконується в останньому рядку, але все, що стоїть після `#`, ігнорується.

Звісно, символ `#` можна використовувати й усередині рядка. Наприклад, Python не прийме за початок коментаря таке:

```python
print("#1")
```

Загалом варто намагатися робити коментарі якомога коротшими, але іноді потрібно написати більше, ніж розумно вміщується в одному рядку. У такому випадку можна продовжити коментар на новому рядку, що також починається з символу `#`:

```python
# Це мій перший скрипт.
# Він виводить фразу "Hello, world."
# Коментарі довші за сам скрипт!
phrase = "Hello, world."
print(phrase)
```

Крім залишення нотаток для себе, коментарі також можна використовувати для тимчасового вимкнення коду під час тестування програми. Іншими словами, додавши `#` на початок рядка коду, можна запускати програму так, ніби цього рядка не існує, без необхідності видаляти код.

Щоб закоментувати ділянку коду в IDLE, виділіть один або кілька рядків і натисніть:

- Windows: `Alt+3`
- macOS: `Ctrl+3`
- Ubuntu Linux: `Ctrl+D`

На початку кожного рядка буде вставлено два символи `#`. Це не відповідає угодам форматування коментарів PEP 8, але завдання виконує!

Щоб розкоментувати код і видалити символи `#` з початку рядків, виділіть закоментований код і натисніть:

- Windows: `Alt+4`
- macOS: `Ctrl+4`
- Ubuntu Linux: `Ctrl+Shift+D`

Тепер розглянемо деякі загальноприйняті угоди щодо коментарів у коді.

### Угоди та антиприклади

Відповідно до PEP 8, коментарі слід завжди писати повними реченнями з одним пробілом між `#` і першим словом коментаря:

```python
# Цей коментар відповідає вимогам PEP 8.

#не робіть так
```

Для рядкових коментарів PEP 8 рекомендує залишати щонайменше два пробіли між кодом і символом `#`:

```python
phrase = "Hello, world"  # Цей коментар відповідає вимогам PEP 8.
print(phrase)# А цей — ні.
```

Серед програмістів великою неприємністю вважаються коментарі, що описують те, що й так очевидно з прочитання коду. Наприклад, такий коментар є зайвим:

```python
# Вивести "Hello, world"
print("Hello, world")
```

Коментар у цьому прикладі непотрібний, оскільки сам код чітко описує те, що відбувається. Коментарі найкраще використовувати для пояснення коду, який може бути незрозумілим, або для обґрунтування того, чому щось зроблено саме так.

Загалом PEP 8 рекомендує використовувати коментарі помірковано. Використовуйте їх лише тоді, коли вони справді підвищують зрозумілість коду, пояснюючи причину того чи іншого рішення. Коментарів, що описують те, що щось робить, часто можна уникнути, використовуючи більш описові імена змінних.

У цьому розділі ви написали і виконали свою першу програму на Python! Ви написали невелику програму, що виводить текст `"Hello, world"` за допомогою функції print().

Ви ознайомилися з трьома концепціями:

1. **Змінні** надають імена значенням у коді за допомогою оператора присвоєння (`=`).

3. **Коментарі** — це рядки коду, що не виконуються, і служать документацією для вас та інших програмістів, які читатимуть ваш код.
