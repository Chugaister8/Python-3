# § 0.2.3 VS Code та розширення

---

## Редактор коду — це не просто текстовий редактор

Блокнот теж відкриває `.py`-файли. Технічно можна писати Python у Блокноті, зберігати, запускати в терміналі — і все працюватиме. Але ніхто так не робить. Не через снобізм, а через просту економіку: хороший редактор коду економить стільки часу і сил, що відмова від нього схожа на принципову відмову від калькулятора на користь арифметики стовпчиком.

Сучасний редактор підказує, яке ім'я функції ти хотів написати, до того як ти дописав перші три літери. Він підкреслює помилки ще до запуску програми. Він форматує код автоматично, щоб ти не думав про відступи і пробіли. Він дозволяє поставити програму на паузу в будь-якій точці і подивитись, що зараз знаходиться в кожній змінній. Він показує, хто і коли востаннє змінив кожен рядок у файлі. Він запускає тести одним кліком і показує які з них провалились і чому.

Це не магія — це інструменти, які звільняють тебе від рутини і дозволяють думати про задачу, а не про синтаксис.

Існує кілька популярних середовищ розробки для Python. PyCharm від JetBrains — потужний, з глибокою інтеграцією саме для Python, але важкий і платний у повній версії. Vim і Neovim — для тих, хто готовий витратити тижні на налаштування і місяці на звикання, але потім отримати неймовірну швидкість роботи. Jupyter Notebook — незамінний для аналізу даних і досліджень, але не підходить для написання звичайних програм.

Для цього курсу ми використовуємо **Visual Studio Code** — і не тому що він «кращий» у якомусь абсолютному сенсі, а тому що він безкоштовний, швидкий, працює однаково на Windows, macOS і Linux, має величезну екосистему розширень і є найпоширенішим редактором серед Python-розробників сьогодні. Те, що ти навчишся тут, — прямо застосовне до реальної роботи.

## Встановлення VS Code

Офіційний сайт — `code.visualstudio.com`. Там є завантаження для всіх трьох основних операційних систем.

На **macOS** після завантаження `.dmg`-файлу перетягни VS Code в папку «Програми». Потім відкрий редактор, натисни `Cmd+Shift+P`, введи `shell command` і обери **«Shell Command: Install 'code' command in PATH»**. Після цього з терміналу можна відкривати будь-яку директорію командою `code .` — крапка означає «поточна директорія». Це зручно: зайшов в папку проєкту, написав `code .` — і редактор відкрився саме там.

На **Linux** є кілька варіантів встановлення: `.deb`-пакет для Ubuntu/Debian, `.rpm`-пакет для Fedora/RHEL, або через snap: `sudo snap install --classic code`. Команда `code .` доступна одразу після встановлення.

На **Windows** запусти завантажений `.exe`-інсталятор. Під час встановлення постав галочку **«Add to PATH»** і **«Add 'Open with Code' action to Windows Explorer»** — остання дозволить відкривати будь-яку папку у VS Code правою кнопкою миші, що дуже зручно.

Після встановлення відкрий термінал і перевір:

```bash
code --version
```

Якщо з'явився номер версії — все в порядку.

## Інтерфейс: що де знаходиться

VS Code має п'ять основних зон, і знати де що лежить — означає не втрачати час на пошуки.

**Бічна панель** зліва — це центр навігації. Вгорі є набір іконок, кожна відкриває свою панель: провідник файлів, пошук, git-інтеграція, запуск і налагодження, розширення. Запам'ятай клавіші: `Ctrl+Shift+E` (або `Cmd+Shift+E` на Mac) — провідник файлів, `Ctrl+Shift+G` — git, `Ctrl+Shift+X` — розширення.

**Редактор** займає центральну частину. Тут відкриваються файли. Можна розділити редактор на кілька колонок — зручно, коли треба дивитись на два файли одночасно: `Ctrl+\` розділяє вертикально.

**Термінал** відкривається внизу через `Ctrl+\`` (зворотній апостроф). Це повноцінний термінал всередині редактора — тут запускаєш програми, керуєш середовищами, працюєш з git. Він вже відкритий у директорії проєкту, тому не потрібно окремо навігувати в терміналі.

**Командна палітра** — `Ctrl+Shift+P` (або `Cmd+Shift+P`). Це найважливіша комбінація у VS Code. Вона відкриває рядок пошуку, через який можна запустити будь-яку команду редактора без використання меню. Ще не знаєш де налаштування? Командна палітра. Не знаєш як перейменувати файл? Командна палітра. Хочеш змінити тему? Командна палітра.

**Швидке відкриття файлу** — `Ctrl+P`. Вводиш частину назви файлу — редактор знаходить його серед усіх файлів проєкту. Набагато швидше ніж клікати у дереві файлів.

## Розширення: що і навіщо встановлювати

Базовий VS Code вміє редагувати текст і трохи підсвічувати синтаксис. Все корисне для Python-розробки додається через розширення — плагіни, які встановлюються в кілька кліків і глибоко інтегруються в редактор.

Відкрий панель розширень (`Ctrl+Shift+X`), знайди і встанови кожне з перерахованих нижче.

---

### Python (Microsoft)

Це перше і найважливіше розширення. Без нього VS Code не «знає» що таке Python — просто редактор тексту з підсвічуванням. З ним з'являється:

**IntelliSense** — автодоповнення коду. Ти починаєш вводити ім'я функції або методу, редактор пропонує варіанти зі списку. Навів курсор на будь-який елемент — з'являється підказка з описом, типами аргументів і поверненим значенням. Це не просто зручність — це спосіб вивчати бібліотеки прямо в процесі роботи, не перемикаючись на документацію.

**Запуск файлів** — кнопка ▶ у правому верхньому куті або `F5`. Програма запускається і виводить результат у вбудований термінал.

**Налагоджувач** — одна з найцінніших функцій. Постав точку зупину (`breakpoint`), клікнувши на номер рядка ліворуч від коду. З'явиться червона крапка. Запусти програму через `F5` — вона зупиниться саме тут, і ти зможеш побачити значення всіх змінних у цей момент, крок за кроком пройти код далі, перевірити що відбувається всередині функції. Це стократно ефективніше ніж вставляти `print()` скрізь де щось незрозуміло.

**Підтримка тестів** — якщо у проєкті є тести, це розширення знаходить їх і дозволяє запускати окремо або всі відразу прямо з редактора з наочним звітом: зелений — пройшов, червоний — впав, і чому.

> ⚠️ **Важливо:** Після встановлення розширення Python одразу попросить або сам виявить інтерпретатор Python. Обов'язково обери той, що знаходиться у твоєму віртуальному середовищі — шлях виду `./venv/bin/python` або `./.venv/bin/python`. Якщо обрати системний Python або не той — автодоповнення не бачитиме встановлених бібліотек, і налагоджувач запускатиме не те середовище. Обрати або змінити інтерпретатор можна через командну палітру: `Python: Select Interpreter`.

### Pylance

Pylance розроблений Microsoft спеціально як мовний сервер для Python — він забезпечує глибший статичний аналіз, ніж базове розширення Python.

На практиці це означає точніше автодоповнення — Pylance краще «розуміє» типи і не пропонує нерелевантних варіантів. Він підкреслює помилки типів ще до запуску: якщо функція очікує число, а ти передаєш рядок — Pylance підкреслить це жовтою або червоною хвилею. Він також надає детальніші підказки при наведенні курсора.

Встановлюється разом з основним розширенням Python або окремо. Після встановлення варто переконатись, що в налаштуваннях вибрано режим перевірки типів. У налаштуваннях VS Code (`Ctrl+,`) знайди `Python > Analysis: Type Checking Mode` і постав значення `basic` — це помірний рівень, який підходить для навчання і не завалює тебе попередженнями з кожного приводу.

---

### Black Formatter

Black — це автоматичний форматувальник коду. Він перетворює будь-який код у чітко визначений стандартний формат: правильні відступи, правильні пробіли навколо операторів, правильне розташування дужок. Він не запитує твою думку — він просто форматує «як треба» і завжди однаково. Звідси і назва: Black, як чорний ящик, у якому немає налаштувань смаку.

Спочатку це може дратувати: «але я хочу написати ось так, а він переробляє». Це відчуття минає через кілька днів. А потім приходить розуміння: ти більше не витрачаєш розумову енергію на те, як розмістити пробіл або де перенести рядок. Ці рішення прийняті за тебе, і ти можеш думати про логіку, а не про оформлення.

Після встановлення розширення налаштуй форматування при збереженні. Відкрий налаштування (`Ctrl+,`), знайди `Editor: Format On Save` і увімкни. Тепер щоразу коли ти зберігаєш файл — Black автоматично приводить код до стандартного вигляду. Жодних зусиль з твого боку.

> 💡 **Порада:** Black форматує тільки стиль, він не перевіряє логічні помилки і не змінює поведінку коду. Якщо написав щось технічно неправильне — Black тихо відмовиться форматувати цей файл. Це непряма підказка: якщо форматування раптом перестало спрацьовувати — мабуть, є синтаксична помилка.

### Ruff

Ruff — це лінтер. Лінтер — інструмент, який аналізує код і вказує на потенційні проблеми: невикористані імпорти, змінні які оголошені але ніде не застосовуються, небезпечні патерни, порушення стилю, логічні підозрілості.

Ruff написаний на Rust і є на сьогодні найшвидшим лінтером для Python — він перевіряє навіть великі проєкти практично миттєво. Він замінює кілька старіших інструментів одночасно: `flake8` для перевірки стилю, `isort` для сортування імпортів, частину перевірок `pylint`. Це зручно: один інструмент замість трьох, одне налаштування, одна команда.

Встановлене розширення Ruff підкреслює проблемні місця безпосередньо в редакторі — тонкою хвилею під рядком. Наводиш курсор — з'являється пояснення що саме не так і код правила (наприклад, `F401` — невикористаний імпорт, `E501` — рядок занадто довгий). Багато виправлень Ruff може зробити автоматично: є лампочка збоку або можна натиснути `Ctrl+.` — і він запропонує виправлення.

Додай у налаштування VS Code (файл `settings.json`, відкрити через командну палітру: `Open User Settings (JSON)`):

```json
{
  "[python]": {
    "editor.defaultFormatter": "charliermarsh.ruff",
    "editor.formatOnSave": true,
    "editor.codeActionsOnSave": {
      "source.fixAll.ruff": "explicit",
      "source.organizeImports.ruff": "explicit"
    }
  }
}
```

Це налаштовує Ruff як форматувальник для Python-файлів і вмикає автоматичне виправлення і сортування імпортів при збереженні.

> ⚠️ **Увага:** Якщо встановлені і Black, і Ruff як форматувальники одночасно — між ними може виникнути конфлікт при збереженні. Обери один. Найпоширеніша комбінація — Black для форматування і Ruff для лінтингу (перевірки помилок). У налаштуваннях вище як форматувальник вказаний Ruff, бо він підтримує і лінтинг, і форматування. Якщо хочеш розділити: `"editor.defaultFormatter": "ms-python.black-formatter"` — для форматування через Black, і окремо тільки linting через Ruff.

### GitLens

Git — система контролю версій, про яку докладно у наступному параграфі. GitLens розширює вбудовану git-інтеграцію VS Code і додає кілька дуже корисних речей.

Найбільш помітна функція — blame-анотації. Прямо в рядку тексту, після коду, блідо з'являється підпис: хто написав цей рядок, коли і з яким коментарем до коміту. Це не просто цікавість — в командній роботі це дозволяє миттєво зрозуміти контекст будь-якого рішення: «Чому тут зроблено саме так? Ага, Марія змінила це три місяці тому у рамках задачі з оптимізацією запитів».

GitLens також показує детальну історію файлу: список усіх комітів, де цей файл змінювався, і що саме змінилось у кожному з них. Є вбудований перегляд відмінностей між версіями прямо в редакторі.

Для початківця найкорисніша функція — візуалізація поточного стану репозиторію: які файли змінені, які готові до коміту, яка гілка активна. Все це видно без відкриття терміналу.

### Thunder Client

Thunder Client — REST API клієнт, вбудований у VS Code. Він дозволяє надсилати HTTP-запити і дивитись відповіді прямо в редакторі, не перемикаючись на зовнішні програми на кшталт Postman або Insomnia.

Навіщо це потрібно на початку навчання? Пізніше в курсі ти будеш писати веб-API на Flask і FastAPI. Щоб перевірити чи правильно відповідає твій сервер, треба надіслати йому запит. Thunder Client дозволяє зробити це в два кліки, не виходячи з редактора.

Після встановлення іконка Thunder Client з'явиться на бічній панелі. Натискаєш «New Request», вибираєш метод (GET, POST, PUT тощо), вводиш URL, при потребі додаєш заголовки або тіло запиту — і натискаєш Send. Відповідь відображається поруч із кодом статусу, заголовками і тілом.

### SQLite Viewer

Проста, але корисна річ. SQLite — легка база даних, яка зберігається в одному файлі і широко використовується для навчання і невеликих проєктів. Без цього розширення, якщо хочеш подивитись що зберігається у файлі `.db` або `.sqlite`, тобі потрібен окремий інструмент.

З SQLite Viewer достатньо двічі клікнути на файл бази даних у провіднику — і відкриється таблиця з даними прямо в редакторі. Можна переглядати всі таблиці, фільтрувати рядки, бачити структуру.

### Docker

Якщо слово «Docker» тобі поки нічого не говорить — не хвилюйся, це тема значно пізніших розділів курсу. Але розширення варто встановити вже зараз: воно маленьке і не заважає, а коли Docker знадобиться — воно вже буде на місці.

Розширення Docker додає у бічну панель інтерфейс для управління контейнерами, образами і мережами. Дозволяє запускати і зупиняти контейнери, переглядати їхні логи і файлову систему, підключатися до їхнього терміналу — все без командного рядка.

## Налаштування, які варто змінити одразу

Базові налаштування VS Code розумні, але кілька речей варто підправити одразу.

Відкрий файл налаштувань через командну палітру: `Open User Settings (JSON)`. Це файл `settings.json`, де можна задати будь-яке налаштування у текстовому форматі.

Ось мінімальний набір корисних налаштувань для Python-розробки:

```json
{
  "editor.fontSize": 14,
  "editor.tabSize": 4,
  "editor.insertSpaces": true,
  "editor.rulers": [88],
  "editor.minimap.enabled": false,
  "editor.renderWhitespace": "boundary",
  "editor.wordWrap": "off",
  "files.autoSave": "onFocusChange",
  "terminal.integrated.defaultProfile.osx": "zsh",
  "python.analysis.typeCheckingMode": "basic",
  "[python]": {
    "editor.defaultFormatter": "ms-python.black-formatter",
    "editor.formatOnSave": true
  }
}
```

Кілька пояснень. `editor.tabSize: 4` і `editor.insertSpaces: true` — Python офіційно рекомендує чотири пробіли як відступ, і саме пробіли (не Tab). `editor.rulers: [88]` — вертикальна лінія на 88 символах, це стандартна довжина рядка для Black. Коли рядок перетинає цю лінію — візуальна підказка що варто перенести. `files.autoSave: "onFocusChange"` — файл зберігається автоматично коли ти переходиш в інше вікно або вкладку. Разом з форматуванням при збереженні це означає: переключився на термінал — код вже збережений і відформатований.

`editor.minimap.enabled: false` — відключає мініатюру коду в правому куті. Вона займає місце і рідко реально корисна, особливо на невеликих моніторах.

## Корисні клавіатурні скорочення

Редактор — це інструмент швидкості. Чим менше рухів мишею, тим швидше і плавніше робота. Ось скорочення, які варто запам'ятати в першу чергу — не всі одразу, а поступово, по кілька на тиждень.

| Дія | macOS | Windows / Linux |
|-----|-------|-----------------|
| Командна палітра | `Cmd+Shift+P` | `Ctrl+Shift+P` |
| Швидке відкриття файлу | `Cmd+P` | `Ctrl+P` |
| Перейти до визначення | `F12` | `F12` |
| Знайти всі використання | `Shift+F12` | `Shift+F12` |
| Перейменувати символ | `F2` | `F2` |
| Відкрити/закрити термінал | `` Cmd+` `` | `` Ctrl+` `` |
| Нова вкладка терміналу | `Cmd+Shift+5` | `Ctrl+Shift+5` |
| Коментувати рядок | `Cmd+/` | `Ctrl+/` |
| Дублювати рядок | `Shift+Alt+↓` | `Shift+Alt+↓` |
| Перемістити рядок вгору/вниз | `Alt+↑` / `Alt+↓` | `Alt+↑` / `Alt+↓` |
| Виділити наступне входження | `Cmd+D` | `Ctrl+D` |
| Виділити всі входження | `Cmd+Shift+L` | `Ctrl+Shift+L` |
| Форматувати документ | `Shift+Alt+F` | `Shift+Alt+F` |
| Пошук у всьому проєкті | `Cmd+Shift+F` | `Ctrl+Shift+F` |
| Замінити у всьому проєкті | `Cmd+Shift+H` | `Ctrl+Shift+H` |
| Розділити редактор | `Cmd+\` | `Ctrl+\` |
| Перейти до рядка | `Ctrl+G` | `Ctrl+G` |

Особливо зверни увагу на `F12` («Перейти до визначення») і `Shift+F12` («Знайти всі використання»). Перша дозволяє миттєво перейти до визначення будь-якої функції чи класу — дуже зручно коли хочеш зрозуміти як щось влаштовано зсередини. Друга показує всі місця в коді де використовується певна змінна, функція або клас — незамінно при рефакторингу.

`F2` — перейменування символу — теж надзвичайно корисне. Якщо ти хочеш перейменувати функцію, яка використовується у двадцяти місцях, `F2` зробить це всюди одразу, включаючи всі файли проєкту.

## Налагоджувач: трохи детальніше

Налагоджувач (debugger) заслуговує окремої уваги, бо це один з тих інструментів, від яких початківці часто відмахуються — і даремно.

Типовий підхід новачка при виявленні помилки: поставити `print(змінна)` в кілька місць і перезапустити програму. Потім ще `print`. Потім ще. Це спрацьовує, але повільно і брудно — потім ці `print` треба видаляти, і якщо забути хоч один — він залишиться в коді назавжди.

Налагоджувач дозволяє зупинити програму в довільній точці і подивитись на стан усіх змінних одразу, без жодних `print`. Це набагато швидше і не залишає слідів.

Основний процес: клікни на номер рядка ліворуч від коду — з'явиться червона крапка, це «точка зупину» (breakpoint). Запусти програму через `F5` замість звичайної кнопки ▶. Коли виконання дійде до цього рядка — програма зупиниться. У лівій панелі з'явиться розділ `Variables` — список усіх змінних і їх поточних значень. У нижній панелі — `Debug Console`, де можна вводити будь-які Python-вирази і бачити результат у контексті зупиненої програми.

Далі є кілька кнопок керування:

`F10` — «Step Over» — виконати поточний рядок і зупинитись на наступному, не заходячи всередину функцій.

`F11` — «Step Into» — зайти всередину функції, яку викликає поточний рядок, і зупинитись на її першому рядку.

`Shift+F11` — «Step Out» — вийти з поточної функції назад у місце виклику.

`F5` — «Continue» — продовжити виконання до наступної точки зупину або до кінця програми.

Спочатку ці кнопки плутаються. Через тиждень практики вони стають інстинктивними.

## Підсумок: що встановити і навіщо

Повна картина одним поглядом:

| Розширення | Для чого |
|------------|----------|
| **Python** (Microsoft) | Основа: IntelliSense, запуск, налагоджувач, тести |
| **Pylance** | Глибший аналіз типів, точніше автодоповнення |
| **Black Formatter** | Автоматичне форматування при збереженні |
| **Ruff** | Лінтинг: пошук помилок і сортування імпортів |
| **GitLens** | Розширена git-інтеграція, blame-анотації |
| **Thunder Client** | Тестування HTTP API без виходу з редактора |
| **SQLite Viewer** | Перегляд SQLite-баз прямо у VS Code |
| **Docker** | Управління контейнерами (знадобиться пізніше) |

Встанови всі вісім, налаштуй `settings.json`, обери правильний інтерпретатор Python — і середовище готове.

Одне застереження під кінець: не захоплюйся встановленням розширень заради самого процесу. Кожне розширення трохи уповільнює запуск редактора і може вступати в конфлікти з іншими. Те, що наведено вище — перевірений і достатній набір для навчання і реальної роботи. Якщо відчуєш потребу в чомусь конкретному — додаєш тоді. Але «на всяк випадок» — ні.
