## Перший живий діалог з Python

Відкрий термінал, переконайся що віртуальне середовище активоване, і запусти Python в інтерактивному режимі:

```bash
python
```

Ти потрапиш у REPL — Read-Eval-Print Loop. Назва розшифровується просто: Read — Python читає те, що ти ввів; Eval — виконує; Print — виводить результат; Loop — і чекає наступного рядка. Це середовище прямої розмови з інтерпретатором, без файлів і без запуску скриптів:

```
Python 3.12.3 (main, Apr 15 2024, 17:43:11)
Type "help", "copyright", "credits" or "license" for more information.
>>>
```

Символ `>>>` — запрошення до введення. Після нього набираєш будь-який Python-вираз, натискаєш Enter — і Python одразу показує результат. Введи кілька речей, щоб відчути як це працює:

```python
>>> 2 + 2
4
>>> 10 * 3
30
>>> "Привіт"
'Привіт'
>>> "При" + "віт"
'Привіт'
>>> type(42)
<class 'int'>
>>> type("hello")
<class 'str'>
```

Зверни увагу: коли ти ввів `"Привіт"` — Python вивів `'Привіт'` в одинарних лапках. Це не помилка. У REPL Python показує так зване repr-представлення значення — технічне відображення, де рядки завжди в одинарних лапках, навіть якщо ти вводив у подвійних. Коли ж ти використовуєш `print()` — відображення вже «людське», без зайвих лапок:

```python
>>> print("Привіт")
Привіт
>>> "Привіт"
'Привіт'
```

Різниця важлива і буде зрозумілішою далі, коли ми поговоримо про рядки докладніше.

REPL — ідеальне місце для експериментів протягом усього курсу. Не закривай його — відкрий поруч з редактором і перевіряй будь-яку ідею прямо тут. Не впевнений що дасть певний вираз? Введи у REPL і побач. Хочеш перевірити чи правильно розумієш поведінку типу? Введи і спостерігай. Це в рази швидше ніж писати файл, запускати, дивитись результат.

Вийти з REPL — команда `exit()` або клавіатурне скорочення `Ctrl+D` на macOS/Linux, `Ctrl+Z` і Enter на Windows.

## Що таке тип даних і навіщо він потрібен

Будь-яка програма працює з даними. Але дані бувають дуже різними: ім'я користувача, ціна товару, кількість елементів у кошику, прапорець «чи авторизований», відсутність результату пошуку. Кожне з цих значень — це не просто набір байтів у пам'яті. Це значення певного **типу**, і тип визначає що з ним можна робити.

З числами можна рахувати: додавати, множити, порівнювати. З рядками можна об'єднувати, шукати підрядки, змінювати регістр. Але не можна рядок помножити на рядок, або додати число до тексту без явного перетворення. Python стежить за цим суворо — і якщо ти спробуєш зробити щось, що не має сенсу для певного типу, він скаже тобі про це негайно, а не дасть програмі тихо дати неправильну відповідь.

Спробуй у REPL:

```python
>>> "5" + 3
TypeError: can only concatenate str (not "int") to str
```

Це не Python примхливий — це він чесний. Рядок `"5"` і число `3` — різні типи, і складання їх разом не має однозначного сенсу. Чи потрібно отримати рядок `"53"`? Чи число `8`? Python не вгадує. Він вимагає щоб ти сказав явно.

У Python є п'ять базових типів з якими починається будь-яка програма: рядок `str`, ціле число `int`, число з плаваючою крапкою `float`, логічне значення `bool` і відсутність значення `None`. Є й інші типи — списки, словники, кортежі, множини — але ці п'ять є фундаментом і зустрічаються скрізь без винятку.

## Що таке змінна

Перш ніж говорити про типи — потрібно поговорити про змінні, бо без них жоден тип не має практичного застосування.

Уяви що ти пишеш програму, яка вітає користувача. Ім'я користувача тобі знадобиться кілька разів: вивести у привітанні, записати в лог, підставити в заголовок сторінки. Можна писати `"Оксана"` в кожному місці окремо. Але якщо завтра Оксана змінить ім'я на Олексія — доведеться знайти і поміняти кожне місце. Це неефективно і ненадійно.

Змінна вирішує цю задачу: ти запам'ятовуєш значення під певним іменем, а потім використовуєш ім'я скільки завгодно разів. Зміниш значення в одному місці — всі посилання оновляться автоматично.

```python
name = "Оксана"
print(name)           # Оксана
print(f"Привіт, {name}!")  # Привіт, Оксана!
print(f"До побачення, {name}!")  # До побачення, Оксана!
```

Рядок `name = "Оксана"` — це оператор присвоєння. Знак `=` тут не означає «рівність» у математичному сенсі. Це команда: «візьми значення праворуч і прив'яжи його до імені ліворуч». В математиці `x = 5` — це твердження що x дорівнює п'яти. У Python це інструкція: зроби так, щоб `name` вказував на рядок `"Оксана"`.

Насправді змінна в Python — це ярлик, а не ящик. Ця різниця тонка, але важлива. Коли ти пишеш `name = "Оксана"`, Python не «кладе `"Оксана"` у комірку на ім'я `name`». Він створює об'єкт-рядок `"Оксана"` десь у пам'яті і прикріплює до нього ярлик `name`. Якщо потім написати `name = "Марія"` — той самий ярлик перекріплюється на новий об'єкт `"Марія"`. Старий рядок `"Оксана"` нікуди не зник — він просто більше не має імені, і Python прибере його згодом.

Ця модель пояснює кілька речей, які спочатку можуть здаватись дивними, і ми ще до неї повернемось.

### Правила іменування змінних

Ім'я змінної може містити латинські літери, цифри і підкреслення — але не може починатись з цифри і не може збігатись з ключовими словами Python:

```python
# Правильні імена
user_name = "Іван"
age2 = 30
_private = "внутрішнє"
total_price = 499.99

# Неправильні імена
2name = "Іван"      # SyntaxError: починається з цифри
if = True           # SyntaxError: if — ключове слово Python
class = "клас"      # SyntaxError: class — ключове слово
```

Ключових слів Python небагато: `if`, `else`, `elif`, `for`, `while`, `def`, `class`, `return`, `import`, `from`, `True`, `False`, `None`, `and`, `or`, `not`, `in`, `is` — та ще кілька. Всі вони зарезервовані для синтаксису мови.

За угодою у Python змінні пишуть у стилі **`snake_case`**: всі слова маленькими літерами, розділені підкресленням. `first_name`, `order_total`, `is_logged_in` — так прийнято. Не `firstName` — це JavaScript-стиль, який у Python виглядає чужорідно. Не `FirstName` — це стиль для класів. Ця угода не є технічною вимогою — Python не відмовить якщо напишеш `firstName`. Але вся Python-екосистема, всі бібліотеки, всі open-source проєкти використовують `snake_case`, тому відхилення від норми робить твій код менш читабельним для інших і для тебе самого через місяць.

Ще одна угода, якої варто дотримуватись з самого початку: ім'я змінної має **пояснювати що в ній зберігається**. `x`, `a`, `tmp`, `val` — погані імена, якщо мова не йде про суто математичний контекст де `x` — очевидна координата. `user_email`, `total_discount`, `is_admin` — хороші імена, що не вимагають пояснень.

```python
# Погано — що таке d? хто такий u?
d = 7
u = "test@example.com"
f = True

# Добре — все зрозуміло без коментарів
days_until_deadline = 7
user_email = "test@example.com"
is_admin = True
```

## str — рядок

Рядок — це послідовність символів. Будь-який текст: ім'я, адреса, повідомлення, HTML-код, номер телефону записаний як текст, JSON-відповідь від API — все це рядки.

Щоб створити рядок, просто беремо текст у лапки. Одинарні або подвійні — Python ставиться до обох однаково:

```python
name       = "Оксана"
city       = 'Київ'
empty      = ""            # порожній рядок — теж рядок
space      = " "           # рядок з одного пробілу
digit_text = "42"          # РЯДОК, не число — бо в лапках
```

Остання деталь надзвичайно важлива і варта того щоб зупинитись на ній окремо. `"42"` і `42` — це принципово різні речі, хоч і виглядають схоже:

```python
>>> type("42")
<class 'str'>
>>> type(42)
<class 'int'>

>>> "42" + "8"
'428'       # конкатенація рядків: просто склеюємо тексти

>>> 42 + 8
50          # арифметика: математичне додавання

>>> "42" + 8
TypeError   # несумісні типи — Python не знає що ти мав на увазі
```

Це не баг і не дивацтво — це послідовна логіка. Рядок `"42"` для Python — це послідовність двох символів `"4"` і `"2"`, і він нічого не знає про те що вони разом утворюють число. Тому щоб робити математику з числами, які прийшли як текст (наприклад, введені користувачем через клавіатуру), їх спочатку потрібно перетворити — і ми це розглянемо у розділі про перетворення типів.

### Одинарні, подвійні і потрійні лапки

Python дозволяє три варіанти і кожен має своє місце:

```python
# Одинарні — коли всередині є апостроф або подвійна лапка
message  = 'Він сказав: "Привіт!"'
apostrophe_ok = 'It is fine'

# Подвійні — загальний випадок, найпоширеніші
name  = "Оксана"
email = "user@example.com"

# Комбінація — щоб не екранувати
tricky1 = "It's a great day"       # апостроф всередині подвійних — ОК
tricky2 = 'She said "hello"'       # подвійні всередині одинарних — ОК
```

Якщо потрібна лапка того ж типу, що обрамляє рядок — використовують зворотний слеш для екранування:

```python
escaped1 = "Він сказав: \"Привіт!\""  # \" всередині подвійних
escaped2 = 'It\'s OK'                  # \' всередині одинарних
```

Але зазвичай простіше просто обрати інший тип лапок.

Потрійні лапки — `"""` або `'''` — дозволяють рядку займати кілька рядків коду, зберігаючи переноси і відступи:

```python
haiku = """Старий ставок —
жаба стрибає,
плюскіт води."""

html_template = """
<html>
    <body>
        <h1>Привіт!</h1>
    </body>
</html>
"""

sql_query = """
    SELECT name, email
    FROM users
    WHERE is_active = TRUE
    ORDER BY created_at DESC
"""
```

Потрійні лапки ще використовуються для документаційних рядків — docstrings — коментарів, що пояснюють функції і класи. Але про це — коли дійдемо до функцій.

### Рядки незмінні — і це важливо

Рядок у Python неможливо змінити після створення. Не можна взяти рядок і поміняти в ньому одну літеру — можна лише створити новий рядок на основі старого:

```python
name = "Оксана"

# Спроба змінити символ — помилка
name[0] = "о"    # TypeError: 'str' object does not support item assignment

# Правильно — створити новий рядок
name = "о" + name[1:]    # "оксана"
```

На практиці це означає що всі методи рядків, які нібито «змінюють» рядок, насправді повертають **новий** рядок, а вихідний залишається незмінним:

```python
original = "  Привіт  "
stripped = original.strip()   # " Привіт " — новий рядок без пробілів

print(original)   # "  Привіт  " — незмінний!
print(stripped)   # "Привіт"    — новий
```

Це частий источник подиву на початку. Якщо написати `original.strip()` без присвоєння результату — нічого не зміниться, результат просто зникне. Завжди зберігай результат у змінну, або присвоюй назад:

```python
text = "  Привіт  "
text = text.strip()    # тепер text = "Привіт"
```

Детально методи рядків розглянемо в § 1.1.2, але ця концепція — незмінності — важлива вже зараз.

### Конкатенація і повторення

Рядки можна «складати» оператором `+` — це об'єднання двох рядків в один:

```python
first = "Добрий"
last  = " вечір"

greeting = first + last       # "Добрий вечір"
```

І «множити» оператором `*` — повторення рядка певну кількість разів:

```python
line      = "-" * 40          # "----------------------------------------"
separator = "= " * 20         # "= = = = = = = = = = = = = = = = = = = = "
indent    = "  " * 3          # "      " — три рівні відступу
```

Повторення рядків здається дрібницею, але насправді дуже зручна і часто використовувана операція — наприклад для малювання розділювачів у консольному виводі або генерації відступів.

### f-рядки: вставити значення в текст

Одна з найчастіших задач при роботі з рядками — підставити туди значення змінних. Уяви: потрібно вивести повідомлення «Замовлення №1234 на суму 549.99 грн оброблено». Числа беруться зі змінних і змінюються для кожного замовлення.

Є три способи це зробити — і один з них значно кращий за інших:

```python
order_id = 1234
total    = 549.99

# Спосіб 1: конкатенація через + (незручно, легко помилитись)
message = "Замовлення №" + str(order_id) + " на суму " + str(total) + " грн оброблено"

# Спосіб 2: .format() (старий стиль, ще зустрічається)
message = "Замовлення №{} на суму {} грн оброблено".format(order_id, total)

# Спосіб 3: f-рядки (сучасний стандарт — використовуй завжди)
message = f"Замовлення №{order_id} на суму {total} грн оброблено"
```

F-рядок — рядок з літерою `f` перед лапками. Усередині фігурних дужок `{}` можна написати будь-який Python-вираз, і він буде обчислений і вставлений у рядок:

```python
name = "Оксана"
age  = 28
items_in_cart = 3

# Просто змінна
print(f"Привіт, {name}!")
# Привіт, Оксана!

# Вираз — арифметика
print(f"Наступного року тобі буде {age + 1} рік.")
# Наступного року тобі буде 29 рік.

# Виклик методу
print(f"Ім'я великими літерами: {name.upper()}")
# Ім'я великими літерами: ОКСАНА

# Умова прямо в рядку
print(f"У кошику {items_in_cart} {'товар' if items_in_cart == 1 else 'товари'}")
# У кошику 3 товари

# Форматування числа: 2 знаки після крапки
price = 199.9
print(f"Ціна: {price:.2f} грн")
# Ціна: 199.90 грн

# Вирівнювання: рядок у 20 символів праворуч
print(f"{'Сума':>20}: {1234.5:.2f}")
#                Сума: 1234.50
```

F-рядки з'явились у Python 3.6 і дуже швидко стали стандартом де-факто. Вони читабельніші за конкатенацію, лаконічніші за `.format()` і значно зручніші в написанні. Якщо колись зустрінеш у чужому коді `"Привіт, %s" % name` або `"Привіт, {}".format(name)` — знай, це старі стилі. Вони працюють, але в новому коді краще не використовуй.

### Escape-послідовності

Є символи, які не можна вставити в рядок напряму — наприклад, перенос рядка або табуляція. Для них є спеціальні послідовності з зворотним слешем:

```python
# \n — перенос рядка
print("Рядок перший\nРядок другий")
# Рядок перший
# Рядок другий

# \t — табуляція (відступ)
print("Ім'я:\tОксана")
print("Вік:\t28")
# Ім'я:   Оксана
# Вік:    28

# \\ — зворотний слеш (щоб вставити сам \)
print("Шлях: C:\\Users\\Оксана\\Documents")
# Шлях: C:\Users\Оксана\Documents

# \' і \" — лапки всередині рядка
print("Він сказав: \"Привіт!\"")
# Він сказав: "Привіт!"
```

Коли зворотні слеші потрібні буквально і не треба їх інтерпретувати — наприклад у Windows-шляхах або регулярних виразах — використовують «сирий рядок» з префіксом `r`:

```python
# Без r — Python інтерпретує \U, \D як escape-послідовності і падає
path = "C:\Users\Desktop"   # проблема!

# З r — зворотні слеші буквальні
path    = r"C:\Users\Desktop"        # C:\Users\Desktop
pattern = r"\d+\.\d+"               # регулярний вираз як є
```

## int — ціле число

`int` — це цілі числа: будь-яке ціле додатне, від'ємне або нуль.

```python
age        = 25
year       = 2025
score      = -15
zero       = 0
population = 8_000_000_000    # населення Землі
```

Підкреслення в числах — лише для читабельності. Python їх ігнорує при обчисленнях, але для людини, що читає код, різниця між `8000000000` і `8_000_000_000` очевидна. Можна ставити підкреслення де завгодно між цифрами — `1_000`, `1_000_000`, `10_00_00` — але перші два варіанти логічніші за третій.

Важлива особливість Python: ціле число може бути **будь-якого розміру**. У більшості мов програмування є максимальне значення для цілого числа — наприклад, у Java `int` максимум 2 147 483 647, і якщо перевищити — число «переповнюється» і стає від'ємним. Python обходить це автоматично: якщо число велике, Python просто виділяє більше пам'яті. Це зручно і рятує від цілого класу непомітних помилок:

```python
>>> 2 ** 100
1267650600228229401496703205376   # число з 31 цифрою — без проблем

>>> factorial = 1
>>> for i in range(1, 21):
...     factorial *= i
>>> factorial
2432902008176640000               # 20! — теж без проблем
```

### Арифметичні оператори

З цілими числами доступні всі стандартні математичні операції плюс кілька специфічних для програмування:

```python
a = 17
b = 5

# Базові
a + b     # 22 — додавання
a - b     # 12 — віднімання
a * b     # 85 — множення
a ** b    # 1419857 — піднесення до степеня (17⁵)

# Ділення — тут є нюанс
a / b     # 3.4  — звичайне ділення, ЗАВЖДИ повертає float
a // b    # 3    — ціле ділення: скільки разів b вміщується в a
a % b     # 2    — залишок від ділення
```

Оператор `/` заслуговує особливої уваги, бо поводиться не так як в деяких інших мовах. У Python 3 навіть якщо обидва числа цілі і результат ділення ціле — `/` завжди повертає `float`:

```python
>>> 10 / 2
5.0        # float, не 5!

>>> 7 / 7
1.0        # теж float

>>> type(10 / 2)
<class 'float'>
```

Якщо потрібне ціле — використовуй `//`. Він відкидає дробову частину і повертає `int`:

```python
>>> 10 // 3
3         # 3.333... → відкидає .333

>>> 17 // 5
3         # 3.4 → відкидає .4

>>> type(10 // 3)
<class 'int'>
```

Оператор `%` — залишок від ділення — використовується частіше ніж здається. Класичний приклад: перевірити чи число парне. Парне число без залишку ділиться на 2, тому `n % 2 == 0` означає «n парне»:

```python
>>> 10 % 2
0     # 10 парне

>>> 7 % 2
1     # 7 непарне

>>> 13 % 5
3     # 13 = 5*2 + 3, залишок 3
```

Це знадобиться дуже скоро — у перших задачах з умовами і циклами.

### Пріоритет операцій

Математичний пріоритет дотримується: `**` виконується першим, потім `*`, `/`, `//`, `%`, потім `+` і `-`. Дужки перекривають пріоритет так само як у математиці:

```python
2 + 3 * 4       # 14 — спочатку множення
(2 + 3) * 4     # 20 — спочатку дужки

2 ** 3 ** 2     # 512 — ** правоасоціативний: 2 ** (3**2) = 2**9 = 512
(2 ** 3) ** 2   # 64  — з дужками: (2**3)**2 = 8**2 = 64
```

Коли є сумнів у пріоритеті — просто постав дужки. Код стане зрозумілішим і не буде помилок.

## float — число з плаваючою крапкою

`float` — це числа з дробовою частиною. Температура, ціна, координата, відсоток, результат ділення — всюди де потрібна точність після коми.

```python
price       = 199.99
pi          = 3.14159265358979
temperature = -7.5
percentage  = 0.87         # 87%
very_small  = 0.000001
```

Є ще наукова нотація — зручна для дуже великих або дуже маленьких чисел. Буква `e` означає «помножити на десять у степені»:

```python
avogadro  = 6.022e23      # 6.022 × 10²³ = 602200000000000000000000
electron  = 9.11e-31      # 9.11 × 10⁻³¹ = 0.000000000000000000000000000000911
light_speed = 3e8         # 300000000.0 — швидкість світла м/с
```

Перевірити у REPL:

```python
>>> 6.022e23
6.022e+23

>>> 9.11e-31
9.11e-31

>>> 3e8 * 2
600000000.0
```

### int і float: як вони взаємодіють

Коли в одному виразі є і `int`, і `float` — Python автоматично перетворює `int` на `float`:

```python
>>> 5 + 2.0
7.0    # float

>>> 10 * 0.5
5.0    # float

>>> type(5 + 2.0)
<class 'float'>
```

Це «розширення» типу — Python обирає «ширший» тип щоб не втратити точність. `int` → `float` є безпечним, бо `float` може представити всі цілі числа. Зворотне — `float` → `int` — вже потребує явного перетворення і при цьому дробова частина відкидається.

### Пастка: float і точність

Є одна особливість `float`, яку треба знати з перших кроків і яка регулярно дивує новачків:

```python
>>> 0.1 + 0.2
0.30000000000000004
```

Здається що Python помилився. Насправді — ні. Це фундаментальна особливість того, як комп'ютери зберігають числа з плаваючою крапкою. Вони зберігаються у двійковій системі числення, і деякі десяткові дроби — зокрема `0.1` і `0.2` — не мають точного двійкового представлення. Це як `1/3` у десятковій: нескінченна дріб `0.333...` Те саме відбувається з `0.1` у двійковій — воно нескінченне, і при зберіганні відбувається незначне округлення. При складанні ці мікро-похибки накопичуються і дають такий результат.

Ще кілька прикладів:

```python
>>> 0.1 + 0.1 + 0.1
0.30000000000000004

>>> 1.0 - 0.9
0.09999999999999998

>>> 0.1 * 3
0.30000000000000004
```

На практиці для більшості задач це незначно — якщо ти рахуєш приблизні відстані або температури, похибка в п'ятнадцятому знаку після коми не важлива. Але якщо пишеш фінансовий застосунок — там копійки мають значення. Для таких випадків є модуль `decimal` зі стандартної бібліотеки:

```python
from decimal import Decimal

# float — неточно
0.1 + 0.2                      # 0.30000000000000004

# Decimal — точно
Decimal("0.1") + Decimal("0.2") # Decimal('0.3')
```

Зверни увагу: `Decimal` ініціалізується рядком `"0.1"`, а не числом `0.1`. Якщо написати `Decimal(0.1)` — передасться вже неточний `float` і точність не допоможе:

```python
>>> Decimal(0.1)
Decimal('0.1000000000000000055511151231257827021181583404541015625')  # неточно!

>>> Decimal("0.1")
Decimal('0.1')  # точно
```

Ще наслідок тієї самої проблеми: ніколи не порівнюй `float` через `==`:

```python
>>> 0.1 + 0.2 == 0.3
False          # несподівано, але факт

>>> 0.1 + 0.2
0.30000000000000004  # трохи більше ніж 0.3
```

Правильний підхід — перевіряти що числа достатньо близькі одне до одного, в межах допустимої похибки. Python має вбудовану функцію для цього:

```python
import math

math.isclose(0.1 + 0.2, 0.3)           # True
math.isclose(0.1 + 0.2, 0.3, rel_tol=1e-9)  # True — можна задати допуск
```

Або вручну — перевірити що абсолютна різниця менша за дуже мале число:

```python
abs((0.1 + 0.2) - 0.3) < 1e-9    # True
```

## bool — логічне значення

`bool` — найпростіший тип за кількістю можливих значень: рівно два. `True` — істина, `False` — хиба. З великої літери — це обов'язково, `true` або `TRUE` Python не розуміє:

```python
is_logged_in   = True
has_permission = False
is_admin       = True
email_verified = False
```

На перший погляд — надто простий тип щоб приділяти йому окремий розділ. Насправді `bool` стоїть за кожним рішенням в програмі: кожне `if`, кожна умова циклу, кожна перевірка — все це опирається на логічні значення.

### Звідки беруться bool-значення

Найчастіше `True` і `False` з'являються не з literal-значень, а як результат порівнянь:

```python
age = 20

age > 18          # True — 20 більше 18
age < 18          # False — 20 не менше 18
age == 20         # True — 20 дорівнює 20
age != 20         # False — 20 не дорівнює 20
age >= 20         # True — 20 більше або рівне 20
age <= 19         # False — 20 не менше або рівне 19
```

Повторю важливу деталь, бо вона з'являлась вище і заслуговує повторення: `==` — це порівняння («чи рівні?»), `=` — це присвоєння («зроби рівним»). Плутати їх на початку — неминуче. Python у більшості випадків одразу каже про помилку, але є ситуації де обидва варіанти синтаксично правильні, але дають різний ефект. Запам'ятай раз і назавжди: **одинарний `=` — присвоєння, подвійний `==` — порівняння**.

Порівнювати можна не тільки числа:

```python
name = "Оксана"

name == "Оксана"   # True
name == "оксана"   # False — регістр важливий!
name != "Марія"    # True

"а" in "Оксана"    # True — пошук підрядка
"z" in "Оксана"    # False

len(name) > 3      # True — довжина рядка більша за 3
```

Оператор `in` — окремо корисний: перевіряє чи є підрядок в рядку, або чи є елемент у списку (про списки — в § 1.2.1).

### Логічні оператори: and, or, not

Умови можна комбінувати:

```python
age = 25
is_citizen = True
has_criminal_record = False

# and — обидві умови мають бути True
can_vote = age >= 18 and is_citizen
# True — вік підходить І громадянин

can_run_for_office = age >= 21 and is_citizen and not has_criminal_record
# True — вік OK, громадянин, і без судимостей

# or — достатньо хоч однієї True
has_access = is_admin or is_moderator
# True — якщо хоча б одна з умов виконана

# not — заперечення
is_not_admin = not is_admin
# False — бо is_admin = True
```

Важливо: `and` і `or` обчислюються ліниво (short-circuit evaluation). `and` зупиняється на першому `False` і не перевіряє решту. `or` зупиняється на першому `True`. Це не просто оптимізація — це поведінка яку можна використовувати:

```python
# Якщо users список порожній — його length не перевіряється
# (але якщо порожній — users вже False, і and не дійде до len)
if users and len(users) > 0:
    print(users[0])

# Це еквівалентно і читабельніше:
if users:
    print(users[0])
```

### bool — підтип int

Є цікава і важлива деталь: у Python `bool` є підтипом `int`. `True` — буквально ціле число `1`, `False` — `0`. Це не просто абстракція — це технічний факт:

```python
>>> True == 1
True

>>> False == 0
True

>>> True + True
2

>>> True + False
1

>>> sum([True, False, True, True, False])
3    # підрахунок кількості True у списку

>>> int(True)
1

>>> int(False)
0
```

Останній приклад — `sum()` для підрахунку True-значень — насправді використовується на практиці. Наприклад: «скільки користувачів зі списку є адмінами» — `sum(u.is_admin for u in users)`.

### Truthy і Falsy: логіка Python ширша за True/False

Python дозволяє використовувати будь-яке значення там, де очікується `bool`. Кожен тип має своє «логічне значення». Загальне правило: **порожнє і нульове — False, все інше — True**:

```python
# False (хибні значення)
bool(False)    # False — очевидно
bool(0)        # False — нуль
bool(0.0)      # False — нульовий float
bool("")       # False — порожній рядок
bool(None)     # False — відсутність значення
# (bool([]), bool({}), bool(()) — порожні колекції — теж False, але про них пізніше)

# True (правдиві значення — все інше)
bool(True)     # True
bool(1)        # True — ненульове число
bool(-5)       # True — від'ємне теж ненульове
bool(0.001)    # True — мікроскопічне але ненульове
bool("hi")     # True — непорожній рядок
bool(" ")      # True — пробіл — це вже непорожній рядок!
bool("False")  # True — рядок з текстом "False" — непорожній!
```

Остання деталь — класична пастка. `bool("False")` дає `True`, бо рядок `"False"` непорожній. Python не читає вміст рядка при перетворенні в `bool` — він лише перевіряє чи рядок є взагалі.

На практиці це дає зручний стиль перевірок:

```python
username = ""

# Довгий спосіб
if username != "" and username is not None:
    print(f"Привіт, {username}!")

# Ідіоматичний Python — те саме коротше
if username:
    print(f"Привіт, {username}!")
else:
    print("Ім'я не вказане")
```

Якщо `username` — порожній рядок або `None` — умова `if username` вже `False`. Якщо непорожній рядок — `True`. Це і є truthy/falsy в дії.

## None — відсутність значення

`None` — спеціальне значення яке означає «нічого». Не нуль. Не порожній рядок. Не `False`. Саме відсутність будь-якого значення.

```python
result   = None
user     = None
response = None
```

Чим `None` відрізняється від «порожніх» значень? Семантикою. `0` — це конкретне число, воно означає «нуль». `""` — порожній рядок, але це рядок. `None` — це «ще нічого немає» або «результату не існує».

### Коли зустрічається None

**Функція без return.** Якщо функція не має явного оператора `return` — вона автоматично повертає `None`. Це частий источник подиву у новачків:

```python
def greet(name):
    print(f"Привіт, {name}!")
    # немає return

result = greet("Оксана")   # функція виконалась, вивела текст
print(result)              # None — бо немає return
```

Функція вивела `"Привіт, Оксана!"` на екран — це її побічний ефект. Але значення яке вона **повернула** — `None`. Якщо очікуєш від функції якесь значення і отримуєш `None` — перевір чи є в ній `return`.

**Відсутність результату пошуку.** Функція шукала щось і не знайшла:

```python
# Метод dict.get() повертає None якщо ключ не знайдений
user_data = {"name": "Оксана", "age": 28}

phone = user_data.get("phone")    # None — ключа "phone" немає
name  = user_data.get("name")     # "Оксана" — є

# Перевірка результату
if phone is None:
    print("Телефон не вказаний")
else:
    print(f"Телефон: {phone}")
```

**Початкове значення до ініціалізації:**

```python
current_user = None

# ... якийсь код, який може авторизувати користувача ...

current_user = authenticate(request)

if current_user is None:
    redirect_to_login()
else:
    show_dashboard(current_user)
```

### is None vs == None

Перевіряти чи значення є `None` треба через `is`, а не через `==`:

```python
x = None

# Правильно
if x is None:
    print("Порожньо")

if x is not None:
    print("Є значення")

# Технічно працює, але не рекомендується
if x == None:
    print("Порожньо")
```

Різниця не лише стилістична. Оператор `is` перевіряє ідентичність — чи два імені вказують на один і той самий об'єкт у пам'яті. `None` в Python — singleton: існує лише один екземпляр `None` на всю програму. Тому `x is None` — точна і однозначна перевірка.

`== None` теж спрацьовує для стандартних типів. Але деякі бібліотеки (наприклад, NumPy для масивів) перевизначають оператор `==` і він може давати несподівані результати. `is None` — завжди надійний.

## type() і isinstance()

Ти вже бачив `type()` у REPL. Тепер розглянемо обидві функції детальніше.

`type()` повертає тип будь-якого об'єкта:

```python
print(type("Привіт"))   # <class 'str'>
print(type(42))         # <class 'int'>
print(type(3.14))       # <class 'float'>
print(type(True))       # <class 'bool'>
print(type(None))       # <class 'NoneType'>
```

`<class 'str'>` — Python каже: «це об'єкт класу str». Про класи поговоримо значно пізніше, поки читай це як «це рядок», «це ціле число» і так далі.

`type()` зручна для налагодження — коли не розумієш що за значення у змінній. Але для перевірок у логіці програми краще `isinstance()`:

```python
x = 42

isinstance(x, int)            # True
isinstance(x, float)          # False
isinstance(x, str)            # False
isinstance(x, (int, float))   # True — перевірка на кілька типів одразу
```

Кілька прикладів де `isinstance()` корисна на практиці:

```python
def process_value(value):
    if isinstance(value, str):
        return value.upper()          # рядки — у верхній регістр
    elif isinstance(value, (int, float)):
        return value * 2              # числа — подвоїти
    elif value is None:
        return "нічого"               # None — замінити на текст
    else:
        return value                  # все інше — повернути як є

print(process_value("привіт"))   # ПРИВІТ
print(process_value(21))         # 42
print(process_value(3.14))       # 6.28
print(process_value(None))       # нічого
```

### Чому isinstance() краща за type() для перевірок

`isinstance()` враховує успадкування. Пам'ятаєш що `bool` є підтипом `int`? Ось як це впливає:

```python
flag = True

type(flag) == int       # False — тип точно bool, не int
isinstance(flag, int)   # True — bool є підтипом int, тому True

type(flag) == bool      # True — точний збіг
isinstance(flag, bool)  # True — теж True
```

У реальному коді рідко потрібно відрізнити `bool` від `int`. Але коли пишеш функцію що має приймати «будь-яке число» — `isinstance(x, (int, float))` правильно включить і `bool`-значення, бо `True` і `False` — теж числа. `type(x) in (int, float)` — не включить.

Загальне правило: **`isinstance()` для перевірок у логіці, `type()` для інформаційних цілей** — наприклад вивести що за тип у змінній під час налагодження.

## Перетворення між типами

Python майже ніколи не перетворює типи автоматично — на відміну від JavaScript де `"5" + 3` дасть `"53"`. Тут це одразу помилка:

```python
>>> "5" + 3
TypeError: can only concatenate str (not "int") to str
```

Але перетворити можна явно, викликавши потрібний тип як функцію:

### str → числа

```python
int("42")           # 42
float("3.14")       # 3.14
int("100")          # 100

# Що не працює
int("3.14")         # ValueError: "3.14" — не ціле число
int("привіт")       # ValueError: не число взагалі
int("")             # ValueError: порожній рядок

# Обхід: float → потім int
int(float("3.14"))  # 3 — спочатку у float, потім відкидаємо дріб
```

На практиці перетворення рядків у числа трапляється постійно: введення користувача завжди приходить як рядок, навіть якщо людина ввела число. Тому `int(input("Введіть вік: "))` — стандартна конструкція.

### числа → str

```python
str(42)       # "42"
str(3.14)     # "3.14"
str(True)     # "True"
str(False)    # "False"
str(None)     # "None"

# Але зазвичай f-рядки зручніші
age = 28
message = f"Вам {age} років"   # краще ніж "Вам " + str(age) + " років"
```

### числа → int

```python
int(3.9)      # 3  — ВІДКИДАЄ дробову частину, не округлює!
int(3.1)      # 3
int(-3.9)     # -3 — теж відкидає, не -4!
int(True)     # 1
int(False)    # 0

# Щоб округлити — round(), не int()
round(3.9)    # 4  — математичне округлення
round(3.5)    # 4  — до парного при .5 (банківське округлення)
round(2.5)    # 2  — до парного
round(3.14159, 2)  # 3.14 — з двома знаками після крапки
```

Різниця між `int()` і `round()` — часта причина помилок. `int(3.9)` дає `3`, не `4`. Якщо потрібне округлення — завжди використовуй `round()`.

### будь-що → bool

```python
bool(0)         # False — нульове ціле
bool(0.0)       # False — нульовий float
bool("")        # False — порожній рядок
bool(None)      # False — відсутність значення

bool(1)         # True — ненульове число
bool(-1)        # True — від'ємне теж ненульове
bool("щось")    # True — непорожній рядок
bool("False")   # True! — непорожній рядок, хоч і "False"
bool("0")       # True! — рядок "0" непорожній, хоч і виглядає як нуль
```

Останні два рядки — класичні пастки. Python при конверсії в `bool` дивиться лише на те чи рядок порожній, а не на його вміст. `"False"` непорожній — значить `True`. `"0"` непорожній — значить `True`.

## Динамічна типізація і анотації

Python — мова з динамічною типізацією. Одна і та сама змінна може послідовно вміщувати значення різних типів:

```python
x = 42           # int
x = "тепер рядок"  # str
x = 3.14         # float
x = True         # bool
x = None         # NoneType
```

Python не заперечує — змінна `x` просто щоразу вказує на новий об'єкт іншого типу. Гнучкість очевидна. Але є і зворотній бік: якщо функція очікує число, а ти передав рядок — Python не попередить заздалегідь. Помилка виявиться лише під час виконання, можливо у зовсім іншому місці коду де цим значенням спробують скористатись.

Для боротьби з цим у Python 3.5 з'явились анотації типів — необов'язкові підказки про очікуваний тип:

```python
name:     str   = "Оксана"
age:      int   = 28
price:    float = 199.99
is_admin: bool  = False
result:   None  = None      # рідко потрібно
```

Синтаксис: ім'я, двокрапка, тип, знак `=`, значення. Python ці анотації **не перевіряє під час виконання** — вони для людини і для статичних аналізаторів на кшталт Pylance або mypy. Але користь від них відчутна:

```python
# Без анотацій — незрозуміло що тут зберігається
d = 7
u = True

# З анотаціями — одразу зрозуміло
days_remaining: int  = 7
is_active:      bool = True
```

Pylance у VS Code читає анотації і підкреслює місця де тип не збігається. Це ловить помилки ще до запуску:

```python
age: int = 28
age = "двадцять вісім"  # Pylance підкреслить: expected int, got str
```

У цьому курсі ми поступово вводитимемо анотації — спочатку там де вони найбільше допомагають читати код, пізніше систематично скрізь.

## Підсумок

П'ять базових типів — `str`, `int`, `float`, `bool`, `None` — це абетка Python. Жодна програма не обходиться без них, і кожна дрібна деталь їхньої поведінки варта розуміння:

`str` — текст у лапках. Незмінний. Підтримує конкатенацію і повторення. F-рядки — стандарт для підстановки значень. `"42"` і `42` — різні речі.

`int` — ціле число будь-якого розміру. `/` завжди дає `float`, для цілого ділення — `//`. `%` — залишок, корисний частіше ніж здається.

`float` — число з дробовою частиною. `0.1 + 0.2 ≠ 0.3` — це не баг, це двійкова арифметика. Для точних фінансових обчислень — `Decimal`. Порівнювати через `math.isclose()`.

`bool` — `True` або `False`. Є підтипом `int`. Будь-яке значення Python може бути truthy або falsy. Порожнє і нульове — `False`, все інше — `True`.

`None` — відсутність значення. Функція без `return` повертає `None`. Перевіряти через `is None`, не `== None`.

**Три речі щоб не забути:**

Перше — `=` це присвоєння, `==` це порівняння. Будеш плутати. Це нормально.

Друге — `float("False")` дає помилку, `bool("False")` дає `True`. Python не читає вміст рядка при конверсії в `bool`.

Третє — REPL завжди поруч. Будь-яку деталь поведінки типів можна перевірити за секунду.
