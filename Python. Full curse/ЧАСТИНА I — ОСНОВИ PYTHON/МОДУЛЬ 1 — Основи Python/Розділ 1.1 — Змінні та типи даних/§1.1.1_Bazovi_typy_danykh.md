# § 1.1.1 Базові типи даних

---

## Від абстракції до конкретності

У попередніх параграфах ми говорили про програмування як ідею: алгоритми, інструкції, мислення. Тепер — перший крок у реальний Python. І починаємо ми з найпростішого і водночас найфундаментальнішого: з типів даних.

Будь-яка програма, що б вона не робила, працює з даними. Числа, тексти, логічні значення, відсутність значення — це сировина, з якою Python оперує щомиті. Перш ніж навчитися складати з цих цеглинок щось більше, потрібно добре зрозуміти самі цеглинки: що вони означають, як їх створювати і як Python про них думає.

У Python є кілька вбудованих базових типів. Ми розглянемо п'ять, з якими ти зіштовхуватимешся щодня: рядок (`str`), ціле число (`int`), число з плаваючою крапкою (`float`), логічне значення (`bool`) і відсутність значення (`None`). Це не весь список типів у мові — але це фундамент, без якого далі нікуди.

---

## Що таке змінна

Перш ніж говорити про типи — одне речення про змінні, бо без них типи нікуди.

Змінна — це іменований контейнер для значення. Ти вигадуєш ім'я, ставиш знак `=` і праворуч пишеш значення, яке хочеш запам'ятати. Python виконає це як команду: «відклади ось це значення і дай йому таке ім'я, щоб я міг до нього звернутись пізніше».

```python
name = "Оксана"
age  = 28
```

Тепер у програмі є два «ящики»: один називається `name` і містить рядок `"Оксана"`, інший називається `age` і містить число `28`. У будь-якому місці програми можна написати `name` або `age` — і Python дістане відповідне значення.

Знак `=` тут не означає «рівність» у математичному сенсі — він означає присвоєння. Це команда «покласти значення у змінну», а не твердження про те, що дві речі рівні. Ця різниця дрібна на перший погляд, але важлива для розуміння.

Ім'я змінної може бути майже будь-яким: літери, цифри, підкреслення — за умови що не починається з цифри і не збігається з ключовими словами Python. `user_name`, `total_price`, `is_active`, `x`, `result_2` — все це валідні імена. `2result`, `if`, `class` — ні. Хорошу традицію щодо іменування ми ще розглянемо докладно, але вже зараз варто пам'ятати: ім'я має пояснювати що в ньому зберігається, а не просто позначати якийсь об'єкт.

## Рядок: str

Рядок — це послідовність символів. Текст, одне слово, речення, порожній рядок, набір цифр у лапках — усе це рядки.

```python
greeting    = "Привіт, світе!"
empty       = ""
single_char = "а"
number_text = "42"       # це рядок, не число — через лапки
multiline   = """Це рядок
на кілька
рядків"""
```

Зверни увагу: `"42"` і `42` — це різні речі. Перше — рядок, другий — число. Python не переплутає, але ти можеш — особливо на початку. З рядком `"42"` не можна рахувати (`"42" + 1` дасть помилку), а з числом `42` — можна.

Рядки можна писати в одинарних або подвійних лапках — Python ставиться до обох однаково. Важливо тільки щоб лапки на початку і в кінці збігались. Зазвичай розробники обирають один варіант і дотримуються його скрізь. У цьому курсі ми здебільшого використовуємо подвійні.

Потрійні лапки — `"""` або `'''` — дозволяють створити рядок на кілька рядків, зберігаючи переноси і відступи. Зручно для довгих текстів, шаблонів, або коли хочеш щоб рядок займав кілька рядків у коді для наочності.

### Дві речі, які потрібно знати про рядки відразу

Перша. Рядки незмінні. Якщо ти «змінюєш» рядок — насправді Python створює новий рядок із потрібним вмістом, а старий залишається недоторканим. Це стане важливим пізніше, коли ти почнеш активно обробляти текстові дані.

Друга. Рядки можна «складати» і «множити»:

```python
first = "Добрий"
second = " ранок"

result = first + second     # "Добрий ранок"
line   = "-" * 20           # "--------------------"
```

Плюс між рядками — це конкатенація: об'єднання двох рядків в один. Множення рядка на число — це повторення. Ці операції інтуїтивно зрозумілі і стануть тобі в нагоді раніше ніж здається.

## Ціле число: int

`int` — це ціле число. Додатне, від'ємне або нуль — без дробової частини.

```python
year     = 2025
score    = -15
zero     = 0
big      = 10_000_000
```

Підкреслення в числах — `10_000_000` — це суто візуальний трюк для читабельності. Python їх ігнорує при обчисленнях, але для людини що читає код різниця між `10000000` і `10_000_000` відчутна. Ти можеш ставити підкреслення у будь-якому місці між цифрами — `1_000`, `100_000`, навіть `1_0_0` — але перший варіант очевидно зрозуміліший.

Python не має обмеження на розмір цілого числа. В більшості мов програмування є максимальне значення для цілого числа — через те як пам'ять машини організована. Python обходить це автоматично: ціле число може бути скільки завгодно великим, мова сама подбає про виділення потрібної пам'яті. Це зручно і рятує від цілого класу непомітних помилок, які переслідують розробників в інших мовах.

Основні арифметичні операції:

```python
a = 17
b = 5

a + b    # 22  — додавання
a - b    # 12  — віднімання
a * b    # 85  — множення
a ** b   # 1419857  — степінь (2 ** 10 = 1024)
a // b   # 3   — ціле ділення (скільки разів b входить у a)
a % b    # 2   — залишок від ділення
a / b    # 3.4 — звичайне ділення (завжди дає float!)
```

Остання операція — `a / b` — заслуговує окремої уваги. У Python навіть якщо обидва операнди є цілими числами і результат ділення теж ціле — звичайний `/` завжди повертає `float`. `10 / 2` дасть `5.0`, а не `5`. Це Python 3 виправив спадщину Python 2, де `/` поводилось непередбачувано залежно від типів. Тепер правило просте: хочеш `float` — використовуй `/`, хочеш ціле — `//`.

## Число з плаваючою крапкою: float

`float` — це число з дробовою частиною. Температура, ціна, координата, відсоток — будь-яке число де важлива точність після коми.

```python
price      = 199.99
pi         = 3.14159
small      = 0.001
negative   = -7.5
scientific = 1.5e6      # 1.5 × 10⁶ = 1500000.0
tiny       = 2.7e-4     # 2.7 × 10⁻⁴ = 0.00027
```

Запис `1.5e6` — це наукова нотація. `e` тут означає «помножити на десять у ступені». Зручно для дуже великих або дуже маленьких чисел.

### Маленька пастка з float

Є одна особливість `float`, про яку варто знати з перших кроків, щоб потім не розгублюватись.

```python
0.1 + 0.2    # 0.30000000000000004
```

Так, це не помилка Python. Це фундаментальна особливість того, як комп'ютери зберігають числа з плаваючою крапкою у двійковій системі числення. Деякі дробові числа не можна представити точно в двійковому форматі — так само як `1/3` не можна точно записати десятковим дробом. `0.1` у двійковому — нескінченний дріб, і при зберіганні відбувається округлення, яке і дає таку поведінку.

На практиці це рідко створює проблеми — зазвичай достатньо округлити результат до потрібної кількості знаків. Але якщо ти пишеш фінансовий застосунок де кожна копійка важлива — використовуй модуль `decimal` зі стандартної бібліотеки, який дає точну арифметику з дробами.

```python
from decimal import Decimal

Decimal("0.1") + Decimal("0.2")    # Decimal('0.3') — точно
```

## Логічне значення: bool

`bool` — це тип який має лише два можливих значення: `True` (істина) і `False` (хиба). Саме з такими значеннями працюють умови: «якщо це правда — зроби те, інакше — зроби інше».

```python
is_logged_in   = True
has_permission = False
is_admin       = True
```

`True` і `False` пишуться з великої літери — це ключові слова Python, і регістр тут принциповий. `true` або `TRUE` — не спрацює.

### bool — підтип int

Цікава деталь: у Python `bool` є підтипом `int`. `True` — це буквально `1`, `False` — це `0`. Тому можна писати:

```python
True + True     # 2
True + False    # 1
True * 5        # 5
```

Це рідко потрібно на практиці, але знати корисно — особливо коли зустрінеш код, де хтось використовує `bool` значення в арифметиці, наприклад для підрахунку кількості виконаних умов у списку.

### Як виникають bool значення

Найчастіше `bool` значення з'являються не з літералів `True`/`False`, а як результат порівнянь і перевірок:

```python
age = 20

age > 18         # True
age == 18        # False  — рівність перевіряється через ==, не =
age != 18        # True   — нерівність
age >= 20        # True
age <= 19        # False

name = "Іван"
name == "Іван"   # True
name == "іван"   # False  — регістр важливий
```

Зверни увагу: подвійне `==` — це порівняння («чи рівні?»), одинарне `=` — це присвоєння («зроби рівним»). Плутати їх — класична і дуже поширена помилка. Python у більшості випадків одразу скаже про помилку, але не завжди.

### Truthiness: не тільки True і False

Python дозволяє використовувати будь-яке значення там, де очікується `bool`. Кожен тип має своє уявлення про «правду» і «хибу»:

```python
bool(1)       # True   — ненульове число
bool(0)       # False  — нуль
bool(-5)      # True   — будь-яке ненульове, навіть від'ємне
bool(0.0)     # False  — нульовий float
bool("")      # False  — порожній рядок
bool("hi")    # True   — непорожній рядок
bool(" ")     # True   — рядок з пробілом вже не порожній
bool(None)    # False
```

Загальне правило: «порожнє» є хибою, «непорожнє» є істиною. Це дає зручний спосіб перевіряти наявність даних:

```python
username = ""

if username:
    print(f"Привіт, {username}!")
else:
    print("Ім'я не вказане")
```

Замість `if username != ""` можна просто написати `if username` — Python сам перевірить чи рядок непорожній. Це ідіоматичний Python — так прийнято писати, і так краще читається.

## Відсутність значення: None

`None` — це спеціальне значення, яке означає «нічого». Не нуль, не порожній рядок, не `False` — саме відсутність будь-якого значення.

```python
result   = None
response = None
```

Навіщо потрібне `None`? У кількох ситуаціях.

По-перше, як значення за замовчуванням до того як щось відбулось. Функція яка шукає запис у базі даних і не знаходить його — повертає `None`. Це чіткий сигнал: «я спробував, але результату немає», на відміну від нуля або порожнього рядка, які є реальними значеннями і можуть щось означати.

По-друге, як початкове значення змінної яка буде заповнена пізніше:

```python
user = None

# ... якийсь код ...

user = get_current_user()

if user is None:
    print("Користувач не авторизований")
```

Перевіряти `None` варто саме через `is None`, а не через `== None`. Технічно обидва варіанти спрацюють, але `is` — правильний Python-спосіб для перевірки на `None`. Оператор `is` перевіряє ідентичність — чи це той самий об'єкт, а не просто рівне значення. `None` в Python — це один-єдиний об'єкт на всю програму, тому `is None` — точніше і читабельніше.

## type() і isinstance(): дізнатись тип

Python — мова з динамічною типізацією. Це означає що одна і та сама змінна може у різний момент містити значення різних типів — Python не забороняє присвоїти спочатку число, а потім рядок в ту саму змінну. Це гнучкість, але і відповідальність: іноді потрібно явно перевірити якого типу значення зараз у змінній.

Функція `type()` повертає тип переданого значення:

```python
name = "Оксана"
age  = 28
pi   = 3.14
flag = True

print(type(name))    # <class 'str'>
print(type(age))     # <class 'int'>
print(type(pi))      # <class 'float'>
print(type(flag))    # <class 'bool'>
print(type(None))    # <class 'NoneType'>
```

`<class 'str'>` — це Python каже: «це об'єкт класу str». Про класи ми поговоримо значно пізніше, поки достатньо читати це просто як «це рядок».

Функція `isinstance()` дещо зручніша для перевірок — вона повертає `True` або `False`:

```python
age = 28

isinstance(age, int)          # True
isinstance(age, float)        # False
isinstance(age, (int, float)) # True — перевірка на кілька типів одразу
```

Третій рядок особливо корисний: можна передати кортеж типів і перевірити чи значення є хоча б одним з них.

### type() проти isinstance(): коли що використовувати

На практиці `isinstance()` надається перевага перед `type()` для перевірок у реальному коді. Причина: `isinstance()` враховує успадкування. Якщо є клас-батько і клас-нащадок — `isinstance()` поверне `True` для обох, `type() == ` — тільки для точного збігу. З `bool` і `int` ця різниця вже помітна:

```python
flag = True

type(flag) == int       # False — тип точно bool
isinstance(flag, int)   # True  — bool є підтипом int
```

Тому загальне правило: `isinstance()` для перевірок типу у логіці програми, `type()` коли потрібно дізнатись точний тип для інформаційних цілей — наприклад, для виведення або логування.

## Перетворення між типами

Python часто не перетворює типи автоматично — треба робити це явно. Для цього є функції `int()`, `float()`, `str()`, `bool()`:

```python
# Рядок → число
int("42")          # 42
float("3.14")      # 3.14
int("3.14")        # ValueError: не можна — "3.14" не ціле число
int(float("3.14")) # 3 — спочатку float, потім обрізати дріб

# Число → рядок
str(42)            # "42"
str(3.14)          # "3.14"
str(True)          # "True"

# Число → int (обрізання дробової частини, не округлення!)
int(3.9)           # 3  — обрізає, а не округляє
int(-3.9)          # -3 — теж обрізає (до нуля)
int(True)          # 1
int(False)         # 0

# Будь-що → bool
bool(0)            # False
bool(42)           # True
bool("")           # False
bool("False")      # True — непорожній рядок, навіть якщо в ньому "False"
```

Остання деталь — важлива пастка для новачків. `bool("False")` дає `True`, бо рядок `"False"` непорожній. Python дивиться не на вміст рядка, а на те чи він є. Якщо хочеш перевірити що в рядку написано — порівнюй через `==`:

```python
user_input = "False"

bool(user_input)           # True  — небажана поведінка
user_input == "True"       # False — правильна перевірка
user_input.lower() == "true" # False — на випадок різного регістру
```

## Анотації типів: підказки Python

Починаючи з Python 3.5, можна явно вказувати тип змінної прямо в коді — не для Python (він все одно не перевіряє це під час виконання), а для людини і для таких інструментів як Pylance або mypy:

```python
name:  str   = "Оксана"
age:   int   = 28
price: float = 199.99
flag:  bool  = True
```

Двокрапка після імені змінної, потім тип, потім `=` і значення. Це не обов'язково — Python запустить код і без анотацій. Але з ними код стає зрозумілішим: ти відразу бачиш що має зберігатись у кожній змінній, а Pylance підкреслить місця де ти передаєш значення не того типу.

У цьому курсі ми будемо поступово додавати анотації типів — спочатку рідко і тільки там де це справді допомагає читати код, а потім все частіше, коли дійдемо до більш складних конструкцій.

## Підсумок

П'ять базових типів Python — це будівельні блоки всього іншого:

`str` зберігає текст. Рядки незмінні, їх можна складати і множити.

`int` зберігає цілі числа без обмеження розміру. Ділення через `/` завжди дає `float`, через `//` — ціле.

`float` зберігає числа з дробовою частиною. Має особливість неточних обчислень для деяких значень — результат двійкового представлення.

`bool` має два значення: `True` і `False`. Є підтипом `int`. Будь-яке значення Python може бути оцінене як `True` або `False`.

`None` означає відсутність значення — не нуль, не порожній рядок, а саме «нічого». Перевіряється через `is None`.

`type()` і `isinstance()` дозволяють дізнатись або перевірити тип. `isinstance()` краще для перевірок у логіці — враховує успадкування.

Перетворення між типами — явне через `int()`, `float()`, `str()`, `bool()`. Неявного приведення типів у Python майже немає — це свідоме рішення мови.
