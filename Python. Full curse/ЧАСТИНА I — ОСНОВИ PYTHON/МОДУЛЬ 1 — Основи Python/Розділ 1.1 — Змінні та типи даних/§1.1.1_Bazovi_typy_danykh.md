# § 1.1.1 Базові типи даних

---

## Модуль 0 позаду. Модуль 1 починається тут

Якщо ти дійшов до цього місця і виконав завдання нульового модуля — у тебе вже є те, чого немає у більшості людей, які «починають вчити Python»: налаштоване середовище, встановлені інструменти, перший репозиторій і розуміння навіщо все це потрібно. pyenv слідкує за версіями Python. Віртуальне середовище ізолює залежності. VS Code з розширеннями підказує і форматує. Git зберігає кожен крок. pre-commit не пропускає неохайний код. Це не дрібниці — це фундамент, на якому будуватиметься всі наступні місяці роботи. І він вже стоїть.

Тепер — інший тип роботи. Модуль 0 був про інструменти і середовище, про те як влаштована майстерня. Модуль 1 — про саму мову. Починаємо писати Python не заради `hello.py` і не заради перевірки що все запустилось, а заради реального розуміння: як Python думає про дані, як він зберігає значення, що з ними можна робити і де чатують типові пастки. Це найважливіший модуль у всьому курсі — не тому що найцікавіший, а тому що все що прийде після будується на тому, що тут.

Варто чесно попередити: деякі речі в цьому модулі здадуться банальними. «Ціле число — це ціле число, чого тут пояснювати?» Але Python — мова з власним характером, і цей характер проявляється у деталях, які на перший погляд непомітні. Чому `0.1 + 0.2` не дорівнює `0.3`. Чому функція повернула `None` там де ти очікував рядок. Чому `"42"` і `42` — принципово різні речі хоч і виглядають схоже. Чому `bool("False")` дає `True`. Кожна з цих деталей колись зупинить програму — і людина яка розуміє чому це відбувається, виправить за хвилину. Людина яка не розуміє — витратить годину на пошук «якогось дивного бага».

Модуль 1 охоплює змінні і типи, рядки і числа, умови і цикли, функції, структури даних — списки, словники, множини, кортежі — і об'єктно-орієнтоване програмування. Це багато, і це займе час. Але кожен крок тут конкретний і практичний: новий розділ — нові інструменти в руках, нові задачі які можна вирішити. До кінця модуля у тебе буде достатньо для написання справжніх, корисних програм — не навчальних вправ, а речей які реально щось роблять.

Є одна порада перед початком, і вона стосується всього модуля: **не просто читай — пиши код**. Кожен приклад у цьому параграфі — введи в REPL або напиши у файлі і запусти. Поміняй значення. Зламай. Подивись що станеться. Програмування — це не знання яке засвоюється через читання, це навичка яка розвивається через практику. Різниця між тим хто читав і тим хто робив руками — через місяць стає очевидною.

## Перший живий діалог з Python

Відкрий термінал, переконайся що віртуальне середовище активоване, і запусти Python в інтерактивному режимі:

```bash
python
```

Ти потрапиш у REPL — Read-Eval-Print Loop. Назва розшифровується просто: Read — Python читає те, що ти ввів; Eval — виконує; Print — виводить результат; Loop — і чекає наступного рядка. Це середовище прямої розмови з інтерпретатором, без файлів і без запуску скриптів:

```
Python 3.12.3 (main, Apr 15 2024, 17:43:11)
Type "help", "copyright", "credits" or "license" for more information.
>>>
```

Символ `>>>` — запрошення до введення. Після нього набираєш будь-який Python-вираз, натискаєш Enter — і Python одразу показує результат. Введи кілька речей, щоб відчути як це працює:

```python
>>> 2 + 2
4
>>> 10 * 3
30
>>> "Привіт"
'Привіт'
>>> type(42)
<class 'int'>
>>> type("hello")
<class 'str'>
```

Зверни увагу: коли ти ввів `"Привіт"` — Python вивів `'Привіт'` в одинарних лапках. Це не помилка. У REPL Python показує так зване repr-представлення значення — технічне відображення, де рядки завжди в одинарних лапках, навіть якщо ти вводив у подвійних. Коли ж ти використовуєш `print()` — відображення вже «людське», без зайвих лапок:

```python
>>> print("Привіт")
Привіт
>>> "Привіт"
'Привіт'
```

REPL — ідеальне місце для експериментів протягом усього курсу. Не закривай його — відкрий поруч з редактором і перевіряй будь-яку ідею прямо тут. Вийти з REPL — команда `exit()` або `Ctrl+D` на macOS/Linux, `Ctrl+Z` і Enter на Windows.

## Що таке тип даних і навіщо він потрібен

Будь-яка програма працює з даними. Але дані бувають дуже різними: ім'я користувача, ціна товару, кількість елементів у кошику, прапорець «чи авторизований», відсутність результату пошуку. Кожне з цих значень — це не просто набір байтів у пам'яті. Це значення певного **типу**, і тип визначає що з ним можна робити.

З числами можна рахувати: додавати, множити, порівнювати. З рядками можна об'єднувати, шукати підрядки, змінювати регістр. Але не можна рядок помножити на рядок, або додати число до тексту без явного перетворення. Python стежить за цим суворо — і якщо ти спробуєш зробити щось, що не має сенсу для певного типу, він скаже тобі про це негайно, а не дасть програмі тихо дати неправильну відповідь.

Спробуй у REPL:

```python
>>> "5" + 3
TypeError: can only concatenate str (not "int") to str
```

Це не Python примхливий — це він чесний. Рядок `"5"` і число `3` — різні типи, і складання їх разом не має однозначного сенсу. Чи потрібно отримати рядок `"53"`? Чи число `8`? Python не вгадує. Він вимагає щоб ти сказав явно.

У Python є п'ять базових типів з якими починається будь-яка програма: рядок `str`, ціле число `int`, число з плаваючою крапкою `float`, логічне значення `bool` і відсутність значення `None`. Є й інші типи — списки, словники, кортежі, множини — але ці п'ять є фундаментом і зустрічаються скрізь без винятку.

## Що таке змінна

Перш ніж говорити про типи — потрібно поговорити про змінні, бо без них жоден тип не має практичного застосування.

Уяви що ти пишеш програму, яка вітає користувача. Ім'я користувача тобі знадобиться кілька разів: вивести у привітанні, записати в лог, підставити в заголовок сторінки. Можна писати `"Оксана"` в кожному місці окремо. Але якщо завтра Оксана змінить ім'я — доведеться знайти і поміняти кожне місце. Це неефективно і ненадійно.

Змінна вирішує цю задачу: ти запам'ятовуєш значення під певним іменем, а потім використовуєш ім'я скільки завгодно разів. Зміниш значення в одному місці — всі посилання оновляться автоматично.

```python
name = "Оксана"
print(name)                       # Оксана
print(f"Привіт, {name}!")         # Привіт, Оксана!
print(f"До побачення, {name}!")   # До побачення, Оксана!
```

Рядок `name = "Оксана"` — це оператор присвоєння. Знак `=` тут не означає «рівність» у математичному сенсі. Це команда: «візьми значення праворуч і прив'яжи його до імені ліворуч». В математиці `x = 5` — це твердження що x дорівнює п'яти. У Python це інструкція: зроби так, щоб `name` вказував на рядок `"Оксана"`.

Насправді змінна в Python — це ярлик, а не ящик. Коли ти пишеш `name = "Оксана"`, Python створює об'єкт-рядок `"Оксана"` десь у пам'яті і прикріплює до нього ярлик `name`. Якщо потім написати `name = "Марія"` — той самий ярлик перекріплюється на новий об'єкт. Старий рядок `"Оксана"` нікуди не зник — він просто більше не має імені, і Python прибере його згодом.

### Правила іменування змінних

Ім'я змінної може містити латинські літери, цифри і підкреслення — але не може починатись з цифри і не може збігатись з ключовими словами Python:

```python
# Правильні імена
user_name   = "Іван"
age2        = 30
_private    = "внутрішнє"
total_price = 499.99

# Неправильні імена
2name = "Іван"   # SyntaxError: починається з цифри
if = True        # SyntaxError: if — ключове слово Python
class = "клас"   # SyntaxError: class — ключове слово
```

Ключових слів Python небагато: `if`, `else`, `elif`, `for`, `while`, `def`, `class`, `return`, `import`, `from`, `True`, `False`, `None`, `and`, `or`, `not`, `in`, `is` — та ще кілька. Всі вони зарезервовані для синтаксису мови.

За угодою у Python змінні пишуть у стилі **`snake_case`**: всі слова маленькими літерами, розділені підкресленням. `first_name`, `order_total`, `is_logged_in` — так прийнято. Не `firstName` — це JavaScript-стиль. Не `FirstName` — це стиль для класів. Ця угода не є технічною вимогою, але вся Python-екосистема її дотримується, тому відхилення робить твій код менш читабельним.

Ім'я змінної має **пояснювати що в ній зберігається**:

```python
# Погано — що таке d? хто такий u?
d = 7
u = "test@example.com"
f = True

# Добре — все зрозуміло без коментарів
days_until_deadline = 7
user_email          = "test@example.com"
is_admin            = True
```

## str — рядок

Рядок — це послідовність символів. Будь-який текст: ім'я, адреса, повідомлення, HTML-код, номер телефону записаний як текст, JSON-відповідь від API — все це рядки.

Щоб створити рядок, просто беремо текст у лапки. Одинарні або подвійні — Python ставиться до обох однаково:

```python
name       = "Оксана"
city       = 'Київ'
empty      = ""       # порожній рядок — теж рядок
digit_text = "42"     # РЯДОК, не число — бо в лапках
```

Остання деталь надзвичайно важлива. `"42"` і `42` — це принципово різні речі, хоч і виглядають схоже:

```python
>>> type("42")
<class 'str'>
>>> type(42)
<class 'int'>

>>> "42" + "8"
'428'       # конкатенація рядків: просто склеюємо тексти

>>> 42 + 8
50          # арифметика: математичне додавання

>>> "42" + 8
TypeError   # несумісні типи
```

Рядок `"42"` для Python — це послідовність двох символів `"4"` і `"2"`, і він нічого не знає про те що вони разом утворюють число. Тому щоб робити математику з числами, які прийшли як текст (наприклад, введені користувачем через клавіатуру), їх спочатку потрібно перетворити.

Потрійні лапки — `"""` або `'''` — дозволяють рядку займати кілька рядків коду, зберігаючи переноси і відступи:

```python
description = """Це рядок
на кілька рядків.
Переноси збережені."""

sql_query = """
    SELECT name, email
    FROM users
    WHERE is_active = TRUE
"""
```

Потрійні лапки ще використовуються для документаційних рядків — docstrings — коментарів, що пояснюють функції і класи. Але про це — коли дійдемо до функцій.

Детально методи рядків, f-рядки, escape-послідовності і конкатенацію розглянемо в § 1.1.2.

## int — ціле число

`int` — це цілі числа: будь-яке ціле додатне, від'ємне або нуль.

```python
age:   int = 25
year:  int = 2025
score: int = -15
big:   int = 10_000_000   # підкреслення для читабельності
```

Підкреслення в числах — лише для читабельності. Python їх ігнорує при обчисленнях, але для людини що читає код різниця між `10000000` і `10_000_000` очевидна.

Важлива особливість Python: ціле число може бути **будь-якого розміру**. У більшості мов програмування є максимальне значення для цілого числа — наприклад, у Java `int` максимум 2 147 483 647, і якщо перевищити — число «переповнюється» і стає від'ємним. Python обходить це автоматично: якщо число велике, Python просто виділяє більше пам'яті:

```python
>>> 2 ** 100
1267650600228229401496703205376   # число з 31 цифрою — без проблем
```

Арифметичні оператори, пріоритет операцій і модуль `math` розглянемо детально в § 1.1.3. Поки запам'ятай одну деталь: ділення `/` в Python **завжди** повертає `float`, навіть якщо обидва числа цілі:

```python
>>> 10 / 2
5.0        # float, не 5!
```

Якщо потрібне ціле ділення — є оператор `//`:

```python
>>> 10 // 2
5          # int
>>> 17 // 5
3          # ціла частина від ділення
```

## float — число з плаваючою крапкою

`float` — це числа з дробовою частиною. Температура, ціна, координата, відсоток, результат ділення — всюди де потрібна точність після коми.

```python
price:       float = 19.99
pi:          float = 3.14159265358979
temperature: float = -7.5
```

Є ще наукова нотація — зручна для дуже великих або дуже маленьких чисел. Буква `e` означає «помножити на десять у степені»:

```python
avogadro    = 6.022e23    # 6.022 × 10²³
light_speed = 3e8         # 300 000 000.0 — швидкість світла м/с
tiny        = 9.11e-31    # 9.11 × 10⁻³¹
```

Коли в одному виразі є і `int`, і `float` — Python автоматично перетворює `int` на `float`:

```python
>>> 5 + 2.0
7.0        # float

>>> type(5 + 2.0)
<class 'float'>
```

### Пастка: float і точність

Є одна особливість `float`, про яку треба знати з перших кроків:

```python
>>> 0.1 + 0.2
0.30000000000000004
```

Це не помилка Python. Деякі десяткові дроби не мають точного двійкового представлення — так само як `1/3` у десятковій: нескінченна дріб `0.333...` Те саме відбувається з `0.1` у двійковій системі, і при складанні мікро-похибки накопичуються. Для більшості задач це незначно. Але якщо пишеш фінансовий застосунок — є модуль `decimal`:

```python
from decimal import Decimal

Decimal("0.1") + Decimal("0.2")   # Decimal('0.3') — точно
```

Зверни увагу: `Decimal` ініціалізується **рядком** `"0.1"`, а не числом `0.1`. Якщо передати `float` — неточність вже там:

```python
>>> Decimal(0.1)    # неточно!
Decimal('0.1000000000000000055511151231257827021181583404541015625')

>>> Decimal("0.1")  # точно
Decimal('0.1')
```

З тієї ж причини ніколи не порівнюй `float` через `==`:

```python
>>> 0.1 + 0.2 == 0.3
False     # несподівано, але факт
```

Про правильне порівняння float через `math.isclose()` — в § 1.1.3.

## bool — логічне значення

`bool` має рівно два значення: `True` — істина, `False` — хиба. З великої літери — це обов'язково, `true` або `TRUE` Python не розуміє:

```python
is_admin:   bool = True
has_access: bool = False
```

`bool` стоїть за кожним рішенням в програмі: кожне `if`, кожна умова циклу, кожна перевірка — все це опирається на логічні значення.

### Звідки беруться bool-значення

Найчастіше `True` і `False` з'являються як результат порівнянь:

```python
age = 20

age > 18    # True
age == 20   # True  — == порівняння, = присвоєння!
age != 18   # True
age >= 20   # True
age <= 19   # False
```

Важлива деталь: `==` — це порівняння («чи рівні?»), `=` — це присвоєння («зроби рівним»). Плутати їх на початку — неминуче. Запам'ятай: **одинарний `=` — присвоєння, подвійний `==` — порівняння**.

Умови можна комбінувати через `and`, `or`, `not`:

```python
age        = 25
is_citizen = True

can_vote = age >= 18 and is_citizen    # True — обидві умови
has_id   = age >= 16 or is_citizen     # True — хоча б одна
```

### bool — підтип int

У Python `bool` є підтипом `int`. `True` — буквально `1`, `False` — `0`:

```python
>>> True == 1
True
>>> True + True
2
>>> int(True)
1
>>> int(False)
0
```

### Truthy і Falsy

Python дозволяє використовувати будь-яке значення там де очікується `bool`. Загальне правило: **порожнє і нульове — False, все інше — True**:

```python
bool(0)      # False — нуль
bool(0.0)    # False — нульовий float
bool("")     # False — порожній рядок
bool(None)   # False — відсутність значення

bool(1)      # True — ненульове число
bool(-5)     # True — від'ємне теж ненульове
bool("hi")   # True — непорожній рядок
bool("False") # True! — рядок "False" непорожній
bool("0")    # True! — рядок "0" непорожній
```

Останні два — класичні пастки. `bool("False")` дає `True`, бо рядок непорожній — Python не читає вміст рядка при конверсії в `bool`, він лише перевіряє чи рядок є взагалі.

На практиці це дає зручний стиль перевірок:

```python
username = ""

if username:          # те саме що if username != ""
    print(f"Привіт, {username}!")
else:
    print("Ім'я не вказане")
```

## None — відсутність значення

`None` — спеціальне значення яке означає «нічого». Не нуль. Не порожній рядок. Не `False`. Саме відсутність будь-якого значення.

```python
result   = None
response = None
```

### Коли зустрічається None

**Функція без return.** Якщо функція не має явного `return` — вона автоматично повертає `None`. Це частий источник подиву у новачків:

```python
def greet(name):
    print(f"Привіт, {name}!")
    # немає return

result = greet("Оксана")   # вивела текст на екран...
print(result)              # None — бо немає return
```

Якщо очікуєш від функції якесь значення і отримуєш `None` — перевір чи є в ній `return` з потрібним значенням.

**Відсутність результату пошуку.** Функція шукала щось і не знайшла:

```python
user_data = {"name": "Оксана", "age": 28}

phone = user_data.get("phone")   # None — ключа "phone" немає
name  = user_data.get("name")    # "Оксана" — є

if phone is None:
    print("Телефон не вказаний")
```

**Початкове значення до ініціалізації:**

```python
current_user = None

# ... якийсь код ...

current_user = authenticate(request)

if current_user is None:
    redirect_to_login()
```

### is None vs == None

Перевіряти чи значення є `None` треба через `is`, а не через `==`:

```python
x = None

if x is None:       # правильно
    print("Порожньо")

if x is not None:   # правильно
    print("Є значення")

if x == None:       # технічно працює, але не рекомендується
    ...
```

`None` в Python — singleton: існує лише один екземпляр `None` на всю програму. Тому `is None` — точна і однозначна перевірка. `== None` теж спрацьовує для стандартних типів, але деякі бібліотеки перевизначають `==` і він може давати несподівані результати.

## type() і isinstance()

`type()` повертає тип будь-якого об'єкта:

```python
name:  str   = "Оксана"
age:   int   = 28
price: float = 19.99
flag:  bool  = True

print(type(name))    # <class 'str'>
print(type(age))     # <class 'int'>
print(type(price))   # <class 'float'>
print(type(flag))    # <class 'bool'>
print(type(None))    # <class 'NoneType'>
```

`isinstance()` зручніша для перевірок у логіці — вона повертає `True` або `False` і враховує успадкування типів:

```python
age = 28

isinstance(age, int)            # True
isinstance(age, float)          # False
isinstance(age, (int, float))   # True — перевірка на кілька типів одразу
```

Саме через успадкування `isinstance()` краща за `type()` для перевірок:

```python
flag = True

type(flag) == int       # False — тип точно bool
isinstance(flag, int)   # True  — bool є підтипом int
```

Загальне правило: **`isinstance()` для перевірок у логіці програми, `type()` для інформаційних цілей** — наприклад вивести що за тип у змінній під час налагодження.

## Перетворення між типами

Python майже ніколи не перетворює типи автоматично — треба робити це явно:

```python
# str → число
int("42")            # 42
float("3.14")        # 3.14
int("3.14")          # ValueError — "3.14" не ціле число
int(float("3.14"))   # 3 — спочатку у float, потім відкидаємо дріб

# число → str
str(42)              # "42"
str(3.14)            # "3.14"
str(True)            # "True"

# число → int (відкидає дробову частину, не округлює!)
int(3.9)             # 3  — відкидає, а не округлює
int(-3.9)            # -3 — теж відкидає (до нуля)
int(True)            # 1
int(False)           # 0

# будь-що → bool
bool(0)              # False
bool(42)             # True
bool("")             # False
bool("False")        # True! — непорожній рядок
bool("0")            # True! — теж непорожній рядок
```

Різниця між `int()` і `round()` — часта причина помилок. `int(3.9)` дає `3`, не `4`. Якщо потрібне округлення — `round()`:

```python
round(3.9)          # 4  — математичне округлення
round(3.14159, 2)   # 3.14
```

## Анотації типів

Починаючи з Python 3.5, можна явно вказувати тип змінної — не для Python (він не перевіряє це під час виконання), а для людини і для статичних аналізаторів на кшталт Pylance або mypy:

```python
name:  str   = "Оксана"
age:   int   = 28
price: float = 19.99
flag:  bool  = True
```

Двокрапка після імені змінної, потім тип, потім `=` і значення. Pylance у VS Code читає анотації і підкреслює місця де тип не збігається — це ловить помилки ще до запуску.

У цьому курсі ми будемо поступово додавати анотації — спочатку там де вони найбільше допомагають читати код, пізніше систематично скрізь.

## Підсумок

П'ять базових типів — `str`, `int`, `float`, `bool`, `None` — це фундамент на якому стоїть все інше. Кожен з них з'являтиметься в кожній програмі.

Три речі щоб не забути:

`=` — присвоєння. `==` — порівняння. Плутати їх буде кожен, і не один раз. Це нормально.

`0.1 + 0.2` у float — не `0.3`. Це не баг, це двійкова арифметика. Для грошей — `Decimal`.

`None` перевіряється через `is None`, не `== None`. Функція без `return` повертає `None` — перевір це якщо результат виявився несподіваним.
