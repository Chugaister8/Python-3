# § 1.1.3 Числа та математика

## Числа — простіші ніж здається, але є нюанси

Здавалося б, числа — найпростіша частина програмування. Додав, відняв, помножив. Все що вчили в школі. І так, більшість арифметики в Python саме така — жодних сюрпризів. Але є кілька місць де Python поводиться не зовсім так як можна очікувати, і краще знати про них заздалегідь ніж натрапити випадково.

Цей параграф — про арифметику Python, пріоритет операцій, вбудовані числові функції і модуль `math`. Деякі речі ти вже бачив у § 1.1.1 — тут вони розкриваються повністю.

---

## Сім арифметичних операторів

Python має сім операторів для роботи з числами. Перші чотири — знайомі з математики:

```python
a = 17
b = 5

a + b    # 22      — додавання
a - b    # 12      — віднімання
a * b    # 85      — множення
a ** b   # 1419857 — піднесення до степеня (17⁵)
```

І три специфічних для програмування:

```python
a / b    # 3.4 — звичайне ділення
a // b   # 3   — ціле ділення
a % b    # 2   — залишок від ділення
```

Про кожен з останніх трьох — окремо, бо кожен має свої нюанси.

---

### / — ділення, яке завжди float

Оператор `/` в Python 3 **завжди** повертає `float`. Навіть якщо обидва числа цілі. Навіть якщо результат без залишку:

```python
>>> 10 / 2
5.0        # float, не int!

>>> 7 / 7
1.0        # теж float

>>> type(10 / 2)
<class 'float'>
```

Якщо ти прийшов з Python 2 або з іншої мови де `10 / 2` дає `5` — ця поведінка спочатку дратує. Але насправді вона правильна. У Python 2 `10 / 2` давало `5`, а `10 / 3` давало `3` — і це ховало помилки: люди писали ділення і не розуміли чому результат цілий. Python 3 вирішив цю проблему радикально: `/` завжди дає точний результат у вигляді `float`. Якщо хочеш ціле — скажи явно через `//`.

---

### // — ціле ділення

`//` ділить і відкидає дробову частину. Повертає `int` якщо обидва операнди `int`, і `float` якщо хоча б один `float`:

```python
>>> 10 // 3
3          # 3.333... → відкидає .333

>>> 17 // 5
3          # 3.4 → відкидає .4

>>> 10 // 2
5          # без залишку — 5, не 5.0

>>> 10.0 // 3
3.0        # float // int → float
```

Важливий момент з від'ємними числами. `//` завжди **округлює вниз** — не до нуля, а до менш нескінченності:

```python
>>> -7 // 2
-4         # не -3! — округлює вниз до -4

>>> 7 // -2
-4         # теж -4 — вниз по числовій осі
```

Більшість людей очікують `-3` — «відкинути дробову частину» від `-3.5`. Але `//` округлює вниз по числовій осі, тому `-3.5` → `-4`. Якщо потрібне «відкидання до нуля» — використовуй `int()`:

```python
int(-7 / 2)    # -3 — int() відкидає дробову саме до нуля
```

---

### % — залишок від ділення

`%` — один з тих операторів, які спочатку здаються рідкісними, але потім зустрічаються постійно.

Основне застосування — перевірити чи число ділиться на інше без залишку. Класичний приклад: **парність**. Парне число ділиться на 2 без залишку, тому `n % 2 == 0` означає «n парне»:

```python
>>> 10 % 2
0     # 10 парне

>>> 7 % 2
1     # 7 непарне

>>> 13 % 5
3     # 13 = 5×2 + 3, залишок 3
```

Але застосувань значно більше. Циклічне переміщення: якщо є список з `n` елементів і потрібно після останнього знову перейти до першого — `index % n` завжди дасть валідний індекс. Конвертація одиниць часу: `хвилини = секунди % 60`, `години = хвилини % 60`. Перевірка на кратність: «кожен п'ятий елемент» — `i % 5 == 0`.

Є ще один пов'язаний інструмент — `divmod()`. Він повертає відразу і частку, і залишок:

```python
>>> divmod(17, 5)
(3, 2)     # (частка, залишок)

hours, minutes = divmod(135, 60)   # 135 хвилин → 2 год 15 хв
print(f"{hours}г {minutes}хв")    # 2г 15хв
```

Зручно коли потрібні обидва значення — не треба рахувати двічі.

---

### Скорочені оператори присвоєння

Коли змінюєш значення змінної на основі поточного — є зручний скорочений запис:

```python
x = 10

x += 5    # те саме що x = x + 5  →  15
x -= 3    # те саме що x = x - 3  →  12
x *= 2    # те саме що x = x * 2  →  24
x //= 5   # те саме що x = x // 5 →  4
x **= 3   # те саме що x = x ** 3 →  64
x %= 10   # те саме що x = x % 10 →  4
```

В Python немає `++` і `--` як в C або JavaScript. Щоб збільшити на одиницю — `x += 1`. Крапка.

## Пріоритет операцій

Python дотримується математичного пріоритету: `**` виконується першим, потім `*`, `/`, `//`, `%`, потім `+` і `-`. Дужки перекривають пріоритет — так само як у математиці:

```python
2 + 3 * 4      # 14 — спочатку множення, потім додавання
(2 + 3) * 4    # 20 — дужки спочатку
```

Більшість цього — очевидно зі школи. Є одна неочевидна деталь: `**` є **правоасоціативним**. Це означає що `2 ** 3 ** 2` обчислюється не зліва направо, а справа наліво:

```python
2 ** 3 ** 2    # 512 — це 2 ** (3**2) = 2 ** 9 = 512
(2 ** 3) ** 2  # 64  — це (2**3) ** 2 = 8 ** 2 = 64
```

Це правильна математична конвенція: `a^b^c` традиційно означає `a^(b^c)`. Але якщо не знаєш про це — результат `512` де очікував `64` буде дуже дивним.

Загальна порада: коли є сумнів у пріоритеті — постав дужки. Код стане зрозумілішим, і не буде ситуацій де ти сам через місяць не розумієш що мав на увазі.

## Вбудовані числові функції

Python має кілька функцій для роботи з числами які завжди під рукою — без жодного `import`:

### abs() — абсолютне значення

```python
abs(-42)     # 42
abs(42)      # 42
abs(-3.14)   # 3.14
```

Простий але корисний. Потрібен завжди коли важлива величина без знаку — відстань між точками, різниця між значеннями, відхилення від норми.

### round() — округлення

```python
round(3.14159)       # 3    — до цілого
round(3.14159, 2)    # 3.14 — до 2 знаків після крапки
round(3.14159, 4)    # 3.1416
round(2.5)           # 2    — ось тут несподіванка
round(3.5)           # 4
```

Зверни увагу на `round(2.5)` — результат `2`, не `3`. Python використовує **банківське округлення** (round half to even): при рівно `.5` округлює до найближчого **парного** числа. `2.5` → `2` (парне), `3.5` → `4` (парне). Це стандарт IEEE 754 і він зменшує систематичну похибку при великій кількості округлень. Але якщо очікуєш звичне шкільне «.5 завжди вгору» — це здивує.

Якщо потрібне традиційне округлення — є спосіб через `decimal`:

```python
from decimal import Decimal, ROUND_HALF_UP

Decimal("2.5").quantize(Decimal("1"), rounding=ROUND_HALF_UP)   # 3
Decimal("3.5").quantize(Decimal("1"), rounding=ROUND_HALF_UP)   # 4
```

### max(), min(), sum()

```python
max(3, 1, 4, 1, 5, 9)        # 9
min(3, 1, 4, 1, 5, 9)        # 1
sum([3, 1, 4, 1, 5, 9])      # 23

numbers = [3, 1, 4, 1, 5, 9]
max(numbers)    # 9 — теж приймає список
min(numbers)    # 1
```

`sum()` приймає тільки ітерабельний об'єкт (список, кортеж), а `max()` і `min()` — і список, і кілька аргументів через кому.

### pow() — степінь з модулем

```python
pow(2, 10)         # 1024 — те саме що 2 ** 10
pow(2, 10, 100)    # 24   — (2 ** 10) % 100 — ефективніше ніж окремо
```

Третій аргумент — це операція взяття залишку після піднесення до степеня. Python обчислює її ефективно через модульне піднесення — не рахує весь степінь і потім бере залишок, а комбінує кроки. Корисно в криптографії де числа бувають дуже великими.

## Модуль math

Для ширшого набору математичних функцій є стандартний модуль `math`. Щоб використати — треба спочатку імпортувати:

```python
import math
```

Чому `import`? Тому що Python не завантажує всі стандартні модулі одразу — тільки ті що потрібні. Це економить пам'ять і час запуску. `math` завжди є, але треба попросити.

### sqrt і pow

```python
math.sqrt(16)     # 4.0  — квадратний корінь (завжди float)
math.sqrt(2)      # 1.4142135623730951
math.sqrt(0)      # 0.0

math.pow(2, 10)   # 1024.0 — степінь (math.pow завжди float, ** може int)
```

`math.sqrt()` завжди повертає `float`. `4 ** 0.5` теж дає квадратний корінь і теж float. Різниця — `math.sqrt()` явніший за намір і трохи швидший.

### ceil і floor — округлення вгору і вниз

```python
math.ceil(3.2)    # 4  — вгору до найближчого цілого
math.ceil(3.8)    # 4  — теж вгору
math.ceil(-3.2)   # -3 — вгору означає до нуля для від'ємних

math.floor(3.8)   # 3  — вниз до найближчого цілого
math.floor(3.2)   # 3  — теж вниз
math.floor(-3.2)  # -4 — вниз означає від нуля для від'ємних
```

`ceil` — «стеля» (ceiling), `floor` — «підлога». Запам'ятовується буквально: стеля завжди вища, підлога завжди нижча.

Зверни увагу як вони поводяться з від'ємними числами. `math.ceil(-3.2)` дає `-3` — бо `-3` більше ніж `-3.2` і є найближчим цілим угору. `math.floor(-3.2)` дає `-4` — бо `-4` менше ніж `-3.2` і є найближчим цілим вниз.

Порівняй три різних функції для одного і того самого від'ємного числа:

```python
x = -3.7

int(x)         # -3 — відкидає дробову (до нуля)
math.ceil(x)   # -3 — вгору (теж до нуля, бо від'ємне)
math.floor(x)  # -4 — вниз (від нуля)
round(x)       # -4 — найближче ціле (банківське)
```

Чотири функції — чотири різних результати. Вибір між ними залежить від того що саме потрібно в конкретній задачі.

### log — логарифм

```python
math.log(100, 10)    # 2.0  — log₁₀(100): 10² = 100
math.log(8, 2)       # 3.0  — log₂(8): 2³ = 8
math.log(math.e)     # 1.0  — натуральний логарифм ln(e)
math.log10(1000)     # 3.0  — зручніший варіант для log₁₀
math.log2(1024)      # 10.0 — зручніший варіант для log₂
```

### Константи і нескінченність

```python
math.pi     # 3.141592653589793
math.e      # 2.718281828459045
math.tau    # 6.283185307179586 — 2π
math.inf    # inf  — нескінченність
-math.inf   # -inf
math.nan    # nan  — «не число» (Not a Number)
```

`math.inf` — нескінченність. Корисна як початкове значення для пошуку мінімуму: починаєш з нескінченності і оновлюєш коли знаходиш щось менше. `math.nan` — результат невизначених операцій на кшталт `0/0` або `math.inf - math.inf`. Особливість `nan`: він не рівний навіть самому собі:

```python
>>> math.nan == math.nan
False    # nan не рівний нічому, навіть собі
>>> math.isnan(math.nan)
True     # для перевірки на nan — окрема функція
```

## Порівняння float: math.isclose()

З § 1.1.1 ти вже знаєш що `0.1 + 0.2 == 0.3` дає `False` через особливості двійкової арифметики. Тут — як з цим правильно жити.

Є два підходи. Перший — вручну перевірити що різниця між числами менша за допустиму похибку:

```python
abs((0.1 + 0.2) - 0.3) < 1e-9    # True
```

Але тут потрібно самому вирішувати яка похибка «допустима». Для маленьких чисел `1e-9` може бути занадто строго, для великих — замало.

Другий підхід — `math.isclose()`, який робить це правильно:

```python
import math

math.isclose(0.1 + 0.2, 0.3)    # True
math.isclose(1.0, 1.0000001)    # True — в межах допустимого
math.isclose(1.0, 1.01)         # False — надто далеко
```

`math.isclose()` за замовчуванням використовує **відносну** похибку `1e-9` — числа вважаються близькими якщо відрізняються менш ніж на одну мільярдну від більшого з них. Добре працює для більшості практичних задач.

Якщо потрібен тонший контроль — є параметри:

```python
math.isclose(a, b, rel_tol=1e-5)   # відносна похибка 0.001%
math.isclose(a, b, abs_tol=1e-9)   # абсолютна похибка
```

Для порівняння з нулем `rel_tol` не підходить — відносна похибка від нуля завжди нескінченна. Тут потрібен `abs_tol`:

```python
math.isclose(0.0, 1e-15, abs_tol=1e-9)    # True — практично нуль
math.isclose(0.0, 0.01, abs_tol=1e-9)     # False — надто далеко від нуля
```

Загальне правило: якщо порівнюєш два `float` — `math.isclose()`. Якщо порівнюєш з нулем — `math.isclose(x, 0, abs_tol=...)` з явним допуском.

## Числа як умови

Числа підпорядковуються тим самим правилам truthy/falsy що й інші типи. Нуль — `False`, будь-яке ненульове число — `True`:

```python
bool(0)      # False
bool(0.0)    # False
bool(1)      # True
bool(-5)     # True — від'ємне теж ненульове
bool(0.001)  # True — мікроскопічне але ненульове
```

На практиці:

```python
count = get_count()

if count:           # те саме що if count != 0
    process(count)
```

Але будь обережним з `float`: `bool(0.0)` — `False`, але `bool(1e-300)` — `True`, бо технічно ненульове. Якщо логіка стосується «чи достатньо велике це число» — краще явна перевірка.

І типова ідіома з `%`:

```python
if number % 2 == 0:
    print("парне")

if i % 5 == 0:
    print("кожен п'ятий")
```

## Підсумок

Числа в Python — прості, але з характером.

`/` завжди `float`. `//` ціле ділення — і округлює вниз, не до нуля, що важливо для від'ємних. `%` залишок — зустрічається частіше ніж здається, особливо для перевірок на кратність і циклічних задач.

`round()` використовує банківське округлення: `round(2.5)` дає `2`. Якщо потрібне «.5 завжди вгору» — треба `Decimal` з явним режимом.

`math` — стандартний модуль для серйозної математики. `sqrt`, `ceil`, `floor`, `log`, константи `pi` і `e`. Потребує `import math`.

`math.isclose()` — для порівняння `float`. Ніколи `==`.
