# § 1.1.2 Рядки та їх методи

---

## Рядок — не просто текст

У попередньому параграфі ми познайомились зі `str` як з одним із п'яти базових типів. Але рядок — надто багата тема для одного вступного абзацу. Python надає рядкам стільки вбудованих можливостей, що майже будь-яку задачу з обробки тексту — пошук, заміна, очищення, розбиття, форматування — можна вирішити кількома рядками без жодних сторонніх бібліотек.

У цьому параграфі ми пройдемось: спочатку незмінність як фундаментальна властивість, потім конкатенація і повторення, потім методи рядків, потім f-рядки і форматування, і нарешті escape-послідовності.

---

## Незмінність: чому це важливо знати одразу

Рядок у Python — **незмінний** об'єкт. Після створення його не можна модифікувати — ні замінити символ, ні вставити щось посередині, ні видалити частину. Можна лише створити новий рядок на основі старого.

Спробуй у REPL:

```python
>>> name = "Оксана"
>>> name[0] = "о"
TypeError: 'str' object does not support item assignment
```

Python буквально кричить: «рядок не підтримує присвоєння елементів». Він незмінний. Якщо потрібно щоб перша літера стала маленькою — треба створити новий рядок:

```python
name = "Оксана"
name = "о" + name[1:]   # "оксана" — взяли все крім першої літери і склеїли з "о"
```

Але найважливіший практичний наслідок незмінності — не це. Він стосується методів рядків.

Кожен метод рядка, який нібито «змінює» рядок — `strip()`, `upper()`, `replace()` та інші — насправді **нічого не змінює**. Він повертає новий рядок з потрібними змінами, а вихідний залишається незайманим:

```python
original = "  Привіт, Світе!  "

# Метод повертає новий рядок
stripped = original.strip()

print(original)   #   Привіт, Світе!   — незмінний!
print(stripped)   # Привіт, Світе!     — новий рядок
```

Звідси — класична помилка початківців:

```python
text = "  зайві пробіли  "
text.strip()   # нічого не сталося! результат повернуто і одразу викинуто

print(text)    # "  зайві пробіли  " — як було, так і є
```

Якщо хочеш щоб зміна «зафіксувалась» — завжди присвоюй результат назад у змінну:

```python
text = "  зайві пробіли  "
text = text.strip()   # тепер text = "зайві пробіли"

print(text)    # зайві пробіли
```

Або зберігай у нову змінну, якщо потрібно тримати обидва варіанти:

```python
raw     = "  введені дані  "
cleaned = raw.strip()

print(raw)      #   введені дані   — оригінал
print(cleaned)  # введені дані     — оброблений
```

Запам'ятай це правило і тримай його в голові коли будеш працювати з методами нижче.

## Конкатенація і повторення

Два найпростіших оператори для рядків — `+` і `*`.

Оператор `+` між рядками — це конкатенація: склеювання двох рядків в один:

```python
first    = "Добрий"
second   = " ранок"
greeting = first + second    # "Добрий ранок"
```

Простий і зрозумілий. Але є одна пастка яка з'являється майже у кожного початківця: спробувати скласти рядок з числом:

```python
age = 25
# Так не можна:
message = "Мені " + age + " років"   # TypeError!

# Потрібне явне перетворення:
message = "Мені " + str(age) + " років"   # "Мені 25 років"
```

Саме тому f-рядки зручніші за конкатенацію — вони не вимагають ручного `str()`. Але про це детально трохи нижче.

Оператор `*` між рядком і числом — повторення рядка задану кількість разів:

```python
line      = "-" * 40     # "----------------------------------------"
border    = "=" * 30     # "=============================="
indent    = "    " * 2   # "        " — 8 пробілів (2 рівні відступу)
dots      = ". " * 10    # ". . . . . . . . . . "
```

Повторення здається дрібницею, але в консольних програмах і при генерації тексту — дуже зручна операція. Одна деталь: число завжди праворуч або ліворуч від рядка, але не рядок між двома рядками:

```python
3 * "ha"    # "hahaha" — теж працює, число зліва
"ha" * 3    # "hahaha" — те саме
"ha" * "3"  # TypeError — не можна множити рядок на рядок
```

## Методи рядків

Методи — це функції, вбудовані в рядок, які викликаються через крапку після імені змінної: `рядок.метод()`. Кожен метод щось робить з вмістом рядка і повертає результат. Пам'ятай: повертає новий рядок, вихідний не змінює.

### strip(), lstrip(), rstrip() — прибрати зайві символи

`strip()` — прибирає пробіли, табуляцію і переноси рядків з обох кінців. Незамінний коли обробляєш введення користувача або читаєш дані з файлів — там завжди є незаплановані пробіли:

```python
text = "  Привіт, Світе!  "

text.strip()     # "Привіт, Світе!"
text.lstrip()    # "Привіт, Світе!  " — тільки зліва
text.rstrip()    # "  Привіт, Світе!" — тільки справа
```

`strip()` може приймати аргумент — рядок символів, які треба обрізати:

```python
"###заголовок###".strip("#")    # "заголовок"
"...крапки...".strip(".")       # "крапки"
"/url/path/".strip("/")         # "url/path"
```

Важливий нюанс: аргумент `strip()` — це **набір символів**, а не підрядок. `"abcdef".strip("ace")` видалить всі символи `a`, `c`, `e` з обох кінців — не рядок `"ace"` цілком:

```python
>>> "aaabbbccc".strip("ac")
'bbb'    # видалені всі 'a' зліва і всі 'c' справа
```

---

### lower(), upper(), capitalize(), title() — зміна регістру

```python
text = "  Привіт, Світе!  "

text.lower()       # "  привіт, світе!  " — всі маленькі
text.upper()       # "  ПРИВІТ, СВІТЕ!  " — всі великі
```

На перший погляд — косметика. Але `lower()` надзвичайно важливий для порівняння рядків без урахування регістру. Якщо хочеш перевірити чи ввів користувач слово «так» — він міг написати «Так», «ТАК», «так», «тАк»:

```python
answer = input("Ви згодні? ").strip().lower()

if answer == "так":
    print("Чудово!")
elif answer == "ні":
    print("Шкода.")
else:
    print("Незрозуміло.")
```

`.strip().lower()` — типова комбінація при обробці будь-якого введення: спочатку прибрати зайві пробіли, потім привести до нижнього регістру.

Є ще два методи регістру:

```python
"привіт, світе".capitalize()   # "Привіт, світе" — перша літера велика, решта малі
"привіт, світе".title()         # "Привіт, Світе" — перша літера кожного слова велика
```

`title()` корисний для форматування імен і заголовків. Але з ним є нюанс: він вважає «словом» будь-яку послідовність після не-літерного символу, тому `"it's"` перетвориться на `"It'S"` — апостроф є межею слова.

---

### replace() — заміна підрядка

`replace(old, new)` — замінює всі входження підрядка `old` на `new`:

```python
text = "  Привіт, Світе!  "

text.replace("Світе", "Python")    # "  Привіт, Python!  "
text.replace(" ", "")              # "Привіт,Світе!" — видалити всі пробіли
text.replace("  ", " ")            # замінити подвійні пробіли одинарними
```

Третій опціональний аргумент — кількість замін. Якщо не вказати — замінює всі входження:

```python
"аааа".replace("а", "б")           # "бббб" — всі
"аааа".replace("а", "б", 2)        # "ббаа" — тільки перші два
```

`replace()` чутливий до регістру:

```python
"Привіт, привіт".replace("привіт", "!")   # "Привіт, !" — тільки маленький
```

Якщо потрібна заміна без урахування регістру — простого `replace()` не вистачить. Для цього зазвичай використовують регулярні вирази (модуль `re`), але це тема пізніших розділів.

### split() — розбити рядок на частини

`split(sep)` — ділить рядок по роздільнику і повертає список рядків:

```python
text = "  Привіт, Світе!  "

text.split(",")        # ['  Привіт', ' Світе!  '] — розбити по комі
"a:b:c:d".split(":")  # ['a', 'b', 'c', 'd']
"один два три".split() # ['один', 'два', 'три'] — без аргументу: по будь-яких пробілах
```

Виклик `split()` без аргументу — дуже корисний. Він розбиває рядок по будь-яких пробілах (пробіл, табуляція, перенос рядка) і автоматично ігнорує зайві пробіли з країв:

```python
"  один   два     три  ".split()    # ['один', 'два', 'три'] — чисто
"  один   два     три  ".split(" ") # ['', '', 'один', '', '', 'два', '', '', '', 'три', '', '']
```

Різниця між ними суттєва — `split()` без аргументу набагато зручніший для обробки введення.

Опціональний другий аргумент `maxsplit` — скільки максимум розбиттів зробити:

```python
"один:два:три:чотири".split(":", 2)   # ['один', 'два', 'три:чотири']
```

Перші два розбиття зроблені, решта — залишилась нерозбитою. Корисно коли потрібно відокремити «заголовок» від «тіла», а тіло саме може містити роздільник.

Зворотня операція — `join()`. Він об'єднує список рядків у один, вставляючи роздільник між елементами:

```python
words = ['один', 'два', 'три']

", ".join(words)    # "один, два, три"
" | ".join(words)   # "один | два | три"
"".join(words)      # "одиндватри" — без роздільника
"-".join(words)     # "один-два-три"
```

`join()` викликається на роздільнику і приймає список — не навпаки. Це трохи незвично на перший погляд, але логічно: роздільник — це рядок, і він є «господарем» операції.

### startswith(), endswith() — перевірка початку і кінця

```python
text = "  Привіт, Світе!  "

text.startswith("  П")   # True — рядок починається з "  П"
text.startswith("Привіт") # False — бо на початку є пробіли

filename = "звіт_2024.pdf"
filename.endswith(".pdf")   # True
filename.endswith(".docx")  # False
```

Обидва методи можуть приймати кортеж рядків — перевіряють на будь-який зі варіантів:

```python
filename = "photo.jpg"

filename.endswith((".jpg", ".jpeg", ".png", ".gif"))   # True — це зображення
filename.startswith(("http://", "https://"))           # False — це не URL
```

Це набагато зручніше ніж кілька `or`:

```python
# Довго і некрасиво
if filename.endswith(".jpg") or filename.endswith(".jpeg") or filename.endswith(".png"):
    ...

# Коротко і ясно
if filename.endswith((".jpg", ".jpeg", ".png")):
    ...
```

### in — пошук підрядка

Оператор `in` перевіряє чи міститься підрядок у рядку. Це не метод — це окремий оператор, але синтаксично виглядає схоже:

```python
text = "  Привіт, Світе!  "

"Python" in text      # False
"Привіт" in text      # True
"привіт" in text      # False — регістр важливий
"Світе" in text       # True

# not in — зворотна перевірка
"Python" not in text  # True
```

`in` для рядків — це перевірка входження підрядка, а не пошук окремого символу. Тобто `"Сві" in text` — теж `True`, навіть якщо це не ціле слово.

Для пошуку позиції підрядка є метод `find()` — він повертає індекс першого входження або `-1` якщо не знайдено:

```python
text = "Привіт, Привіт!"

text.find("Привіт")     # 0 — знайдено на позиції 0
text.find("Привіт", 1)  # 8 — знайти починаючи з позиції 1
text.find("Python")     # -1 — не знайдено
```

Метод `count()` — скільки разів підрядок зустрічається:

```python
"banana".count("a")     # 3
"banana".count("an")    # 2
"hello".count("ll")     # 1
```

### Інші корисні методи

```python
text = "  Привіт, Світе!  "

# Перевірки (повертають bool)
"12345".isdigit()     # True — тільки цифри
"hello".isalpha()     # True — тільки літери
"hello123".isalnum()  # True — літери і цифри
"   ".isspace()       # True — тільки пробільні символи

# Довжина рядка — вбудована функція len(), не метод
len(text)             # 20 — кількість символів включно з пробілами
len("")               # 0
len("Привіт")         # 7 — кирилиця теж рахується посимвольно
```

Зверни увагу: `len()` — це **функція**, а не метод рядка. Тому викликається як `len(text)`, а не `text.len()`. Це один з нюансів Python — `len()` є універсальною функцією яка працює з рядками, списками, словниками та іншими колекціями.

## f-рядки: підставляти значення в текст

F-рядки — стандартний спосіб форматування рядків у Python 3.6+. Буква `f` перед лапками перетворює рядок на шаблон, де `{}` — місця для вставки виразів:

```python
name, age = "Іванко", 20

print(f"Мені {age} років, через рік буде {age + 1}")
# Мені 20 років, через рік буде 21
```

Усередині `{}` може бути будь-який Python-вираз — не лише ім'я змінної:

```python
items = ["яблуко", "груша", "слива"]
price = 49.9876

print(f"Кількість товарів: {len(items)}")
print(f"Перший товар: {items[0].upper()}")
print(f"Сума з ПДВ: {price * 1.2:.2f} грн")
```

### Форматування чисел

Двокрапка всередині `{}` відкриває специфікатор формату — інструкцію як саме відобразити значення:

```python
price = 19.9876

f"Ціна: {price:.2f} грн"        # "Ціна: 19.99 грн"      — 2 знаки після крапки
f"Ціна: {price:.0f} грн"        # "Ціна: 20 грн"         — без дробової частини
f"Ціна: {price:10.2f} грн"      # "Ціна:      19.99 грн" — 10 символів ширини
```

Відсотки і числа з роздільниками:

```python
ratio = 0.8745

f"Конверсія: {ratio:.1%}"          # "Конверсія: 87.5%"
f"Конверсія: {ratio:.2%}"          # "Конверсія: 87.45%"

big_number = 1234567.89
f"Сума: {big_number:,.2f}"         # "Сума: 1,234,567.89"
f"Сума: {big_number:_.2f}"         # "Сума: 1_234_567.89"
```

### Вирівнювання

```python
name  = "Іванко"
score = 95

# < — вирівнювання ліворуч (за замовчуванням для рядків)
# > — вирівнювання праворуч (за замовчуванням для чисел)
# ^ — по центру

f"{'Ім\'я':<15} {'Бал':>5}"   # "Ім'я            Бал"
f"{name:<15} {score:>5}"       # "Іванко          95"

# Заповнення символом
f"{'Заголовок':=^40}"          # "===============Заголовок==============="
f"{'текст':->20}"              # "--------------текст"
f"{'текст':-<20}"              # "текст--------------"
```

Вирівнювання незамінне коли треба вивести таблицю або список у консолі — без нього колонки розповзатимуться.

### repr() через !r

Специфікатор `!r` застосовує до значення функцію `repr()` — технічне відображення, де рядки беруться в лапки, а спеціальні символи відображаються явно:

```python
name = "Іванко"

f"{name}"    # Іванко    — звичайне відображення
f"{name!r}"  # 'Іванко'  — з лапками, технічне

text = "рядок\nз переносом"

f"{text}"    # рядок
             # з переносом   — перенос виконаний
f"{text!r}"  # 'рядок\\nз переносом'  — видно спеціальний символ
```

`!r` корисний для налагодження — коли хочеш точно побачити що у змінній, включно з лапками і спеціальними символами, а не «людське» відображення.

### Старі способи форматування

Якщо колись зустрінеш у чужому коді — знай:

```python
# Старий стиль через %
"Привіт, %s! Тобі %d років." % ("Іванко", 20)

# Стиль через .format()
"Привіт, {}! Тобі {} років.".format("Іванко", 20)
"Привіт, {name}!".format(name="Іванко")
```

Обидва працюють, але в новому коді — тільки f-рядки. Вони читабельніші, швидші і зручніші.

## Escape-послідовності

Деякі символи не можна вставити в рядок напряму — або тому що вони мають особливе значення (лапки), або тому що вони невидимі (перенос рядка). Для них є escape-послідовності — комбінації з зворотним слешем `\`:

```python
print("Рядок перший\nРядок другий\nРядок третій")
# Рядок перший
# Рядок другий
# Рядок третій

print("Колонка 1\tКолонка 2\tКолонка 3")
# Колонка 1   Колонка 2   Колонка 3

print("Він сказав: \"Привіт!\"")
# Він сказав: "Привіт!"

print('It\'s fine')
# It's fine

print("Шлях: C:\\Users\\Іванко\\Documents")
# Шлях: C:\Users\Іванко\Documents
```

Найпоширеніші:

| Послідовність | Значення |
|---|---|
| `\n` | Перенос рядка |
| `\t` | Табуляція |
| `\\` | Зворотний слеш |
| `\"` | Подвійна лапка |
| `\'` | Одинарна лапка |
| `\r` | Повернення каретки (рідко) |
| `\0` | Нульовий символ (рідко) |

### Сирі рядки: r""

Коли зворотні слеші потрібні буквально — наприклад у Windows-шляхах або регулярних виразах — і не хочеш про кожен писати `\\`, використовуй сирий рядок з префіксом `r`:

```python
# Без r — Python інтерпретує \U і \D як unicode escape і падає або дає несподіваний результат
path = "C:\Users\Desktop"       # \U і \D — проблема

# З r — всі зворотні слеші буквальні
path    = r"C:\Users\Desktop"   # C:\Users\Desktop
pattern = r"\d+\.\d+"           # регулярний вираз — \d, \. збережені як є
```

Сирий рядок не може закінчуватись непарним зворотним слешем — `r"path\"` — це синтаксична помилка. Якщо потрібно — додай ще один: `r"path\\"` або просто `"path\\"`.

## Індексація і зрізи

Рядок — це послідовність символів, і до кожного можна звернутись за індексом. Індекси в Python починаються з нуля:

```python
text = "Привіт"
#       0 1 2 3 4 5

text[0]    # "П" — перший символ
text[1]    # "р"
text[5]    # "т" — шостий символ

# Від'ємні індекси — рахують з кінця
text[-1]   # "т" — останній
text[-2]   # "і" — передостанній
text[-6]   # "П" — еквівалентно text[0]
```

Зрізи (slices) дозволяють взяти підрядок:

```python
text = "Привіт"

text[0:3]    # "При" — від 0 до 3 (не включно)
text[2:5]    # "иві"
text[:3]     # "При" — від початку до 3
text[3:]     # "віт" — від 3 до кінця
text[:]      # "Привіт" — копія всього рядка
text[::2]    # "Пии" — кожен другий символ
text[::-1]   # "тівирП" — реверс рядка
```

Синтаксис зрізу: `[start:stop:step]`. Де `start` — включно, `stop` — не включно, `step` — крок. Всі три параметри опціональні.

Реверс через `[::-1]` — популярний трюк: крок `-1` означає «іти у зворотному напрямку».

## Підсумок

Рядок у Python — один з найбагатших типів за кількістю вбудованих можливостей. Ключові речі з цього параграфа:

Незмінність — методи повертають нові рядки, не змінюють вихідний. Завжди зберігай результат у змінну.

Методи за планом і що вони роблять: `strip()` — прибирає пробіли з країв; `lower()` / `upper()` — регістр; `replace(old, new)` — замінює підрядки; `split(sep)` — ділить на список; `startswith()` / `endswith()` — перевіряє початок і кінець; `in` — є підрядок чи ні.

f-рядки — стандарт форматування. `:.2f` — два знаки після крапки. `:>20` — вирівнювання по ширині. `!r` — repr-відображення для налагодження.

Escape-послідовності — `\n` (перенос), `\t` (табуляція), `\\` (слеш). Для сирих шляхів і regex — префікс `r`.
