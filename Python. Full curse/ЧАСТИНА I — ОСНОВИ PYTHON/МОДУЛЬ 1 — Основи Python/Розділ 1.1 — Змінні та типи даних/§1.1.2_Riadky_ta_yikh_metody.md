# § 1.1.2 Рядки та їх методи

---

## Рядок — найбільш людяний тип

З усіх базових типів рядок найближче до того як думає людина. Числа — це числа, `True`/`False` — це логіка, `None` — відсутність. Але рядки — це текст. Імена, повідомлення, адреси, команди, відповіді від сервера, вміст файлів. Майже будь-які реальні дані які приходять у програму ззовні — спочатку рядки. Від користувача через клавіатуру — рядок. З файлу — рядок. З мережі — рядок. Тому вміти працювати з рядками — не опція.

Python надає рядкам величезну кількість вбудованих інструментів. Пошук, заміна, очищення, розбиття, форматування, перевірки — все це є без жодних сторонніх бібліотек. Цей параграф — про те як цими інструментами користуватись. І ми почнемо не з методів, а з одного фундаментального факту, який треба зрозуміти перш ніж торкатись будь-якого методу.

---

## Незмінність: найважливіший факт про рядки

Рядок у Python неможливо змінити. Зовсім. Після того як він створений — його не можна модифікувати: ні поміняти символ, ні вставити щось посередині, ні видалити частину. Спробуй у REPL:

```python
>>> name = "Оксана"
>>> name[0] = "о"
TypeError: 'str' object does not support item assignment
```

Python буквально каже: «рядок не підтримує присвоєння елементів». Він незмінний. Крапка.

Якщо потрібно щоб перша літера стала маленькою — не можна «поміняти» символ на місці. Можна лише створити **новий рядок** на основі старого:

```python
name = "Оксана"
name = "о" + name[1:]    # "оксана"
```

Але це — відносно рідкісна ситуація. Набагато частіше незмінність рядків проявляється інакше, і саме тут вона стає джерелом плутанини у більшості початківців.

### Методи повертають нові рядки — і нічого більше

Усі методи рядків — `strip()`, `lower()`, `upper()`, `replace()` та інші — не змінюють рядок. Вони **повертають новий рядок** із потрібними змінами, а вихідний залишається незайманим. Завжди.

Ось де це стає проблемою:

```python
text = "  зайві пробіли  "
text.strip()              # ← результат повернуто і одразу викинуто
print(text)               # "  зайві пробіли  " — нічого не змінилось!
```

Ти викликав `strip()`, метод спрацював, створив новий рядок без пробілів — але ти нікуди його не зберіг. Він просто зник. `text` залишився незмінним бо рядки незмінні.

Виправлення просте: завжди зберігай результат:

```python
text = "  зайві пробіли  "
text = text.strip()       # присвоїти результат назад в ту саму змінну
print(text)               # "зайві пробіли"
```

Або в нову, якщо потрібно тримати обидва варіанти:

```python
raw     = "  введені дані  "
cleaned = raw.strip()

print(raw)      # "  введені дані  " — оригінал збережений
print(cleaned)  # "введені дані"    — оброблений варіант
```

Це правило стосується **кожного** методу рядків. Буде хибно написати `name.upper()` і очікувати що `name` став великими літерами. Не став. Треба `name = name.upper()`.

Запам'ятай це зараз — і коли з'явиться перша дивна помилка «чому метод не спрацював», ти одразу знатимеш де шукати.

## Конкатенація і повторення

Найпростіші операції з рядками — `+` і `*`.

Плюс між рядками — конкатенація: склеювання двох рядків в один. Ніякої магії:

```python
first    = "Добрий"
second   = " ранок"
greeting = first + second    # "Добрий ранок"
```

Але є одна пастка яка чекає майже на кожного. Рядок плюс число — `TypeError`. Python не перетворює число в рядок автоматично:

```python
age = 25
message = "Мені " + age + " років"    # TypeError!
```

Щоб це заробило через конкатенацію — потрібне явне `str()`:

```python
message = "Мені " + str(age) + " років"    # "Мені 25 років"
```

Саме тому конкатенація — не найкращий спосіб збирати рядки зі змінних. F-рядки, про які мова піде далі, вирішують цю проблему елегантніше.

Зірочка між рядком і числом — повторення:

```python
line   = "-" * 40     # "----------------------------------------"
border = "=" * 30     # "=============================="
indent = "  " * 3     # "      " — 6 пробілів
```

Здається дрібницею, поки не починаєш виводити таблиці в консолі або генерувати шаблонний текст — тоді ця операція стає дуже зручною. І ще: `"ha" * 3` і `3 * "ha"` — однаково `"hahaha"`. А `"ha" * "3"` — `TypeError`, рядок на рядок не множиться.

## Методи рядків

Метод — це функція яка «живе» всередині об'єкта і викликається через крапку: `рядок.метод()`. Методи рядків завжди повертають результат і ніколи не змінюють сам рядок — ти вже знаєш чому.

### strip() — прибрати те що навколо

`strip()` прибирає пробіли, табуляцію і переноси рядків з обох кінців рядка.

Здається незначним. Але подумай скільки разів дані приходять з «брудними» краями: користувач набрав щось і випадково натиснув пробіл в кінці, CSV-файл має зайві пробіли навколо значень, API повернув рядок з `\n` в кінці. `strip()` — перша лінія захисту:

```python
text = "  Привіт, Світе!  "

text.strip()     # "Привіт, Світе!"    — обидва краї
text.lstrip()    # "Привіт, Світе!  "  — тільки лівий
text.rstrip()    # "  Привіт, Світе!"  — тільки правий
```

`strip()` можна передати аргумент — **набір** символів для видалення:

```python
"###заголовок###".strip("#")   # "заголовок"
"/api/users/".strip("/")       # "api/users"
"...текст...".strip(".")       # "текст"
```

Але є нюанс. Аргумент — це набір символів, не підрядок. `"aabbacc".strip("ac")` видалить всі `a` і `c` з обох країв, а не рядок `"ac"` як ціле:

```python
>>> "aaabbbccc".strip("ac")
'bbb'
```

Якщо це не те що потрібно — є `removeprefix()` і `removesuffix()` (Python 3.9+), які видаляють точний підрядок:

```python
"https://example.com".removeprefix("https://")   # "example.com"
"report_2024.pdf".removesuffix(".pdf")            # "report_2024"
```

### lower() і upper() — регістр

```python
text = "  Привіт, Світе!  "

text.lower()    # "  привіт, світе!  "
text.upper()    # "  ПРИВІТ, СВІТЕ!  "
```

На перший погляд — косметика. Але `lower()` є однією з найважливіших функцій при обробці будь-яких даних від користувача.

Уяви: програма питає «Ви згодні?» і чекає «так» або «ні». Користувач може написати «Так», «ТАК», «так», «тАк», «ТАК!» — і технічно кожен варіант відрізняється від строкового `"так"`. Якщо не привести до одного регістру — програма не розпізнає більшість відповідей.

```python
answer = input("Ви згодні? ").strip().lower()

if answer == "так":
    print("Чудово!")
elif answer == "ні":
    print("Шкода.")
else:
    print(f"Не розумію: {answer!r}")
```

`.strip().lower()` — це стандартна пара. Майже будь-яке введення від користувача варто через неї пропускати: спочатку прибрати зайві пробіли, потім привести до нижнього регістру. Ця звичка рятує від величезного класу дрібних але болісних помилок.

Є ще `capitalize()` — перша літера велика, решта маленькі — і `title()` — перша літера кожного слова велика:

```python
"привіт, світе".capitalize()   # "Привіт, світе"
"привіт, світе".title()        # "Привіт, Світе"
```

`title()` корисний для форматування імен. Але з ним є дивна поведінка: він вважає «початком слова» будь-яку позицію після не-літерного символу. Тому `"it's"` перетвориться на `"It'S"` — апостроф є межею, і `s` після нього теж стане великим. Для імен людей це рідко проблема, але варто знати.

### replace() — замінити підрядок

`replace(old, new)` замінює всі входження `old` на `new`:

```python
text = "  Привіт, Світе!  "

text.replace("Світе", "Python")    # "  Привіт, Python!  "
text.replace(" ", "")              # "Привіт,Світе!" — видалити всі пробіли
text.replace("  ", " ")            # прибрати подвійні пробіли
```

Третій аргумент — скільки максимум замін зробити:

```python
"аааа".replace("а", "б")       # "бббб" — всі
"аааа".replace("а", "б", 2)    # "ббаа" — тільки перші два
```

Два важливих моменти. Перший: `replace()` чутливий до регістру. `"Привіт, привіт".replace("привіт", "!")` дасть `"Привіт, !"` — замінить тільки маленьке «привіт», велике залишить. Другий: `replace()` не підтримує шаблони чи регулярні вирази — тільки буквальні рядки. Для складніших замін знадобиться модуль `re`, але це пізніше.

### split() і join() — розбити і зібрати

`split(sep)` ділить рядок по роздільнику і повертає список рядків.

```python
"один,два,три".split(",")     # ['один', 'два', 'три']
"a:b:c:d".split(":")          # ['a', 'b', 'c', 'd']
"  Привіт, Світе!  ".split(",")  # ['  Привіт', ' Світе!  ']
```

Але найцінніший варіант — `split()` без аргументу. Він розбиває по будь-яких пробільних символах (пробіл, табуляція, перенос рядка) і автоматично ігнорує зайві пробіли з країв і посередині:

```python
"  один   два     три  ".split()    # ['один', 'два', 'три'] — чисто
"  один   два     три  ".split(" ") # ['', '', 'один', '', '', 'два', ...]  — брудно
```

Різниця разюча. `split()` без аргументу — для обробки реального введення. `split(" ")` — коли роздільник буквально один пробіл і ніяких зайвих.

Ще є `maxsplit` — обмежити кількість розбиттів:

```python
"один:два:три:чотири".split(":", 2)   # ['один', 'два', 'три:чотири']
```

Перші два розбиття зроблені, решта залишилась нерозбитою. Корисно коли формат такий: `команда:аргумент:деталі` — і ти хочеш відокремити команду, а деталі залишити цілими.

`join()` — зворотня операція. Об'єднує список рядків в один, вставляючи роздільник між елементами:

```python
words = ["один", "два", "три"]

", ".join(words)     # "один, два, три"
" | ".join(words)    # "один | два | три"
"".join(words)       # "одиндватри"
"\n".join(words)     # "один\nдва\nтри" — кожне слово на новому рядку
```

`join()` викликається на **роздільнику**, а не на списку. `", ".join(words)` — не `words.join(", ")`. Це спочатку виглядає незвично, але логіка така: роздільник — це рядок, і він є «господарем» операції.

### startswith() і endswith() — перевірити початок і кінець

```python
filename = "звіт_2024.pdf"

filename.startswith("звіт")     # True
filename.endswith(".pdf")       # True
filename.endswith(".docx")      # False
```

Обидва методи приймають кортеж — і тоді перевіряють на будь-який зі варіантів. Це набагато зручніше ніж кілька `or`:

```python
# Довго і некрасиво
if filename.endswith(".jpg") or filename.endswith(".jpeg") or filename.endswith(".png"):
    process_image(filename)

# Коротко і ясно
if filename.endswith((".jpg", ".jpeg", ".png", ".webp")):
    process_image(filename)
```

Те саме зі `startswith`:

```python
url = "https://example.com"

if url.startswith(("http://", "https://")):
    fetch(url)
```

### in — є підрядок чи ні

Оператор `in` перевіряє чи міститься підрядок у рядку. Не метод — окремий оператор:

```python
text = "  Привіт, Світе!  "

"Python" in text      # False
"Привіт" in text      # True
"привіт" in text      # False — регістр важливий
"Світе" in text       # True
"Python" not in text  # True
```

Якщо потрібно не просто «є чи немає», а **де** — є `find()`. Він повертає індекс першого входження або `-1` якщо не знайдено:

```python
text = "Привіт, Привіт!"

text.find("Привіт")      # 0 — перше входження на позиції 0
text.find("Привіт", 1)   # 8 — шукати починаючи з позиції 1
text.find("Python")      # -1 — не знайдено
```

І `count()` — скільки разів зустрічається:

```python
"banana".count("a")    # 3
"banana".count("an")   # 2
```

### len() і is-методи

`len()` повертає довжину рядка — кількість символів:

```python
len("Привіт")    # 7
len("")           # 0
len("  ")         # 2 — пробіли теж символи
```

Зверни увагу: `len()` — це **функція**, не метод рядка. Тому `len(text)`, а не `text.len()`. Вона є для всіх колекцій у Python — рядків, списків, словників. Це одне з місць де Python відрізняється від деяких інших мов.

Є ще методи-перевірки, які повертають `bool`:

```python
"12345".isdigit()     # True — тільки цифри
"hello".isalpha()     # True — тільки літери
"hello123".isalnum()  # True — літери і цифри
"   ".isspace()       # True — тільки пробільні символи
"Hello".isupper()     # False — не всі великі
"HELLO".isupper()     # True
```

Корисні для валідації введення: перш ніж робити `int(value)` — можна перевірити `value.isdigit()` щоб уникнути `ValueError`.

## f-рядки: підставляти значення в текст

Одна з найчастіших задач при роботі з рядками — зібрати текст зі змінних. Є три способи і один з них значно кращий за інших.

```python
name, age = "Іванко", 20

# Спосіб 1: конкатенація через + (незручно, легко помилитись)
message = "Привіт, " + name + "! Тобі " + str(age) + " років."

# Спосіб 2: .format() (старий стиль, все ще зустрічається)
message = "Привіт, {}! Тобі {} років.".format(name, age)

# Спосіб 3: f-рядки (сучасний стандарт — використовуй завжди)
message = f"Привіт, {name}! Тобі {age} років."
```

F-рядок — рядок з літерою `f` перед лапками. Усе в фігурних дужках `{}` — Python-вирази, які обчислюються і вставляються в рядок:

```python
name, age = "Іванко", 20

print(f"Мені {age} років, через рік буде {age + 1}")
# Мені 20 років, через рік буде 21
```

Усередині `{}` може бути що завгодно — не тільки ім'я змінної. Арифметика, виклик методу, умовний вираз:

```python
items = ["яблуко", "груша", "слива"]
price = 49.9876

print(f"Товарів: {len(items)}")
print(f"Перший: {items[0].upper()}")
print(f"З ПДВ: {price * 1.2:.2f} грн")
```

### Форматування чисел

Двокрапка всередині `{}` відкриває специфікатор формату — як саме відобразити значення:

```python
price = 19.9876

f"Ціна: {price:.2f} грн"      # "Ціна: 19.99 грн"  — 2 знаки після крапки
f"Ціна: {price:.0f} грн"      # "Ціна: 20 грн"     — без дробової частини
f"Ціна: {price:.4f} грн"      # "Ціна: 19.9876 грн"
```

Відсотки:

```python
ratio = 0.8745

f"{ratio:.1%}"    # "87.5%"
f"{ratio:.2%}"    # "87.45%"
```

Великі числа з роздільниками:

```python
big = 1234567.89

f"{big:,.2f}"     # "1,234,567.89" — кома як роздільник тисяч
f"{big:_.2f}"     # "1_234_567.89" — підкреслення
```

### Вирівнювання

Коли треба вивести таблицю або список у консолі — без вирівнювання колонки розповзаються в різні боки. Специфікатори `<`, `>`, `^` і число фіксують ширину:

```python
name  = "Іванко"
score = 95

f"{name:<15}"     # "Іванко         " — ліворуч, ширина 15
f"{score:>5}"     # "   95"           — праворуч, ширина 5
f"{name:^15}"     # "    Іванко     " — по центру

# Заголовок таблиці
print(f"{'Ім\'я':<15} {'Бал':>5}")
print(f"{name:<15} {score:>5}")
# Ім'я              Бал
# Іванко             95
```

Можна задати символ-заповнювач перед `<`, `>` або `^`:

```python
f"{'текст':->20}"    # "--------------текст"
f"{'текст':-<20}"    # "текст--------------"
f"{'текст':-^20}"    # "-------текст-------"
```

### repr() через !r

Специфікатор `!r` застосовує `repr()` — технічне відображення, де рядки беруться в лапки, а спеціальні символи відображаються буквально:

```python
name = "Іванко"
text = "рядок\nз переносом"

f"{name}"     # Іванко             — звичайне відображення
f"{name!r}"   # 'Іванко'           — з лапками
f"{text}"     # рядок              — перенос виконаний
              # з переносом
f"{text!r}"   # 'рядок\\nз переносом'  — видно \n як є
```

`!r` незамінний для налагодження — коли хочеш точно побачити що у змінній, включно з невидимими символами. Ще в § 1.1.1 в підсумку REPL ми бачили `'Привіт'` в одинарних лапках — це саме repr-відображення. `print(f"{value!r}")` дає те саме.

### Старі стилі — розпізнавати, не писати

У чужому коді зустрінеш два інших способи:

```python
"Привіт, %s! Тобі %d років." % ("Іванко", 20)     # через %
"Привіт, {}! Тобі {} років.".format("Іванко", 20)  # через .format()
```

Обидва працюють. Але в новому коді — тільки f-рядки. Вони читабельніші, швидші і не вимагають запам'ятовувати `%s`, `%d`, `%f`.

## Escape-послідовності

Деякі символи не можна вставити в рядок напряму: перенос рядка, табуляція, лапка того ж типу що обрамляє рядок, сам зворотний слеш. Для них є escape-послідовності — `\` і символ після нього:

```python
print("Рядок перший\nРядок другий")
# Рядок перший
# Рядок другий

print("Ім'я:\tІванко")
# Ім'я:   Іванко

print("Він сказав: \"Привіт!\"")
# Він сказав: "Привіт!"

print("Шлях: C:\\Users\\Іванко")
# Шлях: C:\Users\Іванко
```

Найпоширеніші:

| Послідовність | Значення |
|---|---|
| `\n` | Перенос рядка |
| `\t` | Табуляція |
| `\\` | Зворотний слеш |
| `\"` | Подвійна лапка |
| `\'` | Одинарна лапка |

### Сирий рядок: r""

Є ситуації де зворотні слеші потрібні буквально — і ти не хочеш щоб Python їх інтерпретував. Два головних випадки: Windows-шляхи і регулярні вирази.

Без `r` — Python намагається інтерпретувати `\U`, `\D`, `\n` і інші послідовності, і або падає, або дає несподіваний результат:

```python
path = "C:\Users\Desktop"    # \U — не те що ти думав
```

З `r` — всі зворотні слеші буквальні:

```python
path    = r"C:\Users\Desktop"   # C:\Users\Desktop — як є
pattern = r"\d+\.\d+"           # регулярний вираз — \d і \. збережені
```

Одне обмеження: сирий рядок не може закінчуватись непарним зворотним слешем. `r"path\"` — синтаксична помилка. Якщо потрібно — або `r"path\\"`, або `"path\\"`.

## Індексація і зрізи

Рядок — це послідовність символів, і до кожного можна звернутись за номером позиції. Індекси починаються з нуля:

```python
text = "Привіт"
#       0 1 2 3 4 5

text[0]    # "П" — перший символ (нульовий індекс)
text[1]    # "р"
text[5]    # "т" — шостий символ (п'ятий індекс)
```

Від'ємні індекси рахують з кінця — і це дуже зручно коли потрібен останній символ або кілька останніх:

```python
text[-1]   # "т" — останній
text[-2]   # "і" — передостанній
text[-6]   # "П" — те саме що text[0]
```

Запам'ятати просто: `text[-1]` завжди останній, `text[-2]` — передостанній, і так далі.

Зріз дозволяє взяти підрядок:

```python
text = "Привіт"

text[0:3]   # "При" — від індексу 0 до 3, але 3 — не включно
text[2:5]   # "иві"
text[:3]    # "При" — від початку до 3
text[3:]    # "віт" — від 3 до кінця
text[:]     # "Привіт" — копія всього рядка
text[::2]   # "Пии" — кожен другий символ
text[::-1]  # "тівирП" — реверс
```

Синтаксис зрізу: `[start:stop:step]`. `start` включно, `stop` — ні, `step` — крок. Всі три опціональні: `[:]` бере все, `[::2]` — кожен другий.

Реверс через `[::-1]` — один з популярних Python-трюків. Крок `-1` означає «іти у зворотному напрямку від кінця до початку». Спробуй у REPL: введи будь-яке слово і додай `[::-1]`.

## Підсумок

Рядок — найбагатший з базових типів. І найближчий до людської мови — бо більшість даних що приходять у програму є рядками.

Три речі звідси щоб не забути:

**Методи повертають нові рядки.** `text.strip()` без присвоєння — ніяк не змінить `text`. Завжди: `text = text.strip()`.

**`.strip().lower()` — стандартна пара** для обробки введення користувача. Майже завжди потрібно обидва.

**f-рядки — стандарт.** `:.2f` — два знаки, `:.1%` — відсоток, `:>20` — ширина і вирівнювання, `!r` — для налагодження. Старі `%` і `.format()` — впізнавати, не писати.
