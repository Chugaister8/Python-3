# Передмова

Мова програмування Python має унікальні сильні сторони та особливості, які важко осягнути одразу. Багато програмістів, знайомих з іншими мовами, підходять до Python з обмеженим мисленням, замість того щоб повністю прийняти його виразність. Деякі програмісти заходять надто далеко в протилежному напрямку, зловживаючи можливостями Python, що може спричинити серйозні проблеми надалі.

Ця книга дає уявлення про Pythonic-спосіб написання програм: найкращий спосіб використовувати Python. Вона ґрунтується на базовому розумінні мови, яке, я вважаю, у вас вже є. Програмісти-початківці дізнаються про найкращі практики можливостей Python. Досвідчені програмісти навчаться впевнено освоювати особливості нового інструменту.

Моя мета — підготувати вас до того, щоб здійснити значний вплив за допомогою Python.

## Що охоплює ця книга

Кожен розділ цієї книги містить широкий, але пов'язаний набір порад. Ви вільні переходити між порадами та слідувати своєму інтересу. Кожна порада містить стислі та конкретні настанови, що пояснюють, як писати Python-програми ефективніше. Поради включають рекомендації щодо того, що робити, чого уникати, як знайти правильний баланс і чому це є найкращим вибором. Поради посилаються одна на одну, що полегшує заповнення прогалин під час читання.

Це друге видання книги орієнтоване виключно на Python 3 (см. Порада 1: «Знайте, яку версію Python ви використовуєте»), аж до версії 3.8 включно. Більшість оригінальних порад із першого видання було переглянуто та включено, але багато з них зазнали суттєвих оновлень. Для деяких порад мої рекомендації повністю змінилися між двома виданнями книги через еволюцію найкращих практик у міру дорослішання Python. Якщо ви все ще переважно використовуєте Python 2, незважаючи на завершення його підтримки 1 січня 2020 року, попереднє видання книги може бути кориснішим для вас.

Python дотримується підходу «батарейки в комплекті» щодо своєї стандартної бібліотеки — на відміну від багатьох інших мов, що постачаються з невеликою кількістю поширених пакетів та вимагають шукати важливу функціональність деінде. Багато з цих вбудованих пакетів настільки тісно переплетені з ідіоматичним Python, що цілком могли б бути частиною специфікації мови. Повний набір стандартних модулів є надто великим, щоб охопити його в цій книзі, але я включив ті, які вважаю критично важливими для вивчення та використання.

### Розділ 1. Pythonic-мислення

Спільнота Python почала вживати прикметник Pythonic для опису коду, що відповідає певному стилю. Ідіоми Python складалися роками через досвід роботи з мовою та взаємодію з іншими. Цей розділ охоплює найкращий спосіб виконання найпоширеніших операцій у Python.

### Розділ 2. Списки та словники

У Python найпоширенішим способом організації інформації є послідовність значень, збережених у списку. Природним доповненням до списку є словник, що зберігає ключі для пошуку, зіставлені з відповідними значеннями. Цей розділ охоплює побудову програм із цих універсальних будівельних блоків.

### Розділ 3. Функції

Функції в Python мають різноманітні додаткові можливості, що спрощують життя програміста. Деякі схожі на можливості інших мов програмування, але багато є унікальними для Python. Цей розділ охоплює використання функцій для уточнення намірів, сприяння повторному використанню та зменшення кількості помилок.

### Розділ 4. Спискові вирази та генератори

У Python є спеціальний синтаксис для швидкого ітерування по списках, словниках та множинах для створення похідних структур даних. Мова також дозволяє функції поступово повертати потік ітерованих значень. Цей розділ охоплює, як ці можливості можуть забезпечити кращу продуктивність, зменшене використання пам'яті та підвищену читабельність.

### Розділ 5. Класи та інтерфейси

Python — об'єктно-орієнтована мова. Виконання завдань у Python часто вимагає написання нових класів та визначення їхньої взаємодії через інтерфейси та ієрархії. Цей розділ охоплює використання класів для вираження передбачуваної поведінки за допомогою об'єктів.

### Розділ 6. Метакласи та атрибути

Метакласи та динамічні атрибути — потужні можливості Python. Однак вони також дозволяють реалізовувати надзвичайно дивну та несподівану поведінку. Цей розділ охоплює поширені ідіоми використання цих механізмів для забезпечення дотримання принципу найменшої несподіванки.

### Розділ 7. Паралелізм та конкурентність

Python спрощує написання конкурентних програм, що виконують багато різних речей, здавалося б, одночасно. Python також може використовуватися для паралельної роботи через системні виклики, підпроцеси та розширення C. Цей розділ охоплює найкраще використання Python у цих тонко відмінних ситуаціях.

### Розділ 8. Надійність та продуктивність

Python має вбудовані можливості та модулі, що допомагають зробити ваші програми надійнішими. Python також включає інструменти для досягнення вищої продуктивності з мінімальними зусиллями. Цей розділ охоплює використання Python для оптимізації програм з метою максимізації їхньої надійності та ефективності у продуктивному середовищі.

### Розділ 9. Тестування та налагодження

Слід завжди тестувати свій код, незалежно від мови, якою він написаний. Однак динамічні можливості Python можуть унікальним чином підвищувати ризик помилок під час виконання. На щастя, вони також полегшують написання тестів та діагностику несправних програм. Цей розділ охоплює вбудовані інструменти Python для тестування та налагодження.

### Розділ 10. Співпраця

Співпраця над Python-програмами вимагає свідомого підходу до написання коду. Навіть якщо ви працюєте самостійно, вам захочеться розуміти, як використовувати модулі, написані іншими. Цей розділ охоплює стандартні інструменти та найкращі практики, що дозволяють людям спільно працювати над Python-програмами.

## Угоди, прийняті в цій книзі

Фрагменти Python-коду в цій книзі наведені шрифтом моноширинного накреслення та мають підсвічування синтаксису. Коли рядки є довгими, для позначення переносу використовується символ ¯. Деякі фрагменти скорочені за допомогою трикрапки (`...`), що позначає ділянки з кодом, несуттєвим для розуміння суті. Щоб ці скорочені фрагменти коректно виконувалися на вашому комп'ютері, потрібно завантажити повний приклад коду (місце завантаження вказано нижче).

Я дозволяю собі деяку художню вільність щодо настанови зі стилю Python, щоб приклади коду краще відповідали формату книги або щоб виділити найважливіші частини. Я також пропускав вбудовану документацію, щоб зменшити обсяг прикладів коду. Настійно рекомендую не наслідувати цього у своїх проєктах; натомість слід дотримуватися настанови зі стилю (см. Порада 2: «Дотримуйтеся настанов стилю PEP 8») і писати документацію (см. Порада 84: «Пишіть рядки документації для кожної функції, класу та модуля»).

Більшість фрагментів коду в цій книзі супроводжуються відповідним виведенням після виконання коду. Під «виведенням» мається на увазі виведення у консоль або термінал: те, що ви бачите при запуску Python-програми в інтерактивному інтерпретаторі. Секції виведення наведені шрифтом моноширинного накреслення та передуються рядком `>>>` (запрошення інтерактивного Python). Ідея полягає в тому, що ви могли б набрати фрагменти коду в оболонці Python та відтворити очікуване виведення.

Нарешті, є деякі інші секції шрифтом моноширинного накреслення, яким не передує рядок `>>>`. Вони представляють виведення при запуску програм, відмінних від звичайного інтерпретатора Python. Ці приклади часто починаються з символів `$`, що вказує на запуск програм з оболонки командного рядка на кшталт Bash. Якщо ви виконуєте ці команди у Windows або іншій системі, можливо, доведеться відповідно скоригувати імена програм та аргументи.
