# Розділ 1. Pythonic-мислення

Ідіоми мови програмування визначаються її користувачами. З часом спільнота Python почала вживати прикметник *Pythonic* для опису коду, що відповідає певному стилю. Pythonic-стиль не є регламентованим чи примусовим з боку компілятора. Він складався роками через досвід роботи з мовою та взаємодію з іншими розробниками. Python-програмісти прагнуть до явності, надають перевагу простому над складним і максимізують читабельність. (Наберіть `import this` в інтерпретаторі, щоб прочитати «Дзен Python».)

Програмісти, знайомі з іншими мовами, можуть писати на Python так, ніби це C++, Java або будь-яка інша мова, якою вони найкраще володіють. Початківці можуть ще освоюватися з широким діапазоном концепцій, які можна виразити засобами Python. Важливо знати найкращий — Pythonic — спосіб виконання найпоширеніших операцій у Python. Ці шаблони впливатимуть на кожну написану вами програму.

## Порада 1. Знайте, яку версію Python ви використовуєте

У більшості прикладів коду цієї книги використовується синтаксис Python 3.7 (випущений у червні 2018 року). Книга також містить деякі приклади у синтаксисі Python 3.8 (випущений у жовтні 2019 року), щоб виділити нові можливості, які незабаром стануть ширше доступними. Книга не охоплює Python 2.

Багато комп'ютерів постачаються з кількома заздалегідь встановленими версіями стандартного CPython runtime. Однак за замовчуванням значення `python` у командному рядку може бути неоднозначним. `python` зазвичай є псевдонімом для `python2.7`, але іноді може бути псевдонімом для ще старіших версій, як-от `python2.6` або `python2.5`. Щоб дізнатися точну версію Python, яку ви використовуєте, можна скористатися прапором `--version`:

```
$ python --version

```

Python 3 зазвичай доступний під іменем `python3`:

```
$ python3 --version
Python 3.8.0
```

Можна також визначити версію Python під час виконання, перевіривши значення у вбудованому модулі `sys`:

```python
import sys
print(sys.version_info)
print(sys.version)
>>>
sys.version_info(major=3, minor=8, micro=0,
releaselevel='final', serial=0)
3.8.0 (default, Oct 21 2019, 12:51:32)
[Clang 6.0 (clang-600.0.57)]
```

Python 3 активно підтримується розробниками ядра Python та спільнотою й постійно вдосконалюється. Python 3 містить різноманітні потужні нові можливості, розглянуті в цій книзі. Більшість найпоширеніших бібліотек Python з відкритим кодом сумісні з Python 3 та орієнтовані саме на нього. Я настійно рекомендую використовувати Python 3 для всіх ваших Python-проєктів.

Підтримка Python 2 завершується після 1 січня 2020 року, після чого припиняться будь-які виправлення помилок, патчі безпеки та зворотні портування можливостей. Використання Python 2 після цієї дати є ризикованим, оскільки він більше не підтримуватиметься офіційно. Якщо ви все ще працюєте з кодовою базою на Python 2, варто розглянути використання таких корисних інструментів, як `2to3` (що постачається разом із Python) та `six` (доступний як пакет спільноти; см. Порада 82: «Знайте, де шукати модулі, створені спільнотою»), щоб допомогти вам перейти на Python 3.

**Що слід запам'ятати**

## Порада 2. Дотримуйтеся настанов стилю PEP 8

Python Enhancement Proposal #8, відомий як PEP 8, — це настанова зі стилю форматування Python-коду. Ви можете писати код на Python так, як вам заманеться, якщо він синтаксично коректний. Однак використання послідовного стилю робить ваш код більш доступним і легшим для читання. Дотримання спільного стилю з іншими Python-програмістами широкої спільноти сприяє співпраці в проєктах. Але навіть якщо ви єдиний, хто колись читатиме ваш код, дотримання настанови зі стилю полегшить внесення змін надалі й допоможе уникнути багатьох поширених помилок.

PEP 8 містить велику кількість деталей щодо того, як писати зрозумілий Python-код. Він продовжує оновлюватися в міру розвитку мови. Варто прочитати повний посібник онлайн (https://www.python.org/dev/peps/pep-0008/). Нижче наведено кілька правил, яких слід обов'язково дотримуватися.

### Пробіли

У Python пробіли мають синтаксичне значення. Python-програмісти особливо чутливі до впливу пробілів на читабельність коду. Дотримуйтеся таких рекомендацій щодо пробілів:

- Використовуйте пробіли замість табуляцій для відступів.
- Використовуйте чотири пробіли для кожного рівня синтаксично значущого відступу.
- Довжина рядків не повинна перевищувати 79 символів.
- Продовження довгих виразів на наступних рядках слід відступати на чотири додаткові пробіли від звичайного рівня відступу.
- У файлі функції та класи повинні розділятися двома порожніми рядками.
- У класі методи повинні розділятися одним порожнім рядком.
- У словнику не ставте пробілів між ключем і двокрапкою; якщо відповідне значення розміщується в тому самому рядку, перед ним ставте один пробіл.
- Ставте один і тільки один пробіл до і після оператора `=` у присвоєнні змінної.
- Для анотацій типів не ставте пробілів між іменем змінної та двокрапкою, але ставте пробіл перед інформацією про тип.

### Іменування

PEP 8 пропонує унікальні стилі іменування для різних частин мови. Ці угоди дозволяють легко визначати тип, що відповідає кожному імені, під час читання коду. Дотримуйтеся таких рекомендацій щодо іменування:

- Функції, змінні та атрибути повинні мати формат `lowercase_underscore`.
- Захищені атрибути екземпляра повинні мати формат `_leading_underscore`.
- Приватні атрибути екземпляра повинні мати формат `__double_leading_underscore`.
- Класи (включно з винятками) повинні мати формат `CapitalizedWord`.
- Константи рівня модуля повинні мати формат `ALL_CAPS`.
- Методи екземпляра в класах повинні використовувати `self`, що посилається на об'єкт, як ім'я першого параметра.
- Методи класу повинні використовувати `cls`, що посилається на клас, як ім'я першого параметра.

### Вирази і оператори

«Дзен Python» стверджує: «Має існувати один — і бажано тільки один — очевидний спосіб зробити це». PEP 8 намагається закріпити цей стиль у своїх рекомендаціях щодо виразів і операторів:

- Використовуйте заперечення всередині виразу (`if a is not b`), а не заперечення позитивних виразів (`if not a is b`).
- Не перевіряйте порожні контейнери або послідовності (наприклад, `[]` або `''`) порівнянням довжини з нулем (`if len(somelist) == 0`). Використовуйте `if not somelist` і вважайте, що порожні значення неявно обчислюватимуться як `False`.
- Те саме стосується непорожніх контейнерів або послідовностей (наприклад, `[1]` або `'hi'`). Оператор `if somelist` неявно є `True` для непорожніх значень.
- Уникайте однорядкових операторів `if`, циклів `for` та `while`, а також складених операторів `except`. Розбивайте їх на кілька рядків для читабельності.
- Якщо вираз не вміщується в один рядок, оберніть його в дужки та додайте переноси рядків і відступи для полегшення читання.
- Надавайте перевагу дужкам для багаторядкових виразів над символом продовження рядка `\`.

### Імпорти

PEP 8 пропонує деякі рекомендації щодо імпортування модулів та їх використання у коді:

- Завжди розміщуйте оператори `import` (включно з `from x import y`) на початку файлу.
- Завжди використовуйте абсолютні імена модулів при їх імпорті, а не імена, відносні до шляху поточного модуля. Наприклад, щоб імпортувати модуль `foo` з пакету `bar`, слід використовувати `from bar import foo`, а не просто `import foo`.
- Якщо потрібно використовувати відносні імпорти, застосовуйте явний синтаксис `from . import foo`.
- Імпорти повинні бути розміщені у таких секціях у наступному порядку: модулі стандартної бібліотеки, сторонні модулі, власні модулі. Кожна підсекція повинна містити імпорти в алфавітному порядку.

> **Примітка.** Інструмент Pylint (https://www.pylint.org) є популярним статичним аналізатором Python-коду. Pylint забезпечує автоматичне дотримання настанови зі стилю PEP 8 та виявляє багато інших типових помилок у Python-програмах. Багато IDE та редакторів також містять вбудовані засоби перевірки стилю або підтримують відповідні плаґіни.

**Що слід запам'ятати**

- Завжди дотримуйтеся настанови зі стилю Python Enhancement Proposal #8 (PEP 8) при написанні Python-коду.
- Дотримання спільного стилю з ширшою спільнотою Python сприяє співпраці з іншими.
- Використання послідовного стилю полегшує подальше внесення змін до власного коду.

## Порада 3. Знайте відмінності між bytes і str

У Python існує два типи для представлення послідовностей символьних даних: `bytes` та `str`. Екземпляри `bytes` містять беззнакові 8-бітові значення (які часто відображаються у кодуванні ASCII):

```python
a = b'h\x65llo'
print(list(a))
print(a)
>>>
[104, 101, 108, 108, 111]
b'hello'
```

Екземпляри `str` містять кодові точки Unicode, що представляють текстові символи з людських мов:

```python
a = 'a\u0300 propos'
print(list(a))
print(a)
>>>
['a', '`', ' ', 'p', 'r', 'o', 'p', 'o', 's']
à propos
```

Важливо, що екземпляри `str` не мають пов'язаного бінарного кодування, а екземпляри `bytes` не мають пов'язаного текстового кодування. Щоб перетворити дані Unicode на бінарні, потрібно викликати метод `encode` типу `str`. Щоб перетворити бінарні дані на Unicode, потрібно викликати метод `decode` типу `bytes`. Можна явно вказати потрібне кодування для цих методів або прийняти системне за замовчуванням, яким зазвичай є UTF-8 (але не завжди — детальніше про це нижче).

Під час написання Python-програм важливо виконувати кодування та декодування даних Unicode на найдальшому кордоні ваших інтерфейсів; цей підхід часто називають «Unicode-сендвічем» (Unicode sandwich). Ядро вашої програми повинне використовувати тип `str`, що містить дані Unicode, і не робити жодних припущень щодо кодування символів. Такий підхід дозволяє бути дуже лояльним до альтернативних текстових кодувань (наприклад, Latin-1, Shift JIS та Big5), зберігаючи суворість до вихідного текстового кодування (в ідеалі — UTF-8).

Розподіл між типами символів призводить до двох поширених ситуацій у Python-коді:

- Ви хочете працювати з необробленими 8-бітовими послідовностями, що містять рядки у кодуванні UTF-8 (або іншому кодуванні).
- Ви хочете працювати з рядками Unicode без конкретного кодування.

Часто вам знадобляться дві допоміжні функції для перетворення між цими випадками та для забезпечення відповідності типу вхідних значень очікуванням вашого коду.

Перша функція приймає екземпляр `bytes` або `str` і завжди повертає `str`:

```python
def to_str(bytes_or_str):
    if isinstance(bytes_or_str, bytes):
        value = bytes_or_str.decode('utf-8')
    else:
        value = bytes_or_str
    return value  # Екземпляр str

print(repr(to_str(b'foo')))
print(repr(to_str('bar')))
>>>
'foo'
'bar'
```

Друга функція приймає екземпляр `bytes` або `str` і завжди повертає `bytes`:

```python
def to_bytes(bytes_or_str):
    if isinstance(bytes_or_str, str):
        value = bytes_or_str.encode('utf-8')
    else:
        value = bytes_or_str
    return value  # Екземпляр bytes

print(repr(to_bytes(b'foo')))
print(repr(to_bytes('bar')))
```

Існують дві серйозні пастки при роботі з необробленими 8-бітовими значеннями та рядками Unicode у Python.

Перша проблема полягає в тому, що `bytes` та `str` здаються схожими за поведінкою, але їхні екземпляри несумісні між собою, тому потрібно чітко визначати типи символьних послідовностей, які ви передаєте.

За допомогою оператора `+` можна додавати `bytes` до `bytes` та `str` до `str` відповідно:

```python
print(b'one' + b'two')
print('one' + 'two')
>>>
b'onetwo'
onetwo
```

Але не можна додавати екземпляри `str` до екземплярів `bytes`:

```python
b'one' + 'two'
>>>
Traceback ...
TypeError: can't concat str to bytes
```

І не можна додавати екземпляри `bytes` до екземплярів `str`:

```python
'one' + b'two'
>>>
Traceback ...
TypeError: can only concatenate str (not "bytes") to str
```

За допомогою бінарних операторів можна порівнювати `bytes` з `bytes` та `str` з `str` відповідно:

```python
assert b'red' > b'blue'
assert 'red' > 'blue'
```

Але не можна порівнювати екземпляр `str` з екземпляром `bytes`:

```python
assert 'red' > b'blue'
>>>
Traceback ...
TypeError: '>' not supported between instances of 'str' and 'bytes'
```

І не можна порівнювати екземпляр `bytes` з екземпляром `str`:

```python
assert b'blue' < 'red'
>>>
Traceback ...
TypeError: '<' not supported between instances of 'bytes' and 'str'
```

Порівняння екземплярів `bytes` та `str` на рівність завжди дає `False`, навіть коли вони містять однакові символи (у цьому випадку — «foo» у кодуванні ASCII):

```python
print(b'foo' == 'foo')
>>>
False
```

Оператор `%` працює з форматними рядками для кожного типу відповідно:

```python
print(b'red %s' % b'blue')
print('red %s' % 'blue')
>>>
b'red blue'
red blue
```

Але не можна передавати екземпляр `str` до форматного рядка `bytes`, оскільки Python не знає, яке бінарне текстове кодування використовувати:

```python
print(b'red %s' % 'blue')
>>>
Traceback ...
TypeError: %b requires a bytes-like object, or an object that
implements __bytes__, not 'str'
```

Можна передати екземпляр `bytes` до форматного рядка `str` за допомогою оператора `%`, але результат буде не таким, як очікується:

```python
print('red %s' % b'blue')
>>>
red b'blue'
```

Цей код насправді викликає метод `__repr__` (см. Порада 75: «Використовуйте рядки repr для налагодження») на екземплярі `bytes` і підставляє результат замість `%s`, тому `b'blue'` залишається у виведенні в екранованому вигляді.

Друга проблема полягає в тому, що операції з файловими дескрипторами (що повертаються вбудованою функцією `open`) за замовчуванням вимагають рядків Unicode, а не необроблених байтів. Це може призводити до несподіваних збоїв, особливо для програмістів, знайомих із Python 2. Наприклад, припустимо, що потрібно записати деякі бінарні дані у файл. Цей, здавалося б, простий код завершується помилкою:

```python
with open('data.bin', 'w') as f:
    f.write(b'\xf1\xf2\xf3\xf4\xf5')
>>>
Traceback ...
TypeError: write() argument must be str, not bytes
```

Причина винятку в тому, що файл було відкрито в режимі текстового запису (`'w'`) замість бінарного (`'wb'`). Коли файл перебуває в текстовому режимі, операції запису очікують екземпляри `str`, що містять дані Unicode, а не екземпляри `bytes` із бінарними даними. Тут ми виправляємо це, змінюючи режим відкриття на `'wb'`:

```python
with open('data.bin', 'wb') as f:
    f.write(b'\xf1\xf2\xf3\xf4\xf5')
```

Аналогічна проблема виникає і при читанні даних із файлів. Наприклад, тут спробуємо прочитати бінарний файл, записаний вище:

```python
with open('data.bin', 'r') as f:
    data = f.read()
>>>
Traceback ...
UnicodeDecodeError: 'utf-8' codec can't decode byte 0xf1 in
position 0: invalid continuation byte
```

Помилка виникає тому, що файл відкрито в режимі текстового читання (`'r'`) замість бінарного (`'rb'`). Коли дескриптор перебуває в текстовому режимі, він використовує системне кодування за замовчуванням для інтерпретації бінарних даних за допомогою методів `bytes.encode` (для запису) та `str.decode` (для читання). На більшості систем кодування за замовчуванням — UTF-8, яке не може прийняти бінарні дані `b'\xf1\xf2\xf3\xf4\xf5'`, що призводить до наведеної вище помилки. Тут ми вирішуємо проблему, змінюючи режим відкриття на `'rb'`:

```python
with open('data.bin', 'rb') as f:
    data = f.read()
assert data == b'\xf1\xf2\xf3\xf4\xf5'
```

Як альтернативу, можна явно вказати параметр `encoding` у функції `open`, щоб уникнути будь-якої платформо-залежної поведінки. Наприклад, тут ми припускаємо, що бінарні дані у файлі насправді мали бути рядком у кодуванні `'cp1252'` (застаріле кодування Windows):

```python
with open('data.bin', 'r', encoding='cp1252') as f:
    data = f.read()
assert data == 'ñòóôõ'
```

Виняток зник, а рядкова інтерпретація вмісту файлу суттєво відрізняється від тієї, що була повернута при читанні необроблених байтів. Урок полягає в тому, що слід перевіряти кодування за замовчуванням на вашій системі (за допомогою `python3 -c 'import locale; print(locale.getpreferredencoding())'`), щоб розуміти, як воно відрізняється від ваших очікувань. Якщо є сумніви, слід явно передавати параметр `encoding` у функцію `open`.

**Що слід запам'ятати**

- `bytes` містить послідовності 8-бітових значень, а `str` містить послідовності кодових точок Unicode.
- Використовуйте допоміжні функції, щоб переконатися, що вхідні дані, з якими ви працюєте, мають очікуваний тип символьної послідовності (8-бітові значення, рядки у кодуванні UTF-8, кодові точки Unicode тощо).
- Екземпляри `bytes` та `str` не можна використовувати разом з операторами (як-от `>`, `==`, `+` та `%`).
- Якщо потрібно читати або записувати бінарні дані у файл або з файлу, завжди відкривайте файл у бінарному режимі (наприклад, `'rb'` або `'wb'`).
- Якщо потрібно читати або записувати дані Unicode у файл або з файлу, зважайте на системне текстове кодування за замовчуванням. Явно передавайте параметр `encoding` у функцію `open`, щоб уникнути несподіванок.

## Порада 4. Надавайте перевагу інтерпольованим f-рядкам над форматними рядками у стилі C та str.format

Рядки присутні скрізь у Python-кодових базах. Вони використовуються для відображення повідомлень у призначених для користувача інтерфейсах та утилітах командного рядка. Вони застосовуються для запису даних у файли та сокети. Вони слугують для позначення того, що пішло не так, у деталях винятків (см. Порада 27: «Використовуйте спискові вирази замість map та filter»). Вони використовуються при налагодженні (см. Порада 80: «Розгляньте інтерактивне налагодження за допомогою pdb» та Порада 75: «Використовуйте рядки repr для налагодження»).

Форматування — це процес об'єднання попередньо визначеного тексту зі значеннями даних в єдине читабельне для людини повідомлення, що зберігається як рядок. Python має чотири різні способи форматування рядків, вбудованих у мову та стандартну бібліотеку. Усі, крім одного — який розглядається останнім у цій пораді, — мають серйозні недоліки, які слід розуміти та уникати.

Найпоширенішим способом форматування рядка в Python є використання оператора форматування `%`. Шаблон попередньо визначеного тексту надається ліворуч від оператора у форматному рядку. Значення для підстановки в шаблон надаються як одне значення або кортеж кількох значень праворуч від оператора форматування. Наприклад, тут оператор `%` застосовується для перетворення важкочитабельних двійкових та шістнадцяткових значень на цілочисельні рядки:

```python
a = 0b10111011
b = 0xc5f
print('Binary is %d, hex is %d' % (a, b))
>>>
Binary is 187, hex is 3167
```

Форматний рядок використовує специфікатори формату (наприклад, `%d`) як заповнювачі, які будуть замінені значеннями з правої частини форматуючого виразу. Синтаксис специфікаторів формату походить від функції `printf` мови C, успадкованої Python (а також іншими мовами програмування). Python підтримує всі звичайні параметри, які можна очікувати від `printf`, такі як специфікатори `%s`, `%x` та `%f`, а також управління кількістю десяткових знаків, доповненням, заповненням і вирівнюванням. Багато програмістів, які тільки починають вивчати Python, починають саме з форматних рядків у стилі C, оскільки вони знайомі та прості у використанні.

Існує чотири проблеми з форматними рядками у стилі C в Python.

Перша проблема: якщо змінити тип або порядок значень даних у кортежі праворуч від форматуючого виразу, можна отримати помилки через несумісність перетворення типів. Наприклад, цей простий форматуючий вираз працює коректно:

```python
key = 'my_var'
value = 1.234
formatted = '%-10s = %.2f' % (key, value)
print(formatted)
>>>
my_var     = 1.23
```

Але якщо поміняти місцями `key` та `value`, під час виконання виникне виняток:

```python
reordered_tuple = '%-10s = %.2f' % (value, key)
>>>
Traceback ...
TypeError: must be real number, not str
```

Аналогічно, якщо залишити параметри правої частини у початковому порядку, але змінити форматний рядок, виникне та сама помилка:

```python
reordered_string = '%.2f = %-10s' % (key, value)
>>>
Traceback ...
TypeError: must be real number, not str
```

Щоб уникнути цієї пастки, потрібно постійно перевіряти синхронізованість обох частин оператора `%`; цей процес є схильним до помилок, оскільки його доводиться виконувати вручну при кожній зміні.

Друга проблема з форматуючими виразами у стилі C — вони стають важкочитабельними, коли потрібно вносити невеликі зміни до значень перед форматуванням у рядок, а це надзвичайно поширена потреба. Тут виводимо вміст комори без внесення змін у вбудований вираз:

```python
pantry = [
    ('avocados', 1.25),
    ('bananas', 2.5),
    ('cherries', 15),
]
for i, (item, count) in enumerate(pantry):
    print('#%d: %-10s = %.2f' % (i, item, count))
>>>
#0: avocados   = 1.25
#1: bananas    = 2.50
#2: cherries   = 15.00
```

Тепер вносимо кілька змін до форматованих значень, щоб зробити виведене повідомлення більш корисним. Через це кортеж у форматуючому виразі стає настільки довгим, що його доводиться розбивати на кілька рядків, що погіршує читабельність:

```python
for i, (item, count) in enumerate(pantry):
    print('#%d: %-10s = %d' % (
        i + 1,
        item.title(),
        round(count)))
>>>
#1: Avocados   = 1
#2: Bananas    = 2
#3: Cherries   = 15
```

Третя проблема з форматуючими виразами: якщо потрібно використати одне й те саме значення у форматному рядку кілька разів, доведеться повторювати його в кортежі правої частини:

```python
template = '%s loves food. See %s cook.'
name = 'Max'
formatted = template % (name, name)
print(formatted)
>>>
Max loves food. See Max cook.
```

Це особливо незручно і схильне до помилок, якщо потрібно повторювати невеликі зміни до форматованих значень. Наприклад, тут ми не забули викликати метод `title()` обидва рази, але могли б легко додати виклик до одного посилання на `name` і не додати до іншого, що призвело б до неузгодженого виведення:

```python
name = 'brad'
formatted = template % (name.title(), name.title())
print(formatted)
>>>
Brad loves food. See Brad cook.
```

Щоб вирішити частину цих проблем, оператор `%` у Python дозволяє також виконувати форматування зі словником замість кортежу. Ключі зі словника зіставляються зі специфікаторами формату з відповідними іменами, наприклад `%(key)s`. Тут ця функціональність використовується для зміни порядку значень у правій частині форматуючого виразу без впливу на виведення, що вирішує проблему №1 вище:

```python
key = 'my_var'
value = 1.234
old_way = '%-10s = %.2f' % (key, value)
new_way = '%(key)-10s = %(value).2f' % {
    'key': key, 'value': value}  # Оригінальний порядок
reordered = '%(key)-10s = %(value).2f' % {
    'value': value, 'key': key}  # Переставлений порядок
assert old_way == new_way == reordered
```

Використання словників у форматуючих виразах також вирішує проблему №3 вище, дозволяючи кільком специфікаторам формату посилатися на одне й те саме значення, що усуває необхідність надавати це значення більше одного разу:

```python
name = 'Max'
template = '%s loves food. See %s cook.'
before = template % (name, name)    # Кортеж
template = '%(name)s loves food. See %(name)s cook.'
after = template % {'name': name}   # Словник
assert before == after
```

Однак словникові форматні рядки вводять і посилюють інші проблеми. Щодо проблеми №2 вище, пов'язаної з невеликими змінами значень перед форматуванням, форматуючі вирази стають довшими та візуально гучнішими через наявність ключа словника та оператора-двокрапки у правій частині. Тут відображаємо однаковий рядок із використанням словника та без нього, щоб продемонструвати цю проблему:

```python
for i, (item, count) in enumerate(pantry):
    before = '#%d: %-10s = %d' % (
        i + 1,
        item.title(),
        round(count))
    after = '#%(loop)d: %(item)-10s = %(count)d' % {
        'loop': i + 1,
        'item': item.title(),
        'count': round(count),
    }
    assert before == after
```

Використання словників у форматуючих виразах також збільшує багатослівність, що є проблемою №4 з форматними рядками у стилі C у Python. Кожен ключ потрібно вказувати принаймні двічі — один раз у специфікаторі формату, один раз у словнику як ключ, а потенційно ще раз як ім'я змінної, що містить значення словника:

```python
soup = 'lentil'
formatted = 'Today\'s soup is %(soup)s.' % {'soup': soup}
print(formatted)
>>>
Today's soup is lentil.
```

Окрім дублюючих символів, ця надмірність призводить до того, що форматуючі вирази зі словниками стають довгими. Такі вирази часто потребують кількох рядків, при цьому форматні рядки конкатенуються на кількох рядках, а присвоєння значень словника займають по одному рядку:

```python
menu = {
    'soup': 'lentil',
    'oyster': 'kumamoto',
    'special': 'schnitzel',
}
template = ('Today\'s soup is %(soup)s, '
            'buy one get two %(oyster)s oysters, '
            'and our special entrée is %(special)s.')
formatted = template % menu
print(formatted)
>>>
Today's soup is lentil, buy one get two kumamoto oysters, and
our special entrée is schnitzel.
```

Щоб зрозуміти, що цей форматуючий вираз виведе, очі змушені постійно переміщатися між рядками форматного рядка та рядками словника. Такий розрив ускладнює виявлення помилок, а читабельність стає ще гіршою, якщо потрібно вносити невеликі зміни до будь-яких значень перед форматуванням. Має бути кращий спосіб.

### Вбудована функція format та str.format

Python 3 додав підтримку розширеного форматування рядків, більш виразного, ніж старі форматні рядки у стилі C з оператором `%`. Для окремих значень Python ця нова функціональність доступна через вбудовану функцію `format`. Наприклад, тут деякі нові параметри (`,` для роздільників тисяч та `^` для вирівнювання по центру) застосовуються для форматування значень:

```python
a = 1234.5678
formatted = format(a, ',.2f')
print(formatted)
b = 'my string'
formatted = format(b, '^20s')
print('*', formatted, '*')
>>>
1,234.57
*        my string         *
```

Цю функціональність можна використовувати для спільного форматування кількох значень, викликавши новий метод `format` типу `str`. Замість специфікаторів формату у стилі C, як-от `%d`, можна вказати заповнювачі за допомогою `{}`. За замовчуванням заповнювачі у форматному рядку замінюються відповідними позиційними аргументами, переданими методу `format` у порядку їхньої появи:

```python
key = 'my_var'
value = 1.234
formatted = '{} = {}'.format(key, value)
print(formatted)
>>>
my_var = 1.234
```

У кожному заповнювачі можна опціонально вказати символ двокрапки, за яким слідують специфікатори формату для налаштування перетворення значень на рядки (для повного переліку параметрів дивіться `help('FORMATTING')`):

```python
formatted = '{:<10} = {:.2f}'.format(key, value)
print(formatted)
>>>
my_var     = 1.23
```

Принцип роботи такий: специфікатори формату передаються вбудованій функції `format` разом із значенням (у прикладі вище — `format(value, '.2f')`). Результат виклику цієї функції і замінює заповнювач у підсумковому форматованому рядку. Поведінку форматування можна налаштувати для кожного класу за допомогою спеціального методу `__format__`.

У форматних рядках у стилі C потрібно екранувати символ `%` (подвоюванням), щоб він не був помилково інтерпретований як заповнювач. У методі `str.format` аналогічно потрібно екранувати фігурні дужки:

```python
print('%.2f%%' % 12.5)
print('{} replaces {{}}'.format(1.23))
>>>
12.50%
1.23 replaces {}
```

У фігурних дужках також можна вказати позиційний індекс аргументу, переданого методу `format`, для використання у заповнювачі. Це дозволяє оновлювати форматний рядок для зміни порядку виведення без необхідності змінювати праву частину форматуючого виразу, що вирішує проблему №1 вище:

```python
formatted = '{1} = {0}'.format(key, value)
print(formatted)
>>>
1.234 = my_var
```

На той самий позиційний індекс можна також посилатися кілька разів у форматному рядку без необхідності передавати значення методу `format` більше одного разу, що вирішує проблему №3 вище:

```python
formatted = '{0} loves food. See {0} cook.'.format(name)
print(formatted)
>>>
Max loves food. See Max cook.
```

На жаль, новий метод `format` нічого не вирішує щодо проблеми №2 вище, залишаючи код важкочитабельним, коли потрібно вносити невеликі зміни до значень перед форматуванням. Різниця в читабельності між старим і новим варіантами незначна, обидва виглядають подібно:

```python
for i, (item, count) in enumerate(pantry):
    old_style = '#%d: %-10s = %d' % (
        i + 1,
        item.title(),
        round(count))
    new_style = '#{}: {:<10s} = {}'.format(
        i + 1,
        item.title(),
        round(count))
    assert old_style == new_style
```

Для методу `str.format` існують ще більш розширені параметри специфікаторів, такі як використання комбінацій ключів словника та індексів списку в заповнювачах, а також приведення значень до рядків Unicode та `repr`:

```python
formatted = 'First letter is {menu[oyster][0]!r}'.format(
    menu=menu)
print(formatted)
>>>
First letter is 'k'
```

Але ці можливості не допомагають зменшити надмірність повторюваних ключів із проблеми №4 вище. Наприклад, тут порівнюється багатослівність використання словників у форматуючих виразах у стилі C з новим стилем передачі іменованих аргументів методу `format`:

```python
old_template = (
    'Today\'s soup is %(soup)s, '
    'buy one get two %(oyster)s oysters, '
    'and our special entrée is %(special)s.')
old_formatted = template % {
    'soup': 'lentil',
    'oyster': 'kumamoto',
    'special': 'schnitzel',
}
new_template = (
    'Today\'s soup is {soup}, '
    'buy one get two {oyster} oysters, '
    'and our special entrée is {special}.')
new_formatted = new_template.format(
    soup='lentil',
    oyster='kumamoto',
    special='schnitzel',
)
assert old_formatted == new_formatted
```

Цей стиль дещо менш шумний, оскільки усуває кілька лапок у словнику та деякі символи у специфікаторах формату, але навряд чи є переконливим. Більше того, розширені можливості використання ключів словника та індексів у заповнювачах забезпечують лише незначну підмножину виразових можливостей Python. Така обмеженість виразності настільки значна, що підриває загальну цінність методу `format` типу `str`.

З огляду на ці недоліки та проблеми форматуючих виразів у стилі C, що залишилися (проблеми №2 та №4 вище), рекомендується уникати методу `str.format` у загальному випадку. Важливо знати нову міні-мову, що використовується у специфікаторах формату (все, що після двокрапки), та вміти використовувати вбудовану функцію `format`. Але решта методу `str.format` слід розглядати як історичний артефакт, що допомагає зрозуміти, як працюють нові f-рядки Python і чому вони такі чудові.

### Інтерпольовані форматні рядки

Python 3.6 додав інтерпольовані форматні рядки — скорочено f-рядки — для остаточного вирішення цих проблем. Новий синтаксис мови вимагає передувати форматним рядкам символу `f`, що аналогічно тому, як байтові рядки передуються символом `b`, а «сирі» (без екранування) рядки — символом `r`.

F-рядки доводять виразність форматних рядків до крайності, вирішуючи проблему №4 вище, повністю усуваючи надмірність надання ключів і значень для форматування. Цієї лаконічності вони досягають, дозволяючи посилатися на всі імена в поточній Python-області видимості як частину форматуючого виразу:

```python
key = 'my_var'
value = 1.234
formatted = f'{key} = {value}'
print(formatted)
>>>
my_var = 1.234
```

Усі ті самі параметри нової міні-мови вбудованої функції `format` доступні після двокрапки у заповнювачах всередині f-рядка, а також можливість приведення значень до рядків Unicode та `repr`, аналогічно методу `str.format`:

```python
formatted = f'{key!r:<10} = {value:.2f}'
print(formatted)
>>>
'my_var'   = 1.23
```

Форматування за допомогою f-рядків коротше, ніж використання форматних рядків у стилі C з оператором `%` та методом `str.format`, у всіх випадках. Тут показані всі ці варіанти разом у порядку від найкоротшого до найдовшого, з вирівняною лівою стороною присвоєння для зручності порівняння:

```python
f_string = f'{key:<10} = {value:.2f}'
c_tuple  = '%-10s = %.2f' % (key, value)
str_args = '{:<10} = {:.2f}'.format(key, value)
str_kw   = '{key:<10} = {value:.2f}'.format(key=key,
                                             value=value)
c_dict   = '%(key)-10s = %(value).2f' % {'key': key,
                                          'value': value}
assert c_tuple == c_dict == f_string
assert str_args == str_kw == f_string
```

F-рядки також дозволяють вставляти повноцінні Python-вирази у фігурні дужки заповнювача, вирішуючи проблему №2 вище завдяки можливості вносити невеликі зміни до форматованих значень із лаконічним синтаксисом. Те, що займало кілька рядків у стилі C та з методом `str.format`, тепер легко поміщається в один рядок:

```python
for i, (item, count) in enumerate(pantry):
    old_style = '#%d: %-10s = %d' % (
        i + 1,
        item.title(),
        round(count))
    new_style = '#{}: {:<10s} = {}'.format(
        i + 1,
        item.title(),
        round(count))
    f_string = f'#{i+1}: {item.title():<10s} = {round(count)}'
    assert old_style == new_style == f_string
```

Або, якщо так зрозуміліше, можна розбити f-рядок на кілька рядків, спираючись на конкатенацію суміжних рядків (аналогічно до C). Хоча це довше, ніж однорядкова версія, воно все одно значно зрозуміліше, ніж будь-який з інших багаторядкових підходів:

```python
for i, (item, count) in enumerate(pantry):
    print(f'#{i+1}: '
          f'{item.title():<10s} = '
          f'{round(count)}')
>>>
#1: Avocados   = 1
#2: Bananas    = 2
#3: Cherries   = 15
```

Python-вирази можуть також з'являтися у параметрах специфікаторів формату. Наприклад, тут кількість знаків після коми параметризується через змінну замість того, щоб жорстко задавати її у форматному рядку:

```python
places = 3
number = 1.23456
print(f'My number is {number:.{places}f}')
>>>
My number is 1.235
```

Поєднання виразності, лаконічності та зрозумілості, яке забезпечують f-рядки, робить їх найкращим вбудованим варіантом для Python-програмістів. Щоразу, коли вам потрібно форматувати значення в рядки, обирайте f-рядки замість альтернатив.

**Що слід запам'ятати**

- Форматні рядки у стилі C з оператором `%` мають різноманітні пастки та проблеми з багатослівністю.
- Метод `str.format` вводить деякі корисні концепції у своїй міні-мові специфікаторів формату, але в іншому повторює помилки форматних рядків у стилі C і не рекомендується до використання.
- F-рядки — це новий синтаксис для форматування значень у рядки, що вирішує найбільші проблеми форматних рядків у стилі C.
- F-рядки лаконічні та водночас потужні, оскільки дозволяють безпосередньо вставляти довільні Python-вирази у специфікатори формату.

## Порада 5. Пишіть допоміжні функції замість складних виразів

Лаконічний синтаксис Python полегшує написання однорядкових виразів, що реалізують багато логіки. Наприклад, припустимо, що потрібно декодувати рядок запиту з URL. Тут кожен параметр рядка запиту представляє ціле числове значення:

```python
from urllib.parse import parse_qs
my_values = parse_qs('red=5&blue=0&green=',
                     keep_blank_values=True)
print(repr(my_values))
>>>
{'red': ['5'], 'blue': ['0'], 'green': ['']}
```

Деякі параметри рядка запиту можуть мати кілька значень, деякі — одне значення, деякі можуть бути присутніми, але мати порожні значення, а деякі — відсутніми взагалі. Використання методу `get` для словника результатів повертатиме різні значення в кожному з цих випадків:

```python
print('Red:    ', my_values.get('red'))
print('Green:  ', my_values.get('green'))
print('Opacity:', my_values.get('opacity'))
>>>
Red:     ['5']
Green:   ['']
Opacity: None
```

Було б добре, якби значення за замовчуванням `0` присвоювалося, коли параметр не надано або є порожнім. Можна вдатися до булевих виразів, оскільки здається, що ця логіка не заслуговує цілого оператора `if` або допоміжної функції.

Синтаксис Python робить цей вибір надто простим. Хитрість полягає в тому, що порожній рядок, порожній список та нуль неявно обчислюються як `False`. Таким чином, наведені нижче вирази будуть обчислюватися як підвираз після оператора `or`, коли перший підвираз є `False`:

```python
# Для рядка запиту 'red=5&blue=0&green='
red = my_values.get('red', [''])[0] or 0
green = my_values.get('green', [''])[0] or 0
opacity = my_values.get('opacity', [''])[0] or 0
print(f'Red:    {red!r}')
print(f'Green:  {green!r}')
print(f'Opacity: {opacity!r}')
>>>
Red:     '5'
Green:   0
Opacity: 0
```

Випадок із `red` працює, тому що ключ присутній у словнику `my_values`. Значення — це список з одним елементом: рядком `'5'`. Цей рядок неявно обчислюється як `True`, тому `red` присвоюється перша частина виразу `or`.

Випадок із `green` працює, тому що значення у словнику `my_values` — це список з одним елементом: порожній рядок. Порожній рядок неявно обчислюється як `False`, що змушує вираз `or` повертати `0`.

Випадок із `opacity` працює, тому що значення у словнику `my_values` взагалі відсутнє. Поведінка методу `get` — повертати другий аргумент, якщо ключ не існує у словнику (см. Порада 16: «Надавайте перевагу get перед in та KeyError для обробки відсутніх ключів словника»). Значення за замовчуванням у цьому випадку — список з одним елементом: порожній рядок. Коли `opacity` не знаходиться у словнику, цей код виконує те саме, що і у випадку з `green`.

Однак цей вираз важко читати, і він все ще не робить усього необхідного. Також потрібно забезпечити перетворення всіх значень параметрів на цілі числа, щоб можна було негайно використовувати їх у математичних виразах. Для цього кожен вираз потрібно обгорнути вбудованою функцією `int` для аналізу рядка як цілого числа:

```python
red = int(my_values.get('red', [''])[0] or 0)
```

Тепер це надзвичайно важко читати. Тут забагато візуального шуму. Код не є доступним для сприйняття. Новий читач коду витратив би забагато часу, розбираючи вираз, щоб зрозуміти, що він насправді робить. Хоча стислість коду — це добре, не варто намагатися вмістити все це в один рядок.

У Python є умовний — або тернарний — вираз `if/else` для того, щоб зробити подібні випадки зрозумілішими, зберігаючи короткість коду:

```python
red_str = my_values.get('red', [''])
red = int(red_str[0]) if red_str[0] else 0
```

Це краще. У менш складних ситуаціях умовні вирази `if/else` можуть зробити речі дуже зрозумілими. Але наведений вище приклад все одно не такий зрозумілий, як альтернатива з повноцінним оператором `if/else` на кількох рядках. Коли вся логіка розгорнута таким чином, компактна версія здається ще складнішою:

```python
green_str = my_values.get('green', [''])
if green_str[0]:
    green = int(green_str[0])
else:
    green = 0
```

Якщо потрібно повторно використовувати цю логіку — навіть лише два або три рази, як у цьому прикладі, — написання допоміжної функції є правильним рішенням:

```python
def get_first_int(values, key, default=0):
    found = values.get(key, [''])
    if found[0]:
        return int(found[0])
    return default
```

Код виклику значно зрозуміліший за складний вираз із `or` та двохрядкову версію з виразом `if/else`:

```python
green = get_first_int(my_values, 'green')
```

Як тільки вирази ускладнюються, настає час розглянути їх розбиття на менші частини та перенесення логіки до допоміжних функцій. Виграш у читабельності завжди переважає те, що могла б надати стислість. Уникайте того, щоб лаконічний синтаксис Python для складних виразів не завів вас у подібну плутанину. Дотримуйтеся принципу DRY: Don't Repeat Yourself («Не повторюйтеся»).

**Що слід запам'ятати**

- Синтаксис Python полегшує написання однорядкових виразів, які є надмірно складними і важкими для читання.
- Переносьте складні вирази до допоміжних функцій, особливо якщо одну й ту саму логіку потрібно використовувати кілька разів.
- Вираз `if/else` надає більш читабельну альтернативу використанню булевих операторів `or` та `and` у виразах.

## Порада 6. Надавайте перевагу розпакуванню з множинним присвоєнням над індексуванням

У Python є вбудований тип `tuple`, який можна використовувати для створення незмінних упорядкованих послідовностей значень. У найпростішому випадку кортеж — це пара з двох значень, наприклад ключі та значення зі словника:

```python
snack_calories = {
    'chips': 140,
    'popcorn': 80,
    'nuts': 190,
}
items = tuple(snack_calories.items())
print(items)
>>>
(('chips', 140), ('popcorn', 80), ('nuts', 190))
```

Значення в кортежах можна отримати через числові індекси:

```python
item = ('Peanut butter', 'Jelly')
first = item[0]
second = item[1]
print(first, 'and', second)
>>>
Peanut butter and Jelly
```

Після створення кортежу його не можна змінити, присвоївши нове значення за індексом:

```python
pair = ('Chocolate', 'Peanut butter')
pair[0] = 'Honey'
>>>
Traceback ...
TypeError: 'tuple' object does not support item assignment
```

Python також має синтаксис для розпакування, що дозволяє присвоювати кілька значень в одному операторі. Шаблони, які вказуються у присвоєннях із розпакуванням, дуже схожі на спроби змінювати кортежі — що не дозволяється, — але насправді вони працюють зовсім інакше. Наприклад, якщо відомо, що кортеж є парою, замість використання індексів для доступу до значень можна присвоїти його кортежу з двох імен змінних:

```python
item = ('Peanut butter', 'Jelly')
first, second = item  # Розпакування
print(first, 'and', second)
>>>
Peanut butter and Jelly
```

Розпакування має менше візуального шуму, ніж звернення до індексів кортежу, і часто вимагає менше рядків. Той самий синтаксис шаблонного зіставлення при розпакуванні працює при присвоєнні до списків, послідовностей і кількох рівнів довільних ітерованих об'єктів всередині ітерованих. Не рекомендується виконувати наведене нижче у вашому коді, але важливо знати, що це можливо і як це працює:

```python
favorite_snacks = {
    'salty': ('pretzels', 100),
    'sweet': ('cookies', 180),
    'veggie': ('carrots', 20),
}
((type1, (name1, cals1)),
 (type2, (name2, cals2)),
 (type3, (name3, cals3))) = favorite_snacks.items()
print(f'Favorite {type1} is {name1} with {cals1} calories')
print(f'Favorite {type2} is {name2} with {cals2} calories')
print(f'Favorite {type3} is {name3} with {cals3} calories')
>>>
Favorite salty is pretzels with 100 calories
Favorite sweet is cookies with 180 calories
Favorite veggie is carrots with 20 calories
```

Початківці Python можуть бути здивовані, дізнавшись, що розпакування можна використовувати навіть для обміну значеннями на місці без необхідності створювати тимчасові змінні. Тут використовується типовий синтаксис з індексами для обміну значеннями між двома позиціями у списку як частина алгоритму сортування методом бульбашки:

```python
def bubble_sort(a):
    for _ in range(len(a)):
        for i in range(1, len(a)):
            if a[i] < a[i-1]:
                temp = a[i]
                a[i] = a[i-1]
                a[i-1] = temp

names = ['pretzels', 'carrots', 'arugula', 'bacon']
bubble_sort(names)
print(names)
>>>
['arugula', 'bacon', 'carrots', 'pretzels']
```

Однак із синтаксисом розпакування можна обмінювати значення за індексами в одному рядку:

```python
def bubble_sort(a):
    for _ in range(len(a)):
        for i in range(1, len(a)):
            if a[i] < a[i-1]:
                a[i-1], a[i] = a[i], a[i-1]  # Обмін

names = ['pretzels', 'carrots', 'arugula', 'bacon']
bubble_sort(names)
print(names)
>>>
['arugula', 'bacon', 'carrots', 'pretzels']
```

Принцип роботи цього обміну такий: права частина присвоєння (`a[i], a[i-1]`) обчислюється першою, і її значення поміщаються в новий тимчасовий безіменний кортеж (наприклад, `('carrots', 'pretzels')` при першій ітерації циклів). Потім шаблон розпакування з лівої частини присвоєння (`a[i-1], a[i]`) використовується для отримання значення цього кортежу та присвоєння його іменам змінних `a[i-1]` та `a[i]` відповідно. Це замінює `'pretzels'` на `'carrots'` за індексом 0 і `'carrots'` на `'pretzels'` за індексом 1. Нарешті, тимчасовий безіменний кортеж непомітно зникає.

Ще одним цінним застосуванням розпакування є список цілей у циклах `for` та аналогічних конструкціях, таких як вирази-генератори (см. Порада 27: «Використовуйте спискові вирази замість map та filter»). Для порівняння, тут виконується ітерація по списку закусок без використання розпакування:

```python
snacks = [('bacon', 350), ('donut', 240), ('muffin', 190)]
for i in range(len(snacks)):
    item = snacks[i]
    name = item[0]
    calories = item[1]
    print(f'#{i+1}: {name} has {calories} calories')
>>>
#1: bacon has 350 calories
#2: donut has 240 calories
#3: muffin has 190 calories
```

Це працює, але виглядає шумно. Потрібно багато зайвих символів для індексування по різних рівнях структури `snacks`. Тут досягається той самий результат за допомогою розпакування разом із вбудованою функцією `enumerate` (см. Порада 7: «Надавайте перевагу enumerate над range»):

```python
for rank, (name, calories) in enumerate(snacks, 1):
    print(f'#{rank}: {name} has {calories} calories')
>>>
#1: bacon has 350 calories
#2: donut has 240 calories
#3: muffin has 190 calories
```

Це Pythonic-спосіб написання такого циклу — він короткий і легкий для розуміння. Зазвичай немає необхідності звертатися до чого-небудь через індекси. Python надає додаткову функціональність розпакування для конструювання списків (см. Порада 13: «Надавайте перевагу узагальнюючому розпакуванню над зрізами»), аргументів функцій (см. Порада 22: «Зменшуйте візуальний шум за допомогою змінної кількості позиційних аргументів»), іменованих аргументів (см. Порада 23: «Надавайте необов'язкову поведінку за допомогою іменованих аргументів»), множинних значень, що повертаються (см. Порада 19: «Ніколи не розпаковуйте більше трьох змінних при поверненні кількох значень з функцій»), тощо. Розумне використання розпакування дозволить вам уникати індексування там, де це можливо, що призводить до більш зрозумілого та Pythonic-коду.

**Що слід запам'ятати**

- У Python є спеціальний синтаксис, який називається розпакуванням, для присвоєння кількох значень в одному операторі.
- Розпакування є узагальненим у Python і може застосовуватися до будь-якого ітерованого об'єкта, включаючи багато рівнів ітерованих об'єктів всередині ітерованих.
- Зменшуйте візуальний шум і підвищуйте ясність коду, використовуючи розпакування для уникнення явного індексування послідовностей.

## Порада 7. Надавайте перевагу enumerate над range

Вбудована функція `range` корисна для циклів, що ітерують по набору цілих чисел:

```python
from random import randint
random_bits = 0
for i in range(32):
    if randint(0, 1):
        random_bits |= 1 << i
print(bin(random_bits))
>>>
0b11101000100100000111000010000001
```

Коли є структура даних для ітерації, наприклад список рядків, можна безпосередньо ітерувати по послідовності:

```python
flavor_list = ['vanilla', 'chocolate', 'pecan', 'strawberry']
for flavor in flavor_list:
    print(f'{flavor} is delicious')
>>>
vanilla is delicious
chocolate is delicious
pecan is delicious
strawberry is delicious
```

Часто потрібно ітерувати по списку та знати індекс поточного елемента. Наприклад, потрібно виведення рейтингу улюблених смаків морозива. Один із способів зробити це — використовувати `range`:

```python
for i in range(len(flavor_list)):
    flavor = flavor_list[i]
    print(f'{i + 1}: {flavor}')
>>>
1: vanilla
2: chocolate
3: pecan
4: strawberry
```

Це виглядає незграбно порівняно з іншими прикладами ітерування по `flavor_list` або `range`. Потрібно отримати довжину списку. Потрібно здійснювати індексування в масив. Кілька кроків ускладнюють читання.

Python надає вбудовану функцію `enumerate` для вирішення цієї ситуації. `enumerate` огортає будь-який ітератор «ледачим» генератором (см. Порада 30: «Розгляньте використання генераторів замість повернення списків»). `enumerate` видає пари з індексу циклу та наступного значення з переданого ітератора. Тут вручну просуваємо повернутий ітератор за допомогою вбудованої функції `next`, щоб продемонструвати його роботу:

```python
it = enumerate(flavor_list)
print(next(it))
print(next(it))
>>>
(0, 'vanilla')
(1, 'chocolate')
```

Кожну пару, видану `enumerate`, можна лаконічно розпакувати в операторі `for` (см. Порада 6: «Надавайте перевагу розпакуванню з множинним присвоєнням над індексуванням»). Результуючий код значно зрозуміліший:

```python
for i, flavor in enumerate(flavor_list):
    print(f'{i + 1}: {flavor}')
>>>
1: vanilla
2: chocolate
3: pecan
4: strawberry
```

Можна зробити це ще коротше, вказавши число, з якого `enumerate` повинна починати підрахунок (у цьому випадку — 1), як другий параметр:

```python
for i, flavor in enumerate(flavor_list, 1):
    print(f'{i}: {flavor}')
```

**Що слід запам'ятати**

- `enumerate` надає лаконічний синтаксис для ітерування по ітератору та отримання індексу кожного елемента по ходу.
- Надавайте перевагу `enumerate` над ітеруванням по `range` з індексуванням у послідовність.
- Можна передати другий параметр у `enumerate`, щоб вказати число, з якого починати підрахунок (за замовчуванням — нуль).

## Порада 8. Використовуйте zip для паралельної обробки ітераторів

Часто в Python виникають ситуації зі списками пов'язаних об'єктів. Спискові вирази полегшують отримання похідного списку з вихідного шляхом застосування виразу (см. Порада 27: «Використовуйте спискові вирази замість map та filter»):

```python
names = ['Cecilia', 'Lise', 'Marie']
counts = [len(n) for n in names]
print(counts)
>>>
[7, 4, 5]
```

Елементи похідного списку пов'язані з елементами вихідного списку через їхні індекси. Щоб ітерувати по обох списках паралельно, можна ітерувати по довжині вихідного списку `names`:

```python
longest_name = None
max_count = 0
for i in range(len(names)):
    count = counts[i]
    if count > max_count:
        longest_name = names[i]
        max_count = count
print(longest_name)
>>>
Cecilia
```

Проблема в тому, що весь цей оператор циклу є візуально шумним. Індекси у `names` та `counts` ускладнюють читання коду. Індексування в масиви за індексом циклу `i` виконується двічі. Використання `enumerate` (см. Порада 7: «Надавайте перевагу enumerate над range») дещо покращує ситуацію, але все ще не є ідеальним:

```python
for i, name in enumerate(names):
    count = counts[i]
    if count > max_count:
        longest_name = name
        max_count = count
```

Щоб зробити цей код зрозумілішим, Python надає вбудовану функцію `zip`. `zip` огортає два або більше ітераторів «ледачим» генератором. Генератор `zip` видає кортежі, що містять наступне значення з кожного ітератора. Ці кортежі можна безпосередньо розпакувати в операторі `for` (см. Порада 6: «Надавайте перевагу розпакуванню з множинним присвоєнням над індексуванням»). Результуючий код значно чистіший:

```python
for name, count in zip(names, counts):
    if count > max_count:
        longest_name = name
        max_count = count
```

`zip` споживає обгорнуті ітератори по одному елементу за раз, що означає можливість його використання з нескінченно довгими вхідними даними без ризику того, що програма використає занадто багато пам'яті та зупиниться.

Однак слід бути обережними щодо поведінки `zip`, коли вхідні ітератори мають різну довжину. Наприклад, припустимо, що до `names` вище додається ще один елемент, але про оновлення `counts` забуто. Запуск `zip` на двох вхідних списках дасть несподіваний результат:

```python
names.append('Rosalind')
for name, count in zip(names, counts):
    print(name)
>>>
Cecilia
Lise
Marie
```

Нового елемента для `'Rosalind'` немає. Чому? Саме так працює `zip`. Він продовжує видавати кортежі, поки будь-який з обгорнутих ітераторів не вичерпається. Його виведення таке ж довге, як і його найкоротший вхід. Цей підхід добре працює, коли відомо, що ітератори однакової довжини, що часто буває для похідних списків, створених за допомогою спискових виразів.

Але в багатьох інших випадках поведінка `zip` з усіченням є несподіваною і небажаною. Якщо немає впевненості, що довжини списків, переданих у `zip`, будуть однаковими, слід розглянути використання функції `zip_longest` з вбудованого модуля `itertools`:

```python
import itertools
for name, count in itertools.zip_longest(names, counts):
    print(f'{name}: {count}')
>>>
Cecilia: 7
Lise: 4
Marie: 5
Rosalind: None
```

`zip_longest` замінює відсутні значення — у цьому випадку довжину рядка `'Rosalind'` — тим, що передане як `fillvalue`, що за замовчуванням є `None`.

**Що слід запам'ятати**

- Вбудована функція `zip` може використовуватися для паралельного ітерування по кількох ітераторах.
- `zip` створює «ледачий» генератор, що видає кортежі, тому може використовуватися з нескінченно довгими вхідними даними.
- `zip` беззвучно усікає виведення до найкоротшого ітератора, якщо передаються ітератори різної довжини.
- Використовуйте функцію `zip_longest` з вбудованого модуля `itertools`, якщо потрібно використати `zip` для ітераторів нерівної довжини без усічення.

## Порада 9. Уникайте блоків else після циклів for та while

У циклах Python є додаткова можливість, яка недоступна у більшості інших мов програмування: після внутрішнього блоку циклу можна розмістити блок `else`:

```python
for i in range(3):
    print('Loop', i)
else:
    print('Else block!')
>>>
Loop 0
Loop 1
Loop 2
Else block!
```

Дивно, але блок `else` виконується відразу після завершення циклу. Чому ця конструкція називається `else`? Чому не `and`? В операторі `if/else` слово `else` означає: «Виконай це, якщо попередній блок не виконався». В операторі `try/except` слово `except` має те саме визначення: «Виконай це, якщо виконання попереднього блоку завершилося помилкою».

Аналогічно, `else` в `try/except/else` відповідає цьому шаблону (см. Порада 65: «Скористайтеся перевагами кожного блоку в try/except/else/finally»), оскільки означає: «Виконай це, якщо не виникло жодного винятку». `try/finally` також інтуїтивно зрозумілий, оскільки означає: «Завжди виконуй це після спроби виконати попередній блок».

З огляду на всі застосування `else`, `except` та `finally` у Python, новий програміст може припустити, що частина `else` у `for/else` означає: «Виконай це, якщо цикл не завершився повністю». Насправді все відбувається навпаки. Використання оператора `break` у циклі насправді пропускає блок `else`:

```python
for i in range(3):
    print('Loop', i)
    if i == 1:
        break
else:
    print('Else block!')
>>>
Loop 0
Loop 1
```

Ще одна несподіванка: блок `else` виконується негайно, якщо виконується ітерація по порожній послідовності:

```python
for x in []:
    print('Never runs')
else:
    print('For Else block!')
>>>
For Else block!
```

Блок `else` також виконується, коли цикли `while` є спочатку `False`:

```python
while False:
    print('Never runs')
else:
    print('While Else block!')
>>>
While Else block!
```

Обґрунтування такої поведінки полягає в тому, що блоки `else` після циклів корисні при використанні циклів для пошуку чогось. Наприклад, потрібно визначити, чи є два числа взаємно простими (тобто їхній єдиний спільний дільник дорівнює 1). Тут ітеруємо по всіх можливих спільних дільниках і перевіряємо числа. Після перевірки всіх варіантів цикл завершується. Блок `else` виконується, коли числа взаємно прості, оскільки цикл не стикається з `break`:

```python
a = 4
b = 9
for i in range(2, min(a, b) + 1):
    print('Testing', i)
    if a % i == 0 and b % i == 0:
        print('Not coprime')
        break
else:
    print('Coprime')
>>>
Testing 2
Testing 3
Testing 4
Coprime
```

На практиці такий код не рекомендується. Натомість слід написати допоміжну функцію для виконання обчислення. Така допоміжна функція пишеться двома поширеними способами.

Перший підхід — повернення раніше при знаходженні шуканої умови. Якщо цикл завершився без знаходження, повертається значення за замовчуванням:

```python
def coprime(a, b):
    for i in range(2, min(a, b) + 1):
        if a % i == 0 and b % i == 0:
            return False
    return True

assert coprime(4, 9)
assert not coprime(3, 6)
```

Другий спосіб — наявність результуючої змінної, яка вказує, чи знайдено те, що шукалося у циклі. Вихід із циклу відбувається одразу після знаходження:

```python
def coprime_alternate(a, b):
    is_coprime = True
    for i in range(2, min(a, b) + 1):
        if a % i == 0 and b % i == 0:
            is_coprime = False
            break
    return is_coprime

assert coprime_alternate(4, 9)
assert not coprime_alternate(3, 6)
```

Обидва підходи значно зрозуміліші для читачів, незнайомих із кодом. Залежно від ситуації, будь-який з них може бути вдалим вибором. Однак виразність, яку дає блок `else`, не переважує тягаря, який він покладає на людей (включаючи вас самих), які захочуть розібратися у вашому коді в майбутньому. Прості конструкції, наприклад цикли, повинні бути самоочевидними в Python. Слід уникати використання блоків `else` після циклів повністю.

**Що слід запам'ятати**

- У Python є спеціальний синтаксис, що дозволяє розміщувати блоки `else` відразу після внутрішніх блоків циклів `for` та `while`.
- Блок `else` після циклу виконується лише в тому випадку, якщо тіло циклу не зустрічало оператор `break`.
- Уникайте використання блоків `else` після циклів, оскільки їхня поведінка неінтуїтивна і може вводити в оману.

## Порада 10. Запобігайте повторенню за допомогою виразів присвоєння

Вираз присвоєння — також відомий як «моржовий оператор» (walrus operator) — це новий синтаксис, введений у Python 3.8 для вирішення давньої проблеми мови, яка може призводити до дублювання коду. Тоді як звичайні оператори присвоєння записуються як `a = b` і читаються «a дорівнює b», ці присвоєння записуються як `a := b` і читаються «a моржує b» (оскільки `:=` нагадує пару очей і бивнів).

Вирази присвоєння корисні тим, що дозволяють присвоювати змінні в місцях, де оператори присвоєння заборонені, наприклад у умовному виразі оператора `if`. Значення виразу присвоєння дорівнює тому, що було присвоєно ідентифікатору ліворуч від моржового оператора.

Наприклад, є кошик зі свіжими фруктами для управління соковим баром. Тут визначається вміст кошика:

```python
fresh_fruit = {
    'apple': 10,
    'banana': 8,
    'lemon': 5,
}
```

Коли клієнт підходить до стойки, щоб замовити лимонад, потрібно переконатися, що в кошику є принаймні один лимон для вичавлювання. Тут це робиться шляхом отримання кількості лимонів з подальшою перевіркою ненульового значення в операторі `if`:

```python
def make_lemonade(count):
    ...
def out_of_stock():
    ...
count = fresh_fruit.get('lemon', 0)
if count:
    make_lemonade(count)
else:
    out_of_stock()
```

Проблема цього, здавалося б, простого коду в тому, що він є шумнішим, ніж потрібно. Змінна `count` використовується лише у першому блоці оператора `if`. Визначення `count` перед оператором `if` створює враження, що вона є важливішою, ніж є насправді, — ніби весь наступний код, включаючи блок `else`, матиме доступ до змінної `count`, хоча насправді це не так.

Цей шаблон отримання значення, перевірки його ненульовості, а потім використання є надзвичайно поширеним у Python. Багато програмістів намагаються обійтися з кількома посиланнями на `count` за допомогою різних прийомів, що погіршують читабельність (см. Порада 5: «Пишіть допоміжні функції замість складних виразів»). На щастя, до мови були додані вирази присвоєння саме для спрощення такого коду. Тут цей приклад переписується за допомогою моржового оператора:

```python
if count := fresh_fruit.get('lemon', 0):
    make_lemonade(count)
else:
    out_of_stock()
```

Хоча це лише на один рядок коротше, воно значно читабельніше, оскільки тепер зрозуміло, що `count` стосується лише першого блоку оператора `if`. Вираз присвоєння спочатку присвоює значення змінній `count`, а потім обчислює це значення в контексті оператора `if`, щоб визначити порядок керування потоком. Ця двоступенева поведінка — присвоїти, а потім обчислити — є фундаментальною природою моржового оператора.

Лимони досить потужні, тому для рецепту лимонаду потрібен лише один, що означає достатність ненульової перевірки. Якщо ж клієнт замовляє сидр, потрібно переконатися в наявності щонайменше чотирьох яблук. Тут це робиться шляхом отримання кількості з словника `fruit_basket` та використання порівняння в умовному виразі оператора `if`:

```python
def make_cider(count):
    ...
count = fresh_fruit.get('apple', 0)
if count >= 4:
    make_cider(count)
else:
    out_of_stock()
```

Тут та сама проблема, що і у прикладі з лимонадом: присвоєння `count` відволікає зайву увагу на цю змінну. Тут читабельність коду покращується також за допомогою моржового оператора:

```python
if (count := fresh_fruit.get('apple', 0)) >= 4:
    make_cider(count)
else:
    out_of_stock()
```

Це працює як очікується і робить код на один рядок коротшим. Важливо зазначити, що вираз присвоєння потрібно оточити дужками для порівняння з `4` в операторі `if`. У прикладі з лимонадом дужки не потрібні, оскільки вираз присвоєння стоїть самостійно як ненульова перевірка; він не є підвиразом більшого виразу. Як і в інших виразах, слід уникати дужок навколо виразів присвоєння там, де це можливо.

Ще один поширений варіант цього повторюваного шаблону виникає, коли потрібно присвоїти змінну в зовнішньому scope залежно від деякої умови, а потім незабаром посилатися на цю змінну у виклику функції. Наприклад, клієнт замовляє банановий смузі. Щоб його приготувати, потрібно мати щонайменше дві порції нарізаних бананів, інакше буде піднято виняток `OutOfBananas`. Тут ця логіка реалізується типовим способом:

```python
def slice_bananas(count):
    ...
class OutOfBananas(Exception):
    pass
def make_smoothies(count):
    ...

pieces = 0
count = fresh_fruit.get('banana', 0)
if count >= 2:
    pieces = slice_bananas(count)
try:
    smoothies = make_smoothies(pieces)
except OutOfBananas:
    out_of_stock()
```

Інший поширений варіант — розмістити присвоєння `pieces = 0` у блоці `else`:

```python
count = fresh_fruit.get('banana', 0)
if count >= 2:
    pieces = slice_bananas(count)
else:
    pieces = 0
try:
    smoothies = make_smoothies(pieces)
except OutOfBananas:
    out_of_stock()
```

Цей другий підхід може видатися дивним, оскільки він означає, що змінна `pieces` може бути спочатку визначена в двох різних місцях — у кожному блоці оператора `if`. Це розділене визначення технічно працює через правила scope у Python (см. Порада 21: «Знайте, як замикання взаємодіють зі scope змінних»), але воно не є легким для читання та виявлення, тому багато хто надає перевагу першій конструкції, де присвоєння `pieces = 0` є першим.

Моржовий оператор знову може скорочувати цей приклад на один рядок коду. Ця невелика зміна усуває зайву увагу до змінної `count`. Тепер зрозуміліше, що `pieces` матиме значення поза оператором `if`:

```python
pieces = 0
if (count := fresh_fruit.get('banana', 0)) >= 2:
    pieces = slice_bananas(count)
try:
    smoothies = make_smoothies(pieces)
except OutOfBananas:
    out_of_stock()
```

Використання моржового оператора також підвищує читабельність при розбитті визначення `pieces` між обома частинами оператора `if`. Змінну `pieces` легше відстежувати, коли визначення `count` більше не передує оператору `if`:

```python
if (count := fresh_fruit.get('banana', 0)) >= 2:
    pieces = slice_bananas(count)
else:
    pieces = 0
try:
    smoothies = make_smoothies(pieces)
except OutOfBananas:
    out_of_stock()
```

Одним із розчарувань програмістів, які тільки починають вивчати Python, є відсутність гнучкого оператора `switch/case`. Загальний стиль для імітації такої функціональності — глибоке вкладення кількох операторів `if`, `elif` та `else`. Наприклад, потрібно реалізувати систему пріоритетів, щоб кожен клієнт автоматично отримував найкращий наявний сік без необхідності робити замовлення. Тут визначається логіка, за якою спочатку подаються банановий смузі, потім яблучний сидр, а потім — лимонад:

```python
count = fresh_fruit.get('banana', 0)
if count >= 2:
    pieces = slice_bananas(count)
    to_enjoy = make_smoothies(pieces)
else:
    count = fresh_fruit.get('apple', 0)
    if count >= 4:
        to_enjoy = make_cider(count)
    else:
        count = fresh_fruit.get('lemon', 0)
        if count:
            to_enjoy = make_lemonade(count)
        else:
            to_enjoy = 'Nothing'
```

Такі некрасиві конструкції дивовижно часто зустрічаються в Python-коді. На щастя, моржовий оператор надає елегантне рішення, яке може бути майже таким же гнучким, як спеціальний синтаксис для операторів `switch/case`:

```python
if (count := fresh_fruit.get('banana', 0)) >= 2:
    pieces = slice_bananas(count)
    to_enjoy = make_smoothies(pieces)
elif (count := fresh_fruit.get('apple', 0)) >= 4:
    to_enjoy = make_cider(count)
elif count := fresh_fruit.get('lemon', 0):
    to_enjoy = make_lemonade(count)
else:
    to_enjoy = 'Nothing'
```

Версія з виразами присвоєння лише на п'ять рядків коротша за оригінал, але покращення читабельності значне завдяки зменшенню вкладеності та відступів. Якщо в коді виникають подібні некрасиві конструкції, рекомендується, якщо можливо, перевести їх на використання моржового оператора.

Ще одним поширеним розчаруванням нових Python-програмістів є відсутність конструкції циклу `do/while`. Наприклад, потрібно розливати сік у пляшки у міру надходження нових фруктів, поки фрукти не закінчаться. Тут ця логіка реалізується за допомогою циклу `while`:

```python
def pick_fruit():
    ...
def make_juice(fruit, count):
    ...

bottles = []
fresh_fruit = pick_fruit()
while fresh_fruit:
    for fruit, count in fresh_fruit.items():
        batch = make_juice(fruit, count)
        bottles.extend(batch)
    fresh_fruit = pick_fruit()
```

Це повторюється, оскільки вимагає двох окремих викликів `fresh_fruit = pick_fruit()`: один перед циклом для встановлення початкових умов, і ще один наприкінці циклу для поповнення списку доставлених фруктів. Стратегія підвищення повторного використання коду в цій ситуації — використання ідіоми «цикл із половиною» (loop-and-a-half). Це усуває зайві рядки, але також підриває роль циклу `while`, перетворюючи його на нескінченний. Тепер весь контроль потоку циклу залежить від умовного оператора `break`:

```python
bottles = []
while True:             # Цикл
    fresh_fruit = pick_fruit()
    if not fresh_fruit:  # і половина
        break
    for fruit, count in fresh_fruit.items():
        batch = make_juice(fruit, count)
        bottles.extend(batch)
```

Моржовий оператор усуває необхідність в ідіомі «цикл із половиною», дозволяючи змінній `fresh_fruit` повторно присвоюватися та умовно обчислюватися при кожному проходженні циклу `while`. Це рішення є коротким і легким для читання, і воно повинне бути пріоритетним у вашому коді:

```python
bottles = []
while fresh_fruit := pick_fruit():
    for fruit, count in fresh_fruit.items():
        batch = make_juice(fruit, count)
        bottles.extend(batch)
```

Існують інші ситуації, де вирази присвоєння можуть використовуватися для усунення надмірності (см. Порада 29: «Уникайте повторної роботи у спискових виразах за допомогою виразів присвоєння»). Загалом, коли одиніший і той самий вираз або присвоєння повторюється кілька разів у межах групи рядків, настає час розглянути використання виразів присвоєння для підвищення читабельності.

**Що слід запам'ятати**

- Вирази присвоєння використовують моржовий оператор (`:=`) для одночасного присвоєння та обчислення імен змінних в одному виразі, зменшуючи таким чином повторення.
- Коли вираз присвоєння є підвиразом більшого виразу, його слід оточити дужками.
- Хоча оператори `switch/case` та цикли `do/while` недоступні у Python, їхню функціональність можна значно зрозуміліше імітувати за допомогою виразів присвоєння.
