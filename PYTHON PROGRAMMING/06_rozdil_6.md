## 6.1 Роль функцій

Програми, які ми писали досі, складаються з однієї функції, що зазвичай називається main. Ми також використовували заздалегідь написані функції і методи, включаючи вбудовані функції Python (наприклад, print, abs), функції і методи стандартних бібліотек Python (наприклад, math.sqrt), а також методи модуля graphics (наприклад, myPoint.getX()). Функції — важливий інструмент для побудови складних програм. Цей розділ охоплює причини і способи проектування власних функцій, щоб зробити програми простішими у написанні та розумінні.

У розділі 4 ми розглядали графічне розв'язання задачі майбутньої вартості. Нагадаємо, що ця програма використовує графічну бібліотеку для побудови стовпчастої діаграми, що відображає зростання інвестиції. Ось програма, якою ми її залишили:

```python
# futval_graph2.py

from graphics import *


def main():
    # Введення
    print("Ця програма будує графік зростання 10-річної інвестиції.")

    # Отримати основну суму і відсоткову ставку
    principal = float(input("Введіть початкову суму: "))
    apr = float(input("Введіть річну відсоткову ставку: "))

    # Створити графічне вікно з мітками на лівому краї
    win = GraphWin("Investment Growth Chart", 320, 240)
    win.setBackground("white")
    win.setCoords(-1.75, -200, 11.5, 10400)
    Text(Point(-1, 0), ' 0.0K').draw(win)
    Text(Point(-1, 2500), ' 2.5K').draw(win)
    Text(Point(-1, 5000), ' 5.0K').draw(win)
    Text(Point(-1, 7500), ' 7.5k').draw(win)
    Text(Point(-1, 10000), '10.0K').draw(win)

    # Намалювати стовпець для початкової основної суми
    bar = Rectangle(Point(0, 0), Point(1, principal))
    bar.setFill("green")
    bar.setWidth(2)
    bar.draw(win)

    # Намалювати стовпець для кожного наступного року
    for year in range(1, 11):
        principal = principal * (1 + apr)
        bar = Rectangle(Point(year, 0), Point(year + 1, principal))
        bar.setFill("green")
        bar.setWidth(2)
        bar.draw(win)

    input("Натисніть <Enter>, щоб вийти.")
    win.close()

main()
```

Це, безумовно, робоча програма, але є питання стилю програмування, яке слід вирішити. Зверніть увагу, що ця програма малює стовпці в двох різних місцях. Початковий стовпець малюється безпосередньо перед циклом, а наступні — всередині циклу.

Наявність схожого коду в двох місцях має певні недоліки. Очевидно, одна проблема — це необхідність писати код двічі. Більш тонка проблема полягає в тому, що код доводиться підтримувати в двох різних місцях. Якщо ми вирішимо змінити колір або інші аспекти стовпців, нам потрібно переконатися, що ці зміни відбуваються в обох місцях. Нездатність тримати пов'язані частини коду у синхронізованому стані — поширена проблема при супроводі програм.

Функції можна використовувати для зменшення дублювання коду і для того, щоб зробити програми більш зрозумілими і простими в обслуговуванні. Перш ніж виправляти програму майбутньої вартості, розглянемо, що можуть запропонувати функції.


## 6.2 Функції неформально

Функцію можна уявити як підпрограму — маленьку програму всередині програми. Основна ідея функції полягає в тому, що ми пишемо послідовність операторів і даємо цій послідовності ім'я. Потім інструкції можуть бути виконані в будь-якій точці програми, якщо посилатися на ім'я функції.

Частина програми, що створює функцію, називається визначенням функції. Коли функція згодом використовується в програмі, говорять, що визначення викликається або запускається. Одне визначення функції може викликатися в багатьох різних точках програми.

Розглянемо конкретний приклад. Припустімо, ви хочете написати програму, що виводить слова пісні «Happy Birthday». Стандартні слова виглядають так:

```
Happy birthday to you!
Happy birthday to you!
Happy birthday, dear <insert-name>.
Happy birthday to you!
```

Ми будемо грати з цим прикладом в інтерактивному середовищі Python. Можливо, вам захочеться запустити Python і спробувати деякі з цих прикладів самостійно.

Простий підхід до цієї проблеми — використати чотири оператори print. Ось інтерактивний сеанс, що створює програму для виконання «Happy Birthday» для Фреда:

```python
>>> def main():
        print("Happy birthday to you!")
        print("Happy birthday to you!")
        print("Happy birthday, dear Fred.")
        print("Happy birthday to you!")
```

Потім ми можемо запустити цю програму і отримати слова пісні:

```python
>>> main()
Happy birthday to you!
Happy birthday to you!
Happy birthday, dear Fred.
Happy birthday to you!
```

Очевидно, у цій програмі є дублюючий код. Для такої простої програми це не велика проблема, але навіть тут трохи дратує знову і знову передруковувати один і той самий рядок. Введемо функцію, що виводить слова першого, другого і четвертого рядків:

```python
>>> def happy():
        print("Happy birthday to you!")
```

Ми визначили нову функцію з іменем happy. Ось приклад того, що вона робить:

```python
>>> happy()
Happy birthday to you!
```

Виклик команди happy змушує Python вивести рядок пісні.

Тепер ми можемо переробити куплет для Фреда, використовуючи happy. Назвемо нашу нову версію singFred:

```python
>>> def singFred():
        happy()
        happy()
        print("Happy birthday, dear Fred.")
        happy()
```

Ця версія потребувала значно менше введення завдяки команді happy. Спробуємо вивести слова для Фреда, щоб переконатися, що все працює:

```python
>>> singFred()
Happy birthday to you!
Happy birthday to you!
Happy birthday, dear Fred.
Happy birthday to you!
```

Непогано. Тепер припустімо, що також день народження у Люсі, і ми хочемо виконати куплет для Фреда, а потім для Люсі. Куплет для Фреда ми вже маємо; підготуємо і для Люсі:

```python
>>> def singLucy():
        happy()
        happy()
        print("Happy birthday, dear Lucy.")
        happy()
```

Тепер ми можемо написати основну програму, яка співає для Фреда і Люсі:

```python
>>> def main():
        singFred()
        print()
        singLucy()
```

Голий print між двома викликами функцій вставляє пробіл між куплетами у виведенні. А ось кінцевий продукт у дії:

```python
>>> main()
Happy birthday to you!
Happy birthday to you!
Happy birthday, dear Fred.
Happy birthday to you!

Happy birthday to you!
Happy birthday to you!
Happy birthday, dear Lucy.
Happy birthday to you!
```

Що ж, це, мабуть, спрацьовує, і ми усунули частину дублювання, визначивши функцію happy. Однак щось досі відчувається не зовсім правильним. У нас є дві функції — singFred і singLucy, що майже ідентичні. Дотримуючись цього підходу, додавання куплету для Елмера змусило б нас створити функцію singElmer, що виглядає так само, як функції для Фреда і Люсі. Чи не можемо ми щось зробити з цим розростанням куплетів?

Зверніть увагу, що єдина відмінність між singFred і singLucy — ім'я наприкінці третього оператора print. Куплети повністю однакові за виключенням цієї однієї змінної частини. Ми можемо об'єднати ці дві функції, використавши параметр. Напишемо загальну функцію sing:

```python
>>> def sing(person):
        happy()
        happy()
        print("Happy Birthday, dear", person + ".")
        happy()
```

Ця функція використовує параметр з іменем `person`. Параметр — це змінна, яка ініціалізується при виклику функції. Ми можемо використовувати функцію sing для виведення куплету як для Фреда, так і для Люсі. Нам потрібно лише вказати ім'я як параметр при виклику функції:

```python
>>> sing("Fred")
Happy birthday to you!
Happy birthday to you!
Happy Birthday, dear Fred.
Happy birthday to you!

>>> sing("Lucy")
Happy birthday to you!
Happy birthday to you!
Happy Birthday, dear Lucy.
Happy birthday to you!
```

Завершимо програмою, яка співає для всіх трьох іменинників:

```python
>>> def main():
        sing("Fred")
        print()
        sing("Lucy")
        print()
        sing("Elmer")
```

Простіше нікуди.

Ось повна програма як файл модуля:

```python
# happy.py

def happy():
    print("Happy Birthday to you!")

def sing(person):
    happy()
    happy()
    print("Happy birthday, dear", person + ".")
    happy()

def main():
    sing("Fred")
    print()
    sing("Lucy")
    print()
    sing("Elmer")

main()
```


## 6.3 Майбутня вартість за допомогою функції

Тепер, коли ви побачили, як визначення функцій може допомогти вирішити проблему дублювання коду, повернемося до графіка майбутньої вартості. Пам'ятайте, проблема в тому, що стовпці діаграми малюються в двох різних місцях програми. Код безпосередньо перед циклом виглядає так:

```python
# Намалювати стовпець для початкової основної суми
bar = Rectangle(Point(0, 0), Point(1, principal))
bar.setFill("green")
bar.setWidth(2)
bar.draw(win)
```

А код всередині циклу такий:

```python
bar = Rectangle(Point(year, 0), Point(year + 1, principal))
bar.setFill("green")
bar.setWidth(2)
bar.draw(win)
```

Спробуємо об'єднати ці два фрагменти в одну функцію, що малює стовпець на екрані.

Щоб намалювати стовпець, нам потрібна певна інформація. А саме, ми маємо знати, для якого року малюється стовпець, якою буде його висота і в яке вікно він малюватиметься. Ці три значення будуть надані як параметри функції. Ось визначення функції:

```python
def drawBar(window, year, height):
    # Намалювати стовпець у window для заданого року із заданою висотою
    bar = Rectangle(Point(year, 0), Point(year + 1, height))
    bar.setFill("green")
    bar.setWidth(2)
    bar.draw(window)
```

Щоб використовувати цю функцію, нам просто потрібно вказати значення для трьох параметрів. Наприклад, якщо `win` — це GraphWin, ми можемо намалювати стовпець для року 0 і основної суми $2 000, викликавши drawBar так:

```python
drawBar(win, 0, 2000)
```

Включаючи функцію drawBar, ось остання версія нашої програми майбутньої вартості:

```python
# futval_graph3.py
from graphics import *

def drawBar(window, year, height):
    # Намалювати стовпець у window починаючи з year із заданою висотою
    bar = Rectangle(Point(year, 0), Point(year + 1, height))
    bar.setFill("green")
    bar.setWidth(2)
    bar.draw(window)

def main():
    # Введення
    print("Ця програма будує графік зростання 10-річної інвестиції.")

    # Отримати основну суму і відсоткову ставку
    principal = float(input("Введіть початкову суму: "))
    apr = float(input("Введіть річну відсоткову ставку: "))

    # Створити графічне вікно з мітками на лівому краї
    win = GraphWin("Investment Growth Chart", 320, 240)
    win.setBackground("white")
    win.setCoords(-1.75, -200, 11.5, 10400)
    Text(Point(-1, 0), ' 0.0K').draw(win)
    Text(Point(-1, 2500), ' 2.5K').draw(win)
    Text(Point(-1, 5000), ' 5.0K').draw(win)
    Text(Point(-1, 7500), ' 7.5k').draw(win)
    Text(Point(-1, 10000), '10.0K').draw(win)

    drawBar(win, 0, principal)
    for year in range(1, 11):
        principal = principal * (1 + apr)
        drawBar(win, year, principal)

    input("Натисніть <Enter>, щоб вийти.")
    win.close()

main()
```

Ви можете бачити, як drawBar усунула дублюючий код. Якщо ми захочемо змінити зовнішній вигляд стовпців на діаграмі, нам потрібно змінити код лише в одному місці — у визначенні drawBar. Не хвилюйтеся, якщо ви ще не розумієте кожну деталь цього прикладу. Вам ще є чому навчитися про функції.


## 6.4 Функції і параметри: захопливі подробиці

Можливо, вас цікавить вибір параметрів для функції drawBar. Очевидно, рік, для якого малюється стовпець, і висота стовпця — це змінні частини при малюванні стовпця. Але чому `window` теж є параметром цієї функції? Зрештою, ми малюватимемо всі стовпці в одному вікні; схоже, воно не змінюється.

Причина того, що `window` є параметром, пов'язана з областю видимості змінних у визначеннях функцій. Область видимості — це місця в програмі, де може посилатися на задану змінну. Пам'ятайте, кожна функція — це своя маленька підпрограма. Змінні, що використовуються всередині однієї функції, є локальними для цієї функції, навіть якщо вони мають те саме ім'я, що й змінні, що з'являються в іншій функції.

Єдиний спосіб для функції побачити змінну з іншої функції — щоб ця змінна була передана як параметр.¹ Оскільки GraphWin (присвоєний змінній `win`) створюється всередині main, він не є безпосередньо доступним у drawBar. Однак параметр `window` у drawBar отримує значення `win` з main при виклику drawBar. Щоб зрозуміти, як це відбувається, нам потрібно детальніше розглянути процес виклику функції.

Визначення функції виглядає так:

```
def <ім'я>(<формальні-параметри>):
    <тіло>
```

Ім'я функції повинно бути ідентифікатором, а `формальні-параметри` — це (можливо порожня) послідовність імен змінних (також ідентифікаторів). Формальні параметри, як і всі змінні, що використовуються у функції, доступні лише в тілі функції. Змінні з однаковими іменами в іншому місці програми є відмінними від формальних параметрів і змінних усередині тіла функції.

Функція викликається за допомогою її імені, за яким слідує список фактичних параметрів або аргументів:

```
<ім'я>(<фактичні-параметри>)
```

Коли Python доходить до виклику функції, він ініціює чотирикроковий процес:

1. Програма, що викликає, призупиняє виконання у точці виклику.
2. Формальним параметрам функції присвоюються значення, надані фактичними параметрами у виклику.
3. Виконується тіло функції.
4. Управління повертається до точки відразу після місця виклику функції.

Повертаючись до прикладу «Happy Birthday», простежимо виконання двох куплетів. Ось частина тіла main:

```python
sing("Fred")
print()
sing("Lucy")
```

Коли Python доходить до `sing("Fred")`, виконання main тимчасово призупиняється. У цей момент Python знаходить визначення sing і бачить, що вона має один формальний параметр `person`. Формальному параметру присвоюється значення фактичного параметра, тому це рівносильно виконанню такого оператора:

```python
person = "Fred"
```

На рисунку 6.1 показано стан ситуації. Зверніть увагу, що змінна `person` всередині sing щойно була ініціалізована.

У цей момент Python починає виконувати тіло sing. Перший оператор — ще один виклик функції, цього разу — happy. Python призупиняє виконання sing і передає управління викликаній функції. Тіло happy складається з одного print. Цей оператор виконується, а потім управління повертається до місця, де sing зупинилася. На рисунку 6.2 показано знімок виконання до цього моменту.

Виконання продовжується в такому самому порядку: Python робить ще дві поїздки в happy для завершення виконання sing. Коли Python доходить до кінця sing, управління повертається до main і продовжується відразу після виклику функції. На рисунку 6.3 показано, де ми знаходимося в цей момент. Зверніть увагу, що змінна `person` у sing зникла. Пам'ять, зайнята локальними змінними функції, звільняється після її завершення. Локальні змінні не зберігають жодних значень від одного виконання функції до наступного.

Наступний оператор для виконання — голий print у main. Він виробляє порожній рядок у виведенні. Потім Python зустрічає ще один виклик sing. Як і раніше, управління передається у визначення функції. Цього разу формальний параметр — «Lucy». На рисунку 6.4 показано ситуацію, коли sing починає виконуватися вдруге.

Тепер перемотаємо до кінця. Тіло функції sing виконується для Люсі (з трьома поїздками через happy), і управління повертається до main відразу після точки виклику функції. Тепер ми досягли кінця нашого фрагмента коду, як показано на рисунку 6.5. Ці три оператори в main спричинили виконання sing двічі і виконання happy шість разів. Загалом було згенеровано дев'ять рядків виведення.

Сподіваємося, ви починаєте розуміти, як працюють виклики функцій. Один момент, якого цей приклад не торкнувся, — це використання кількох параметрів. Зазвичай, коли визначення функції має кілька параметрів, фактичні параметри зіставляються з формальними за позицією. Перший фактичний параметр присвоюється першому формальному, другий — другому і т.д. Можливо змінити цю поведінку за допомогою ключових параметрів, що зіставляються за іменем (наприклад, `end=""` у виклику print). Однак у всіх наших прикладних функціях ми покладатимемося на позиційне зіставлення.

Як приклад, знову поглянемо на використання функції drawBar з програми майбутньої вартості. Ось виклик для малювання початкового стовпця:

```python
drawBar(win, 0, principal)
```

Коли Python передає управління у drawBar, ці параметри зіставляються з формальними параметрами в заголовку функції:

```python
def drawBar(window, year, height):
```

Сукупний ефект такий, ніби тіло функції було попереджено трьома операторами присвоювання:

```python
window = win
year = 0
height = principal
```

При виклику функції завжди потрібно уважно стежити за тим, щоб фактичні параметри розташовувалися в правильному порядку відповідно до визначення функції.

---
¹ Технічно можливо посилатися на змінну з функції, вкладеної в іншу функцію, але вкладання функцій виходить за рамки цього обговорення.


## 6.5 Функції, що повертають значення

Ви бачили, що передача параметрів надає механізм для ініціалізації змінних у функції. У певному сенсі параметри діють як введення до функції. Ми можемо викликати функцію багато разів і отримувати різні результати, змінюючи вхідні параметри. Нерідко ми також хочемо отримувати інформацію з функції. Насправді фундаментальні ідеї та словник функцій запозичені з математики, де функція вважається відношенням між вхідними та вихідними змінними. Наприклад, математик міг би визначити функцію f, що обчислює квадрат свого аргументу. Математично це можна записати так:

```
f(x) = x²
```

Це показує, що f — функція, яка оперує над однією змінною (тут названою x) і виробляє значення, що є квадратом x.

Як і функції Python, математики використовують нотацію зі скобками для позначення застосування функції. Наприклад, f(5) = 25 стверджує, що при застосуванні f до 5, результат дорівнює 25. Математичні функції не обмежені одним аргументом. Можна, наприклад, визначити функцію, що використовує теорему Піфагора для отримання довжини гіпотенузи прямокутного трикутника за довжинами катетів. Назвемо цю функцію h:

```
h(x, y) = √(x² + y²)
```

З цього визначення ви повинні переконатися, що h(3, 4) = 5.

До цього часу ми обговорювали деталі функцій Python на прикладах, де функції використовуються як нові команди і викликаються для виконання команд. Але з математичної точки зору виклики функцій — це, по суті, вирази, що виробляють результат. Ми можемо легко розширити наш погляд на функції Python, щоб врахувати цю ідею. Насправді ви вже бачили численні приклади функцій такого типу. Наприклад, розгляньте такий виклик функції sqrt із бібліотеки math:

```python
discRt = math.sqrt(b * b - 4 * a * c)
```

Тут значення `b*b - 4*a*c` — це фактичний параметр функції `math.sqrt`. Оскільки виклик функції знаходиться в правій частині оператора присвоювання, він є виразом. Функція `math.sqrt` виробляє значення, яке потім присвоюється змінній `discRt`. Технічно кажуть, що sqrt повертає квадратний корінь свого аргументу.

Писати функції, що повертають значення, дуже легко. Ось реалізація функції Python, що повертає квадрат свого аргументу:

```python
def square(x):
    return x ** 2
```

Бачите, наскільки це визначення функції схоже на математичну версію (f(x)) вище? Тіло функції Python складається з одного оператора return. Коли Python зустрічає return, він негайно виходить з поточної функції і повертає управління до точки відразу після місця виклику функції. Крім того, значення, надане в операторі return, надсилається назад до того, хто викликав, як результат виразу. По суті, це просто додає одну невелику деталь до чотирикрокового процесу виклику функції, описаного раніше: значення, що повертається функцією, використовується як результат виразу.

Ефект полягає в тому, що ми можемо використовувати функцію square у будь-якому місці нашого коду, де допустимий вираз. Ось кілька інтерактивних прикладів:

```python
>>> square(3)
9
>>> print(square(4))
16
>>> x = 5
>>> y = square(x)
>>> print(y)
25
>>> print(square(x) + square(3))
34
```

Скористаємося функцією square для написання ще однієї функції — тієї, що знаходить відстань між двома точками. Враховуючи дві точки (x₁, y₁) і (x₂, y₂), відстань між ними обчислюється як √((x₂ - x₁)² + (y₂ - y₁)²). Ось функція Python для обчислення відстані між двома об'єктами Point:

```python
def distance(p1, p2):
    dist = math.sqrt(square(p2.getX() - p1.getX())
                     + square(p2.getY() - p1.getY()))
    return dist
```

Використовуючи функцію distance, ми можемо доповнити інтерактивну програму для трикутника з розділу 4 для обчислення периметра трикутника. Ось повна програма:

```python
# triangle2.py
import math
from graphics import *

def square(x):
    return x ** 2

def distance(p1, p2):
    dist = math.sqrt(square(p2.getX() - p1.getX())
                     + square(p2.getY() - p1.getY()))
    return dist

def main():
    win = GraphWin("Draw a Triangle")
    win.setCoords(0.0, 0.0, 10.0, 10.0)
    message = Text(Point(5, 0.5), "Клацніть на три точки")
    message.draw(win)

    # Отримати та намалювати три вершини трикутника
    p1 = win.getMouse()
    p1.draw(win)
    p2 = win.getMouse()
    p2.draw(win)
    p3 = win.getMouse()
    p3.draw(win)

    # Використати об'єкт Polygon для малювання трикутника
    triangle = Polygon(p1, p2, p3)
    triangle.setFill("peachpuff")
    triangle.setOutline("cyan")
    triangle.draw(win)

    # Обчислити периметр трикутника
    perim = distance(p1, p2) + distance(p2, p3) + distance(p3, p1)
    message.setText("Периметр: {0:0.2f}".format(perim))

    # Чекати ще одного кліку для виходу
    win.getMouse()
    win.close()

main()
```

Ви можете бачити, як distance викликається тричі в одному рядку для обчислення периметра трикутника. Використання функції тут заощаджує чимало стомлюючого кодування. Функції, що повертають значення, є надзвичайно корисними і гнучкими, оскільки їх можна комбінувати у виразах таким чином.

До речі, порядок визначень функцій у програмі не важливий. Все спрацювало б так само, якби функція main була визначена вгорі, наприклад. Нам лише потрібно переконатися, що функція визначена до того, як програма фактично намагається її виконати. Оскільки виклик `main()` не відбувається до самого останнього рядка модуля, всі функції будуть визначені до того, як програма фактично почне виконуватися.

Як ще один приклад, повернемося до програми «Happy Birthday». У вихідній версії ми використовували кілька функцій, що містять оператори print. Замість того, щоб допоміжні функції виконували виведення, ми могли б просто мати функції, що повертають значення — рядки в цьому випадку — які потім виводяться main. Розгляньте цю версію програми:

```python
# happy2.py

def happy():
    return "Happy Birthday to you!\n"

def verseFor(person):
    lyrics = happy() * 2 + "Happy birthday, dear " + person + ".\n" + happy()
    return lyrics

def main():
    for person in ["Fred", "Lucy", "Elmer"]:
        print(verseFor(person))

main()
```

Зверніть увагу, що весь вивід виконується в одному місці (main), тоді як happy і verseFor відповідають лише за створення і повернення відповідних рядків. Завдяки «магії» функцій, що повертають значення, ми спростили програму так, що весь куплет будується в одному рядковому виразі:

```python
lyrics = happy() * 2 + "Happy birthday, dear " + person + ".\n" + happy()
```

Обов'язково уважно вивчіть і зрозумійте цей рядок коду; він справді ілюструє силу і красу функцій, що повертають значення.

Крім того, що ця версія програми є більш елегантною, вона також більш гнучка, ніж оригінал, тому що виведення більше не розподілено по кількох функціях. Наприклад, ми можемо легко змінити програму, щоб записати результати у файл замість виведення на екран. Все, що нам потрібно зробити, — відкрити файл для запису і додати параметр `file=` до оператора print. Ніякої ревізії інших функцій не потрібно. Ось повна модифікація:

```python
def main():
    outf = open("Happy_Birthday.txt", "w")
    for person in ["Fred", "Lucy", "Elmer"]:
        print(verseFor(person), file=outf)
    outf.close()
```

Загалом майже завжди краще (тобто гнучкіше) мати функції, що повертають значення, а не виводять інформацію на екран. Таким чином, той, хто викликає, може вибирати — виводити інформацію чи використовувати її якось інакше.

Іноді функції потрібно повернути більше одного значення. Це можна зробити, просто перерахувавши більше одного виразу в операторі return. Як несерйозний приклад, ось функція, що обчислює як суму, так і різницю двох чисел:

```python
def sumDiff(x, y):
    sum = x + y
    diff = x - y
    return sum, diff
```

Як ви можете бачити, цей return повертає два значення. При виклику цієї функції ми поміщаємо її в одночасне присвоювання:

```python
num1, num2 = input("Будь ласка, введіть два числа (num1, num2) ").split(", ")
s, d = sumDiff(float(num1), float(num2))
print("Сума:", s, "а різниця:", d)
```

Як і у випадку з параметрами, коли з функції повертається кілька значень, вони присвоюються змінним за позицією. У цьому прикладі `s` отримає перше значення, перераховане в return (sum), а `d` — друге значення (diff).

Це, по суті, все, що потрібно знати про функції, що повертають значення, у Python. Є одне «підводне каміння», про яке слід попередити. Технічно всі функції Python повертають значення, незалежно від того, чи містить функція фактично оператор return. Функції без return завжди повертають спеціальний об'єкт, що позначається `None`. Цей об'єкт часто використовується як своєрідне значення за замовчуванням для змінних, що наразі не містять нічого корисного. Поширена помилка, яку роблять нові (і не такі нові) програмісти, — писати те, що має бути функцією, що повертає значення, але забувати включити оператор return наприкінці.

Припустімо, ми забули включити оператор return наприкінці функції distance:

```python
def distance(p1, p2):
    dist = math.sqrt(square(p2.getX() - p1.getX())
                     + square(p2.getY() - p1.getY()))
```

Запуск переглянутої програми трикутника з цією версією distance генерує таке повідомлення про помилку Python:

```
Traceback (most recent call last):
  File "triangle2.py", line 42, in <module>
    main()
  File "triangle2.py", line 35, in main
    perim = distance(p1, p2) + distance(p2, p3) + distance(p3, p1)
TypeError: unsupported operand type(s) for +: 'NoneType' and 'NoneType'
```

Проблема тут у тому, що ця версія distance не повертає число; вона завжди повертає значення `None`. Додавання не визначене для `None` (що має спеціальний тип `NoneType`), тому Python скаржиться. Якщо ваші функції, що повертають значення, видають дивні повідомлення про помилки, пов'язані з `None`, або якщо ваші програми виводять загадковий «None» в середині виведення, перевірте, чи не пропустили ви оператор return.


## 6.6 Функції, що змінюють параметри

Значення, що повертаються, — це основний спосіб надсилання інформації з функції назад до частини програми, що викликала функцію. У деяких випадках функції також можуть взаємодіяти зі стороною, що викликає, шляхом внесення змін до параметрів функції. Розуміння того, коли і як це можливо, вимагає засвоєння деяких тонких деталей про те, як працює присвоювання в Python і який вплив це має на зв'язок між фактичними і формальними параметрами, що використовуються у виклику функції.

Почнемо з простого прикладу. Припустімо, ви пишете програму для управління банківськими рахунками або інвестиціями. Одним із поширених завдань є нарахування відсотків на рахунок (як ми робили в програмі майбутньої вартості). Ми могли б розглянути написання функції, яка автоматично додає відсотки до балансу рахунку. Ось перша спроба такої функції:

```python
# addinterest1.py
def addInterest(balance, rate):
    newBalance = balance * (1 + rate)
    balance = newBalance
```

Намір цієї функції — встановити баланс рахунку на значення, оновлене з урахуванням суми відсотків.

Спробуємо нашу функцію, написавши невелику тестову програму:

```python
def test():
    amount = 1000
    rate = 0.05
    addInterest(amount, rate)
    print(amount)
```

Що, на вашу думку, виведе ця програма? Наш намір — додати 5% до `amount`, що дасть результат 1050. Ось що насправді відбувається:

```python
>>> test()
1000
```

Як ви можете бачити, `amount` не змінився! Що пішло не так?

Насправді нічого не пішло не так. Якщо ви уважно розглянете те, що ми обговорили до цього щодо функцій і параметрів, ви побачите, що це саме той результат, якого слід очікувати. Простежимо виконання цього прикладу, щоб побачити, що відбувається. Перші два рядки функції test створюють дві локальні змінні `amount` і `rate`, яким надаються початкові значення 1000 і 0.05 відповідно.

Далі управління передається у функцію addInterest. Формальним параметрам `balance` і `rate` присвоюються значення фактичних параметрів `amount` і `rate`. Пам'ятайте, навіть попри те, що ім'я `rate` з'являється в обох функціях, це дві окремі змінні. Ситуація на початку виконання addInterest показана на рисунку 6.6. Зверніть увагу, що присвоювання параметрів змушує змінні `balance` і `rate` у addInterest посилатися на значення фактичних параметрів.

Виконання першого рядка addInterest створює нову змінну `newBalance`. Тепер настає ключовий крок. Наступний оператор у addInterest присвоює `balance` те саме значення, що й `newBalance`. Результат показаний на рисунку 6.7. Зверніть увагу, що `balance` тепер посилається на те саме значення, що й `newBalance`, але це не вплинуло на `amount` у функції test.

У цей момент виконання addInterest завершено і управління повертається до test. Локальні змінні (включаючи параметри) у addInterest зникають, але `amount` і `rate` у функції test все ще посилаються на початкові значення 1000 і 0.05 відповідно. Звичайно, програма виводить `amount` як 1000.

Підсумовуючи ситуацію: формальні параметри функції лише отримують значення фактичних параметрів. Функція не має доступу до змінної, що містить фактичний параметр; тому присвоювання нового значення формальному параметру не впливає на змінну, що містить фактичний параметр. У термінах мов програмування Python передає всі параметри за значенням (by value).

Деякі мови програмування (наприклад, C++ та Ada) дозволяють передавати самі змінні як параметри функції. Такий механізм називається передачею параметрів за посиланням (by reference). Коли змінна передається за посиланням, присвоювання нового значення формальному параметру фактично змінює значення параметрної змінної у програмі, що викликає.

Оскільки Python не дозволяє передачу параметрів за посиланням, очевидна альтернатива — змінити нашу функцію addInterest так, щоб вона повертала `newBalance`. Це значення потім може бути використане для оновлення `amount` у функції test. Ось робоча версія (`addinterest2.py`):

```python
def addInterest(balance, rate):
    newBalance = balance * (1 + rate)
    return newBalance

def test():
    amount = 1000
    rate = 0.05
    amount = addInterest(amount, rate)
    print(amount)
```

Ви повинні легко простежити виконання цієї програми, щоб побачити, як ми отримаємо такий результат:

```python
>>> test()
1050
```

Тепер припустімо, що замість розгляду одного рахунку ми пишемо програму, яка має справу з багатьма банківськими рахунками. Ми могли б зберігати баланси рахунків у списку Python. Було б добре мати функцію addInterest, яка додає нараховані відсотки до всіх балансів у списку. Якщо `balances` — список балансів рахунків, ми можемо оновити першу суму у списку (ту, що знаходиться за індексом 0), рядком коду:

```python
balances[0] = balances[0] * (1 + rate)
```

Пам'ятайте, це працює, тому що списки є змінними. Цей рядок коду по суті говорить: «Помнож значення на 0-й позиції списку на (1 + rate) і збережи результат назад у 0-ю позицію списку». Звичайно, дуже схожий рядок коду оновив би баланс наступної позиції в списку; нам просто потрібно замінити нулі на одиниці:

```python
balances[1] = balances[1] * (1 + rate)
```

Більш загальний спосіб оновлення всіх балансів у списку — використати цикл, що проходить по позиціях 0, 1, ..., довжина−1. Розгляньте `addinterest3.py`:

```python
def addInterest(balances, rate):
    for i in range(len(balances)):
        balances[i] = balances[i] * (1 + rate)

def test():
    amounts = [1000, 2200, 800, 360]
    rate = 0.05
    addInterest(amounts, rate)
    print(amounts)
```

Витратьте хвилину на вивчення цієї програми. Функція test починається із встановлення `amounts` як списку чотирьох значень. Потім викликається функція addInterest з передачею `amounts` як першого параметра. Після виклику функції виводиться значення `amounts`. Чого ви очікуєте побачити? Запустимо програму і подивимося, що відбувається:

```python
>>> test()
[1050.0, 2310.0, 840.0, 378.0]
```

Хіба не цікаво? У цьому прикладі здається, що функція змінює значення змінної `amounts`. Але я щойно сказав вам, що Python передає параметри за значенням, тому сама змінна (`amounts`) не може бути змінена функцією. То що ж тут відбувається?

Перші два рядки test створюють змінні `amounts` і `rate`, а потім управління передається у функцію addInterest. Ситуація в цей момент зображена на рисунку 6.8.

Зверніть увагу, що значення змінної `amounts` — це тепер об'єкт-список, який сам містить чотири значення int. Саме цей об'єкт-список передається до addInterest і, отже, також є значенням `balances`.

Далі виконується addInterest. Цикл проходить по кожному індексу в діапазоні 0, 1, ..., довжина−1 і оновлює цей елемент у `balances`. Результат показаний на рисунку 6.9.

Ви помітите на схемі, що я залишив старі значення (1000, 2200, 800, 360) просто висіти в повітрі. Я зробив це, щоб підкреслити, що числа у значеннях не змінилися. Натомість відбулося те, що були створені нові значення, і присвоювання у список змусило його посилатися на нові значення. Старі значення фактично будуть очищені, коли Python виконає збирання сміття.

Тепер повинно бути зрозуміло, чому версія програми addInterest зі списком дає такий результат. Коли addInterest завершується, список, збережений у `amounts`, тепер містить нові баланси, і саме вони виводяться. Важливий момент тут — змінна `amounts` ніколи не змінювалася. Вона все ще посилається на той самий список, що й до виклику addInterest. Те, що відбулося, — це те, що стан цього списку змінився, і ця зміна видима у програмі, що викликала.

Тепер ви дійсно знаєте все про те, як Python передає параметри до функцій. Параметри завжди передаються за значенням. Однак якщо фактичний параметр — це змінна, значенням якої є змінний об'єкт (як список або графічний об'єкт), то зміни стану об'єкта будуть видимі програмі, що викликала. Ця ситуація — ще один приклад проблеми псевдонімування, що обговорювалася в розділі 4.


## 6.7 Функції і структура програми

До цього часу ми обговорювали функції як механізм для зменшення дублювання коду, тим самим скорочуючи і спрощуючи наші програми. Дивовижно, але функції часто використовуються навіть тоді, коли це фактично робить програму довшою. Друга причина використання функцій — зробити програми більш модульними.

Коли алгоритми, які ви проектуєте, стають складнішими, програми дедалі важче розуміти. Люди досить добре справляються з відстеженням восьми-десяти речей одночасно. Зіткнувшись з алгоритмом, що складається з сотень рядків, навіть найкращі програмісти здатимуться.

Один зі способів впоратися з цією складністю — розбити алгоритм на менші підпрограми, кожна з яких має сенс сама по собі. Я ще чимало скажу про це пізніше, коли будемо обговорювати проектування програм у розділі 9. Поки що просто розглянемо приклад. Повернемося до задачі майбутньої вартості ще раз. Ось основна програма, якою ми її залишили:

## 6.8 Підсумок розділу

- Функція — це своєрідна підпрограма. Програмісти використовують функції для зменшення дублювання коду і для структурування або модуляризації програм. Після визначення функція може викликатися кілька разів з різних місць програми. Параметри дозволяють функціям мати змінні частини. Параметри, що з'являються у визначенні функції, називаються формальними параметрами, а вирази, що з'являються у виклику функції, відомі як фактичні параметри.

- Виклик функції ініціює чотирикроковий процес:
  1. Програма, що викликає, призупиняється.
  2. Значення фактичних параметрів присвоюються формальним параметрам.
  3. Виконується тіло функції.
  4. Управління повертається до точки відразу після виклику функції у програмі, що викликала. Значення, що повертається функцією, використовується як результат виразу.

- Область видимості змінної — це область програми, де на неї можна посилатися. Формальні параметри та інші змінні всередині визначень функцій є локальними для функції. Локальні змінні відрізняються від змінних із тим самим іменем, що можуть використовуватися в іншому місці програми.

- Функції можуть повідомляти інформацію назад тому, хто викликав, через значення, що повертаються. У Python функції можуть повертати кілька значень. Функції, що повертають значення, як правило, слід викликати зсередини виразу. Функції, що явно не повертають значення, повертають спеціальний об'єкт `None`.

- Python передає параметри за значенням. Якщо значення, що передається, є змінним об'єктом, то зміни, внесені до об'єкта, можуть бути видимі тому, хто викликав.
