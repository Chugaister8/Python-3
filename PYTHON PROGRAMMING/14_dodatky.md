## Розділ 5: Послідовності: рядки, списки та файли

### Введення (рядки)

```python
<змінна> = input(<запрошення>)
```

### Операції над послідовностями (рядки і списки)

| оператор | значення |
|---|---|
| `<посл> + <посл>` | Повертає конкатенацію послідовностей. Послідовності повинні бути одного типу. |
| `<посл> * <n>` | Повертає послідовність, конкатеновану з собою n разів. n має бути int. |
| `<посл>[<n>]` | Повертає елемент на позиції n (від 0 зліва). n має бути int. |
| `<посл>[<n>]` де n < 0 | Повертає елемент на позиції n (від 1 справа). n має бути int. |
| `len(<посл>)` | Повертає довжину послідовності. |
| `<посл>[<початок>:<кінець>]` | Повертає підпослідовність від початку до (не включаючи) кінця. |
| `for <змінна> in <посл>:` | Ітерує по елементах послідовності. |

### Методи рядків

| функція | значення |
|---|---|
| `s.capitalize()` | Копія s лише з першою літерою у верхньому регістрі. |
| `s.center(width)` | Копія s, вирівняна по центру в полі заданої ширини. |
| `s.count(sub)` | Підрахунок входжень `sub` у s. |
| `s.find(sub)` | Знайти першу позицію входження `sub` у s. |
| `s.join(list)` | Конкатенувати список у рядок, використовуючи s як роздільник. |
| `s.ljust(width)` | Як center, але s вирівняне ліворуч. |
| `s.lower()` | Копія s у нижньому регістрі. |
| `s.lstrip()` | Копія s зі видаленим початковим пробілом. |
| `s.replace(oldsub, newsub)` | Замінити всі входження `oldsub` у s на `newsub`. |
| `s.rfind(sub)` | Як find, але повертає крайню праву позицію. |
| `s.rjust(width)` | Як center, але s вирівняне праворуч. |
| `s.rstrip()` | Копія s зі видаленим кінцевим пробілом. |
| `s.split()` | Розбити s на список підрядків (дивіться текст). |
| `s.title()` | Копія s із першою літерою кожного слова у верхньому регістрі. |
| `s.upper()` | Копія s з усіма символами у верхньому регістрі. |

### Додавання до списку

```python

```

### Функції перетворення типів

| функція | значення |
|---|---|
| `float(<вираз>)` | Перетворює вираз у значення з рухомою комою. |
| `int(<вираз>)` | Перетворює вираз у цілочисельне значення. |
| `str(<вираз>)` | Повертає рядкове представлення виразу. |
| `eval(<рядок>)` | Обчислює рядок як вираз. |

### Форматування рядків

**Синтаксис виразу:**

```python
<рядок-шаблон>.format(<значення0>, <значення1>, <значення2>, ...)
```

**Синтаксис специфікатора:**

```
{<індекс>}
{<індекс>:<ширина>}
{<індекс>:<ширина>.<точність>}
{<індекс>:<ширина>.<знаки>f}
```

**Примітки:**
- Остання форма — для фіксованої кількості десяткових знаків.
- Ширина 0 означає використовувати скільки треба місця.
- Ширина з провідним 0 означає доповнювати нулями (пробіл за замовчуванням).
- Перед шириною може стояти `<` для вирівнювання ліворуч, `>` — праворуч, `^` — по центру.

### Обробка файлів

**Відкриття і закриття файлів:**

```python
<змінна> = open(<ім'я>, <режим>)
# Режим: "r" — читання, "w" — запис, "a" — додавання

<файловий-об'єкт>.close()
```

**Читання файлу:**

`<файл>.read()` — Повертає весь залишок вмісту файлу як один (потенційно великий, багаторядковий) рядок.

`<файл>.readline()` — Повертає наступний рядок файлу (весь текст до наступного символу нового рядка включно).

`<файл>.readlines()` — Повертає список рядків, що залишилися у файлі. Кожен елемент — один рядок із символом нового рядка наприкінці.

*Примітка: файловий об'єкт також може використовуватися в циклі `for`, де він трактується як послідовність рядків.*

**Запис у файл:**

```python
print(..., file=<вихідний-файл>)
```

# Додаток B. Глосарій

**абстракція** (abstraction) — Навмисне приховування або ігнорування певних деталей для того, щоб зосередитися на тих, що є суттєвими.

**аксесор** (accessor method) — Метод, що повертає значення однієї або кількох змінних примірника об'єкта, але не змінює об'єкт.

**шаблон акумулятора** (accumulator pattern) — Поширений шаблон програмування, в якому кінцева відповідь будується по частинах у циклі.

**змінна-акумулятор** (accumulator variable) — Змінна, що використовується для зберігання результату в шаблоні програмування з акумулятором.

**фактичний параметр** (actual parameter) — Значення, що передається функції під час виклику.

**алгоритм** (algorithm) — Детальна послідовність кроків для виконання деякого процесу. Рецепт.

**псевдонімування** (aliasing) — Ситуація, коли дві або більше змінних посилаються на один і той самий об'єкт. Якщо об'єкт є змінним, то зміни, внесені через одну змінну, будуть видимі для інших.

**аналіз** (analysis) — 1) У контексті циклу розробки програмного забезпечення — процес вивчення задачі і визначення того, що комп'ютерна програма могла б зробити для її вирішення. 2) Математичне вивчення задачі або алгоритму для визначення деяких його властивостей, наприклад часової ефективності.

**and** — Бінарний булевий оператор, що повертає true, коли обидва підвирази є істинними.

**API** (application programming interface, інтерфейс прикладного програмування) — Специфікація функціональності, яку надає бібліотечний модуль. Програміст повинен розуміти API, щоб мати можливість використовувати модуль.

**аргумент** (argument) — Фактичний параметр.

**масив** (array) — Колекція схожих об'єктів, до яких можна звертатися через індексування. Зазвичай масиви мають фіксований розмір і є однорідними (всі елементи одного типу). Порівняйте зі списком.

**ASCII** (American Standard Code for Information Interchange, американський стандартний код обміну інформацією) — Стандарт для кодування тексту, де кожен символ представлений числом від 0 до 127.

**присвоювання** (assignment) — Процес надання значення змінній.

**асоціативний масив** (associative array) — Колекція, де значення асоційовані з ключами. У Python називається словником.

**атрибути** (attributes) — Змінні примірника і методи об'єкта.

**базовий випадок** (base case) — В рекурсивній функції або визначенні — ситуація, коли рекурсія не потрібна. Всі правильні рекурсії повинні мати один або більше базових випадків.

**пакетна обробка** (batch) — Режим обробки, в якому введення і виведення виконуються через файли, а не інтерактивно.

**двійкова система числення** (binary) — Система числення з основою 2, де єдиними цифрами є 0 і 1.

**бінарний пошук** (binary search) — Дуже ефективний алгоритм пошуку елементів у відсортованій колекції. Потребує часу, пропорційного log₂n, де n — розмір колекції.

**біт** (bit) — Двійкова цифра (binary digit). Фундаментальна одиниця інформації. Зазвичай представляється за допомогою 0 і 1.

**тіло** (body) — Загальний термін для блоку операторів усередині управляючої конструкції — циклу або рішення.

**булева алгебра** (Boolean algebra) — Правила, що регулюють спрощення і перетворення булевих виразів.

**булевий вираз** (Boolean expression) — Твердження про істину. Булевий вираз обчислюється або як true, або як false.

**булеві операції** (Boolean operations) — Зв'язки для побудови булевих виразів. У Python — `and`, `or` і `not`.

**помилка** (bug) — Помилка в програмі.

**байт-код** (byte code) — Проміжна форма мови комп'ютера. Мови високого рівня іноді компілюються в байт-код, який потім інтерпретується. У Python файли з розширенням `.pyc` — це байт-код.

**виклик** (call) — Процес звернення до визначення функції.

**центральний процесор** (CPU, Central Processing Unit) — «Мозок» комп'ютера, де виконуються числові та логічні операції.

**клас** (class) — Клас описує набір пов'язаних об'єктів. Механізм класів у Python використовується як «фабрика» для виробництва об'єктів.

**клієнт** (client) — У програмуванні модуль, що взаємодіє з іншим компонентом, називається клієнтом для цього компонента.

**ін'єкція коду** (code injection) — Форма комп'ютерної атаки, коли зловмисний користувач вводить комп'ютерні інструкції у виконувану програму, змушуючи застосунок відхилятися від свого первісного задуму.

**кодування** (coding) — Процес перетворення алгоритму в комп'ютерну програму.

**коментар** (comment) — Текст, що розміщується в програмі для читачів. Коментарі ігноруються комп'ютером.

**компілятор** (compiler) — Складна програма, що перекладає програму, написану мовою високого рівня, на машинну мову, яку може виконати конкретний комп'ютер.

**комп'ютер** (computer) — Машина, що зберігає і маніпулює інформацією під управлінням програми, що допускає зміни.

**комп'ютерна наука** (computer science) — Наука про те, що можна обчислити.

**умовна конструкція** (conditional) — Інша назва для управляючої конструкції рішення.

**конструктор** (constructor) — Функція, що створює новий об'єкт. У класі Python — це метод `__init__`.

**управляючі коди** (control codes) — Спеціальні символи, що не виводяться на екран, але використовуються при обміні інформацією.

**управляюча конструкція** (control structure) — Оператор мови програмування, що управляє виконанням інших операторів (наприклад, `if` і `while`).

**координатне перетворення** (coordinate transformation) — У графічному програмуванні — математика зміни точки або набору точок з однієї системи координат в іншу.

**цикл із лічильником** (counted loop) — Цикл, написаний для ітерації певну кількість разів.

**криптографія** (cryptography) — Вивчення технік для кодування інформації, щоб зберегти її в безпеці.

**дані** (data) — Інформація, якою маніпулює комп'ютерна програма.

**тип даних** (data type) — Конкретний спосіб представлення даних. Тип даних елемента визначає, які значення він може мати і які операції підтримує.

**налагодження** (debugging) — Процес знаходження і усунення помилок у програмі.

**структура рішення** (decision structure) — Управляюча конструкція, що дозволяє різним частинам програми виконуватися в залежності від конкретної ситуації.

**дерево рішень** (decision tree) — Складна структура рішень, де початкове рішення розгалужується на більше рішень.

**визначений цикл** (definite loop) — Вид циклу, де кількість ітерацій відома на момент початку виконання циклу.

**проектування** (design) — Процес розробки системи, що може вирішити деяку задачу. Також продукт цього процесу.

**словник** (dictionary) — Невпорядкований об'єкт-колекція Python, що дозволяє асоціювати значення з довільними ключами.

**рядок документації** (docstring) — Техніка документування в Python, яка пов'язує рядок із компонентом програми.

**порожній рядок** (empty string) — Об'єкт типу даних string, що не містить жодного символу (`""`).

**інкапсуляція** (encapsulation) — Приховування деталей чогось. Зазвичай — розмежування між реалізацією і використанням об'єкта або функції.

**шифрування** (encryption) — Процес кодування інформації для збереження її конфіденційності.

**цикл до кінця файлу** (end-of-file loop) — Шаблон програмування, що використовується для рядкового читання файлу.

**подія** (event) — У GUI-програмуванні — зовнішня дія, як-от клік миші, що спричиняє виконання чогось у програмі.

**керований подіями стиль** (event-driven) — Стиль програмування, в якому програма чекає подій і реагує на них. Часто використовується в GUI-програмуванні.

**обробка винятків** (exception handling) — Механізм мови програмування, що дозволяє програмісту граційно обробляти помилки, виявлені під час виконання програми.

**виконати** (execute) — Запустити програму або фрагмент програми.

**алгоритм експоненційного часу** (exponential time) — Алгоритм, що потребує кількості кроків, пропорційної функції, в якій розмір задачі з'являється в показнику степеня. Такі алгоритми, як правило, вважаються нерозв'язними на практиці.

**вираз** (expression) — Частина програми, що виробляє дані.

**цикл завантаження-виконання** (fetch-execute cycle) — Процес, який комп'ютер виконує для запуску програми машинного коду.

**float** — Тип даних для представлення чисел із дробовою частиною. Скорочення від «floating point» (числа з рухомою комою).

**блок-схема** (flowchart) — Графічне зображення потоку управління в програмі або алгоритмі.

**функція** (function) — Підпрограма всередині програми. Функції приймають параметри як введення і можуть повертати значення.

**функціональна декомпозиція** (functional decomposition) — Дивіться низхідне проектування.

**збирання сміття** (garbage collection) — Процес, виконуваний динамічними мовами програмування (Python, Lisp, Java), в якому місця в пам'яті, що містять більше непотрібні значення, звільняються для зберігання нових.

**графічний інтерфейс користувача** (GUI, graphical user interface) — Стиль взаємодії з комп'ютерним застосунком, що передбачає широке використання графічних компонентів: вікон, меню і кнопок.

**графічне вікно** (graphics window) — Вікно на екрані, де можна малювати графіку.

**проблема зупинки** (halting problem) — Знаменита нерозв'язна задача. Програма, що визначає, чи зупиниться інша програма на заданому введенні.

**апаратне забезпечення** (hardware) — Фізичні компоненти обчислювальної системи.

**хеш** (hash) — Інша назва для асоціативного масиву або словника.

**неоднорідний** (heterogeneous) — Здатний містити більше одного типу даних одночасно. Наприклад, списки Python.

**однорідний** (homogeneous) — Здатний містити значення лише одного типу.

**ідентифікатори** (identifiers) — Імена, що надаються програмним сутностям.

**оператор if** (if statement) — Управляюча конструкція для реалізації рішень у програмі.

**оператор import** (import statement) — Оператор, що робить зовнішній бібліотечний модуль доступним для використання в програмі.

**невизначений цикл** (indefinite loop) — Цикл, для якого кількість потрібних ітерацій не обов'язково відома на момент початку виконання циклу.

**індексування** (indexing) — Вибір одного елемента з послідовності на основі його відносної позиції.

**нескінченний цикл** (infinite loop) — Цикл, що не завершується.

**успадкування** (inheritance) — Визначення нового класу як спеціалізації іншого класу.

**введення, обробка, виведення** (input, process, output) — Поширений шаблон програмування. Програма запрошує введення, обробляє його і видає відповідь.

**перевірка введення** (input validation) — Процес перевірки значень, наданих користувачем, щоб переконатися в їхній коректності перед виконанням обчислень.

**примірник** (instance) — Конкретний об'єкт деякого класу.

**змінна примірника** (instance variable) — Частина даних, збережена всередині об'єкта.

**int** — Тип даних для представлення чисел без дробової частини. Скорочення від «integer» (ціле число).

**ціле число** (integer) — Позитивне або негативне ціле число. Дивіться int.

**інтерактивний цикл** (interactive loop) — Цикл, що дозволяє частині програми повторюватися за бажанням користувача.

**інтерфейс** (interface) — З'єднання між двома компонентами. Для функції або методу — це ім'я, параметри і значення, що повертаються. Для об'єкта — набір методів (та їхніх інтерфейсів) для маніпулювання ним.

**інтерпретатор** (interpreter) — Комп'ютерна програма, що симулює поведінку комп'ютера, який «розуміє» мову високого рівня.

**нерозв'язна на практиці задача** (intractable) — Задача, яку надто складно вирішити на практиці, зазвичай через надто тривалий час виконання.

**викликати** (invoke) — Скористатися функцією.

**ітерувати** (iterate) — Виконувати кілька разів. Кожне виконання тіла циклу називається ітерацією.

**ключ** (key) — 1) У шифруванні — спеціальне значення, необхідне для кодування або декодування повідомлення. 2) У контексті колекцій даних — спосіб знаходження значення в словнику.

**лексикографічний порядок** (lexicographic) — Порядок рядків, подібний до алфавітного, але заснований на базових числових кодах символів рядка.

**бібліотека** (library) — Зовнішня колекція корисних функцій або класів, яку можна імпортувати і використовувати в програмі.

**лінійний пошук** (linear search) — Процес пошуку, що послідовно переглядає елементи в колекції.

**алгоритм лінійного часу** (linear time algorithm) — Алгоритм, що потребує кількості кроків, пропорційної розміру вхідних даних.

**список** (list) — Загальний тип даних Python для представлення послідовних колекцій. Списки є неоднорідними і можуть зростати і скорочуватися за потреби.

**літерал** (literal) — Позначення для запису конкретного значення в мові програмування. Наприклад, 3 — це літерал int, а «Hello» — літерал рядка.

**локальна змінна** (local variable) — Змінна, визначена всередині функції. На неї можна посилатися лише всередині визначення функції. Дивіться область видимості.

**алгоритм логарифмічного часу** (log time algorithm) — Алгоритм, що потребує кількості кроків, пропорційної логарифму розміру вхідних даних.

**цикл з виходом посередині** (loop and a half) — Структура циклу, що має вихід десь посередині тіла циклу. У Python це реалізується комбінацією `while True:` / `break`.

**цикл** (loop) — Управляюча конструкція для виконання фрагментів програми кілька разів.

**індексна змінна циклу** (loop index) — Змінна, що використовується для управління циклом. В операторі `for i in range(n)` `i` використовується як індексна змінна.

**машинний код** (machine code) — Програма машинною мовою.

**машинна мова** (machine language) — Низькорівневі (двійкові) інструкції, які може виконати конкретний CPU.

**оперативна пам'ять** (main memory) — Місце зберігання всіх даних і програмних інструкцій, з якими CPU зараз працює.

**відображення** (mapping) — Загальна асоціація між ключами і значеннями. Словники Python реалізують відображення.

**злиття** (merge) — Процес об'єднання двох відсортованих списків в один відсортований список.

**сортування злиттям** (merge sort) — Ефективний алгоритм сортування «розділяй і владарюй».

**метод** (method) — Функція, що живе всередині об'єкта. Об'єктами маніпулюють, викликаючи їхні методи.

**вираз зі змішаними типами** (mixed-typed expression) — Вираз, що включає більше одного типу даних. Зазвичай використовується в контексті комбінування int і float у числових обчисленнях.

**архітектура модель-представлення** (model-view architecture) — Розподіл GUI-програми шляхом відокремлення задачі (моделі) від інтерфейсу користувача (представлення).

**модальний** (modal) — Вікно або діалог є модальними, якщо вони вимагають від користувача певної взаємодії перед продовженням роботи із застосунком.

**модульний** (modular) — Що складається з кількох відносно незалежних частин, які працюють разом.

**модуль** (module) — Загалом будь-яка відносно незалежна частина програми. У Python — також файл із кодом, який можна імпортувати і виконувати.

**схема ієрархії модулів** (module hierarchy chart) — Діаграма, що показує структуру функціональної декомпозиції програми.

**Монте-Карло** (Monte Carlo) — Техніка симуляції, що включає ймовірнісні (випадкові або псевдовипадкові) елементи.

**змінний** (mutable) — Такий, що допускає зміни. Об'єкт, стан якого може змінюватися.

**мутатор** (mutator method) — Метод, що змінює стан об'єкта (тобто модифікує одну або більше змінних примірника).

**алгоритм O(n log n)** (n log n algorithm) — Алгоритм, що потребує кількості кроків, пропорційної розміру вхідних даних, помноженому на логарифм розміру вхідних даних.

**квадратичний алгоритм** (n-squared algorithm) — Алгоритм, що потребує кількості кроків, пропорційної квадрату розміру вхідних даних.

**помилка імені** (name error) — Виняток, що виникає, коли Python просять отримати значення змінної, якій не присвоєно значення.

**простір імен** (namespace) — Асоціація між ідентифікаторами і тим, що вони представляють у програмі.

**вкладання** (nesting) — Процес розміщення однієї управляючої конструкції всередині іншої. Цикли і рішення можуть довільно вкладатися.

**символ нового рядка** (newline) — Спеціальний символ, що позначає розрив між рядками у файлі або багаторядковому рядку. У Python позначається `"\n"`.

**not** — Унарний булевий оператор для заперечення виразу.

**об'єкт** (object) — Програмна сутність, що має деякі дані і набір операцій для маніпулювання ними.

**об'єктно-базований** (object-based) — Проектування і програмування, що використовує об'єкти як основну форму абстракції.

**об'єктно-орієнтований** (object-oriented) — Об'єктно-базоване проектування або програмування, що включає характеристики поліморфізму і успадкування.

**відкриття файлу** (open) — Процес асоціювання файлу у вторинній пам'яті зі змінною в програмі, через яку файлом можна маніпулювати.

**оператор** (operator) — Функція для об'єднання виразів у більш складні вирази.

**or** — Бінарний булевий оператор, що повертає true, коли хоча б один із підвиразів є істинним.

**перевизначення** (override) — Термін, що застосовується в ситуації, коли підклас змінює поведінку успадкованого методу.

**параметри** (parameters) — Спеціальні змінні у функції, що ініціалізуються під час виклику інформацією, переданою від того, хто викликає.

**передача за значенням** (pass by value) — Техніка передачі параметрів, що використовується в Python. Формальним параметрам присвоюються значення фактичних параметрів. Функція не може змінювати, на який об'єкт посилається змінна-фактичний параметр.

**передача за посиланням** (pass by reference) — Техніка передачі параметрів, що використовується в деяких мовах програмування і дозволяє значенню змінної, що використовується як фактичний параметр, бути зміненим викликуваною функцією.

**піксель** (pixel) — Скорочення від «picture element». Одна крапка на графічному дисплеї.

**відкритий текст** (plaintext) — У шифруванні — незакодоване повідомлення.

**поліморфізм** (polymorphism) — Буквально «багато форм». В ООП — здатність конкретного рядка коду реалізовуватися різними методами залежно від типу даних задіяного об'єкта.

**переносимість** (portability) — Здатність запускати програму без змін на різних системах.

**цикл із кінцевою перевіркою** (post-test loop) — Циклічна конструкція, де умова циклу не перевіряється до виконання тіла циклу.

**цикл із попередньою перевіркою** (pre-test loop) — Циклічна конструкція, де умова циклу перевіряється перед виконанням тіла циклу.

**точність** (precision) — Кількість цифр точності в числі.

**завантажувальне читання** (priming read) — У сторожовому циклі — читання до перевірки умови циклу.

**програма** (program) — Детальний набір інструкцій для комп'ютера.

**програмування** (programming) — Процес створення комп'ютерної програми для вирішення деякої задачі.

**середовище програмування** (programming environment) — Спеціальна комп'ютерна програма, що надає засоби для полегшення програмування. IDLE (у стандартній дистрибуції Python) — приклад простого середовища.

**мова програмування** (programming language) — Нотація для написання комп'ютерних програм.

**запрошення** (prompt) — Виведене повідомлення, що сигналізує користувачу програми про очікування введення.

**прототип** (prototype) — Початкова спрощена версія програми.

**псевдокод** (pseudocode) — Нотація для запису алгоритмів із використанням точної природної мови замість мови комп'ютера.

**псевдовипадкові числа** (pseudo-random) — Послідовності чисел, генеровані комп'ютерними алгоритмами і використовувані для симуляції випадкових подій.

**відкритий ключ** (public key) — Форма шифрування, що використовує два різних ключі. Повідомлення, закодоване відкритим ключем, може бути декодовано лише окремим закритим ключем.

**оперативна пам'ять** (RAM, random access memory) — Дивіться оперативна пам'ять.

**випадкове блукання** (random walk) — Процес симуляції, в якому рух деякого об'єкта визначається ймовірнісно.

**читання** (read) — Термін для позначення комп'ютерного введення. Програма читає інформацію з клавіатури або файлу.

**запис** (record) — Колекція інформації про окрему особу або об'єкт.

**рекурсивний** (recursive) — Що має властивість (для функції або визначення) посилатися на себе. Дивіться рекурсивний. (Це самопосилальне визначення — навмисний жарт автора.)

**рекурсивна функція** (recursive function) — Функція, що викликає себе, прямо або опосередковано.

**реляційний оператор** (relational operator) — Оператор, що порівнює значення і повертає true або false (наприклад, `<`, `<=`, `==`, `>=`, `>`, `!=`).

**зарезервовані слова** (reserved words) — Ідентифікатори, що є частиною вбудованого синтаксису мови.

**роздільна здатність** (resolution) — Кількість пікселів на графічному екрані. Зазвичай виражається як горизонталь на вертикаль (наприклад, 640×480).

**значення RGB** (RGB value) — Представлення кольору трьома числами (як правило, в діапазоні 0–255), що вказують яскравість червоного, зеленого і синього компонентів пікселя.

**область видимості** (scope) — Область програми, де можна посилатися на задану змінну. Змінні, визначені у функціях, мають локальну область видимості.

**скрипт** (script) — Інша назва для програми. Зазвичай використовується для відносно простих програм.

**пошук** (search) — Процес знаходження конкретного елемента в колекції.

**вторинна пам'ять** (secondary memory) — Загальний термін для енергонезалежних пристроїв зберігання: жорстких дисків, компакт-дисків тощо.

**початкове значення** (seed) — Значення для запуску генерації псевдовипадкової послідовності.

**сортування вибором** (selection sort) — Алгоритм сортування з квадратичним часом.

**параметр self** (self parameter) — У Python — перший параметр методу. Посилання на об'єкт, до якого застосовується метод.

**семантика** (semantics) — Значення конструкції.

**сторож** (sentinel) — Спеціальне значення, що сигналізує про кінець серії введень.

**сторожовий цикл** (sentinel loop) — Цикл, що продовжується, поки не зустрінеться спеціальне значення.

**обчислення з коротким замиканням** (short-circuit evaluation) — Процес обчислення, що повертає відповідь, щойно результат стає відомим, не обчислюючи всіх підвиразів.

**сигнатура** (signature) — Інша назва для інтерфейсу функції. Сигнатура включає ім'я, параметри і значення, що повертаються.

**симуляція** (simulation) — Програма, призначена для абстрактного імітування деякого реального процесу.

**одночасне присвоювання** (simultaneous assignment) — Оператор, що дозволяє присвоювати кілька змінних за один крок. Наприклад, `x, y = y, x` — обмін змінними.

**зрізування** (slicing) — Виймання підпослідовності рядка, списку або іншого об'єкта-послідовності.

**програмне забезпечення** (software) — Комп'ютерні програми.

**сортування** (sorting) — Процес розміщення послідовності елементів у заданому порядку.

**вихідний код** (source code) — Текст програми мовою високого рівня.

**спіральна розробка** (spiral design) — Створення системи шляхом спочатку проектування спрощеного прототипу з поступовим додаванням функцій.

**оператор** (statement) — Одна команда мови програмування.

**поетапне уточнення** (step-wise refinement) — Процес проектування системи, що починається з дуже високорівневого, абстрактного опису і поступово додає деталі.

**рядок** (string) — Тип даних для представлення послідовності символів (тексту).

**структурна схема** (structure chart) — Дивіться схема ієрархії модулів.

**підклас** (subclass) — Коли один клас успадковує від іншого, клас, що успадковує, називається підкласом.

**підрядок** (substring) — Послідовність сусідніх символів усередині рядка. Дивіться зрізування.

**суперклас** (superclass) — Клас, від якого виконується успадкування.

**синтаксис** (syntax) — Форма мови.

**tkinter** — Стандартний GUI-фреймворк, що постачається з Python. Модуль `graphics.py`, що використовується в цій книзі, побудований на його основі.

**низхідне проектування** (top-down design) — Процес побудови системи, що починається з дуже високорівневого алгоритму, що описує рішення через підпрограми. Кожна підпрограма потім проектується по черзі. Інші назви — поетапне уточнення і функціональна декомпозиція.

**таблиця істинності** (truth table) — Таблиця, що показує значення булевого виразу для всіх можливих комбінацій значень його підвиразів.

**кортеж** (tuple) — Тип послідовності Python, що поводиться як незмінний список.

**унарний** (unary) — Оператор, що діє на один операнд.

**Unicode** — Альтернатива ASCII, що кодує символи практично всіх писемностей світу. Unicode розроблений для сумісності з ASCII.

**модульне тестування** (unit testing) — Перевірка компонента програми незалежно від інших частин.

**розпакування** (unpack) — У Python — присвоювання елементів послідовності незалежним змінним. Наприклад, список або кортеж із двох значень можна розпакувати у змінні: `x, y = myList`.

**змінна** (variable) — Ідентифікатор, що позначає значення для подальшого використання. Значення змінної може бути змінено через присвоювання.

**віджет** (widget) — Компонент інтерфейсу користувача в GUI.

**запис** (write) — Процес виведення інформації. Наприклад, дані записуються у файл.
