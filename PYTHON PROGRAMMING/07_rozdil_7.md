## 7.1 Прості рішення

До цього часу ми переважно розглядали комп'ютерні програми як послідовності інструкцій, що виконуються одна за одною. Послідовність — фундаментальна концепція програмування, але сама по собі вона недостатня для розв'язання будь-якої задачі. Нерідко необхідно змінювати послідовний потік програми відповідно до потреб конкретної ситуації. Це виконується за допомогою спеціальних операторів, відомих як управляючі конструкції. У цьому розділі ми розглянемо структури розгалуження — оператори, що дозволяють програмі виконувати різні послідовності інструкцій для різних випадків, фактично дозволяючи програмі «вибирати» відповідний курс дій.


### 7.1.1 Приклад: попередження про температуру

Почнемо з того, щоб змусити комп'ютер прийняти просте рішення. Для простого прикладу повернемося до програми перетворення температур із Цельсія у Фаренгейти з розділу 2. Пам'ятайте, її написала Сьюзен Комп'ютервелл, щоб допомогти собі визначати, як одягатися щоранку в Європі. Ось програма, якою ми її залишили:

```python
# convert.py
#     Програма для перетворення температур Цельсія у Фаренгейти
# Автор: Сьюзен Комп'ютервелл

def main():
    celsius = float(input("Яка температура в градусах Цельсія? "))
    fahrenheit = 9/5 * celsius + 32
    print("Температура становить", fahrenheit, "градусів Фаренгейта.")

main()
```

Це непогана програма, але ми хочемо її покращити. Сьюзен Комп'ютервелл не дуже рання пташка, і хоча вона має програму для перетворення температур, іноді вона не звертає особливої уваги на результати. Наше покращення програми забезпечить, щоб у разі екстремальних температур програма виводила відповідне попередження, щоб Сьюзен зверталася увагу.

Перший крок — повністю задати покращення. Екстремальна температура — це або дуже жарко, або дуже холодно. Скажімо, будь-яка температура вище 90 градусів Фаренгейта заслуговує попередження про спеку, а температура нижче 30 градусів — попередження про холод. Маючи цю специфікацію, ми можемо спроектувати розширений алгоритм:

```
Ввести температуру в градусах Цельсія (назвати її celsius)
Обчислити fahrenheit як 9/5 celsius + 32
Вивести fahrenheit
якщо fahrenheit > 90
    вивести попередження про спеку
якщо fahrenheit < 30
    вивести попередження про холод
```

У цьому новому проекті є два прості рішення наприкінці. Відступ вказує, що крок слід виконувати лише в тому разі, якщо виконується умова, перелічена в попередньому рядку. Ідея тут у тому, що рішення вводить альтернативний потік управління через програму. Точний набір кроків, що виконуються алгоритмом, залежатиме від значення `fahrenheit`.

На рисунку 7.1 показана блок-схема можливих шляхів через алгоритм. Ромбоподібні блоки показують умовні рішення. Якщо умова хибна, управління переходить до наступного оператора в послідовності (того, що нижче). Якщо ж умова виконується, управління передається до інструкцій у блоці праворуч. Після завершення цих інструкцій управління переходить до наступного оператора.

Ось як новий проект перекладається у код Python:

```python
# convert2.py
#     Програма для перетворення температур Цельсія у Фаренгейти.
#     Ця версія виводить попередження про спеку та холод.

def main():
    celsius = float(input("Яка температура в градусах Цельсія? "))
    fahrenheit = 9/5 * celsius + 32
    print("Температура становить", fahrenheit, "градусів Фаренгейта.")

    # Вивести попередження для екстремальних температур
    if fahrenheit > 90:
        print("Там дійсно дуже спекотно. Будьте обережні!")
    if fahrenheit < 30:
        print("Бррр. Обов'язково одягніться тепло!")

main()
```

Ви можете бачити, що оператор Python `if` використовується для реалізації рішення. Форма `if` дуже схожа на псевдокод в алгоритмі:

```
if <умова>:
    <тіло>
```

Тіло — це просто послідовність одного або більше операторів з відступом під заголовком `if`. У `convert2.py` є два оператори `if`, обидва з одним оператором у тілі.

Семантика `if` має бути зрозумілою з наведеного прикладу. Спочатку обчислюється умова в заголовку. Якщо умова є істинною, виконується послідовність операторів у тілі, а потім управління переходить до наступного оператора у програмі. Якщо умова хибна, оператори в тілі пропускаються. На рисунку 7.2 показана семантика `if` у вигляді блок-схеми. Зверніть увагу, що тіло `if` або виконується, або ні залежно від умови. У будь-якому разі управління потім переходить до наступного оператора після `if`. Це одностороннє, або просте, рішення.


### 7.1.2 Утворення простих умов

Один момент, що ще не обговорювався, — це те, як виглядає умова. Поки що наші програми використовуватимуть прості умови, що порівнюють значення двох виразів: `<вираз> <реляційний-оператор> <вираз>`. Реляційний оператор — це лише вишуканий термін для математичних концепцій типу «менше ніж» або «дорівнює». У Python є шість реляційних операторів:

| Python | математика | значення |
|--------|-----------|----------|
| `<` | < | менше ніж |
| `<=` | ≤ | менше або дорівнює |
| `==` | = | дорівнює |
| `>=` | ≥ | більше або дорівнює |
| `>` | > | більше ніж |
| `!=` | ≠ | не дорівнює |

Зверніть особливу увагу на використання `==` для рівності. Оскільки Python використовує знак `=` для позначення оператора присвоювання, для поняття рівності потрібен інший символ. Поширена помилка в програмах Python — використання `=` в умовах, де потрібен `==`.

Умови можуть порівнювати як числа, так і рядки. При порівнянні рядків впорядкування є лексикографічним. По суті, це означає, що рядки впорядковуються за алфавітним порядком відповідно до базових значень Unicode. Тому всі великі латинські літери передують відповідним малим (наприклад, «Bbbb» передує «aaaa», оскільки «B» передує «a»).

Слід зазначити, що умови є, по суті, різновидом виразу, що називається булевим виразом (на честь Джорджа Буля, англійського математика XIX ст.). Коли булевий вираз обчислюється, він виробляє значення або «істина» (умова виконується), або «хиба» (вона не виконується). У деяких мовах, таких як C++ і старіші версії Python, для представлення цих значень просто використовуються цілі числа 1 і 0. Інші мови, як-от Java і сучасний Python, мають спеціальний тип даних для булевих виразів.

У Python булеві вирази мають тип `bool`, а булеві значення «істина» і «хиба» представляються літералами `True` і `False`. Ось кілька інтерактивних прикладів:

```python
>>> 3 < 4
True
>>> 3 * 4 < 3 + 4
False
>>> "hello" == "hello"
True
>>> "hello" < "hello"
False
>>> "Hello" < "hello"
True
```


### 7.1.3 Приклад: умовне виконання програми

Ще в розділі 1 я згадував, що є кілька різних способів запуску програм Python. Деякі файли модулів Python призначені для безпосереднього запуску. Їх зазвичай називають «програмами» або «скриптами». Інші модулі Python призначені переважно для імпортування та використання іншими програмами; їх часто називають «бібліотеками». Іноді нам хочеться створити свого роду гібридний модуль, який можна використовувати і як самостійну програму, і як бібліотеку, що імпортується іншими програмами.

До цього часу більшість наших програм мали рядок внизу для виклику функції main:

```python
main()
```

Як ви знаєте, саме це запускає програму. Ці програми підходять для безпосереднього запуску. У віконному середовищі ви можете запустити файл, (двічі) клацнувши на його значку. Або ввести команду типу `python <мій_файл>.py`.

Оскільки Python обчислює рядки модуля в процесі імпортування, наші поточні програми також запускаються, коли їх імпортують в інтерактивний сеанс Python або в іншу програму Python. Загалом краще, щоб модулі не запускалися при імпортуванні. При інтерактивному тестуванні програми звичайний підхід — спочатку імпортувати модуль, а потім щоразу викликати main (або якусь іншу функцію), коли хочемо запустити його.

У програмі, що призначена або для імпортування (без запуску), або для безпосереднього запуску, виклик main в кінці повинен бути умовним. Просте рішення впорається із цим:

```python
if <умова>:
    main()
```

Нам просто потрібно з'ясувати відповідну умову.

Щоразу, коли модуль імпортується, Python створює всередині цього модуля спеціальну змінну `__name__` і присвоює їй рядок, що представляє ім'я модуля. Ось приклад взаємодії, що показує, що відбувається з бібліотекою math:

```python
>>> import math
>>> math.__name__
'math'
```

Ви можете бачити, що при імпортуванні змінній `__name__` всередині модуля math присвоюється рядок `'math'`.

Однак коли код Python виконується безпосередньо (не імпортується), Python встановлює значення `__name__` рівним `'__main__'`. Щоб побачити це в дії, просто запустіть оболонку Python і подивіться на значення:

```python
>>> __name__
'__main__'
```

Таким чином, якщо модуль імпортується, код всередині цього модуля побачить змінну `__name__`, значенням якої є ім'я модуля. Коли файл запускається безпосередньо, код побачить, що `__name__` має значення `'__main__'`. Модуль може визначити спосіб свого використання, перевіривши цю змінну.

Склавши все разом, ми можемо змінити кінцеві рядки наших програм так:

```python
if __name__ == '__main__':
    main()
```

Це гарантує, що main автоматично запускатиметься, коли програма запускається безпосередньо, але не запускатиметься, якщо модуль імпортується. Рядок коду, подібний до цього, ви побачите в кінці практично кожної програми Python.


## 7.2 Двосторонні рішення

Тепер, маючи спосіб вибірково виконувати певні оператори в програмі за допомогою рішень, час повернутися і привести до ладу розв'язувач квадратних рівнянь із розділу 3. Ось програма, якою ми її залишили:

```python
# quadratic.py
#     Програма, що обчислює дійсні корені квадратного рівняння.
#     Примітка: ця програма аварійно завершується, якщо рівняння не має дійсних коренів.

import math

def main():
    print("Ця програма знаходить дійсні розв'язки квадратного рівняння\n")

    a = float(input("Введіть коефіцієнт a: "))
    b = float(input("Введіть коефіцієнт b: "))
    c = float(input("Введіть коефіцієнт c: "))

    discRoot = math.sqrt(b * b - 4 * a * c)
    root1 = (-b + discRoot) / (2 * a)
    root2 = (-b - discRoot) / (2 * a)

    print("\nРозв'язки:", root1, root2)

main()
```

Як зазначено в коментарях, ця програма аварійно завершується, коли їй задаються коефіцієнти квадратного рівняння, що не має дійсних коренів. Проблема в тому, що коли b² − 4ac менше 0, програма намагається обчислити квадратний корінь від'ємного числа. Оскільки від'ємні числа не мають дійсних коренів, бібліотека math повідомляє про помилку. Ось приклад:

```
>>> main()
Ця програма знаходить дійсні розв'язки квадратного рівняння

Введіть коефіцієнт a: 1
Введіть коефіцієнт b: 2
Введіть коефіцієнт c: 3
Traceback (most recent call last):
  File "quadratic.py", line 23, in <module>
    main()
  File "quadratic.py", line 16, in main
    discRoot = math.sqrt(b * b - 4 * a * c)
ValueError: math domain error
```

Ми можемо використати рішення для перевірки цієї ситуації і переконатися, що програма не може аварійно завершитися. Ось перша спроба:

```python
# quadratic2.py
import math

def main():
    print("Ця програма знаходить дійсні розв'язки квадратного рівняння\n")
    a = float(input("Введіть коефіцієнт a: "))
    b = float(input("Введіть коефіцієнт b: "))
    c = float(input("Введіть коефіцієнт c: "))

    discrim = b * b - 4 * a * c
    if discrim >= 0:
        discRoot = math.sqrt(discrim)
        root1 = (-b + discRoot) / (2 * a)
        root2 = (-b - discRoot) / (2 * a)
        print("\nРозв'язки:", root1, root2)

main()
```

Ця версія спочатку обчислює значення дискримінанта (b² − 4ac) і потім перевіряє, що він не є від'ємним. Лише після цього програма обчислює квадратний корінь і знаходить розв'язки. Ця програма ніколи не намагатиметься викликати `math.sqrt`, коли `discrim` від'ємний.

На жаль, ця оновлена версія є не зовсім повним розв'язанням. Чи бачите ви, що відбувається, коли рівняння не має дійсних коренів? Відповідно до семантики простого `if`, коли `b*b - 4*a*c` менше нуля, програма просто пропустить обчислення і перейде до наступного оператора. Оскільки наступного оператора немає, програма просто завершується. Ось приклад інтерактивного сеансу:

```
>>> main()
Ця програма знаходить дійсні розв'язки квадратного рівняння

Введіть коефіцієнт a: 1
Введіть коефіцієнт b: 2
Введіть коефіцієнт c: 3
>>>
```

Це майже гірше, ніж попередня версія, тому що вона не дає користувачам жодного пояснення про те, що пішло не так; вона просто залишає їх ні з чим. Краща програма виводила б повідомлення, що повідомляє користувачам: їхнє конкретне рівняння не має дійсних розв'язків. Ми могли б зробити це, додавши ще одне просте рішення в кінець програми:

```python
if discrim < 0:
    print("Рівняння не має дійсних коренів!")
```

Це, звичайно, вирішить нашу проблему, але таке розв'язання просто не відчувається правильним. Ми запрограмували послідовність двох рішень, але два результати є взаємовиключними. Якщо `discrim >= 0` є істинним, то `discrim < 0` має бути хибним, і навпаки. У нас дві умови в програмі, але є насправді лише одне рішення для прийняття. Виходячи зі значення `discrim`, програма повинна або вивести, що дійсних коренів немає, або обчислити і відобразити корені. Це приклад двостороннього рішення. На рисунку 7.3 ілюструється ситуація.

У Python двостороннє рішення може бути реалізоване шляхом додавання до оператора `if` гілки `else`. Результат називається оператором `if-else`:

```
if <умова>:
    <оператори>
else:
    <оператори>
```

Коли інтерпретатор Python зустрічає цю структуру, він спочатку обчислює умову. Якщо умова істинна, виконуються оператори під `if`. Якщо умова хибна, виконуються оператори під `else`. У будь-якому разі управління потім переходить до оператора, що слідує після `if-else`.

Використання двостороннього рішення в розв'язувачі квадратних рівнянь дає більш елегантне розв'язання:

```python
# quadratic3.py
import math

def main():
    print("Ця програма знаходить дійсні розв'язки квадратного рівняння\n")

    a = float(input("Введіть коефіцієнт a: "))
    b = float(input("Введіть коефіцієнт b: "))
    c = float(input("Введіть коефіцієнт c: "))

    discrim = b * b - 4 * a * c
    if discrim < 0:
        print("\nРівняння не має дійсних коренів!")
    else:
        discRoot = math.sqrt(b * b - 4 * a * c)
        root1 = (-b + discRoot) / (2 * a)
        root2 = (-b - discRoot) / (2 * a)
        print("\nРозв'язки:", root1, root2)

main()
```

Ця програма цілком відповідає вимогам. Ось зразок сеансу, що запускає нову програму двічі:

```
>>> main()

Введіть коефіцієнт a: 1
Введіть коефіцієнт b: 2
Введіть коефіцієнт c: 3

Рівняння не має дійсних коренів!
>>> main()
Ця програма знаходить дійсні розв'язки квадратного рівняння

Введіть коефіцієнт a: 2
Введіть коефіцієнт b: 4
Введіть коефіцієнт c: 1

Розв'язки: -0.2928932188134524 -1.7071067811865475
```


## 7.3 Багатосторонні рішення

Найновіша версія розв'язувача квадратних рівнянь — безумовно велике покращення, але вона все ще має деякі дивацтва. Ось ще один приклад запуску:

```
>>> main()
Ця програма знаходить дійсні розв'язки квадратного рівняння

Введіть коефіцієнт a: 1
Введіть коефіцієнт b: 2
Введіть коефіцієнт c: 1

Розв'язки: -1.0 -1.0
```

Це технічно правильно; дані коефіцієнти дають рівняння з подвійним коренем у −1. Однак виведення може бути незрозумілим для деяких користувачів. Здається, що програма помилково вивела одне і те саме число двічі. Можливо, програма повинна бути трохи більш інформативною, щоб уникнути плутанини.

Ситуація з подвійним коренем виникає, коли `discrim` рівний рівно 0. У цьому випадку `discRoot` також дорівнює 0, і обидва корені мають значення `-b/(2a)`. Якщо ми хочемо виявити цей особливий випадок, нашій програмі фактично потрібне тристороннє рішення. Ось швидка замальовка проекту:

```
Перевірити значення discrim
    коли < 0: обробити випадок відсутності коренів
    коли = 0: обробити випадок подвійного кореня
    коли > 0: обробити випадок двох різних коренів.
```

Один зі способів закодувати цей алгоритм — використати два оператори `if-else`. Тіло гілки `if` або `else` може містити будь-які допустимі оператори Python, включаючи інші `if` або `if-else`. Розміщення одного складеного оператора всередині іншого називається вкладанням. Ось фрагмент коду, що використовує вкладання для досягнення тристороннього рішення:

```python
if discrim < 0:
    print("Рівняння не має дійсних коренів")
else:
    if discrim == 0:
        root = -b / (2 * a)
        print("Існує подвійний корінь:", root)
    else:
        # Обробити випадок двох коренів
```

Якщо ви уважно простежите цей код, побачите, що є рівно три можливі шляхи. Послідовність визначається значенням `discrim`. На рисунку 7.4 показана блок-схема цього розв'язання.

Знову ж таки, ми маємо робоче розв'язання, але реалізація відчувається не зовсім правильною. Ми вирішили тристороннє рішення, використовуючи два двосторонніх. Отриманий код не відображає справжнього трибічного рішення вихідної задачі. Уявіть, якби нам потрібно було зробити п'ятистороннє рішення. Структури `if-else` вкладалися б чотири рівні вглибину, і код Python сповзав би за правий край сторінки.

Є інший спосіб записати багатосторонні рішення в Python, що зберігає семантику вкладених структур, але надає їм більш привабливого вигляду. Ідея полягає в тому, щоб об'єднати `else`, що відразу слідує за `if`, у єдину гілку, що називається `elif` (вимовляється «ель-іф»):

```
if <умова1>:
    <оператори випадку 1>
elif <умова2>:
    <оператори випадку 2>
elif <умова3>:
    <оператори випадку 3>
...
else:
    <оператори за замовчуванням>
```

Ця форма використовується для виокремлення будь-якої кількості блоків коду, що взаємно виключають один одного. Python обчислює кожну умову по черзі в пошуку першої, що є істинною. Якщо знайдено істинну умову, виконуються оператори з відступом під цією умовою, і управління переходить до наступного оператора після всього `if-elif-else`. Якщо жодна з умов не є істинною, виконуються оператори під `else`. Гілка `else` є необов'язковою; якщо вона відсутня, можливо, що жоден блок операторів з відступом не буде виконаний.

Використання `if-elif-else` для відображення тристороннього рішення в нашому розв'язувачі квадратних рівнянь дає гарно завершену програму:

```python
# quadratic4.py
import math

def main():
    print("Ця програма знаходить дійсні розв'язки квадратного рівняння\n")

    a = float(input("Введіть коефіцієнт a: "))
    b = float(input("Введіть коефіцієнт b: "))
    c = float(input("Введіть коефіцієнт c: "))

    discrim = b * b - 4 * a * c
    if discrim < 0:
        print("\nРівняння не має дійсних коренів!")
    elif discrim == 0:
        root = -b / (2 * a)
        print("\nІснує подвійний корінь:", root)
    else:
        discRoot = math.sqrt(b * b - 4 * a * c)
        root1 = (-b + discRoot) / (2 * a)
        root2 = (-b - discRoot) / (2 * a)
        print("\nРозв'язки:", root1, root2)

main()
```


## 7.4 Обробка винятків

Наша програма для квадратних рівнянь використовує структури розгалуження, щоб уникнути обчислення квадратного кореня від'ємного числа і генерації помилки під час виконання. Це поширений шаблон у багатьох програмах: використання рішень для захисту від рідкісних, але можливих помилок.

У випадку розв'язувача квадратних рівнянь ми перевіряли дані перед викликом функції `sqrt`. Іноді самі функції перевіряють можливі помилки і повертають спеціальне значення, що сигналізує про невдалу операцію. Наприклад, інша операція квадратного кореня могла б повертати від'ємне число (скажімо, −1) для сигналізації про помилку. Оскільки функція квадратного кореня завжди повинна повертати невід'ємний корінь, це значення могло б використовуватися для сигналізування про виникнення помилки. Програма перевіряла б результат операції за допомогою рішення:

```python
discRt = otherSqrt(b * b - 4 * a * c)
if discRt < 0:
    print("Дійсних коренів немає.")
else:
    ...
```

Іноді програми настільки перенасичені рішеннями для перевірки особливих випадків, що основний алгоритм обробки звичайних випадків здається повністю загубленим. Розробники мов програмування придумали механізми обробки винятків, що допомагають вирішити цю проблему проектування. Ідея механізму обробки винятків полягає в тому, що програміст може писати код, що перехоплює і обробляє помилки, що виникають під час виконання програми. Замість явної перевірки успішності кожного кроку алгоритму, програма з обробкою винятків може, по суті, сказати: «Виконай ці кроки, і якщо виникне будь-яка проблема, обробляй її ось так».

Ми не будемо обговорювати всі деталі механізму обробки винятків Python тут, але я хочу навести конкретний приклад, щоб ви могли зрозуміти, як обробка винятків працює. У Python обробка винятків виконується за допомогою спеціальної управляючої конструкції, схожої на рішення. Почнемо з конкретного прикладу, а потім розглянемо загальний підхід.

Ось версія програми для квадратних рівнянь, що використовує механізм винятків Python для перехоплення потенційних помилок у функції `math.sqrt`:

```python
# quadratic5.py
import math

def main():
    print("Ця програма знаходить дійсні розв'язки квадратного рівняння\n")

    try:
        a = float(input("Введіть коефіцієнт a: "))
        b = float(input("Введіть коефіцієнт b: "))
        c = float(input("Введіть коефіцієнт c: "))
        discRoot = math.sqrt(b * b - 4 * a * c)
        root1 = (-b + discRoot) / (2 * a)
        root2 = (-b - discRoot) / (2 * a)
        print("\nРозв'язки:", root1, root2)
    except ValueError:
        print("\nДійсних коренів немає")

main()
```

Зверніть увагу, що це по суті перша версія програми для квадратних рівнянь із додаванням `try...except` навколо її ядра. Оператор `try` має загальну форму:

```
try:
    <тіло>
except <ТипПомилки>:
    <обробник>
```

Коли Python зустрічає оператор `try`, він намагається виконати оператори всередині тіла. Якщо ці оператори виконуються без помилок, управління переходить до наступного оператора після `try...except`. Якщо де-небудь у тілі виникає помилка, Python шукає гілку `except` із відповідним типом помилки. Якщо відповідна гілка `except` знайдена, виконується код-обробник.

Оригінальна програма без обробки винятків виробляла таку помилку:

```
Traceback (most recent call last):
  File "quadratic.py", line 23, in <module>
    main()
  File "quadratic.py", line 16, in main
    discRoot = math.sqrt(b * b - 4 * a * c)
ValueError: math domain error
```

Останній рядок цього повідомлення про помилку вказує на тип згенерованої помилки — а саме `ValueError`. Оновлена версія програми надає гілку `except` для перехоплення `ValueError`. Ось як це виглядає в дії:

```
Ця програма знаходить дійсні розв'язки квадратного рівняння

Введіть коефіцієнт a: 1
Введіть коефіцієнт b: 2
Введіть коефіцієнт c: 3

Дійсних коренів немає
```

Замість аварійного завершення, обробник винятків перехоплює помилку і виводить повідомлення, що рівняння не має дійсних коренів.

Цікаво, що наша нова програма також перехоплює помилки, спричинені введенням користувачем неприпустимих значень. Запустимо програму знову і цього разу введемо «x» як перше введення:

```
Ця програма знаходить дійсні розв'язки квадратного рівняння

Введіть коефіцієнт a: x

Дійсних коренів немає
```

Чи бачите, що тут відбулося? Python підняв `ValueError` при виконанні `float("x")`, тому що `"x"` не може бути перетворений у float. Це змусило програму вийти з `try` і перейти до гілки `except` для цієї помилки. Звичайно, кінцеве повідомлення тут виглядає трохи дивно. Ось остання версія програми, що перевіряє, якого роду помилка виникла:

```python
# quadratic6.py
import math

def main():
    print("Ця програма знаходить дійсні розв'язки квадратного рівняння\n")
    try:
        a = float(input("Введіть коефіцієнт a: "))
        b = float(input("Введіть коефіцієнт b: "))
        c = float(input("Введіть коефіцієнт c: "))
        discRoot = math.sqrt(b * b - 4 * a * c)
        root1 = (-b + discRoot) / (2 * a)
        root2 = (-b - discRoot) / (2 * a)
        print("\nРозв'язки:", root1, root2)
    except ValueError as excObj:
        if str(excObj) == "math domain error":
            print("Дійсних коренів немає")
        else:
            print("Введено неприпустимий коефіцієнт")
    except:
        print("\nЩось пішло не так, вибачте!")

main()
```

Кілька гілок `except` схожі на `elif`. Якщо виникає помилка, Python по черзі перевіряє кожну гілку `except`, шукаючи таку, що відповідає типу помилки. Гола гілка `except` внизу в цьому прикладі діє як `else` і буде використовуватися як значення за замовчуванням, якщо жоден попередній тип помилки `except` не відповідає. Якщо значення за замовчуванням немає і жоден із типів `except` не відповідає помилці, програма аварійно завершується і Python виводить повідомлення про помилку.

Зверніть увагу, як я обробив два різних типи `ValueError`. Винятки — це, власне, різновид об'єкта. Якщо після типу помилки ввести `as <змінна>` у гілці `except`, Python присвоїть цій змінній фактичний об'єкт виняток. У цьому випадку я перетворив виняток на рядок і подивився на повідомлення, щоб побачити, що спричинило `ValueError`. Зверніть увагу, що цей текст — саме те, що Python виводить, якщо помилка не перехоплена (наприклад, `ValueError: math domain error`). Якщо виняток не є `ValueError`, ця програма просто виводить загальне вибачення.

Ви можете бачити, як оператор `try...except` дозволяє нам писати «куленепробивні» програми. Ви можете використовувати ту саму техніку, спостерігаючи за повідомленнями про помилки, які виводить Python, і проектуючи гілки `except` для їхнього перехоплення та обробки. Чи потрібно вам вдаватися до всіх цих заморочок, залежить від типу програми, яку ви пишете. У ваших початкових програмах вам, мабуть, не варто надто турбуватися про неправильне введення; однак програмне забезпечення професійного рівня повинне максимально можливо захищати користувачів від несподіваних результатів.


## 7.5 Дослідження в проектуванні: максимум із трьох

Тепер, маючи рішення, які можуть змінювати потік управління програми, наші алгоритми звільнені від одноманітності покрокової, строго послідовної обробки. Це і благо, і прокляття. Позитивний бік — ми тепер можемо розробляти складніші алгоритми, як ми це зробили для нашого розв'язувача квадратних рівнянь. Негативний бік — проектування цих складніших алгоритмів значно важче. У цьому розділі ми крок за кроком пройдемо через проектування більш складної задачі з рішеннями, щоб проілюструвати деякі виклики та захоплення процесу проектування.

Припустімо, нам потрібен алгоритм для знаходження найбільшого із трьох чисел. Цей алгоритм міг би бути частиною більшої задачі, наприклад визначення оцінок або обчислення податків, але нас цікавить лише суть задачі. Тобто як комп'ютер може визначити, яке з трьох введених користувачем значень є найбільшим? Ось простий контур програми:

```python
def main():
    x1, x2, x3 = eval(input("Будь ласка, введіть три значення: "))

    # відсутній код встановлює maxval рівним значенню найбільшого

    print("Найбільше значення:", maxval)
```

Зверніть увагу, що я використовую `eval` як швидкий і грубий спосіб отримати три числа; у виробничому коді (програмах для інших користувачів) `eval`, звичайно, слід, як правило, уникати. Тут це нормально, оскільки ми лише розробляємо і тестуємо деякі ідеї алгоритму.

Нам просто потрібно заповнити відсутню частину. Перш ніж читати наступний аналіз, можливо, ви захочете спробувати розв'язати цю задачу самостійно.


### 7.5.1 Стратегія 1: порівняти кожне з усіма

Очевидно, ця програма ставить перед нами задачу рішення. Нам потрібна послідовність операторів, що встановлює значення `maxval` рівним найбільшому з трьох введень — x1, x2 і x3. На перший погляд, це виглядає як тристороннє рішення; нам потрібно виконати одне з таких присвоювань:

```python
maxval = x1
maxval = x2
maxval = x3
```

Здається, нам просто потрібно передувати кожному відповідними умовами, щоб воно виконувалося лише у відповідній ситуації.

Розглянемо першу можливість — що x1 є найбільшим. Щоб визначити, що x1 справді є найбільшим, просто потрібно перевірити, що воно принаймні таке ж велике, як два інших. Ось перша спроба:

```python
if x1 >= x2 >= x3:
    maxval = x1
```

Виявляється, Python дозволяє таку складену умову, і вона поводиться точно так само, як математичне відношення x1 ≥ x2 ≥ x3. Тобто умова є істинною, коли x1 принаймні таке ж велике, як x2, а x2 — принаймні таке ж велике, як x3. Отже, Python не матиме проблем із цією умовою.

Щоразу, коли ви пишете рішення, слід ставити собі два ключових питання. По-перше, чи ви абсолютно впевнені, що виконання тіла рішення є правильною дією, коли умова є істинною? У цьому випадку умова чітко стверджує, що x1 принаймні таке ж велике, як x2 і x3, тому присвоювання його значення `maxval` повинно бути правильним. Завжди звертайте особливу увагу на граничні значення. Зверніть увагу, що наша умова включає рівне, а також більше. Слід переконатися, що це правильно. Припустімо, x1, x2 і x3 усі однакові; ця умова повертає true. Це нормально, тому що не важливо, яке ми вибираємо; перше принаймні таке ж велике, як інші, і, отже, є максимумом.

Друге питання — зворотне до першого. Чи ми впевнені, що ця умова є істинною у всіх випадках, де x1 є максимальним? На жаль, наша умова не відповідає цьому тесту. Припустімо, значення 5, 2 і 4. Очевидно, x1 є найбільшим, але наша умова повертає false, оскільки відношення 5 ≥ 2 ≥ 4 не виконується. Нам потрібно виправити це.

Нам потрібно переконатися, що x1 є найбільшим, але нас не цікавить відносний порядок x2 і x3. Що нам насправді потрібно — два окремих тести, щоб визначити, що `x1 >= x2` і що `x1 >= x3`. Python дозволяє тестувати кілька умов подібним чином, поєднуючи їх ключовим словом `and`. Семантику `and` ми детально обговоримо в розділі 8. Інтуїтивно наступна умова видається тим, що нам потрібно:

```python
if x1 >= x2 and x1 >= x3:  # x1 більше або дорівнює кожному з інших
    maxval = x1
```

Щоб завершити програму, нам просто потрібно реалізувати аналогічні тести для інших можливостей:

```python
if x1 >= x2 and x1 >= x3:
    maxval = x1
elif x2 >= x1 and x2 >= x3:
    maxval = x2
else:
    maxval = x3
```

Підсумовуючи цей підхід: наш алгоритм, по суті, перевіряє кожну можливу умову і порівнює кожне значення з обома іншими. Звідси і назва цього підходу.


### 7.5.2 Стратегія 2: дерево рішень

Можливо, є кращий спосіб вирішити цю проблему. Інший підхід використовує структуру вкладених рішень, що нагадує схему дерева — звідси і назва «дерево рішень». Ідея така: якщо x1 >= x2, то ми вже знаємо, що x2 не є максимумом, і ми можемо порівняти x1 і x3, щоб вирішити між ними. Якщо x1 < x2, то x1 не є максимумом, і нам потрібно порівняти x2 і x3:

```python
if x1 >= x2:
    if x1 >= x3:
        maxval = x1
    else:
        maxval = x3
else:
    if x2 >= x3:
        maxval = x2
    else:
        maxval = x3
```

На рисунку 7.5 показана ситуація у вигляді блок-схеми. Цей підхід вирішує задачу, зробивши лише два порівняння. Проте з цим алгоритмом виникає проблема: якщо ми хочемо вирішити задачу максимуму із чотирьох або п'яти значень, дерево рішень стає дуже складним. Стратегія дерева рішень добре масштабується лише у виняткових випадках.


### 7.5.3 Стратегія 3: послідовна обробка

До цього часу ми спроектували два дуже різних алгоритми, але жоден не здається особливо елегантним. Можливо, є ще третій спосіб. При проектуванні алгоритму хорошою відправною точкою є запитати себе, як ви вирішували б задачу, якби вас попросили виконати роботу. Знаходячи максимум із трьох чисел, у вас, мабуть, немає дуже хороший інтуїції щодо кроків, через які ви проходите. Ви просто поглянули б на числа і знали б, яке є найбільшим. Але що, якби вам вручили книгу, що містить сотні чисел без певного порядку? Як знайти найбільше в цій колекції?

Зіткнувшись із більшою задачею, більшість людей розробляє просту стратегію. Проглядайте числа, поки не знайдете велике, і покладіть на нього палець. Продовжуйте переглядати; якщо знаходите число більше, ніж те, на яке покладено палець, — переміщуйте палець на нове. Дійшовши до кінця списку, ваш палець залишиться на найбільшому значенні. Коротко кажучи, ця стратегія — послідовно проглядати список, відстежуючи найбільше число, знайдене до цього моменту.

У комп'ютера немає пальців, але ми можемо використати змінну для відстеження поточного максимуму. Насправді найпростіший підхід — просто використати `maxval` для цього. Таким чином, коли ми дійдемо до кінця, `maxval` автоматично міститиме найбільше значення. Блок-схема, що відображає цю стратегію для задачі максимуму із трьох, показана на рисунку 7.6.

Ось відповідний код Python:

```python
maxval = x1
if x2 > maxval:
    maxval = x2
if x3 > maxval:
    maxval = x3
```

Очевидно, послідовний підхід є найкращим із наших трьох алгоритмів. Сам код досить простий і містить лише два прості рішення, а послідовність легша для розуміння, ніж вкладення, що використовувалося в попередньому алгоритмі. Крім того, ідея добре масштабується для більших задач. Наприклад, додавання четвертого елемента вимагає лише ще одного оператора:

```python
maxval = x1
if x2 > maxval:
    maxval = x2
if x3 > maxval:
    maxval = x3
if x4 > maxval:
    maxval = x4
```

Не повинно дивувати, що останнє розв'язання масштабується до більших задач; ми придумали алгоритм, явно розглядаючи, як вирішити більш складну задачу. Насправді ви бачите, що код дуже повторюваний. Ми можемо легко написати програму, яка дозволяє користувачу знайти найбільше із n чисел, включивши наш алгоритм у цикл. Замість окремих змінних для x1, x2, x3 і т.д., ми можемо просто отримувати значення по одному і продовжувати повторно використовувати одну змінну x. Щоразу ми порівнюємо нове x з поточним значенням `maxval`, щоб перевірити, чи воно більше:

```python
# maxn.py
#    Знаходить максимум серії чисел

def main():
    n = int(input("Скільки чисел є? "))

    # Встановити max рівним першому значенню
    maxval = float(input("Введіть число >> "))

    # Тепер порівняти наступні n-1 значень
    for i in range(n - 1):
        x = float(input("Введіть число >> "))
        if x > maxval:
            maxval = x

    print("Найбільше значення:", maxval)

main()
```

Цей код використовує рішення, вкладене всередину циклу, для виконання роботи. На кожній ітерації циклу `maxval` містить найбільше значення, знайдене до цього моменту.


### 7.5.4 Стратегія 4: використати Python

Перш ніж залишити цю задачу, слід зазначити, що жодна з ретельно розробленої нами розробки алгоритмів не була необхідною. Python насправді має вбудовану функцію `max`, що повертає найбільший із своїх параметрів. Ось найпростіша версія нашої програми:

```python
def main():
    x1, x2, x3 = eval(input("Будь ласка, введіть три значення: "))
    print("Найбільше значення:", max(x1, x2, x3))
```

Звичайно, ця версія взагалі не вимагала розробки алгоритму, що дещо суперечить меті вправи! Іноді Python просто занадто простий для нашого власного блага.

## 7.6 Підсумок розділу

- Структури розгалуження — це управляючі конструкції, що дозволяють програмі виконувати різні послідовності інструкцій для різних випадків.

- Рішення реалізуються в Python операторами `if`. Прості рішення реалізуються звичайним `if`. Двосторонні рішення, як правило, використовують `if-else`. Багатосторонні рішення реалізуються за допомогою `if-elif-else`.

- Рішення ґрунтуються на обчисленні умов, що є простими булевими виразами. Булевий вираз є або істинним, або хибним. Python має спеціальний тип даних `bool` з літералами `True` і `False`. Умови утворюються за допомогою реляційних операторів: `<`, `<=`, `!=`, `==`, `>` і `>=`.

- Деякі мови програмування надають механізми обробки винятків, що допомагають зробити програми більш «куленепробивними». Python надає оператор `try-except` для обробки винятків.

- Алгоритми, що включають рішення, можуть стати досить складними, оскільки структури розгалуження вкладаються. Зазвичай є кілька можливих розв'язань, і ретельно потрібно продумати, щоб отримати правильну, ефективну і зрозумілу програму.
