Почнемо з простого прикладу: колекції чисел. Ще в розділі 8 ми написали просту, але корисну програму для обчислення середнього значення набору чисел, введених користувачем. Припустімо, ми хочемо розширити цю програму так, щоб вона обчислювала не лише середнє, а й два інших стандартних статистичних показники — медіану і стандартне відхилення.

**Медіана** — це значення, що ділить набір даних на рівні частини. Для даних [2, 4, 6, 9, 13] медіана дорівнює 6, оскільки є два значення, більші за 6, і два менших. Один зі способів обчислення медіани — зберегти всі числа і розташувати їх у порядку, щоб ідентифікувати середнє значення.

**Стандартне відхилення** — це міра того, наскільки дані розкидані відносно середнього. Якщо дані щільно зосереджені навколо середнього, стандартне відхилення мале. Стандартне відхилення s визначається як:

```
s = sqrt(sum((x - xi)^2) / (n - 1))
```

де x̄ — середнє, xi — i-те значення, n — кількість значень.

Обчислення стандартного відхилення потребує пам'ятати всі введені значення, тому нам потрібен спосіб зберігати і маніпулювати всією колекцією чисел.


## 11.2 Застосування списків

### 11.2.1 Списки і масиви

Як ви знаєте, списки Python — це впорядковані послідовності елементів. Ідеї та позначення для маніпулювання списками запозичені з математичного поняття послідовності. Маючи список, ми можемо використовувати єдину змінну для представлення цілої послідовності, а доступ до окремих елементів можна здійснювати через індексування:

```python
total = 0
for i in range(n):
    total = total + s[i]
```

Практично всі мови програмування мають деяку структуру послідовності, аналогічну списку Python; в інших мовах вона називається масивом (array). Масиви в інших мовах, як правило, мають фіксований розмір і є однорідними (можуть зберігати об'єкти лише одного типу даних). На відміну від них, списки Python є динамічними і можуть рости і скорочуватися на вимогу. Вони також є неоднорідними — можна змішувати довільні типи даних в одному списку.


### 11.2.2 Операції над списками

Оскільки списки є послідовностями, всі вбудовані операції над послідовностями Python застосовуються до списків:

| оператор | значення |
|---|---|
| `<посл> + <посл>` | конкатенація |
| `<посл> * <цілочисл-вираз>` | повторення |
| `<посл>[]` | індексування |
| `len(<посл>)` | довжина |
| `<посл>[:]` | зрізування |
| `for <змінна> in <посл>:` | ітерація |
| `<вираз> in <посл>` | перевірка приналежності (повертає bool) |

Операція перевірки приналежності може використовуватися, щоб перевірити, чи певне значення з'являється де-небудь у послідовності:

```python
>>> lst = [1, 2, 3, 4]
>>> 3 in lst
True
>>> 5 in lst
False
>>> ans = 'Y'
>>> ans in 'Yy'
True
```

Пам'ятайте, що важлива відмінність між списками і рядками полягає в тому, що списки є змінними. Можна використовувати присвоювання для зміни значення елементів у списку:

```python
>>> lst = [1, 2, 3, 4]
>>> lst[3] = 'Hello'
>>> lst
[1, 2, 3, 'Hello']
>>> lst[2] = 7
>>> lst
[1, 2, 7, 'Hello']
>>> lst[1:3] = ["Slice", "Assignment"]
>>> lst
[1, 'Slice', 'Assignment', 'Hello']
```

Як показує останній приклад, можна навіть змінити цілу підпослідовність у списку, присвоївши список у зріз.

Списки можна створювати перерахуванням елементів у квадратних дужках:

```python
odds = [1, 3, 5, 7, 9]
food = ["spam", "eggs", "back bacon"]
silly = [1, "spam", 4, "U"]
empty = []
```

Список ідентичних елементів можна створити за допомогою оператора повторення:

```python
zeroes = [0] * 50
```

Методи для маніпулювання списками:

| метод | значення |
|---|---|
| `<список>.append(x)` | Додає елемент x у кінець списку. |
| `<список>.sort()` | Сортує (впорядковує) список. |
| `<список>.reverse()` | Змінює порядок списку на зворотний. |
| `<список>.index(x)` | Повертає індекс першого входження x. |
| `<список>.insert(i, x)` | Вставляє x у список за індексом i. |
| `<список>.count(x)` | Повертає кількість входжень x у список. |
| `<список>.remove(x)` | Видаляє перше входження x зі списку. |
| `<список>.pop(i)` | Видаляє i-й елемент списку і повертає його значення. |

Списки також можуть скорочуватися, коли елементи видаляються. Окремі елементи або цілі зрізи можна видаляти зі списку за допомогою оператора `del`:

```python
>>> myList
[34, 26, 0, 10]
>>> del myList[1]
>>> myList
[34, 0, 10]
>>> del myList[1:3]
>>> myList
[34]
```


### 11.2.3 Статистика зі списками

Тепер, знаючи більше про списки, ми готові вирішити нашу маленьку статистичну задачу. Один очевидний підхід — зберігати числа у списку. Ми можемо написати серію функцій — `mean`, `stdDev` і `median` — що приймають список чисел і обчислюють відповідні статистичні показники.

Почнемо з функції, що отримує числа від користувача:

```python
def getNumbers():
    nums = []  # починаємо з порожнього списку
    # сторожовий цикл для отримання чисел
    xStr = input("Введіть число (<Enter> для виходу) >> ")
    while xStr != "":
        x = float(xStr)
        nums.append(x)  # додаємо значення до списку
        xStr = input("Введіть число (<Enter> для виходу) >> ")
    return nums
```

Потім функція для обчислення середнього:

```python
def mean(nums):
    total = 0.0
    for num in nums:
        total = total + num
    return total / len(nums)
```

Функція для обчислення стандартного відхилення:

```python
def stdDev(nums, xbar):
    sumDevSq = 0.0
    for num in nums:
        dev = num - xbar
        sumDevSq = sumDevSq + dev * dev
    return sqrt(sumDevSq / (len(nums) - 1))
```

Зверніть увагу, що середнє (`xbar`) передається як параметр, тому що основна програма обчислюватиме і середнє, і стандартне відхилення, і повторно обчислювати середнє всередині `stdDev` було б неефективно.

Нарешті, функція для медіани:

```python
def median(nums):
    nums.sort()
    size = len(nums)
    midPos = size // 2
    if size % 2 == 0:
        med = (nums[midPos] + nums[midPos - 1]) / 2.0
    else:
        med = nums[midPos]
    return med
```

Ретельно вивчіть цей код, щоб переконатися, що розумієте, як він вибирає правильну медіану з відсортованого списку. Середня позиція обчислюється як `size // 2`. Якщо розмір дорівнює 3, `midPos` — 1 (це правильна середня позиція, оскільки три значення мають індекси 0, 1, 2). Якщо розмір дорівнює 4, `midPos` — 2, і медіана знаходиться усередненням значень у позиціях `midPos` і `midPos−1`.

Ось повна програма:

```python
# stats.py
from math import sqrt


def getNumbers():
    nums = []  # починаємо з порожнього списку
    # сторожовий цикл для отримання чисел
    xStr = input("Введіть число (<Enter> для виходу) >> ")
    while xStr != "":
        x = float(xStr)
        nums.append(x)  # додаємо значення до списку
        xStr = input("Введіть число (<Enter> для виходу) >> ")
    return nums


def mean(nums):
    total = 0.0
    for num in nums:
        total = total + num
    return total / len(nums)


def stdDev(nums, xbar):
    sumDevSq = 0.0
    for num in nums:
        dev = num - xbar
        sumDevSq = sumDevSq + dev * dev
    return sqrt(sumDevSq / (len(nums) - 1))


def median(nums):
    nums.sort()
    size = len(nums)
    midPos = size // 2
    if size % 2 == 0:
        med = (nums[midPos] + nums[midPos - 1]) / 2.0
    else:
        med = nums[midPos]
    return med


def main():
    print("Ця програма обчислює середнє, медіану і стандартне відхилення.")

    data = getNumbers()
    xbar = mean(data)
    std = stdDev(data, xbar)
    med = median(data)

    print("\nСереднє:", xbar)
    print("Стандартне відхилення:", std)
    print("Медіана:", med)


if __name__ == '__main__': main()
```


## 11.3 Списки записів

Всі розглянуті нами приклади списків включали списки простих типів, як-от числа і рядки. Однак список може використовуватися для зберігання колекцій будь-якого типу. Особливо корисне застосування — зберігання колекції записів. Проілюструємо цю ідею, побудувавши на програмі обробки даних про GPA студентів з минулого розділу.

Нагадаємо, що наша попередня програма зчитувала файл із інформацією про оцінки студентів, щоб знайти і вивести відомості про студента з найвищим GPA. Одна з найпоширеніших операцій над такими даними — сортування. Напишемо програму, яка сортує файл студентів за GPA.

Щоб зчитати студентів у список:

```python
def readStudents(filename):
    infile = open(filename, 'r')
    students = []
    for line in infile:
        students.append(makeStudent(line))
    infile.close()
    return students
```

Щоб записати список студентів назад у файл:

```python
def writeStudents(students, filename):
    outfile = open(filename, 'w')
    for s in students:
        print("{0}\t{1}\t{2}".format(
            s.getName(), s.getHours(), s.getQPoints()),
              file=outfile)
    outfile.close()
```

Тепер нам потрібно з'ясувати, як сортувати записи за GPA. Вбудований метод `sort` дозволяє вказати ключ, що використовується при сортуванні, передаючи необов'язковий ключовий параметр `key`. Ключова функція повинна приймати елемент списку як параметр і повертати значення ключа для цього елемента:

```python
def use_gpa(aStudent):
    return aStudent.gpa()
```

Однак, оскільки метод `gpa` у класі Student вже є відповідною функцією (він приймає один параметр `self` і повертає обчислений GPA), ми можемо безпосередньо використати його як ключ:

```python
data.sort(key=Student.gpa)
```

Ось повна програма:

```python
# gpasort.py
#     Програма для сортування інформації про студентів у порядку GPA.

from gpa import Student, makeStudent


def readStudents(filename):
    infile = open(filename, 'r')
    students = []
    for line in infile:
        students.append(makeStudent(line))
    infile.close()
    return students


def writeStudents(students, filename):
    outfile = open(filename, 'w')
    for s in students:
        print("{0}\t{1}\t{2}".format(
            s.getName(), s.getHours(), s.getQPoints()),
              file=outfile)
    outfile.close()


def main():
    print("Ця програма сортує інформацію про студентів за GPA")
    filename = input("Введіть ім'я файлу з даними: ")
    data = readStudents(filename)
    data.sort(key=Student.gpa)
    filename = input("Введіть ім'я вихідного файлу: ")
    writeStudents(data, filename)
    print("Дані записано до", filename)


if __name__ == '__main__':
    main()
```

Важлива деталь: при передачі функції методу sort ми не ставимо дужки після імені функції (`use_gpa`, а не `use_gpa()`). Ми не хочемо викликати функцію — ми надсилаємо її методу sort, і він викликатиме її тоді, коли потрібно порівняти два елементи.


## 11.4 Проектування зі списками і класами

Списки і класи разом дають нам потужні інструменти для структурування даних у наших програмах. Пригадаємо клас DieView з минулого розділу. Щоб відображати шість можливих значень кістки, кожен об'єкт DieView відстежує сім кіл, що представляють позиції крапок. У попередній версії ми зберігали ці кола, використовуючи змінні примірника `pip1`, `pip2`, `pip3` тощо.

Розглянемо, як виглядає код із використанням колекції об'єктів-кіл, збережених у списку. Основна ідея — замінити сім окремих змінних примірника єдиним списком `pips`:

```python
self.pips = [self.__makePip(cx - offset, cy - offset),
             self.__makePip(cx - offset, cy),
             self.__makePip(cx - offset, cy + offset),
             self.__makePip(cx, cy),
             self.__makePip(cx + offset, cy - offset),
             self.__makePip(cx + offset, cy),
             self.__makePip(cx + offset, cy + offset)]
```

Перевага списку крапок полягає в тому, що значно простіше виконувати дії над усім набором. Наприклад, ми можемо вимкнути кістку, встановивши для всіх крапок колір фону:

```python
for pip in self.pips:
    pip.setFill(self.background)
```

Дві рядки коду замість семи!

Аналогічно ми можемо ввімкнути набір крапок, перебираючи відповідні індекси:

```python
for i in [0, 3, 6]:
    self.pips[i].setFill(self.foreground)
```

Ще далі ми можемо використати підхід, керований таблицями. Ідея — використати список, де кожен елемент сам є списком індексів крапок:

```python
onTable = [[], [3], [2, 4], [2, 3, 4],
           [0, 2, 4, 6], [0, 2, 3, 4, 6], [0, 1, 2, 4, 5, 6]]
```

Тоді метод setValue стає дуже компактним:

```python
for pip in self.pips:
    pip.setFill(self.background)
on = onTable[value]
for i in on:
    self.pips[i].setFill(self.foreground)
```

Ця версія зводить 36 рядків попереднього методу setValue до семи. Крім того, її значно простіше змінювати: якщо потрібно змінити, які крапки відображаються для різних значень, просто змінюємо записи в `onTable`.

Ось оновлений клас з таблицею `onTable`, збереженою як змінна примірника в конструкторі:

```python
# dieview2.py
from graphics import *


class DieView:
    """DieView — це віджет, що відображає графічне представлення
    стандартної шестигранної кістки."""

    def __init__(self, win, center, size):
        """Створити вигляд кістки, наприклад:
             d1 = GDie(myWin, Point(40, 50), 20)
        створює кістку з центром у (40,50) зі сторонами довжиною 20."""

        # спочатку визначити деякі стандартні значення
        self.win = win
        self.background = "white"   # колір грані кістки
        self.foreground = "black"   # колір крапок
        self.psize = 0.1 * size     # радіус кожної крапки
        hsize = size / 2.0          # половина розміру кістки
        offset = 0.6 * hsize        # відстань від центру до зовнішніх крапок

        # створити квадрат для грані
        cx, cy = center.getX(), center.getY()
        p1 = Point(cx - hsize, cy - hsize)
        p2 = Point(cx + hsize, cy + hsize)
        rect = Rectangle(p1, p2)
        rect.draw(win)
        rect.setFill(self.background)

        # Створити 7 кіл для стандартних позицій крапок
        self.pips = [self.__makePip(cx - offset, cy - offset),
                     self.__makePip(cx - offset, cy),
                     self.__makePip(cx - offset, cy + offset),
                     self.__makePip(cx, cy),
                     self.__makePip(cx + offset, cy - offset),
                     self.__makePip(cx + offset, cy),
                     self.__makePip(cx + offset, cy + offset)]

        # Створити таблицю для визначення, які крапки увімкнені для кожного значення
        self.onTable = [[], [3], [2, 4], [2, 3, 4],
                        [0, 2, 4, 6], [0, 2, 3, 4, 6], [0, 1, 2, 4, 5, 6]]

        self.setValue(1)

    def __makePip(self, x, y):
        """Внутрішній допоміжний метод для малювання крапки у (x,y)."""
        pip = Circle(Point(x, y), self.psize)
        pip.setFill(self.background)
        pip.setOutline(self.background)
        pip.draw(self.win)
        return pip

    def setValue(self, value):
        """Встановити цю кістку для відображення value."""
        # Вимкнути всі крапки
        for pip in self.pips:
            pip.setFill(self.background)

        # Увімкнути відповідні крапки
        for i in self.onTable[value]:
            self.pips[i].setFill(self.foreground)
```

Цей приклад також демонструє переваги інкапсуляції з розділу 10. Ми значно покращили реалізацію класу DieView, але не змінили набір методів, які він підтримує. Ми можемо замінити цю поліпшену версію в будь-яку програму, що використовує DieView, без необхідності змінювати будь-який інший код.


## 11.5 Приклад: калькулятор Python

Переопрацьований клас DieView показує, як списки можна ефективно використовувати як змінні примірника об'єктів. Цікаво, що наші списки `pips` і `onTable` містять кола і списки, відповідно, які самі є об'єктами. Поєднуючи і вкладаючи колекції і об'єкти, ми можемо розробляти елегантні способи зберігання даних.

Ми можемо навіть піти ще далі і розглядати саму програму як колекцію структур даних і набір алгоритмів, що оперують ними. Якщо програма містить дані і операції, природний спосіб організувати її — трактувати весь застосунок як об'єкт.


### 11.5.1 Калькулятор як об'єкт

Як приклад розробимо програму, що реалізує простий калькулятор Python. Наш калькулятор матиме кнопки для десяти цифр (0–9), десяткової крапки (.), чотирьох операцій (+, -, *, /) і кількох спеціальних кнопок: C для очищення дисплея, <- для видалення одного символу і = для обчислення.

При клацанні кнопок відповідні символи з'являтимуться на дисплеї, дозволяючи користувачу скласти формулу. При натисканні = формула обчислюється і результат відображається на дисплеї.

Розділимо функціонування калькулятора на дві частини: створення інтерфейсу і взаємодія з користувачем. Інтерфейс складається з віджета дисплея і купи кнопок. Ми будемо відстежувати ці GUI-віджети за допомогою змінних примірника. Взаємодія з користувачем керуватиметься набором методів.

Для реалізації цього ми створимо клас Calculator.


### 11.5.2 Побудова інтерфейсу

Конструктор класу Calculator будує весь інтерфейс. Оскільки кнопок багато і вони схожі, ми використаємо список для їхнього зберігання. Замість безпосереднього створення кожної кнопки, спочатку створимо список специфікацій кнопок (`bSpecs`), де кожна специфікація — **кортеж** (tuple) із координат центру і мітки:

```python
bSpecs = [(2, 1, '0'), (3, 1, '.'),
          (1, 2, '1'), (2, 2, '2'), (3, 2, '3'), (4, 2, '+'), (5, 2, '-'),
          (1, 3, '4'), (2, 3, '5'), (3, 3, '6'), (4, 3, '*'), (5, 3, '/'),
          (1, 4, '7'), (2, 4, '8'), (3, 4, '9'), (4, 4, '<-'), (5, 4, 'C')]
```

**Кортеж** — це послідовність, схожа на список, але вона обмежується круглими дужками `()` і є незмінною (immutable). Якщо вміст послідовності не змінюватиметься після створення, кортеж є ефективнішим за список.

Потім перебираємо специфікації, розпаковуючи кортеж при кожній ітерації:

```python
for (cx, cy, label) in bSpecs:
    self.buttons.append(Button(self.win, Point(cx, cy),
                                .75, .75, label))
```

Коли кортеж змінних використовується в лівій частині присвоювання, відповідні компоненти кортежу в правій частині «розпаковуються» у змінні лівої частини. Насправді саме так Python реалізує всі одночасні присвоювання.


### 11.5.3 Обробка натискань кнопок

Отримання натискань клавіш виконується через очікування кліків миші, поки один із кліків не потрапить на кнопку. Для визначення, чи натиснута кнопка, ми перебираємо список кнопок і перевіряємо кожну:

```python
def getButton(self):
    # Очікує натискання кнопки і повертає мітку натиснутої кнопки.
    while True:
        p = self.win.getMouse()
        for b in self.buttons:
            if b.clicked(p):
                return b.getLabel()  # вихід із методу
```

Маючи кнопки у списку — велика перевага, тому що можна використати цикл `for` для їх перебору.

Ось повна програма:

```python
# calc.pyw -- Чотирифункціональний калькулятор з використанням арифметики Python.
#      Ілюструє використання об'єктів і списків для побудови простого GUI.

from graphics import *
from button import Button


class Calculator:
    # Цей клас реалізує простий GUI калькулятора

    def __init__(self):
        # створити вікно для калькулятора
        win = GraphWin("calculator")
        win.setCoords(0, 0, 6, 7)
        win.setBackground("slategray")
        self.win = win
        # Тепер створити віджети
        self.__createButtons()
        self.__createDisplay()

    def __createButtons(self):
        # створити список кнопок
        # bSpecs задає координати центру і мітку кнопок
        bSpecs = [(2, 1, '0'), (3, 1, '.'),
                  (1, 2, '1'), (2, 2, '2'), (3, 2, '3'), (4, 2, '+'), (5, 2, '-'),
                  (1, 3, '4'), (2, 3, '5'), (3, 3, '6'), (4, 3, '*'), (5, 3, '/'),
                  (1, 4, '7'), (2, 4, '8'), (3, 4, '9'), (4, 4, '<-'), (5, 4, 'C')]
        self.buttons = []
        for (cx, cy, label) in bSpecs:
            self.buttons.append(Button(self.win, Point(cx, cy), .75, .75, label))
        # створити більшу кнопку '='
        self.buttons.append(Button(self.win, Point(4.5, 1), 1.75, .75, "="))
        # активувати всі кнопки
        for b in self.buttons:
            b.activate()

    def __createDisplay(self):
        bg = Rectangle(Point(.5, 5.5), Point(5.5, 6.5))
        bg.setFill('white')
        bg.draw(self.win)
        text = Text(Point(3, 6), '')
        text.draw(self.win)
        text.setFace("courier")
        text.setStyle("bold")
        text.setSize(16)
        self.display = text

    def getButton(self):
        # Очікує натискання кнопки і повертає мітку натиснутої кнопки.
        while True:
            p = self.win.getMouse()
            for b in self.buttons:
                if b.clicked(p):
                    return b.getLabel()  # вихід із методу

    def processButton(self, key):
        # Оновлює дисплей калькулятора при натисканні цієї клавіші
        text = self.display.getText()
        if key == 'C':
            self.display.setText("")
        elif key == '<-':
            # Backspace, відрізати останній символ.
            self.display.setText(text[:-1])
        elif key == '=':
            # Обчислити вираз і відобразити результат.
            # try...except "перехоплює" помилки у формулі.
            try:
                result = eval(text)
            except:
                result = 'ERROR'
            self.display.setText(str(result))
        else:
            # Звичайне натискання клавіші — додати до кінця дисплея
            self.display.setText(text + key)

    def run(self):
        # Нескінченний цикл подій для обробки кліків кнопок.
        while True:
            key = self.getButton()
            self.processButton(key)


# Це запускає програму.
if __name__ == '__main__':
    # Спочатку створити об'єкт калькулятора
    theCalc = Calculator()
    # Тепер викликати метод run калькулятора.
    theCalc.run()
```

Зверніть особливу увагу на самий кінець програми. Щоб запустити застосунок, ми створюємо примірник класу Calculator і потім викликаємо його метод `run`.


## 11.6 Приклад: покращена анімація гарматного ядра

Приклад з калькулятором використовував список об'єктів Button для спрощення коду. У цьому випадку підтримка колекції схожих об'єктів у вигляді списку була суто програмістською зручністю, оскільки вміст списку кнопок ніколи не змінювався. Списки стають незамінними, коли колекція динамічно змінюється під час виконання програми.

Розглянемо анімацію гарматного ядра з минулого розділу. Як ми її залишили, програма могла показувати лише один постріл одночасно. У цьому розділі ми розширимо програму, щоб дозволити кілька пострілів. Для цього потрібно відстежувати всі ядра, що зараз летять. Це постійно змінювана колекція, і для її управління ми будемо використовувати список.


### 11.6.1 Створення лаунчера

Перш ніж вирішувати задачу анімації кількох пострілів, нам потрібно оновити інтерфейс програми. Для нової версії ми хочемо додати новий віджет, що дозволяє користувачу швидко стріляти пострілами з різними початковими кутами і швидкостями — більше схоже на відеогру.

Клас Launcher відстежуватиме поточний кут і швидкість у змінних примірника. Ось основні методи:

```python
class Launcher:

    def adjAngle(self, amt):
        """Змінити кут пострілу на amt градусів."""
        self.angle = self.angle + radians(amt)
        self.redraw()

    def adjVel(self, amt):
        """Змінити швидкість пострілу на amt."""
        self.vel = self.vel + amt
        self.redraw()

    def redraw(self):
        """Перемалювати стрілку для відображення поточного кута і швидкості."""
        self.arrow.undraw()
        pt2 = Point(self.vel * cos(self.angle),
                    self.vel * sin(self.angle))
        self.arrow = Line(Point(0, 0), pt2).draw(self.win)
        self.arrow.setArrow("last")
        self.arrow.setWidth(3)

    def fire(self):
        return ShotTracker(self.win, degrees(self.angle), self.vel, 0.0)

    def __init__(self, win):
        # намалювати базову кулю лаунчера
        base = Circle(Point(0, 0), 3)
        base.setFill("red")
        base.setOutline("red")
        base.draw(win)

        # зберегти вікно і задати початковий кут та швидкість
        self.win = win
        self.angle = radians(45.0)
        self.vel = 40.0

        # створити початкову «пусту» стрілку (потрібна для redraw)
        self.arrow = Line(Point(0, 0), Point(0, 0)).draw(win)
        # замінити правильною стрілкою
        self.redraw()
```

Метод `fire` просто повертає відповідний об'єкт ShotTracker. Цикл анімації не є відповідним місцем для методу `fire` — взаємодія з лаунчером не повинна бути модальною.


### 11.6.2 Відстеження кількох пострілів

З лаунчером у нас можна перейти до цікавого питання — кілька об'єктів одночасно. Ми хочемо мати можливість регулювати лаунчер і стріляти новими пострілами, поки деякі постріли ще в польоті. Цикл подій має виконувати подвійний обов'язок, слугуючи також циклом анімації для всіх пострілів, що «живі».

Ми будемо використовувати підхід, аналогічний калькулятору, і створимо об'єкт застосунку — `ProjectileApp`:

```python
class ProjectileApp:

    def __init__(self):
        # створити графічне вікно зі шкальною лінією внизу
        self.win = GraphWin("Projectile Animation", 640, 480)
        self.win.setCoords(-10, -10, 210, 155)
        Line(Point(-10, 0), Point(210, 0)).draw(self.win)
        for x in range(0, 210, 50):
            Text(Point(x, -7), str(x)).draw(self.win)
            Line(Point(x, 0), Point(x, 2)).draw(self.win)

        # додати лаунчер у вікно
        self.launcher = Launcher(self.win)

        # починаємо з порожнім списком «живих» пострілів
        self.shots = []
```

Ось метод `run`, що реалізує цикл подій/анімації:

```python
def run(self):
    # головний цикл подій/анімації
    while True:
        self.updateShots(1 / 30)

        key = self.win.checkKey()
        if key in ["q", "Q"]:
            break

        if key == "Up":
            self.launcher.adjAngle(5)
        elif key == "Down":
            self.launcher.adjAngle(-5)
        elif key == "Right":
            self.launcher.adjVel(5)
        elif key == "Left":
            self.launcher.adjVel(-5)
        elif key in ["f", "F"]:
            self.shots.append(self.launcher.fire())

        update(30)

    self.win.close()
```

Зверніть, наскільки легко насправді вистрілити ядро! Коли користувач натискає клавішу `f`, ми отримуємо об'єкт ShotTracker від лаунчера і просто додаємо його до списку живих пострілів.

Метод `updateShots` займається анімацією пострілів і прибиранням тих, що «померли»:

```python
def updateShots(self, dt):
    alive = []
    for shot in self.shots:
        shot.update(dt)
        if shot.getY() >= 0 and -10 < shot.getX() < 210:
            alive.append(shot)
        else:
            shot.undraw()
    self.shots = alive
```

Зверніть, як цей код накопичує список живих пострілів і потім оновлює `self.shots` після завершення циклу. Важливий момент: ніколи не модифікуйте список, через який ітерує цикл `for` — це може призводити до непередбачуваної поведінки. Замість цього ми накопичуємо результат у новому списку (`alive`) і в кінці присвоюємо його `self.shots`.

```python
if __name__ == '__main__':
    ProjectileApp().run()
```

Ключ до розуміння фінальної програми — пам'ятати, що робить кожний клас і як вони всі взаємодіють. Схема класів (аналог структурних схем із розділу 9 в об'єктно-орієнтованому контексті) показує залежності між класами без занурення в усі деталі коду.


## 11.7 Непослідовні колекції

Python надає ряд вбудованих типів даних для колекцій. Після списків найбільш широко використовується тип колекції під назвою **словник** (dictionary).


### 11.7.1 Основи словників

Списки дозволяють зберігати і витягувати елементи з послідовних колекцій. Ми шукаємо елемент за індексом — його позицією. Але багато застосунків вимагають більш гнучкого способу пошуку інформації. Наприклад, нам може знадобитися отримати інформацію про студентів за їхніми ідентифікаційними номерами. У термінах програмування це пара «ключ-значення» (key-value pair). Ми отримуємо доступ до значення (інформація про студента), асоційованого з конкретним ключем (ідентифікаційний номер).

Колекція, що дозволяє шукати інформацію, асоційовану з довільними ключами, називається відображенням (mapping). Словники Python є відображеннями. Деякі інші мови програмування надають схожі структури, що називаються хешами або асоціативними масивами.

Словник можна створити в Python, перерахувавши пари «ключ-значення» всередині фігурних дужок. Ось простий словник, що зберігає деякі вигадані імена користувачів і паролі:

```python
>>> passwd = {"guido": "superprogrammer", "turing": "genius",
              "bill": "monopoly"}
```

Зверніть увагу, що ключі та значення поєднуються символом «:», а пари відокремлюються комами.

Основне використання словника — пошук значення, асоційованого з конкретним ключем. Це виконується через нотацію індексування:

```python
>>> passwd["guido"]
'superprogrammer'
>>> passwd["bill"]
'monopoly'
```

Загалом `<словник>[<ключ>]` повертає об'єкт, асоційований із заданим ключем.

Словники є змінними; значення, асоційоване з ключем, можна змінити через присвоювання:

```python
>>> passwd["bill"] = "blue screen"
>>> passwd
{'turing': 'genius', 'bill': 'blue screen', 'guido': 'superprogrammer'}
```

Зверніть також, що словник виводиться в іншому порядку, ніж був створений. Це не помилка. Відображення за своєю природою є невпорядкованими. Якщо потрібно зберігати колекцію елементів у певному порядку, потрібна послідовність, а не відображення.

Підсумовуючи: словники — це змінні колекції, що реалізують відображення ключів у значення. Ключами можуть бути будь-які незмінні типи, а значеннями — будь-який тип.


### 11.7.2 Операції зі словниками

Python надає наступні методи для словників:

| метод | значення |
|---|---|
| `<ключ> in <словник>` | Повертає true, якщо словник містить вказаний ключ. |
| `<словник>.keys()` | Повертає послідовність ключів. |
| `<словник>.values()` | Повертає послідовність значень. |
| `<словник>.items()` | Повертає послідовність кортежів (ключ, значення). |
| `<словник>.get(<ключ>, <за-замовчуванням>)` | Якщо словник має ключ, повертає його значення; інакше — значення за замовчуванням. |
| `del <словник>[<ключ>]` | Видаляє вказаний запис. |
| `<словник>.clear()` | Видаляє всі записи. |
| `for <змінна> in <словник>:` | Цикл по ключах. |

Інтерактивний сеанс з нашим словником паролів:

```python
>>> list(passwd.keys())
['turing', 'bill', 'newuser', 'guido']
>>> list(passwd.values())
['genius', 'blue screen', 'ImANewbie', 'superprogrammer']
>>> list(passwd.items())
[('turing', 'genius'), ('bill', 'blue screen'),
 ('newuser', 'ImANewbie'), ('guido', 'superprogrammer')]
>>> "bill" in passwd
True
>>> 'fred' in passwd
False
>>> passwd.get('bill', 'unknown')
'blue screen'
>>> passwd.get('john', 'unknown')
'unknown'
```


### 11.7.3 Приклад програми: частота слів

Напишемо програму, що аналізує текстові документи і підраховує, скільки разів кожне слово з'являється в документі. Нам потрібен лічильник для кожного слова в документі — саме тут стає в пригоді словник Python.

Використаємо словник, де ключами є рядки, що представляють слова, а значеннями — цілі числа, що рахують входження слова:

```python
counts[w] = counts.get(w, 0) + 1
```

Якщо `w` ще немає в словнику, `get` поверне 0, і результат — запис для `w` встановлюється в 1.

Перш ніж розбивати текст, корисно привести всі літери до нижнього регістру і видалити розділові знаки:

```python
fname = input("Файл для аналізу: ")
text = open(fname, "r").read()
text = text.lower()
for ch in '!"#$%&\'()*+,-./:;<=>?@[\\]^_`{|}~':
    text = text.replace(ch, " ")
words = text.split()
```

Тоді побудувати словник лічильників:

```python
counts = {}
for w in words:
    counts[w] = counts.get(w, 0) + 1
```

Для виведення n найчастіших слів спочатку отримаємо список пар «ключ-значення» і відсортуємо їх за частотою:

```python
def byFreq(pair):
    return pair[1]

items = list(counts.items())
items.sort()                              # упорядковує пари за алфавітом
items.sort(key=byFreq, reverse=True)      # упорядковує за частотою
```

Перше сортування — за алфавітом — потрібне тому, що Python виконує **стабільне** сортування. Стабільний алгоритм сортування не змінює відносний порядок елементів із рівними ключами. Тому після першого алфавітного сортування слова з однаковою частотою залишатимуться в алфавітному порядку після другого сортування за частотою.

Ось повна програма:

```python
def byFreq(pair):
    return pair[1]


def main():
    print("Ця програма аналізує частоту слів у файлі")
    print("і виводить звіт про n найчастіших слів.\n")

    # отримати послідовність слів із файлу
    fname = input("Файл для аналізу: ")
    text = open(fname, 'r').read()
    text = text.lower()
    for ch in '!"#$%&\'()*+,-./:;<=>?@[\\]^_`{|}~':
        text = text.replace(ch, ' ')
    words = text.split()

    # побудувати словник лічильників слів
    counts = {}
    for w in words:
        counts[w] = counts.get(w, 0) + 1

    # вивести аналіз n найчастіших слів
    n = eval(input("Вивести аналіз скількох слів? "))
    items = list(counts.items())
    items.sort()
    items.sort(key=byFreq, reverse=True)
    for i in range(n):
        word, count = items[i]
        print("{0:<15}{1:>5}".format(word, count))


if __name__ == '__main__': main()
```


## 11.8 Підсумок розділу

- Об'єкт типу «список» — це змінна послідовність довільних об'єктів. До елементів можна звертатися за допомогою індексування і зрізування. Елементи списку можна змінювати за допомогою присвоювання.

- Списки Python схожі на масиви в інших мовах програмування. Списки Python є більш гнучкими, оскільки їхній розмір може змінюватися і вони є неоднорідними. Списки Python також підтримують ряд корисних методів.

- Особливо важлива операція обробки даних — сортування. Метод `sort` у списках Python може бути налаштований шляхом надання відповідної ключової функції. Це дозволяє програмам сортувати списки довільних об'єктів.

- Класи можуть використовувати списки для зберігання колекцій у змінних примірника. Нерідко використання списку є більш гнучким, ніж використання окремих змінних примірника.

- Весь застосунок можна розглядати як колекцію даних і набір операцій — об'єкт. Це поширений підхід до структурування GUI-застосунків.

- Словник Python реалізує довільне відображення ключів у значення. Це дуже корисно для представлення непослідовних колекцій.
