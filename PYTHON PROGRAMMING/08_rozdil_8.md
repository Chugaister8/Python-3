## 8.1 Цикли for: короткий огляд

У розділі 7 ми детально розглянули оператор Python `if` і його використання для реалізації шаблонів програмування — таких як односторонні, двосторонні та багатосторонні рішення. У цьому розділі ми завершимо огляд управляючих конструкцій, детально розглянувши цикли і булеві вирази.

Ви вже знаєте, що оператор Python `for` надає вид циклу. Він дозволяє ітерувати через послідовність значень:

```
for <змінна> in <послідовність>:
    <тіло>
```

Індексна змінна циклу `<змінна>` набуває по черзі кожного значення в послідовності, а оператори в тілі циклу виконуються по одному разу для кожного значення.

Припустімо, ми хочемо написати програму, яка може обчислити середнє значення серії чисел, введених користувачем. Щоб зробити програму загальною, вона повинна працювати для будь-якої кількості чисел. Ви знаєте, що середнє значення обчислюється підсумовуванням чисел і діленням на їхню кількість. Нам не потрібно відстежувати всі введені числа; нам лише потрібна поточна сума, щоб ми могли обчислити середнє наприкінці.

Цей опис задачі повинен нагадати вам деякі вже знайомі шаблони проектування. Ми маємо справу з серією чисел — вони оброблятимуться деяким видом циклу. Якщо є n чисел, цикл повинен виконуватися n разів; можна використати шаблон циклу з лічильником. Нам також потрібна поточна сума; це вимагає акумулятора в циклі. Поєднуючи ці дві ідеї, ми можемо створити проект цієї задачі:

```
ввести кількість чисел, n
ініціалізувати total нулем
цикл n разів
    ввести число, x
    додати x до total
вивести середнє як total / n
```

Сподіваємося, ви бачите, як в цьому проекті поєднані шаблони циклу з лічильником і акумулятора. Ми можемо майже безпосередньо перекласти цей проект на реалізацію Python:

```python
# average1.py

def main():
    n = int(input("Скільки у вас чисел? "))
    total = 0.0
    for i in range(n):
        x = float(input("Введіть число >> "))
        total = total + x
    print("\nСереднє значення чисел:", total / n)

main()
```

Поточна сума починається з 0, і кожне число додається по черзі. Після циклу total ділиться на n для обчислення середнього.

Ось програма в дії:

```
Скільки у вас чисел? 5
Введіть число >> 32
Введіть число >> 45
Введіть число >> 34
Введіть число >> 76
Введіть число >> 45

Середнє значення чисел: 46.4
```

Непогано. Знання кількох поширених шаблонів — циклу з лічильником і акумулятора — привело нас до робочої програми з мінімальними труднощами у проектуванні та реалізації.


## 8.2 Невизначені цикли

Наша програма усереднення, безумовно, функціональна, але вона не має найкращого інтерфейсу. Вона починається з запиту про кількість чисел. Для невеликої кількості чисел це нормально, але що, якщо мені потрібно усереднити цілу сторінку чисел? Підрахунок їх може стати значним навантаженням.

Було б значно зручніше, якби комп'ютер міг сам подбати про підрахунок чисел. На жаль, як ви, мабуть, пам'ятаєте, цикл `for` (у звичайній формі) є визначеним циклом, і це означає, що кількість ітерацій визначається при запуску циклу. Ми не можемо використати визначений цикл, якщо не знаємо кількість ітерацій заздалегідь, а ми не можемо знати, скільки ітерацій потрібно цьому циклу, поки не введено всі числа. Здається, ми в глухому куті.

Розв'язання цієї дилеми лежить в іншому виді циклу — невизначеному, або умовному. Невизначений цикл продовжує ітерацію, поки не будуть виконані певні умови. Наперед немає гарантії щодо того, скільки разів цикл обійде.

У Python невизначений цикл реалізується за допомогою оператора `while`. Синтаксично `while` дуже простий:

```
while <умова>:
    <тіло>
```

Тут умова — це булевий вираз, як і в операторах `if`. Тіло — це, як завжди, послідовність одного або більше операторів.

Семантика `while` є прямолінійною. Тіло циклу виконується повторно, поки умова залишається істинною. Коли умова стає хибною, цикл завершується. На рисунку 8.1 показана блок-схема для `while`. Зверніть увагу, що умова завжди перевіряється у верхній частині циклу, перед виконанням тіла циклу. Така структура називається циклом з попередньою перевіркою (pre-test loop). Якщо умова циклу спочатку хибна, тіло циклу взагалі не виконуватиметься.

Ось приклад простого циклу `while`, що лічить від 0 до 10:

```python
i = 0
while i <= 10:
    print(i)
    i = i + 1
```

Цей код матиме такий самий результат, ніби ми написали цикл `for` на кшталт:

```python
for i in range(11):
    print(i)
```

Зверніть увагу, що версія з `while` вимагає від нас ініціалізувати `i` перед циклом і збільшувати `i` внизу тіла циклу. У циклі `for` змінна циклу обробляється автоматично.

Простота оператора `while` робить його і потужним, і небезпечним. Оскільки він менш жорсткий, він є більш універсальним; він може робити більше, ніж просто ітерувати через послідовності. Але він також є поширеним джерелом помилок.

Припустімо, ми забули збільшити `i` у кінці тіла циклу в прикладі з підрахунком:

```python
i = 0
while i <= 10:
    print(i)
```

Яким буде виведення цієї програми? Коли Python доходить до циклу, `i` дорівнюватиме 0, що менше 10, тому тіло циклу виконується, виводячи 0. Тепер управління повертається до умови; `i` все ще 0, тому тіло циклу виконується знову, виводячи 0. Тепер управління повертається до умови; `i` все ще 0, тому тіло циклу виконується знову...

Ви розумієте. Це приклад нескінченного циклу. Зазвичай нескінченні цикли — погана річ. Очевидно, що ця версія програми нічого корисного не робить.

Як програміст-початківець, не дивно, якщо ви випадково напишете кілька програм із нескінченними циклами — це обряд посвяти для програмістів. Навіть більш досвідчені програмісти час від часу роблять це. Зазвичай можна вийти з циклу, натиснувши `<Ctrl>-C` (утримуючи клавішу `<Ctrl>` і натискаючи C). Якщо ваш цикл дуже щільний, це може не спрацювати, і доведеться вдатися до більш радикальних засобів (наприклад, `<Ctrl>-<Alt>-<Delete>` на ПК). Якщо нічого не допомагає, є завжди надійна кнопка скидання на вашому комп'ютері. Найкраща ідея — взагалі уникати написання нескінченних циклів.


## 8.3 Поширені шаблони циклів

### 8.3.1 Інтерактивні цикли

Одне хороше використання невизначеного циклу — написання інтерактивних циклів. Ідея інтерактивного циклу полягає в тому, що він дозволяє користувачу повторювати певні частини програми за бажанням. Розглянемо цей шаблон циклу в контексті нашої задачі з усередненням чисел.

Нагадаємо, що попередня версія програми змушувала користувача підраховувати, скільки чисел потрібно усередняти. Ми хочемо змінити програму так, щоб вона сама відстежувала кількість чисел. Це можна зробити за допомогою ще одного акумулятора — назвемо його `count` — що починається з 0 і збільшується на 1 при кожному проходи циклу.

Щоб дозволити користувачу зупинитися в будь-який час, кожна ітерація циклу питатиме, чи є ще дані для обробки. Загальний шаблон інтерактивного циклу виглядає так:

```
встановити moredata в "yes"
while moredata is "yes"
    отримати наступний елемент даних
    обробити елемент
    запитати користувача, чи є ще дані
```

Поєднуючи шаблон інтерактивного циклу з акумуляторами для total і count, отримуємо такий алгоритм:

```
ініціалізувати total нулем
ініціалізувати count нулем
встановити moredata в "yes"
while moredata is "yes"
    ввести число, x
    додати x до total
    додати 1 до count
    запитати користувача, чи є ще moredata
вивести total / count
```

Ось відповідна програма Python:

```python
# average2.py

def main():
    total = 0.0
    count = 0
    moredata = "yes"
    while moredata[0] == "y":
        x = float(input("Введіть число >> "))
        total = total + x
        count = count + 1
        moredata = input("Є ще числа (так або ні)? ")
    print("\nСереднє значення чисел:", total / count)

main()
```

Зверніть увагу, що ця програма використовує індексування рядків (`moredata[0]`) для аналізу лише першої літери введення користувача. Це дозволяє давати різноманітні відповіді, як-от «так», «т», «тут» тощо. Важливо лише, щоб перша літера була «т».

У цій версії користувачу не потрібно рахувати значення, але інтерфейс все ще не дуже зручний. Користувача, мабуть, дратуватиме постійний запит на нові дані. Інтерактивний цикл має багато хороших застосувань; це — не одне з них.


### 8.3.2 Сторожові цикли

Кращим розв'язанням задачі усереднення чисел є шаблон, відомий як сторожовий цикл. Сторожовий цикл продовжує обробляти дані, поки не зустріне спеціальне значення, що сигналізує про кінець. Це спеціальне значення називається сторожем (sentinel). Будь-яке значення може бути обрано як сторож. Єдине обмеження — воно повинне відрізнятися від фактичних значень даних. Сторож не обробляється як частина даних.

Ось загальний шаблон для проектування сторожових циклів:

```
отримати перший елемент даних
while елемент не є сторожем
    обробити елемент
    отримати наступний елемент даних
```

Зверніть увагу, як цей шаблон уникає обробки сторожового елемента. Перший елемент отримується до початку циклу. Це іноді називається завантажувальним читанням (priming read), оскільки воно запускає процес. Якщо перший елемент є сторожем, цикл негайно завершується і дані не обробляються. В іншому разі елемент обробляється і зчитується наступний. Перевірка циклу вгорі гарантує, що наступний елемент не є сторожем перед обробкою. Коли досягнуто сторожа, цикл завершується.

Ми можемо застосувати шаблон сторожа до нашої задачі усереднення чисел. Перший крок — вибрати сторожа. Припустімо, ми використовуємо програму для усереднення оцінок за іспит. У цьому випадку можна безпечно припустити, що жодна оцінка не буде нижче 0. Користувач може ввести від'ємне число, щоб сигналізувати про кінець даних:

```python
# average3.py

def main():
    total = 0.0
    count = 0
    x = float(input("Введіть число (від'ємне для виходу) >> "))
    while x >= 0:
        total = total + x
        count = count + 1
        x = float(input("Введіть число (від'ємне для виходу) >> "))
    print("\nСереднє значення чисел:", total / count)

main()
```

Я змінив запрошення, щоб користувач знав, як сигналізувати про кінець даних. Зверніть увагу, що запрошення однакове і в завантажувальному читанні, і в кінці тіла циклу.

Ось програма в дії:

```
Введіть число (від'ємне для виходу) >> 32
Введіть число (від'ємне для виходу) >> 45
Введіть число (від'ємне для виходу) >> 34
Введіть число (від'ємне для виходу) >> 76
Введіть число (від'ємне для виходу) >> 45
Введіть число (від'ємне для виходу) >> -1

Середнє значення чисел: 46.4
```

Ця версія забезпечує зручність інтерактивного циклу без необхідності щоразу вводити «так». Сторожовий цикл — дуже зручний шаблон для розв'язання всіляких задач обробки даних. Це ще один кліше, який слід запам'ятати.

Це розв'язання з сторожовим циклом досить хороше, але є ще одне обмеження. Програму не можна використовувати для усереднення набору чисел, що містить від'ємні значення. Нам потрібне сторожове значення, що відрізняється від будь-якого можливого дійсного числа. Звичайно, це неможливо, поки ми обмежуємося роботою з числами. Незалежно від того, яке число ми обираємо як сторожа, завжди можливо, що якийсь набір даних містить таке число.

Щоб мати справді унікальний сторож, нам потрібно розширити можливі введення. Припустімо, ми отримуємо введення від користувача у вигляді рядка. Ми можемо мати відмітний нечисловий рядок, що вказує на кінець введення; всі інші рядки перетворюватимуться на числа і оброблятимуться як дані. Просте розв'язання — використати порожній рядок як сторожове значення. Пам'ятайте, порожній рядок представляється в Python як `""` (лапки без пробілу між). Якщо користувач вводить порожній рядок у відповідь на введення (просто натискає `<Enter>`), Python повертає порожній рядок. Ось алгоритм:

```
ініціалізувати total нулем
ініціалізувати count нулем
ввести елемент даних як рядок, xStr
while xStr is not empty
    перетворити xStr на число, x
    додати x до total
    додати 1 до count
    ввести наступний елемент даних як рядок, xStr
вивести total / count
```

Перекладаючи на Python:

```python
# average4.py

def main():
    total = 0.0
    count = 0
    xStr = input("Введіть число (<Enter> для виходу) >> ")
    while xStr != "":
        x = float(xStr)
        total = total + x
        count = count + 1
        xStr = input("Введіть число (<Enter> для виходу) >> ")
    print("\nСереднє значення чисел:", total / count)

main()
```

Цей код не перетворює введення на число (через float), поки не перевірить, що введення не є сторожем (`""`).

Ось приклад запуску, що показує: тепер можна усереднювати довільні набори чисел:

```
Введіть число (<Enter> для виходу) >> 34
Введіть число (<Enter> для виходу) >> 23
Введіть число (<Enter> для виходу) >> 0
Введіть число (<Enter> для виходу) >> -25
Введіть число (<Enter> для виходу) >> -34.4
Введіть число (<Enter> для виходу) >> 22.7
Введіть число (<Enter> для виходу) >>

Середнє значення чисел: 3.38333333333
```

Нарешті ми маємо відмінне розв'язання нашої вихідної задачі.


### 8.3.3 Цикли для файлів

Один недолік усіх представлених до цього програм усереднення — вони є інтерактивними. Уявіть, що ви намагаєтеся усередняти 87 чисел і випадково робите помилку ближче до кінця. З нашою інтерактивною програмою вам доведеться починати все заново.

Кращий підхід — ввести всі числа у файл. Дані у файлі можна переглядати і редагувати перед відправкою до програми, що генерує звіт. Цей файлоорієнтований підхід зазвичай використовується для застосунків обробки даних.

Ще в розділі 5 ми розглядали читання даних із файлів, використовуючи файловий об'єкт як послідовність у циклі `for`. Ми можемо безпосередньо застосувати цю техніку до задачі усереднення чисел. Припускаючи, що числа введено у файл по одному на рядок, ми можемо обчислити середнє за допомогою цієї програми:

```python
# average5.py

def main():
    fileName = input("У якому файлі знаходяться числа? ")
    infile = open(fileName, 'r')
    total = 0.0
    count = 0
    for line in infile:
        total = total + float(line)
        count = count + 1
    print("\nСереднє значення чисел:", total / count)

main()
```

У цьому коді змінна циклу `line` ітерує по файлу як послідовності рядків; кожен рядок перетворюється на число і додається до поточної суми.

У багатьох мовах програмування немає спеціального механізму для циклічного перебору файлів. У таких мовах рядки файлу можна зчитувати по одному за допомогою форми сторожового циклу. Ми можемо проілюструвати цей метод у Python, використовуючи `readline()`. Пам'ятайте, метод `readline()` отримує наступний рядок із файлу у вигляді рядка. Наприкінці файлу `readline()` повертає порожній рядок, який ми можемо використати як сторожове значення. Ось загальний шаблон циклу до кінця файлу з `readline()` у Python:

```python
line = infile.readline()
while line != "":
    # обробити рядок
    line = infile.readline()
```

На перший погляд, можна підозрювати, що цей цикл передчасно зупиняється, якщо зустрічає порожній рядок у файлі. Це не так. Пам'ятайте, порожній рядок у текстовому файлі містить один символ нового рядка (`"\n"`), а метод readline включає символ нового рядка у своє значення, що повертається. Оскільки `"\n" != ""`, цикл продовжиться.

Ось код, що отримується при застосуванні сторожового циклу до кінця файлу до нашої задачі усереднення:

```python
# average6.py

def main():
    fileName = input("У якому файлі знаходяться числа? ")
    infile = open(fileName, 'r')
    total = 0.0
    count = 0
    line = infile.readline()
    while line != "":
        total = total + float(line)
        count = count + 1
        line = infile.readline()
    print("\nСереднє значення чисел:", total / count)

main()
```

Очевидно, ця версія є не такою лаконічною, як та, що використовує цикл `for`. У Python можна так само добре використовувати останній, але все одно корисно знати про цикли до кінця файлу на випадок, якщо ви застрягнете програмувати менш елегантною мовою.


### 8.3.4 Вкладені цикли

У минулому розділі ви бачили, як управляючі конструкції — такі як рішення і цикли — можуть вкладатися одна в одну для отримання складних алгоритмів. Особливо корисна, але дещо хитра техніка — вкладання циклів.

Розглянемо приклад програми. Знову звернемося до задачі усереднення чисел — обіцяю, це востаннє.¹ Припустімо, ми дещо змінюємо специфікацію задачі усереднення на основі файлу. Цього разу замість того, щоб вводити числа по одному на рядок, ми дозволимо будь-яку кількість значень на рядку. Коли на рядку з'являється кілька значень, вони розділяються комами.

На верхньому рівні базовий алгоритм — якийсь цикл обробки файлу, що обчислює поточну суму і кількість. Для практики скористаємося циклом до кінця файлу. Ось код, що складає цикл верхнього рівня:

```python
total = 0.0
count = 0
line = infile.readline()
while line != "":
    # оновити total і count для значень у рядку
    line = infile.readline()
print("\nСереднє значення чисел:", total / count)
```

Тепер нам потрібно з'ясувати, як оновлювати `total` і `count` у тілі циклу. Оскільки кожен окремий рядок файлу містить одне або більше чисел, розділених комами, ми можемо розбити рядок на підрядки, кожен з яких представляє число. Потім нам потрібно пройти по цих підрядках, перетворити кожен на число і додати до total. Ось фрагмент коду, що обробляє рядок:

```python
for xStr in line.split(", "):
    total = total + float(xStr)
    count = count + 1
```

Зверніть увагу, що ітерація циклу `for` у цьому фрагменті керується значенням `line`, яке є змінною управління циклом для циклу обробки файлу, описаного вище. Пов'язавши ці два цикли разом, отримуємо нашу програму:

```python
# average7.py

def main():
    fileName = input("У якому файлі знаходяться числа? ")
    infile = open(fileName, 'r')
    total = 0.0
    count = 0
    line = infile.readline()
    while line != "":
        # оновити total і count для значень у рядку
        for xStr in line.split(", "):
            total = total + float(xStr)
            count = count + 1
        line = infile.readline()
    print("\nСереднє значення чисел:", total / count)

main()
```

Як ви можете бачити, цикл, що обробляє числа в рядку, має відступ всередині циклу обробки файлу. Зовнішній цикл `while` ітерує по одному разу для кожного рядка файлу. На кожній ітерації зовнішнього циклу внутрішній цикл `for` ітерує стільки разів, скільки чисел у цьому рядку. Коли внутрішній цикл завершується, зчитується наступний рядок файлу, і зовнішній цикл виконує наступну ітерацію.

Окремі фрагменти цієї задачі не є складними самі по собі, але кінцевий результат є досить складним. Найкращий спосіб проектувати вкладені цикли — слідувати процесу, який ми застосовували тут. Спочатку спроектуйте зовнішній цикл, не турбуючись про те, що всередині. Потім спроектуйте те, що всередині, ігноруючи зовнішній цикл. Нарешті складіть частини разом, слідкуючи за збереженням вкладання. Якщо окремі цикли правильні, вкладений результат теж буде правильним — довіряйте цьому. З невеликою практикою ви будете реалізовувати подвійні і навіть потрійні вкладені цикли без труднощів.

## 8.4 Обчислення з булевими виразами

Тепер у нас є дві управляючі конструкції — `if` і `while` — що використовують умови, тобто булеві вирази. Концептуально булевий вираз обчислюється як одне з двох значень: хиба або істина. У Python ці значення представляються літералами `False` і `True`. До цього часу ми використовували прості булеві вирази, що порівнюють два значення (наприклад, `while x >= 0`).


### 8.4.1 Булеві оператори

Іноді прості умови, які ми використовували, здаються недостатньо виразними. Наприклад, припустімо, що вам потрібно визначити, чи знаходяться два об'єкти Point в одній позиції — тобто вони мають однакові координати x і y. Один зі способів обробки цього — вкладене рішення:

```python
if p1.getX() == p2.getX():
    if p1.getY() == p2.getY():
        # точки однакові
    else:
        # точки різні
else:
    # точки різні
```

Ви бачите, наскільки це незграбно.

Замість того, щоб вирішувати цю проблему через структуру рішень, інший підхід — побудувати більш складний вираз за допомогою булевих операцій. Як і більшість мов програмування, Python надає три булеві оператори: `and`, `or` і `not`. Розглянемо ці три оператори, а потім побачимо, як вони можуть спростити нашу задачу.

Булеві оператори `and` і `or` використовуються для поєднання двох булевих виразів і отримання булевого результату:

```
<вираз> and <вираз>
<вираз> or <вираз>
```

`and` двох виразів є істинним саме тоді, коли обидва вирази є істинними. Ми можемо представити це визначення у таблиці істинності:

| P | Q | P and Q |
|---|---|---------|
| T | T | T |
| T | F | F |
| F | T | F |
| F | F | F |

У цій таблиці P і Q представляють менші булеві вирази. Оскільки кожен вираз має два можливих значення, існує чотири можливих комбінації значень, кожна показана як один рядок. Останній стовпець дає значення `P and Q` для кожної можливої комбінації. За визначенням, `and` є істинним лише у випадку, коли і P, і Q є істинними.

`or` двох виразів є істинним, коли хоча б один вираз є істинним. Ось таблиця істинності для `or`:

| P | Q | P or Q |
|---|---|--------|
| T | T | T |
| T | F | T |
| F | T | T |
| F | F | F |

`or` є хибним лише тоді, коли обидва вирази є хибними. Зверніть особливо увагу, що `or` є істинним, коли обидва вирази є істинними. Це математичне визначення `or`, але слово «або» іноді вживається у виключному сенсі в повсякденній розмовній мові.

Оператор `not` обчислює протилежне від булевого виразу. Це унарний оператор, тобто він оперує над одним виразом. Таблиця істинності дуже проста:

| P | not P |
|---|-------|
| T | F |
| F | T |

Використовуючи булеві оператори, можна будувати довільно складні булеві вирази. Як і для арифметичних операторів, точне значення складного виразу залежить від правил пріоритету. Розглянемо вираз:

```python
a or not b and c
```

Як він обчислюється? Python дотримується стандартної угоди, що порядок пріоритету від вищого до нижчого: спочатку `not`, потім `and`, потім `or`. Тому вираз еквівалентний такій версії з дужками:

```python
(a or ((not b) and c))
```

На відміну від арифметики, більшість людей не схильна знати або пам'ятати правила пріоритету для булевих виразів. Рекомендується завжди використовувати дужки для складних виразів, щоб уникнути плутанини.

Тепер повернемося до нашої задачі. Для тестування збігу розташування двох точок можна використати операцію `and`:

```python
if p1.getX() == p2.getX() and p2.getY() == p1.getY():
    # точки однакові
else:
    # точки різні
```

Тут весь вираз буде істинним лише тоді, коли обидві прості умови є істинними. Це гарантує, що і координати x, і координати y збігаються. Очевидно, це значно простіше і зрозуміліше, ніж версія з вкладеними `if`.

Розглянемо трохи складніший приклад. У наступному розділі ми будемо розробляти симуляцію гри в ракетбол. Частина симуляції повинна визначати, коли гра закінчилася. Припустімо, `scoreA` і `scoreB` представляють рахунки двох гравців. Гра закінчується, щойно один із гравців досягає 15 очок. Ось булевий вираз, що є істинним, коли гра закінчена:

```python
scoreA == 15 or scoreB == 15
```

Коли будь-який рахунок досягає 15, одна з двох простих умов стає істинною, і за визначенням `or` весь булевий вираз є істинним. Поки обидві умови залишаються хибними (жоден гравець не досяг 15), весь вираз є хибним.

Наша симуляція потребуватиме циклу, що продовжується, поки гра не закінчена. Ми можемо побудувати відповідну умову циклу, взявши заперечення умови закінчення гри:

```python
while not (scoreA == 15 or scoreB == 15):
    # продовжувати грати
```

Ми також можемо будувати більш складні булеві вирази, що відображають різні можливі умови зупинки. Деякі гравці в ракетбол грають на «всухо». Для цих гравців гра також закінчується, коли один із гравців досягає 7, а інший ще не набрав жодного очка. Для стислості використаємо `a` для `scoreA` і `b` для `scoreB`. Ось вираз для закінчення гри зі «всухо»:

```python
a == 15 or b == 15 or (a == 7 and b == 0) or (b == 7 and a == 0)
```

Ви бачите, як ми додали ще дві ситуації до вихідної умови? Нові частини відображають два можливих способи настання «всухо», і кожен вимагає перевірки обох рахунків.

Спробуємо ще один приклад. Припустімо, ми пишемо симуляцію для волейболу. Традиційний волейбол не має «всухо», але вимагає, щоб команда перемогла принаймні з різницею в два очки. Якщо рахунок 15:14 або навіть 21:20 — гра продовжується.

Ось один підхід до умови закінчення волейбольного матчу:

```python
(a >= 15 and a - b >= 2) or (b >= 15 and b - a >= 2)
```

Цей вираз означає: гра закінчена, коли команда A перемогла (набрала принаймні 15 і веде принаймні на 2) або коли команда B перемогла.

Ось інший спосіб:

```python
(a >= 15 or b >= 15) and abs(a - b) >= 2
```

Ця версія дещо лаконічніша. Вона стверджує, що гра закінчена, коли одна з команд досягла переможної суми і різниця в рахунку принаймні 2. Пам'ятайте, `abs` повертає абсолютне значення виразу.


### 8.4.2 Булева алгебра

Всі рішення в комп'ютерних програмах зводяться до відповідних булевих виразів. Вміння формулювати, маніпулювати і міркувати про ці вирази — важлива навичка для програмістів і комп'ютерних учених. Булеві вирази підпорядковуються певним алгебраїчним законам, подібним до тих, що застосовуються до числових операцій. Ці закони називаються булевою логікою або булевою алгеброю.

Розглянемо кілька прикладів. У наведеній нижче таблиці показані деякі правила алгебри та їхні аналоги в булевій алгебрі:

| алгебра | булева алгебра |
|---------|----------------|
| a × 0 = 0 | a and False == False |
| a × 1 = a | a and True == a |
| a + 0 = a | a or False == a |

З цих прикладів видно, що `and` має схожість із множенням, `or` — із додаванням, а 0 і 1 відповідають `False` і `True`.

Ось деякі інші цікаві властивості булевих операцій. Що завгодно в `or` із `True` просто дорівнює `True`:

```python
(a or True) == True
```

І `and`, і `or` дистрибутивні щодо один одного:

```python
(a or (b and c)) == ((a or b) and (a or c))
(a and (b or c)) == ((a and b) or (a and c))
```

Подвійне заперечення скасовується:

```python
(not (not a)) == a
```

Наступні два тотожності відомі як закони Де Моргана:

```python
(not (a or b)) == ((not a) and (not b))
(not (a and b)) == ((not a) or (not b))
```

Зверніть увагу, як оператор змінюється між `and` і `or`, коли `not` «вштовхується» у вираз.

Одна приємна властивість булевої алгебри — прості тотожності такого роду легко перевіряються таблицею істинності. Оскільки завжди є скінченна комбінація можливих значень для змінних, ми можемо систематично перерахувати всі можливості і обчислити значення виразів. Наприклад, наведена нижче таблиця демонструє перший закон Де Моргана:

| a | b | a or b | not (a or b) | not a | not b | (not a) and (not b) |
|---|---|--------|--------------|-------|-------|---------------------|
| T | T | T | F | F | F | F |
| T | F | T | F | F | T | F |
| F | T | T | F | T | F | F |
| F | F | F | T | T | T | T |

Рядки представляють чотири різних випадки для змінних a і b, а стовпці показують значення істинності підвиразів у тотожності. Зверніть увагу, що виділені стовпці ідентичні, тим самим доводячи, що тотожність завжди виконується.

Важливе застосування булевої алгебри — аналіз і спрощення булевих виразів усередині програм. Наприклад, повернемося до ракетбольного матчу ще раз. Вище ми розробили умову циклу для продовження гри:

```python
while not (scoreA == 15 or scoreB == 15):
    # продовжувати грати
```

Застосовуючи закон Де Моргана, знаємо, що вираз еквівалентний такому:

```python
(not scoreA == 15) and (not scoreB == 15)
```

Пам'ятайте, ми маємо змінювати `or` на `and` при «розподілі» `not`. Ця умова не краще за першу, але ми можемо зробити ще один крок, «вштовхнувши» `not` у самі умови:

```python
while scoreA != 15 and scoreB != 15:
    # продовжувати грати
```

Тепер у нас є версія, яку значно легше розуміти. Це читається просто: поки гравець A не досяг 15 і гравець B не досяг 15 — продовжувати грати.

Цей конкретний приклад ілюструє загальний корисний підхід до умов циклу. Іноді легше з'ясувати, коли цикл повинен зупинитися, ніж коли він повинен продовжуватися. У такому випадку просто запишіть умову завершення циклу і поставте `not` перед нею. Одне-два застосування законів Де Моргана можуть привести вас до простішої, але еквівалентної версії, придатної для використання в операторі `while`.


## 8.5 Інші поширені структури

Взяті разом, структура рішень (`if`) і цикл із попередньою перевіркою (`while`) надають повний набір управляючих конструкцій. Це означає, що кожен алгоритм може бути виражений лише за їх допомогою. Освоївши `while` і `if`, ви можете написати будь-який мислимий алгоритм. Однак для певних видів задач альтернативні структури іноді можуть бути зручними. Цей розділ описує деякі з цих альтернатив.


### 8.5.1 Цикл із кінцевою перевіркою

Припустімо, ви пишете алгоритм введення, який повинен отримати від користувача невід'ємне число. Якщо користувач вводить неправильне введення, програма запитує ще одне значення. Вона продовжує повторний запит, поки користувач не введе дійсне значення. Цей процес називається перевіркою введення. Добре спроектовані програми перевіряють введення, коли це можливо.

Ось простий алгоритм:

```
repeat
    отримати число від користувача
until число >= 0
```

Ідея в тому, що цикл продовжує отримувати введення, поки значення не є прийнятним. Блок-схема цього проекту показана на рисунку 8.2. Зверніть увагу, як цей алгоритм містить цикл, де перевірка умови відбувається після тіла циклу. Це цикл із кінцевою перевіркою (post-test loop). Цикл із кінцевою перевіркою завжди повинен виконувати тіло циклу принаймні один раз.

На відміну від деяких інших мов, Python не має оператора, що безпосередньо реалізує цикл із кінцевою перевіркою. Однак цей алгоритм може бути реалізований за допомогою `while` шляхом «заповнення» умови циклу для першої ітерації:

```python
number = -1  # Починаємо з недопустимого значення, щоб потрапити в цикл.
while number < 0:
    number = float(input("Введіть додатне число: "))
```

Це змушує тіло циклу виконатися принаймні один раз і еквівалентне алгоритму з кінцевою перевіркою. Зверніть увагу, що це схоже на структуру, наведену раніше для шаблону інтерактивного циклу. Інтерактивні цикли природно підходять до реалізації з кінцевою перевіркою.

Деякі програмісти вважають за краще більш безпосередньо симулювати цикл із кінцевою перевіркою, використовуючи оператор Python `break`. Виконання `break` змушує Python негайно вийти з циклу, що містить цей оператор. Нерідко `break` використовується для виходу з того, що синтаксично виглядає як нескінченний цикл.

Ось той самий алгоритм, реалізований з `break`:

```python
while True:
    number = float(input("Введіть додатне число: "))
    if number >= 0: break  # Вийти з циклу, якщо число дійсне.
```

Перший рядок може здатися вам дивним. Пам'ятайте, цикл `while` продовжується, поки вираз у заголовку циклу обчислюється як істинний. Оскільки `True` завжди є істинним, це виглядає як нескінченний цикл. Однак коли значення `number` є невід'ємним, виконується оператор `break`, який завершує цикл. Зверніть увагу, що я розмістив `break` у тому самому рядку, що й `if`. Це допустимо, коли тіло `if` містить лише один оператор. Часто можна побачити однорядкову комбінацію `if-break`, що використовується як вихід із циклу.

Навіть цей невеликий приклад можна покращити. Було б непогано, якби програма видавала попередження, що пояснює, чому введення було неприпустимим. У версії `while` цикл із кінцевою перевіркою це дещо незручно:

```python
number = -1  # Починаємо з недопустимого значення, щоб потрапити в цикл.
while number < 0:
    number = float(input("Введіть додатне число: "))
    if number < 0:
        print("Введене вами число було не додатним")
```

Чи бачите ви, як перевірка дійсності повторюється в двох місцях?

Додавання попередження до версії з `break` вимагає лише додавання `else` до існуючого `if`:

```python
while True:
    number = float(input("Введіть додатне число: "))
    if number >= 0:
        break  # Вийти з циклу, якщо число дійсне.
    else:
        print("Введене вами число було не додатним")
```


### 8.5.2 Цикл з виходом посередині

Деякі програмісти вирішили б проблему попередження з попереднього розділу, використовуючи дещо інший стиль:

```python
while True:
    number = float(input("Введіть додатне число: "))
    if number >= 0: break  # Вихід із циклу
    print("Введене вами число було не додатним")
```

Тут вихід із циклу фактично знаходиться посередині тіла. Це називається «цикл з виходом посередині» (loop and a half). Деякі пуристи нарікають на виходи посередині циклу, але цей шаблон може бути досить зручним.

«Цикл з виходом посередині» — елегантний спосіб уникнути завантажувального читання в сторожовому циклі. Ось загальний шаблон сторожового циклу, реалізованого як «цикл з виходом посередині»:

```python
while True:
    отримати наступний елемент даних
    if the item is the sentinel: break
    process the item
```

На рисунку 8.3 показана блок-схема цього підходу до сторожових циклів. Ви можете бачити, що ця реалізація є вірною першому правилу сторожових циклів: уникати обробки сторожового значення.

Вибір — використовувати оператори `break` чи ні — значною мірою є питанням смаку. Обидва стилі є прийнятними. Спокуси, якої, як правило, слід уникати, — це рясне вкраплення тіла циклу кількома операторами `break`. Логіку циклу легко втратити, коли є кілька виходів. Однак бувають випадки, коли навіть це правило слід порушити, щоб отримати найбільш елегантне розв'язання задачі.


### 8.5.3 Булеві вирази як рішення

До цього часу ми говорили про булеві вирази лише в контексті інших управляючих конструкцій. Іноді самі булеві вирази можуть виступати як управляючі конструкції. Насправді булеві вирази настільки гнучкі в Python, що іноді можуть призводити до тонких помилок програмування.

Розглянемо написання інтерактивного циклу, що продовжується, поки відповідь користувача починається з «y». Щоб дозволити користувачу вводити відповідь великими або малими літерами, можна використати такий цикл:

```python
while response[0] == "y" or response[0] == "Y":
```

Слід бути уважним і не скорочувати цю умову так, як ви могли б думати про неї по-англійськи: «While the first letter is 'y' or 'Y'». Наступна форма не працює:

```python
while response[0] == "y" or "Y":
```

Насправді це нескінченний цикл. Розуміння, чому ця умова завжди є істинною, вимагає вивчення деяких особливостей булевих виразів Python.

Ви вже знаєте, що Python має тип `bool`. Насправді це відносно недавнє доповнення до мови (версія 2.3). До цього Python просто використовував цілі числа 1 і 0 для представлення істини та хиби. Насправді тип `bool` — це просто «спеціальний» `int`, де значення 0 і 1 виводяться як `False` і `True`.

Ми використовували булеві літерали `True` і `False` для представлення булевих значень. Проте Python є насправді дуже гнучким щодо того, який тип даних може виступати як булевий вираз. Будь-який вбудований тип може бути інтерпретований як булевий. Для чисел (int і float) нульове значення вважається хибним; будь-що, відмінне від нуля, вважається істинним. Ось кілька прикладів:

```python
>>> bool(0)
False
>>> bool(1)
True
>>> bool(32)
True
>>> bool("hello")
True
>>> bool("")
False
>>> bool([1, 2, 3])
True
>>> bool([])
False
```

Як ви можете бачити, для типів-послідовностей порожня послідовність інтерпретується як хибна, тоді як будь-яка непорожня послідовність — як істинна.

Гнучкість булевих виразів Python поширюється на булеві оператори. Хоча основне використання цих операторів — формування булевих виразів, вони мають операційні визначення, що роблять їх корисними й для інших цілей. Ця таблиця підсумовує поведінку операторів:

| оператор | операційне визначення |
|----------|-----------------------|
| `x and y` | Якщо x є хибним, повернути x. Інакше — повернути y. |
| `x or y` | Якщо x є істинним, повернути x. Інакше — повернути y. |
| `not x` | Якщо x є хибним, повернути `True`. Інакше — повернути `False`. |

Ці операційні визначення показують, що булеві оператори Python — це оператори короткого замикання (short-circuit operators). Це означає, що значення істина або хиба повертається, щойно результат стає відомим. У виразі `and`, де перший вираз є хибним, і в `or`, де перший вираз є істинним, Python не обчислюватиме навіть другий вираз.

Тепер поглянемо на нашу проблему нескінченного циклу:

```python
response[0] == "y" or "Y"
```

Це завжди обчислюється як істинне. Перш за все, зверніть увагу, що булевий оператор поєднує два вирази; перший — проста умова, а другий — рядок. Ось еквівалентна версія з дужками:

```python
(response[0] == "y") or ("Y")
```

За операційним описом `or`, цей вираз повертає або `True` (повернутий `==`, коли `response[0]` є `"y"`) або `"Y"` (коли `response[0]` не є `"y"`). Будь-який з цих результатів інтерпретується Python як істинний. Оскільки `"Y"` — непорожній рядок, Python завжди інтерпретуватиме його як істинний.

Ця поведінка зумовлена деякими особливостями визначень булевих операторів. Це одне з небагатьох місць, де проектування Python може стати пасткою для програміста-початківця.

Гнучкість Python дозволяє деякі лаконічні ідіоми програмування. Часто програми пропонують значення за замовчуванням для відповіді:

```python
ans = input("Який смак ви хочете [ванільний]: ")
if ans != "":
    flavor = ans
else:
    flavor = "vanilla"
```

Використовуючи той факт, що рядок у `ans` може бути оброблений як булевий, умову в цьому коді можна спростити:

```python
ans = input("Який смак ви хочете [ванільний]: ")
if ans:
    flavor = ans
else:
    flavor = "vanilla"
```

Якщо користувач просто натискає `<Enter>`, `ans` буде порожнім рядком, який Python інтерпретує як хибне. У цьому випадку порожній рядок буде замінений на `"vanilla"` у гілці `else`.

Ту саму ідею можна більш лаконічно закодувати, трактуючи самі рядки як булеві і використовуючи `or`:

```python
ans = input("Який смак ви хочете [ванільний]: ")
flavor = ans or "vanilla"
```

Насправді цю задачу легко виконати в одному рядку коду:

```python
flavor = input("Який смак ви хочете [ванільний]: ") or "vanilla"
```

Не знаю, чи справді варто заощаджувати кілька рядків коду за допомогою булевих операторів у такий спосіб. Якщо вам подобається цей стиль — використовуйте його. Тільки переконайтеся, що ваш код не стає настільки хитрим, що іншим (або вам) важко його зрозуміти.


## 8.6 Приклад: простий цикл обробки подій

Ще в розділі 4 я згадував, що сучасні програми з графічними інтерфейсами користувача (GUI) загалом пишуться в стилі, керованому подіями. Програма відображає графічний інтерфейс і потім «чекає» подій від користувача — клацання по меню або натискання клавіші на клавіатурі. Програма реагує, обробляючи подію. Під капотом механізм, що рухає цей стиль програми, — так званий цикл обробки подій (event loop). Загальний контур GUI-програми виглядає так:

```
намалювати GUI
while True:
    отримати наступну подію
    if подія є "сигналом виходу":
        break
    обробити подію
прибрати і вийти
```

По суті, ми маємо сторожовий цикл (тут виражений як «цикл з виходом посередині»), де сторожем є просто спеціальна подія, наприклад натискання клавіші `q`, що спричиняє вихід із програми.

Як простий приклад, розглянемо програму, яка відкриває графічне вікно і дозволяє користувачу змінювати його колір, натискаючи різні клавіші — r для червоного, g для сірого тощо. Користувач може вийти в будь-який час, натиснувши клавішу `q`. Ми можемо закодувати це як простий цикл обробки подій, використовуючи `getKey()` для обробки натискань клавіш:

```python
# event_loop1.py  -- вікно зі зміною кольору за клавіатурою

from graphics import *

def main():
    win = GraphWin("Color Window", 500, 500)

    # Цикл подій: обробляти натискання клавіш, поки користувач не натисне "q".
    while True:
        key = win.getKey()
        if key == "q":  # вихід із циклу
            break

        # обробити клавішу
        if key == "r":
            win.setBackground("pink")
        elif key == "w":
            win.setBackground("white")
        elif key == "g":
            win.setBackground("lightgray")
        elif key == "b":
            win.setBackground("lightblue")

    # вийти з програми
    win.close()

main()
```

Зверніть, що щоразу при проходженні циклу подій ця програма очікуватиме, поки користувач натисне клавішу на клавіатурі.

Більш гнучкий інтерфейс користувача міг би дозволяти взаємодію різними способами — набором на клавіатурі, вибором пункту меню, наведенням миші на значок або клацанням кнопки. У цьому випадку цикл подій мав би перевіряти кілька типів подій, а не чекати однієї конкретної. Розширимо наше просте вікно зі зміною кольору, додавши взаємодію з мишею для введення тексту — свого роду вдосконалену версію прикладу «клацання і введення» з розділу 4.

Змішуючи управління мишею і клавіатурою, ми відразу стикаємося з проблемою. Ми не можемо більше покладатися на наші основні методи введення `getMouse` і `getKey`. Якщо ми викликаємо `win.getKey()`, програма зупиняється, поки користувач натисне клавішу. Що, якщо він вирішить скористатися мишею? Нічого — програма зупинена в очікуванні натискання клавіші. Навпаки, якщо ми викликаємо `getMouse()`, клавіатурне введення блокується. У термінах проектування інтерфейсу ми називаємо ці методи введення модальними, тому що вони прив'язують користувача до певного режиму взаємодії. Введення є немодальним (краще сказати мультимодальним), коли користувач сам керує способом взаємодії.

У нашому прикладі ми можемо зробити цикл подій немодальним, використавши альтернативні методи `checkKey` і `checkMouse`. Ці методи подібні до своїх аналогів, за винятком того, що вони не чекають від користувача жодних дій. Розглянемо такий оператор:

```python
key = win.checkKey()
```

Python перевірить, чи натиснуто клавішу, і якщо так — поверне рядок, що представляє клавішу. Однак він не чекає. Якщо жодна клавіша не натиснута, `checkKey` негайно поверне порожній рядок. Перевіряючи значення `key`, програма може визначити, чи натиснуто клавішу, не зупиняючись в очікуванні.

Використовуючи методи перевірки, ми можемо легко окреслити немодальний цикл подій:

```
намалювати GUI
while True:
    key = checkKey()
    if key is quit signal: break
    
    if key is a valid key:
        process key
    
    click = checkMouse()
    if click is valid:
        process click
прибрати і вийти
```

Кожного разу через цикл програма перевіряє натискання клавіші або клацання миші і обробляє їх відповідно. Якщо немає жодної події для обробки, вона не чекає — просто йде по колу і перевіряє знову. Коли програма начебто терпляче чекає дій користувача, насправді вона зайнята кружлянням по циклу.

Ось перший крок до нашої вдосконаленої програми «клацання і введення» — перегляд вікна зі зміною кольору з розширеним циклом подій:

```python
# event_loop2.py --- вікно зі зміною кольору

from graphics import *

def handleKey(k, win):
    if k == "r":
        win.setBackground("pink")
    elif k == "w":
        win.setBackground("white")
    elif k == "g":
        win.setBackground("lightgray")
    elif k == "b":
        win.setBackground("lightblue")

def handleClick(pt, win):
    pass

def main():
    win = GraphWin("Click and Type", 500, 500)

    # Цикл подій: обробляти натискання клавіш та кліки мишею,
    #    поки користувач не натисне клавішу "q".
    while True:
        key = win.checkKey()
        if key == "q":  # вихід із циклу
            break

        if key:
            handleKey(key, win)

        pt = win.checkMouse()
        if pt:

    win.close()

main()
```

Я використовував функції для модуляризації програми і наголошення на тому, як структура відповідає розширеному алгоритму циклу подій. Оскільки я ще не визначив точно, що робити з кліками миші, я визначив функцію `handleClick` з тілом, що містить оператор `pass`. Оператор `pass` нічого не робить; він просто заповнює місце, де Python синтаксично очікує наявності оператора. Це дозволяє мені запустити й протестувати програму і побачити, що вона поводиться так само, як і попередня версія.

Також уважно зверніть на умови, використані в операторах `if`. Коли немає введення, виклики `checkKey()` і `checkMouse()` обидва повертають значення, що Python інтерпретує як хибне; для `checkKey` — це порожній рядок, а для `checkMouse()` — спеціальний об'єкт `None`. Це дозволяє лаконічно перевіряти, чи дійсно мала місце взаємодія з користувачем. Ми можемо просто написати `if key:` замість `if key != "":` і `if pt:` замість `if pt != None`. Не всі програмісти використовують ці ідіоми, але мені подобається, як вони читаються.

Тепер, оновивши нашу програму зміни кольорів немодальним циклом подій, ми готові додати частину обробки миші. Ми хочемо дозволити користувачам розміщувати текст у вікні. Замість роботи з одним символом за раз через існуючий цикл подій, зручніше мати користувачів, що насправді вводять у об'єкт Entry. Тому клацання по вікну ініціює базовий трикроковий алгоритм:

1. Відобразити поле Entry там, де клацнув користувач.
2. Дозволити користувачу ввести текст у поле; введення завершується натисканням клавіші `<Enter>`.
3. Поле Entry зникає, і введений текст з'являється безпосередньо у вікні.

У кроці 2 відбувається щось цікаве. Ми хочемо, щоб текст, що вводить користувач, з'являвся у полі Entry, але не хочемо, щоб ці натискання клавіш інтерпретувалися як команди верхнього рівня. Наприклад, введення `"q"` у Entry не повинно спричиняти завершення програми! Нам потрібно, щоб програма «стала модальною». Відповідь — ще один цикл. Всередині основного циклу подій ми вкладаємо інший цикл, що «споживає» всі натискання клавіш, поки користувач не натисне `<Enter>`. Ось код для оновленого `handleClick`:

```python
def handleClick(pt, win):
    # створити Entry для введення користувача
    entry = Entry(pt, 10)
    entry.draw(win)

    # Стати модальним: цикл, поки користувач не натисне <Enter>
    while True:
        key = win.getKey()
        if key == "Return": break

    # стерти Entry і створити та намалювати Text
    entry.undraw()
    typed = entry.getText()
    Text(pt, typed).draw(win)

    # очистити (ігнорувати) будь-яке клацання миші, що відбулося під час введення
    win.checkMouse()
```

Вивчіть цей код, щоб переконатися, що ви розумієте, як він реалізує трикроковий алгоритм. Варто зазначити, що `getKey` повертає `"Return"` для клавіші `<Enter>`. Оскільки ми просто чекаємо `"Return"`, я міг би спростити модальний цикл до чогось такого:

```python
while win.getKey() != "Return":
    pass
```

У цій версії тіло циклу буквально нічого не робить. Цикл просто продовжує перевіряти умову, поки не натиснуто `<Enter>` і вона не стане хибною. Потім цикл завершується, дозволяючи програмі продовжитися. Обидві версії виконують роботу. Друга хитріша, але перша здається більш очевидною. Мабуть, краще обирати очевидне над хитрим.

Самий останній рядок цієї функції необхідний, щоб забезпечити справжню модальність введення тексту. Будь-які кліки миші, що могли відбутися до натискання клавіші `<Enter>`, мають просто ігноруватися. Оскільки `checkMouse` повертає лише кліки миші, що відбулися з моменту останнього виклику `checkMouse`, виклик функції тут наприкінці має ефект очищення будь-якого кліку, що міг відбутися, але ще не був перевірений.

Я б настійно рекомендував запустити та вивчити остаточну версію цієї програми — `event_loop3.py`. Серед варіантів для спроб — закоментувати `checkMouse` наприкінці `handleClick` і подивитися, чи можна відтворити ситуацію, коли програма поводиться дивно. Ще одна хороша вправа — модифікація, що дозволяє користувачу скасувати введення тексту в будь-який час, натиснувши клавішу `Escape` (`<Esc>`). Фактично `"Escape"` стає ще одним сторожем для модального циклу, але при його використанні об'єкт Text не створюється.


## 8.7 Підсумок розділу

- Цикл Python `for` — це визначений цикл, що ітерує через послідовність.

- Оператор Python `while` — приклад невизначеного циклу. Він продовжує ітерацію, поки умова циклу залишається істинною. Використовуючи невизначений цикл, програмісти повинні убезпечитися від можливості випадкового написання нескінченного циклу.

- Одне важливе використання невизначеного циклу — реалізація шаблону «інтерактивний цикл». Інтерактивний цикл дозволяє повторювати частини програми за бажанням користувача.

- Сторожовий цикл — це цикл, що обробляє введення, поки не зустріне спеціальне значення (сторожа). Сторожові цикли — поширений шаблон програмування. При написанні сторожового циклу програміст повинен слідкувати за тим, щоб сторожового значення не обробляли.

- Цикли корисні для читання файлів. Python розглядає файл як послідовність рядків, тому особливо зручно обробляти файл рядок за рядком за допомогою циклу `for`. В інших мовах цикл для файлу, як правило, реалізується за допомогою шаблону сторожового циклу.

- Цикли, як і інші управляючі конструкції, можуть вкладатися. При проектуванні алгоритмів із вкладеними циклами найкраще розглядати цикли по одному.

- Складні булеві вирази можна будувати з простих умов за допомогою булевих операторів `and`, `or` і `not`. Булеві оператори підпорядковуються правилам булевої алгебри. Закони Де Моргана описують, як заперечувати булеві вирази, що включають `and` і `or`.

- Нестандартні циклічні структури, такі як «цикл з виходом посередині», можна будувати за допомогою циклу `while` з умовою `True` і оператором `break` для забезпечення виходу з циклу.

- Булеві оператори Python `and` і `or` застосовують обчислення з коротким замиканням. Вони також мають операційні визначення, що дозволяють використовувати їх у певних контекстах рішень. Хоча Python має вбудований тип даних `bool`, інші типи даних (наприклад, `int`) також можуть використовуватися там, де очікуються булеві вирази.

- GUI-програми зазвичай є програмами, керованими подіями, і реалізують ретельно спроектовані цикли обробки подій для управління взаємодією з користувачем. Взаємодія називається немодальною, коли користувач сам визначає, що відбудеться далі, і модальною, коли застосунок диктує, що користувач повинен зробити далі.

    Лінія регресії — це лінія з рівнянням:

    де m = (Σxᵢyᵢ − nx̄ȳ) / (Σxᵢ² − nx̄²)  
    x̄ — середнє x-значень, ȳ — середнє y-значень, n — кількість точок.

14. Напишіть програму для перетворення кольорового зображення в чорно-біле. Користувач вказує ім'я файлу із зображенням GIF або PPM, програма завантажує і відображає файл. По кліку миші програма перетворює зображення на чорно-біле. Потім від користувача запрошується ім'я файлу для збереження чорно-білого зображення.  
    Базова ідея перетворення — пройти по пікселях зображення і перетворити кожен із кольорового на відповідний відтінок сірого. Сірий піксель створюється встановленням однакового значення яскравості для його червоного, зеленого і синього компонентів. Щоб визначити яскравість сірого, слід використати зважене середнє вихідних RGB-значень. Ось псевдокод алгоритму:  
    ```
    for each row in the image:
        for each column in the image:
            r, g, b = get pixel information for current row and column
            brightness = int(round(0.299*r + 0.587*g + 0.114*b))
            set pixel to color_rgb(brightness, brightness, brightness)
        update the image  # to see progress row by row
    ```  
    Примітка: операції над пікселями в класі Image є досить повільними, тому для тестування програми використовуйте відносно невеликі зображення.

15. Напишіть програму для перетворення зображення на його кольоровий негатив. Загальна форма програми буде схожа на попередню задачу. Негатив пікселя утворюється вирахуванням кожного кольорового значення з 255. Тому новий колір пікселя: `color_rgb(255-r, 255-g, 255-b)`.
